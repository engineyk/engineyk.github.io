# 生成器模式（Builder Pattern）

## 概念简介

生成器模式是一种创建型设计模式，主要用于“分步骤”构建一个复杂对象。它将一个复杂对象的构建与它的表示分离，使同样的构建过程可以创建不同的表示。

---

## 结构图

```
┌──────────────┐
│   指挥者     │
└──────┬───────┘
       │
┌──────▼───────┐
│   生成器接口 │
└──────┬───────┘
       │
 ┌─────┴──────┐
 │ 具体生成器 │
 └─────┬──────┘
       │
┌──────▼───────┐
│   产品对象   │
└──────────────┘
```

## 代码示例（C#）

```csharp
// 产品类
public class Product
{
    public string PartA { get; set; }
    public string PartB { get; set; }
    public string PartC { get; set; }

    public void Show()
    {
        Console.WriteLine($"部件A: {PartA}");
        Console.WriteLine($"部件B: {PartB}");
        Console.WriteLine($"部件C: {PartC}");
    }
}

// 抽象生成器
public abstract class Builder
{
    protected Product product = new Product();
    public abstract void BuildPartA();
    public abstract void BuildPartB();
    public abstract void BuildPartC();
    public Product GetResult()
    {
        return product;
    }
}

// 具体生成器
public class ConcreteBuilder : Builder
{
    public override void BuildPartA()
    {
        product.PartA = "具体部件A";
    }
    public override void BuildPartB()
    {
        product.PartB = "具体部件B";
    }
    public override void BuildPartC()
    {
        product.PartC = "具体部件C";
    }
}

// 指挥者
public class Director
{
    public void Construct(Builder builder)
    {
        builder.BuildPartA();
        builder.BuildPartB();
        builder.BuildPartC();
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        Builder builder = new ConcreteBuilder();
        Director director = new Director();
        director.Construct(builder);
        Product product = builder.GetResult();
        product.Show();
    }
}
```

---

## 优缺点

**优点：**
- 可以分步构建复杂对象，便于控制细节。
- 同样的构建过程可以创建不同的表示（产品）。
- 封装性好，客户端不需要关注具体的建造过程和细节。

**缺点：**
- 产品的内部变化会导致生成器的修改，类数量较多。
- 不适合产品构建步骤差异较大的场景。

---

## 应用场景

- 需要生成复杂对象，且对象的各个部分经常变化。
- 客户端不需要知道产品的具体构建细节。
