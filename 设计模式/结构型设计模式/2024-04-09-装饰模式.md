# 装饰模式（Decorator Pattern）—— C# 实现

装饰模式是一种结构型设计模式，允许在不改变原有对象结构的情况下，动态地为对象添加功能。它通过将对象包装在装饰器类中，把附加的职责或行为附加到对象上。

## 结构

- **Component（抽象组件）**：定义对象的接口。
- **ConcreteComponent（具体组件）**：实现Component接口，定义被装饰的原始对象。
- **Decorator（抽象装饰器）**：实现Component接口，持有一个Component对象的引用。
- **ConcreteDecorator（具体装饰器）**：继承Decorator，扩展Component的功能。

## UML类图

```
Component
  ├── ConcreteComponent
  └── Decorator
        └── ConcreteDecorator
```

## C# 示例代码

```csharp
using System;

// 抽象组件
public abstract class Coffee
{
    public abstract string GetDescription();
    public abstract double GetCost();
}

// 具体组件
public class SimpleCoffee : Coffee
{
    public override string GetDescription() => "Simple Coffee";
    public override double GetCost() => 5.0;
}

// 抽象装饰器
public abstract class CoffeeDecorator : Coffee
{
    protected Coffee coffee;
    public CoffeeDecorator(Coffee coffee)
    {
        this.coffee = coffee;
    }
    public override string GetDescription() => coffee.GetDescription();
    public override double GetCost() => coffee.GetCost();
}

// 具体装饰器A
public class MilkDecorator : CoffeeDecorator
{
    public MilkDecorator(Coffee coffee) : base(coffee) { }
    public override string GetDescription() => coffee.GetDescription() + ", Milk";
    public override double GetCost() => coffee.GetCost() + 1.5;
}

// 具体装饰器B
public class SugarDecorator : CoffeeDecorator
{
    public SugarDecorator(Coffee coffee) : base(coffee) { }
    public override string GetDescription() => coffee.GetDescription() + ", Sugar";
    public override double GetCost() => coffee.GetCost() + 0.5;
}

// 使用示例
class Program
{
    static void Main()
    {
        Coffee coffee = new SimpleCoffee();
        Console.WriteLine($"{coffee.GetDescription()} : {coffee.GetCost()}");

        coffee = new MilkDecorator(coffee);
        Console.WriteLine($"{coffee.GetDescription()} : {coffee.GetCost()}");

        coffee = new SugarDecorator(coffee);
        Console.WriteLine($"{coffee.GetDescription()} : {coffee.GetCost()}");
    }
}
```

## 输出结果

```
Simple Coffee : 5
Simple Coffee, Milk : 6.5
Simple Coffee, Milk, Sugar : 7
```

## 适用场景

- 希望在运行时为对象添加功能。
- 需要动态地撤销/更换功能。
- 继承体系导致类爆炸，装饰器可替代子类扩展。

## 优缺点

**优点：**
- 比继承更加灵活，支持动态扩展对象功能。
- 可以组合多个装饰器，实现功能叠加。

**缺点：**
- 增加了系统中类的数量，理解和调试较复杂。
- 多层装饰可能导致排查问题困难。

## 总结

装饰模式是运行时功能扩展的优雅方案，适用于“增强对象功能”的场景，而无需修改原有代码结构。
