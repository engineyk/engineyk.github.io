# 命令模式（Command Pattern）—— C# 实现

命令模式是一种行为型设计模式，它将请求（命令）封装为对象，从而可以用不同的请求、队列、日志、撤销等操作参数化其他对象。

## 结构

- **Command（命令接口）**：声明执行操作的接口。
- **ConcreteCommand（具体命令）**：实现命令接口，绑定接收者并调用相应操作。
- **Receiver（接收者）**：知道如何实施与执行一个请求的操作。
- **Invoker（调用者）**：要求命令对象执行请求。
- **Client（客户端）**：创建具体命令对象并设置其接收者。

## UML 类图

```
[Client] ——> [Invoker] ——> [Command] <—— [ConcreteCommand] ——> [Receiver]
```

## C# 示例代码

```csharp
using System;

// 命令接口
public interface ICommand
{
    void Execute();
}

// 接收者
public class Light
{
    public void On() => Console.WriteLine("Light is ON");
    public void Off() => Console.WriteLine("Light is OFF");
}

// 具体命令：开灯
public class LightOnCommand : ICommand
{
    private Light _light;
    public LightOnCommand(Light light) { _light = light; }
    public void Execute() => _light.On();
}

// 具体命令：关灯
public class LightOffCommand : ICommand
{
    private Light _light;
    public LightOffCommand(Light light) { _light = light; }
    public void Execute() => _light.Off();
}

// 调用者
public class RemoteControl
{
    private ICommand _command;
    public void SetCommand(ICommand command) { _command = command; }
    public void PressButton() { _command.Execute(); }
}

// 使用示例
class Program
{
    static void Main()
    {
        Light light = new Light();
        ICommand lightOn = new LightOnCommand(light);
        ICommand lightOff = new LightOffCommand(light);

        RemoteControl remote = new RemoteControl();

        remote.SetCommand(lightOn);
        remote.PressButton(); // 输出：Light is ON

        remote.SetCommand(lightOff);
        remote.PressButton(); // 输出：Light is OFF
    }
}
```

## 输出结果

```
Light is ON
Light is OFF
```

## 适用场景

- 需要将调用操作的对象与实际实施操作的对象解耦。
- 需要支持撤销、重做操作。
- 需要支持命令的排队、日志记录等功能。

## 优缺点

**优点：**
- 解耦调用者与接受者。
- 容易扩展新的命令。
- 支持撤销、重做、事务等。

**缺点：**
- 增加了系统中类的数量。
- 设计过度会导致类爆炸。

## 总结

命令模式将请求封装为对象，极大地提高了系统的灵活性和可扩展性，常用于事务、队列、日志、遥控器等场景。
