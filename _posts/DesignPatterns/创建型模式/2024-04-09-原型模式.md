# 原型模式（Prototype Pattern）

## 概念简介

原型模式是一种创建型设计模式，它通过复制已有对象来创建新对象，而不是通过 new 关键字实例化。这种模式适用于创建成本较高或复杂对象的场景，并且允许对象在运行时动态复制自身。

---

## 结构图

```
┌─────────────┐
│   原型接口  │◄────────────┐
└──────┬──────┘             │
       │                    │
┌──────▼──────┐      ┌──────▼──────┐
│具体原型A    │      │具体原型B    │
└──────┬──────┘      └──────┬──────┘
       │                    │
       ▼                    ▼
   克隆对象A            克隆对象B
```

---

## 代码示例（C#）

```csharp
// 原型接口
public interface IPrototype<T>
{
    T Clone();
}

// 具体原型类
public class ConcretePrototype : IPrototype<ConcretePrototype>
{
    public string Data { get; set; }

    public ConcretePrototype(string data)
    {
        Data = data;
    }

    // 实现克隆方法
    public ConcretePrototype Clone()
    {
        // 浅拷贝
        return (ConcretePrototype)this.MemberwiseClone();
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        ConcretePrototype prototype = new ConcretePrototype("原型数据");
        ConcretePrototype clone = prototype.Clone();
        Console.WriteLine(prototype.Data); // 输出：原型数据
        Console.WriteLine(clone.Data);     // 输出：原型数据
        // 修改克隆对象不会影响原型对象（浅拷贝下引用类型需注意）
        clone.Data = "克隆数据";
        Console.WriteLine(prototype.Data); // 输出：原型数据
        Console.WriteLine(clone.Data);     // 输出：克隆数据
    }
}
```

---

## 优缺点

**优点：**
- 可以在运行时动态创建对象，无需关心具体类型。
- 创建新对象更高效，尤其是复杂对象。
- 简化对象的创建过程。

**缺点：**
- 克隆过程需要考虑深拷贝与浅拷贝，复杂对象需特别注意。
- 对象中存在复杂引用关系时，克隆实现较为困难。

---

## 应用场景

- 需要大量相似对象时。
- 创建对象成本较高（如初始化耗时或资源昂贵）。
- 希望运行时决定实例化哪些类。
