# 桥接模式（Bridge Pattern）

## 概念简介

桥接模式是一种结构型设计模式。它将抽象部分与实现部分分离，使它们都可以独立地变化。桥接模式通过组合的方式建立两个类层次结构之间的桥梁，从而实现解耦，通常用于处理系统中多维度变化的场景。

---

## 结构图

```
        抽象化（Abstraction）
              ▲
              │
      ┌───────┴────────┐
      │                │
扩展抽象化    ──►  实现化接口（Implementor）
      │                │
      └─────┬───────┬──┘
            │       │
     具体实现A  具体实现B
```

---

## 代码示例（C#）

```csharp
// 实现化接口
public interface IImplementor
{
    void OperationImpl();
}

// 具体实现A
public class ConcreteImplementorA : IImplementor
{
    public void OperationImpl()
    {
        Console.WriteLine("具体实现A的方法执行");
    }
}

// 具体实现B
public class ConcreteImplementorB : IImplementor
{
    public void OperationImpl()
    {
        Console.WriteLine("具体实现B的方法执行");
    }
}

// 抽象化
public abstract class Abstraction
{
    protected IImplementor implementor;

    public Abstraction(IImplementor implementor)
    {
        this.implementor = implementor;
    }

    public abstract void Operation();
}

// 扩展抽象化
public class RefinedAbstraction : Abstraction
{
    public RefinedAbstraction(IImplementor implementor) : base(implementor) { }

    public override void Operation()
    {
        Console.WriteLine("扩展抽象化操作：");
        implementor.OperationImpl();
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        IImplementor implA = new ConcreteImplementorA();
        Abstraction abstractionA = new RefinedAbstraction(implA);
        abstractionA.Operation();

        IImplementor implB = new ConcreteImplementorB();
        Abstraction abstractionB = new RefinedAbstraction(implB);
        abstractionB.Operation();
    }
}
```

---

## 优缺点

**优点：**
- 实现了抽象与实现的分离，解耦多维度变化。
- 扩展性强，可以独立扩展抽象层和实现层。
- 符合开闭原则。

**缺点：**
- 增加了系统结构的复杂性。
- 对于单一维度变化的系统不适合使用。

---

## 应用场景

- 系统存在多维度变化（如功能变化和平台变化），需要解耦。
- 不希望在抽象和实现之间有静态继承关系。
- 希望通过组合替代继承以减少类层次的数量。
