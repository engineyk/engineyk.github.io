- 镜面反射光(<span style="color:#F3F300">黄线</span>): 直接从表面反射出去，不进入物体内部。
- 漫反射光(<span style="color:red">红线</span>): 折射的光和颗粒碰撞后，从物体表面射出的光。
- 金属(Metallic)材质会立即吸收所有折射光，故而金属只有镜面反射，而没有折射光引起的漫反射。

```c++

float kD = 1.0 - kS;                        // 折射/漫反射部分
```

## 1.2 Energy Conservation 能量守恒
- 入射光的总能量 ≤ 出射光的总能量
- 最终颜色 = 漫反射 + 镜面反射 + GI <= 1
- 光能传播过程中的能量守衡体现在三个方面。
  - 一个对象反射出来的光照信息，不可能超过它接受到的信息。也就是说，全反射是一个物体的极限。
  - 一个物体越光亮，那么它的颜色信息应该越少。（refection和diffuse是一个插值关系）

    ```c++
    oneMinusReflectivity = 1 - lerp(dielectricSpec, 0, metallic)
    specColor = lerp(dielectricSpec, albedo, metallic)
    diffColor = albedo * (oneMinusReflectivity)
    ```

  - 一个物体越平滑，那么它的高亮点会越小，越亮。

## 1.3 Metallic-Roughness 工作流

| 参数                   | 范围         | 描述                                         |
| ---------------------- | ------------ | -------------------------------------------- |
| **BaseColor (Albedo)** | sRGB 色值    | 非金属：漫反射颜色；金属：F0 反射色          |
| **Metallic**           | [0, 1]       | 0 = 电介质（非金属），1 = 导体（金属）       |
| **Roughness**          | [0, 1]       | 0 = 完全光滑（镜面），1 = 完全粗糙（漫反射） |
| **AO**                 | [0, 1]       | 环境光遮蔽，模拟间接光被遮挡的程度           |
| **Normal**             | 切线空间法线 | 微表面法线扰动                               |

**Metallic 对材质的影响：**

```c++
// F0 derivation
vec3 F0 = vec3(0.04);                  // dielectric base reflectance
F0 = mix(F0, albedo, metallic);        // metal uses albedo as F0

// diffuse is zeroed out for metals
kD *= 1.0 - metallic;
```

- 非金属（metallic=0）：F0 ≈ 0.04，漫反射 = albedo
- 金属（metallic=1）：F0 = albedo，漫反射 = 0（金属吸收所有折射光）

## 1.4 Common Pitfalls

1. **Linear vs sRGB**：PBR 计算必须在线性空间，最终输出做 Gamma 校正
2. **Energy Conservation**：确保 kD + kS ≤ 1，金属 kD = 0
3. **Roughness Remapping**：`α = roughness²`（Disney/UE4 convention）
4. **Divide by Zero**：分母加 epsilon（如 0.001）防止除零
5. **HDR + Tone Mapping**：PBR 输出可能超过 [0,1]，需要 Tone Mapping（如 ACES）

```c++
// ACES Tone Mapping
vec3 ACESFilm(vec3 x)
{
    float a = 2.51;
    float b = 0.03;
    float c = 2.43;
    float d = 0.59;
    float e = 0.14;
    return clamp((x * (a * x + b)) / (x * (c * x + d) + e), 0.0, 1.0);
}

// Gamma Correction
color = pow(color, vec3(1.0 / 2.2));
```

## 2.1 BxDF

- **BRDF**（Bidirectional Reflectance Distribution Function，双向反射分布函数）
- 基于物理的BRDF（Use a physically based BRDF)

### 2.1.1 概念

$$
f(l, v) = \frac{dL_o(v)}{dE(l)}
$$

其中：
- \( f(l, v) \)：BRDF函数，表示从入射方向 \( l \) 到出射方向 \( v \) 的反射属性
- \( dL_o(v) \)：**出射方向** \( v \) 上的辐射亮度微分
- \( dE(l) \)：**入射方向** \( l \) 上的照度微分

（1）概念理解
- 入射光 l 照到物体表面上，反射光线为v，那么反射光的亮度（辐射率）和入射光的能量(辐照度)会成一个比例，而这个比例就是BDRF。
- 可以理解为，在某一个特定的角度观看某个点时，各个方向的入射光对该点的最终光亮度产生的贡献比例，其实它就是一些零点几的小数。
  
（2）为什么是辐射率和辐射度微分的比值，而不是辐射率和辐射率，或者辐照度和辐照度的微分比值？
见参考资料的两种解释，比较好理解的是从数学角度来说，只有用Irradiance的微分，这个比值才会有意义，否则直接用出射光的dLo比上入射光的L，会趋于0，如果是dE，则是Li * cosθ * dw，也是趋于0，是有意义的。

- 描述物体表面如何反射光线的方程
- 表示了当给定一条入射光的时候，某一条特定的出射光线的性质是怎么样的。
- 它的精确定义是**出射光辐射率**(Radiance)的微分和**入射光辐照度**(Irradiance)的微分之比。


### 2.1.2 公式

$$
L_o(p, \omega_o) = \int_\Omega \left( k_d \frac{c}{\pi} + \frac{DFG}{4(\omega_o \cdot \mathbf{n})(\omega_i \cdot \mathbf{n})} \right) L_i(p, \omega_i) (\mathbf{n} \cdot \omega_i) d\omega_i
$$

- 左侧 \(L_o(p, \omega_o)\)：表示在表面点 p 出射方向 ωₒ 的出射辐亮度。
- 积分区域 \(\Omega\)：对所有可能的入射方向（单位半球）积分。
- \(k_d \frac{c}{\pi}\)：表示朗伯漫反射模型，对应表面的漫反射部分。
  - kd： 为漫反射系数
  - c：物体颜色Albedo
- \(\frac{DFG}{4(\omega_o \cdot \mathbf{n})(\omega_i \cdot \mathbf{n})}\)：Cook-Torrance模型描述的高光反射部分，DFG代表微面元分布、菲涅尔效应和几何遮蔽项的组合。
- \(L_i(p, \omega_i)\)：\(L_i(p, l)\)表示点 p 在入射方向 ωᵢ 上的辐入亮度。
  - 环境光、全局光、由光照系统、材质BSDF、GI、自动计算，包括环境贴图、ShadowMap采样、反射探针等等
  - \(p\)，空间点p
  - \(\omega_i\)，点光源、环境光的出射幅亮度
- \((\mathbf{n} \cdot \omega_i)\)：\((\mathbf{n} \cdot l)\)：法线与入射方向的夹角余弦，用于修正能量贡献。
- \(d\omega_i\)：对单位半球微分方向积分。
![alt text](https://res.cloudinary.com/dyjxpjxro/image/upload/v1775053988/灯光方向_gycrob.png)


$$
f_r = k_d f_{lambert} + k_s f_{cook-torrance}
$$
- \(k_s \)：入射光被镜面反射比例
- \(k_d \)：入射光中被折射的比例
  ```c++
  float kS = calculateSpecularComponent(...); // 反射/镜面部分
  float kD = 1.0 - kS;                        // 折射/漫反射部分
  ```
- \(\omega_i \)：入射光线lightDir
- \(\omega_o \)：出射光线viewDir
-  \(f_{lambert}\)：**漫反射** 伦勃朗漫反射（Lambertian Diffuse） \(f_{lambert} = \frac{c}{\pi}\)
   - \(c\)： Albedo、表面颜色、漫反射纹理
- \(f_\text{cook-torrance}\)：**镜面反射** \(f_\text{cook-torrance} = \frac{DFG}{4(\omega_o \cdot \mathbf{n})(\omega_i \cdot \mathbf{n})}\)

  - \(Cook-Torrance(\mathbf{v}, \mathbf{l}, \mathbf{n}) = \frac{D(\mathbf{h}) \cdot G(\mathbf{v}, \mathbf{l}, \mathbf{n}) \cdot F(\mathbf{v}, \mathbf{l}, \mathbf{n})}{4 \cdot (\mathbf{v} \cdot \mathbf{n}) \cdot (\mathbf{l} \cdot \mathbf{n})}\)

### 2.1.5 Cook-Torrance

```c++
float3 BRDF(float3 v, float3 l, float3 n, float3 f0, float roughness) {
    float h  = normalize(l + v); // 光源和视角的中间
    float nh  = max(dot(n, h), 0.0);
    float nl  = max(dot(n, v), 0.0);
    // 计算法线分布函数
    float D = D_GGX(n, h, roughness);
    // 计算菲涅尔函数
    float3 F = F_Schlick(v, h, f0);
    // 计算几何遮挡函数
    float G = G_SchlickGGX(n, v, l, roughness);
    // 计算BRDF
    float3 brdf = (D * G * F) / (4.0 * nv * nl);
    return brdf;
}
```

$$
Cook-Torrance(\mathbf{v}, \mathbf{l}, \mathbf{n}) = \frac{D(\mathbf{h}) \cdot F(\mathbf{v}, \mathbf{h}) \cdot G(\mathbf{v}, \mathbf{l}, \mathbf{n})}{4 \cdot (\mathbf{n} \cdot \mathbf{v}) \cdot (\mathbf{n} \cdot \mathbf{l})}
$$

#### 2.3.1.1 D - NDF (Normal Distribution Function，法线分布函数)

> NDF (Normal Distribution Function，法线分布函数) Trowbridge-Reitz GGX

$$
D_{\text{GGX（Trowbridge-Reitz）}}(\mathbf{n}, \mathbf{h}, \mathbf{a}) = \frac{\alpha^2}{\pi \left((n*h)^2 (\alpha^2 - 1) + 1\right)^2}
$$

- $$ \alpha $$  是粗糙度参数，决定了表面的光泽度。
  - <img src="https://res.cloudinary.com/dyjxpjxro/image/upload/v1775053988/NDF_rcdnfu.png" alt="Logo" width="232" height="56">
  - 粗糙度改变高光分布，比较强的质感，缺乏立体感，缺乏遮挡V
- 粗糙度低时，h取向在微平面集中在小的半径范围内
- 粗糙度高时，h取向在方向随机

``` c++
// Trowbridge-Reitz GGX
// 粗糙的物体a=1：1
// 光滑的物体a=0：无穷小
// Epic GLSL
float DistributionGGX(vec3 N, vec3 H, float roughness)
{
    float a   = roughness * roughness; // a:粗糙度
    float a2  = a * a;
    float nh  = max(dot(N, H), 0.0);
    float nh2 = nh*nh;
    float d = (nh2 * (a2 - 1.0) + 1.0);
    return a2 / (PI * d * d);
}
```

```c++
// GGX（Trowbridge-Reitz）
float D_GGX(float a2, float nh) // a:粗糙度
{
  float d = (nh * a2 - nh) * nh + 1;  // 2 mad
  return a2 / (PI * d * d);           // 4 mul, 1 rcp
}
```

#### 2.3.1.2 G - Visibility（遮挡）Smith's Schlick-GGX

>GGX和Schlick-Beckmann组合而成的模拟函数Schlick-GGX
>Smith，考虑两个视角，实现方向和光线方向，使用GeometrySmith

从几何学上统计相互遮蔽的概率，消耗光线能量。
描述阴影的属性，当微平面粗糙时，微平面可能会挡住其他微平面，从而减少反射的光线。

$$
{SchlickGGX}(\mathbf{N}, \mathbf{v}, k) = \frac{\mathbf{n} \cdot \mathbf{v}}{(\mathbf{n} \cdot \mathbf{v})(1 - k) + k}
$$

<!--
- n 是表面法线。
- v 是方向向量（可以是观察方向或光线方向）。
$$
k = \left(\frac{\alpha + 1}{8}\right)^2
$$
-->

**系数K**

$$
k = \frac{(a + 1)^2}{8}
$$

- 如果用 Disney/Unreal 公式，则 $$ k \in [0.125, 0.25] $$

| 材质     | a   | k                  | G                 | 结果值域      |
| -------- | --- | ------------------ | ----------------- | ------------- |
| 光滑物体 | 0   | 值小：0.125（1/8） | 1 （n*v / n*v）） | 无遮挡，接近0 |
| 粗糙物体 | 1   | 值大：0.5（4/8）   | nv （n*v / 1））  | 有遮挡，接近1 |

- α : 粗糙度参数，用于控制遮挡效果。
- 若 k ∈ [0, 1]，则 G ≈ [1, n*v]
  - 光滑物体，k值小，G值为1，无遮挡
  - 粗糙物体，k值大，G值为n*v，有遮挡

- 若简化实现，则 \( k \in [0, 0.5] \)
$$
k = \frac{a^2}{2}
$$

> 同时考虑两个视角，视线方向（几何遮挡）跟光线方向（几何阴影）

$$
G(n, v, l, k) = G_{sub}(n, v, k) G_{sub}(n, l, k)
$$

```c++
float GeometrySmith(vec3 N, vec3 V, vec3 L, float roughness) {
  float nv = max(dot(N, V), 0.0);
  float nl = max(dot(N, L), 0.0);
  float ggx1 = SchlickGGX(nl, roughness); // 几何阴影
  float ggx2 = SchlickGGX(nv, roughness); // 几何遮挡
  return ggx1 * ggx2;
}

float SchlickGGX(float nv, float roughness) {
  float r = roughness + 1.0;
  float k = (r * r) / 8.0;
  float denom = nv * (1.0 - k) + k;
  return nv / denom;
}
```
#### 2.3.1.3 F - Fresnel（菲涅尔）-Schlick近似法(Approximation)

```c++
// 菲涅尔 F（Schlick近似）
// 纯金属F0为1, F0 + 0 * ..., 只计算F0
// 非金属F0为0，0 + pow(1-vh, 5)，不计算F0，只有pow(vh, 5)
// VdotH为0的时候反射最高，
// vec3 F0 = mix(vec3(0.04), baseColor, metallic); // 非金属用0.04，金属用baseColor

float3 F_Schlick(float3 F0, float vh)
{
    return F0 + (1.0 - F0) * pow(1.0 - vh, 5.0);
}
```

$$
F_{Schlick}(h,v,F_0) = F_0 + (1 - F_0)(1 - (h \cdot v))^5
$$

- 不同角度有不同强度的反射光
- 水的材质现象,材质表面对于光线的反射
  - N和V的角度
  - **正交时**当光线垂直入射的时候（沿着法线入射的时候）是最弱的，
  - **非垂直**而当光线与法线的夹角逐渐增大，则反射也会增强。
- 类似物理碰撞，如光线是子弹，如水是钢盔
- 如果是正面，冲撞最大，直接穿透。
- 如果是侧面，冲撞最小，会反弹。
- 纯金属的反射率F0为其颜色，纯金属的颜色因为反射率不同造成
- 非金属的反射率F0为其明暗，非金属的颜色因为不同波长的光的吸收率不同造成

- 正交时,夹角为90°, 视线V和法线夹角为0时，h*v=1，代入公式，二项式为0，值为F0
- 非垂直,夹角<90°, 视线V和法线夹角为1时，h*v=0，代入公式，二项式为0，值为0

### 2.1.6 DualSpecularGGX

> BRDF1 的增强版，主要用于 PC / 主机端高品质渲染。
> 单个 GGX Lobe 无法准确模拟某些材质（如皮肤、车漆、涂层材质）的双层高光现象，Dual Specular 通过叠加两个不同粗糙度的 GGX 分布来解决这个问题。

DualSpecularGGX 在 UE4/UE5 中主要用于：
- 皮肤（Skin） — 表皮层（窄高光）+ 油脂层（宽高光）
- 车漆（Car Paint） — 清漆层（锐利反射）+ 底漆层（模糊反射）
- 涂层材质（Clear Coat） — 类似双层结构
- 眼球（Eye） — 角膜层 + 虹膜层

```c++
// Lobe0Alpha2：较窄的高光（AverageToRoughness0 控制）
// Lobe1Alpha2：较宽的高光（AverageToRoughness1 控制）
// DUAL_SPECULAR.z：两个 Lobe 的混合权重
vec3 DualSpecularGGX(float NoL, float NoV, float NoH, float LoH, float AverageRoughness, vec3 SpecularColor)
{
  //AverageRoughness = GBuffer.Roughness
  float AverageToRoughness0 = DUAL_SPECULAR.x * 2.0;
  float AverageToRoughness1 = DUAL_SPECULAR.y * 2.0;

  float Lobe0Roughness = max(clamp((AverageRoughness * AverageToRoughness0), 0.0, 1.0), 0.02);
  float Lobe1Roughness = clamp((AverageRoughness * AverageToRoughness1), 0.0, 1.0);

  float AverageAlpha2 = pow((AverageRoughness), 2.0);
  float Lobe0Alpha2 = pow((Lobe0Roughness), 2.0);
  float Lobe1Alpha2 = pow((Lobe1Roughness), 2.0);

  // Generalized microfacet specular
  float D = mix(GGXTermNotNormalized(NoH, Lobe0Alpha2), GGXTermNotNormalized(NoH, Lobe1Alpha2), DUAL_SPECULAR.z);
  float Vis = SmithJointGGXVisibilityTerm(NoL, NoV, AverageAlpha2); // Average visibility well approximates using two separate ones (one per lobe).
  vec3 F = FresnelTerm(SpecularColor, LoH);
  return (D * Vis) * F;
}

float GGXTermNotNormalized (float NdotH, float roughness)
{
    float a2 = roughness * roughness;
    float d = (NdotH * a2 - NdotH) * NdotH + 1.0; // 2 mad
    return min(a2 / (d * d + 1e-5), 128.0);
    // This function is not intended to be running on Mobile,
    // therefore epsilon is smaller than what can be represented by half
}
float SmithJointGGXVisibilityTerm(float nl, float nv, float roughness)
{
    float a = roughness;
    float lambdaV = nl * (nv * (1.0 - a) + a);
    float lambdaL = nv * (nl * (1.0 - a) + a);
    return min(0.5 / (lambdaV + lambdaL + 1e-5), 128.0);
}

vec3 FresnelTerm (vec3 F0, float cosA)
{
    float t = Pow5 (1.0 - cosA);   // ala Schlick interpoliation
    return F0 + (1-F0) * t;
}
```

### 2.1.7 BRDF2

> Minimalist CookTorrance BRDF — 移动端/中端 GPU 常用

- NDF: Modified GGX 或 BlinnPhong
- Visibility: Modified Kelemen-Szirmay-Kalos（简化 Smith）
- Fresnel: 用 `1/LdotH` 近似，省去 pow5 计算

```c++
// Based on Minimalist CookTorrance BRDF
// Implementation is slightly different from original derivation: http://www.thetenthplanet.de/archives/255
//
// * NDF (depending on UNITY_BRDF_GGX):
//  a) BlinnPhong
//  b) [Modified] GGX
// * Modified Kelemen and Szirmay-​Kalos for Visibility term
// * Fresnel approximated with 1/LdotH

// Simplified Visibility (Kelemen)
float V_Kelemen(float LoH)
{
    return 0.25 / (LoH * LoH + 1e-5);
}

// Simplified Fresnel
vec3 F_Approx(vec3 F0, float LoH)
{
    float t = pow(1.0 - LoH, 5.0);
    return F0 + (1.0 - F0) * t;
}

// BRDF2 final
vec3 BRDF2(vec3 N, vec3 V, vec3 L, vec3 F0, float roughness)
{
    vec3 H = normalize(V + L);
    float NoH = max(dot(N, H), 0.0);
    float LoH = max(dot(L, H), 0.0);
    float NoL = max(dot(N, L), 0.0);

    float D = DistributionGGX(N, H, roughness);
    float Vis = V_Kelemen(LoH);
    vec3  F = F_Approx(F0, LoH);

    return D * Vis * F;
}
```

### 2.1.8 BRDF3

**使用LookUp Texture**

**BRDF LUT（查找表）**

> X 轴通常为粗糙度（roughness）
> Y 轴为 N·V（法线与视线夹角）。
> LUT 存储预积分的 Scale 和 Bias 两个值（RG 通道）

BRDF LUT 的核心思想是将 Cook-Torrance BRDF 的镜面反射积分拆分为两部分：

$$
\int_\Omega f_r(l, v) \cos\theta_l \, d\omega_l \approx F_0 \cdot \text{Scale}(NdotV, roughness) + \text{Bias}(NdotV, roughness)
$$

- **Scale (R通道)**：F0 的缩放系数
- **Bias (G通道)**：F0 的偏移量

```c++
// Google Filamen
// 以 Filament brdf.cpp 为例
for (int y = 0; y < size; ++y) {       // Y = NdotV
    float NdotV = float(y + 0.5) / size;
    for (int x = 0; x < size; ++x) {   // X = roughness
        float roughness = float(x + 0.5) / size;

        // 进行多次蒙特卡洛采样
        float2 brdf = integrateBRDF(NdotV, roughness, sampleCount);
        // 存储到贴图，RG通道
        data[y * size + x] = brdf;
    }
}
```

## 2.2 IBL（Image-Based Lighting）基于图像的光照

> IBL 使用环境贴图（CubeMap）作为光源，提供间接光照（漫反射 + 镜面反射）

### 2.2.1 漫反射 IBL（Irradiance Map）

对环境贴图在半球上做余弦加权积分，预计算为 Irradiance CubeMap：

$$
L_{diffuse} = k_d \cdot \frac{albedo}{\pi} \cdot \int_\Omega L_i(\omega_i)(n \cdot \omega_i) \, d\omega_i
$$

```c++
// Sample irradiance map for diffuse IBL
vec3 irradiance = texture(irradianceMap, N).rgb;
vec3 diffuseIBL = kD * albedo * irradiance;
```
### 2.2.2 镜面反射 IBL（Pre-filtered Environment Map + BRDF LUT）

**Split-Sum Approximation（Epic Games）：**

$$
L_{specular} \approx \underbrace{L_{prefilt}(R, roughness)}_{\text{Pre-filtered Map}} \cdot \underbrace{(F_0 \cdot Scale + Bias)}_{\text{BRDF LUT}}
$$

- **Pre-filtered Environment Map**：按不同粗糙度级别（mip level）预卷积环境贴图
- **BRDF LUT**：预积分的菲涅尔缩放和偏移（见第5章）

```c++
// Specular IBL
vec3 R = reflect(-V, N);
float lod = roughness * MAX_REFLECTION_LOD;
vec3 prefilteredColor = textureLod(prefilterMap, R, lod).rgb;
vec2 envBRDF = texture(brdfLUT, vec2(NdotV, roughness)).rg;
vec3 specularIBL = prefilteredColor * (F0 * envBRDF.x + envBRDF.y);
```

### 2.2.3 IBL 完整合成

```c++
// Combine direct + indirect lighting
vec3 F = fresnelSchlickRoughness(NdotV, F0, roughness);
vec3 kS = F;
vec3 kD = (1.0 - kS) * (1.0 - metallic);

// Indirect diffuse
vec3 irradiance = texture(irradianceMap, N).rgb;
vec3 diffuseIBL = kD * albedo * irradiance;

// Indirect specular
vec3 R = reflect(-V, N);
vec3 prefilteredColor = textureLod(prefilterMap, R, roughness * MAX_REFLECTION_LOD).rgb;
vec2 envBRDF = texture(brdfLUT, vec2(NdotV, roughness)).rg;
vec3 specularIBL = prefilteredColor * (F0 * envBRDF.x + envBRDF.y);

// Final
vec3 ambient = (diffuseIBL + specularIBL) * ao;
vec3 color = ambient + Lo;  // Lo = direct lighting

// Fresnel with roughness correction for IBL
vec3 fresnelSchlickRoughness(float cosTheta, vec3 F0, float roughness)
{
    return F0 + (max(vec3(1.0 - roughness), F0) - F0) * pow(1.0 - cosTheta, 5.0);
}
```

```c++
vec3 fresnelSchlick(float cosTheta, vec3 F0)
{
  return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
} 

float DistributionGGX(vec3 N, vec3 H, float roughness)
{
  float a      = roughness*roughness;
  float a2     = a*a;
  float NdotH  = max(dot(N, H), 0.0);
  float NdotH2 = NdotH*NdotH;

  float num   = a2;
  float denom = (NdotH2 * (a2 - 1.0) + 1.0);
  denom = PI * denom * denom;

  return num / denom;
}

float GeometrySchlickGGX(float NdotV, float roughness)
{
  float r = (roughness + 1.0);
  float k = (r*r) / 8.0;

  float num   = NdotV;
  float denom = NdotV * (1.0 - k) + k;

  return num / denom;
}

float GeometrySmith(vec3 N, vec3 V, vec3 L, float roughness)
{
  float NdotV = max(dot(N, V), 0.0);
  float NdotL = max(dot(N, L), 0.0);
  float ggx2  = GeometrySchlickGGX(NdotV, roughness);
  float ggx1  = GeometrySchlickGGX(NdotL, roughness);

  return ggx1 * ggx2;
}
```
