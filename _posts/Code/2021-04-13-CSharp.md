## 一、类型系统

### 1.1 值类型 vs 引用类型

| 维度          | 值类型 (Value Type)                      | 引用类型 (Reference Type)                           |
| ------------- | ---------------------------------------- | --------------------------------------------------- |
| **存储位置**  | 栈 (Stack) 或内联在包含对象中            | 堆 (Heap)，栈上存引用（指针）                       |
| **赋值语义**  | 深拷贝（复制全部数据）                   | 浅拷贝（复制引用，共享数据）                        |
| **默认值**    | 0 / false / '\0'                         | null                                                |
| **继承**      | 隐式继承 `System.ValueType`              | 继承 `System.Object`                                |
| **代表类型**  | `int`, `float`, `bool`, `struct`, `enum` | `class`, `string`, `array`, `delegate`, `interface` |
| **装箱/拆箱** | 值类型 → object 时发生装箱               | 不涉及                                              |
| **GC 压力**   | 栈上分配无 GC 压力                       | 堆上分配有 GC 压力                                  |

#### 面试题：struct 和 class 的区别？

```csharp
// struct: value type, stack allocation, no inheritance, no default constructor override
struct Vector3
{
    public float x, y, z;
    
    // struct cannot have parameterless constructor (before C# 10)
    public Vector3(float x, float y, float z)
    {
        this.x = x;
        this.y = y;
        this.z = z;
    }
}

// class: reference type, heap allocation, supports inheritance
class Transform
{
    public Vector3 Position { get; set; }
    public Vector3 Rotation { get; set; }
}
```

**关键区别总结：**

| 特性         | struct                        | class              |
| ------------ | ----------------------------- | ------------------ |
| 类型         | 值类型                        | 引用类型           |
| 继承         | 不支持（可实现接口）          | 支持               |
| 默认构造函数 | 不可自定义（C# 10 前）        | 可自定义           |
| null         | 不可为 null（除非 Nullable）  | 可为 null          |
| 适用场景     | 小型、不可变数据（≤16 bytes） | 复杂对象、需要多态 |

#### 面试题：装箱和拆箱的性能影响？

```csharp
// Boxing: value type -> heap allocation + copy
int value = 42;
object boxed = value;  // Boxing: allocate on heap, copy value

// Unboxing: type check + copy back to stack
int unboxed = (int)boxed;  // Unboxing

// Performance trap: ArrayList causes boxing for value types
ArrayList list = new ArrayList();
for (int i = 0; i < 10000; i++)
{
    list.Add(i);  // Boxing on every iteration! 10000 heap allocations
}

// Solution: use generic List<T>
List<int> genericList = new List<int>();
for (int i = 0; i < 10000; i++)
{
    genericList.Add(i);  // No boxing, stored as int directly
}
```

**装箱开销：**
- 堆上分配内存（对象头 + 方法表指针 + 数据）
- 复制值到堆
- 返回引用
- 增加 GC 压力
- 单次装箱约 **20-30ns**，大量装箱会严重影响性能

### 1.2 string 的不可变性

```csharp
// string is immutable reference type
string a = "Hello";
string b = a;        // b points to same string object
a += " World";       // creates NEW string object, a points to new one
// b still points to "Hello"

// String interning: compile-time string literals share same reference
string s1 = "Hello";
string s2 = "Hello";
bool same = object.ReferenceEquals(s1, s2);  // true! (interned)

// Performance: string concatenation in loop
// BAD: O(n^2) time, n allocations
string result = "";
for (int i = 0; i < 1000; i++)
{
    result += i.ToString();  // new string object every iteration
}

// GOOD: O(n) time, minimal allocations
var sb = new StringBuilder();
for (int i = 0; i < 1000; i++)
{
    sb.Append(i);
}
string result2 = sb.ToString();
```

#### 面试题：string 是值类型还是引用类型？为什么表现得像值类型？

**答：** string 是引用类型，但因为**不可变性 (Immutability)** 表现得像值类型：
- 任何修改操作都返回新对象
- `==` 运算符被重载为比较内容而非引用
- 字符串驻留 (String Interning) 使相同字面量共享引用

## 二、垃圾回收 (GC)

### 2.1 GC 分代模型

```
┌─────────────────────────────────────────────────┐
│                  Managed Heap                   │
├──────────┬──────────────┬───────────────────────┤
│  Gen 0   │    Gen 1     │        Gen 2          │
│ (~256KB) │  (~2MB)      │    (variable)         │
│          │              │                       │
│ new obj  │ survived     │ long-lived objects    │
│ short    │ Gen0 once    │ survived Gen1         │
│ lived    │              │ static fields         │
│          │              │ large objects (>85KB) │
├──────────┴──────────────┴───────────────────────┤
│         Large Object Heap (LOH)                 │
│         objects >= 85,000 bytes                 │
│         only collected with Gen 2               │
│         NOT compacted by default                │
└─────────────────────────────────────────────────┘
```

**GC 触发条件：**
1. Gen 0 分配阈值达到（~256KB）
2. 显式调用 `GC.Collect()`
3. 系统内存不足
4. AppDomain 卸载

**GC 过程：**
1. **Mark（标记）：** 从 GC Root 出发，标记所有可达对象
2. **Sweep（清除）：** 回收未标记对象的内存
3. **Compact（压缩）：** 移动存活对象，消除内存碎片（LOH 默认不压缩）

**GC Root 包括：**
- 栈上的局部变量和参数
- 静态字段
- GC Handle（GCHandle.Alloc）
- Finalizer 队列中的对象

### 2.2 GC 对游戏的影响

```csharp
// Unity specific: GC.Alloc in Update is the #1 performance killer

// BAD: allocates every frame
void Update()
{
    // string concatenation: 2 allocations per frame
    string info = "HP: " + health.ToString();
    
    // LINQ: allocates enumerator + closure
    var enemies = allUnits.Where(u => u.IsEnemy).ToList();
    
    // new array every frame
    var hits = Physics.RaycastAll(ray);
}

// GOOD: zero allocation patterns
private StringBuilder _sb = new StringBuilder(64);
private List<Unit> _enemyCache = new List<Unit>(32);
private RaycastHit[] _hitBuffer = new RaycastHit[16];

void Update()
{
    // reuse StringBuilder
    _sb.Clear();
    _sb.Append("HP: ").Append(health);
    
    // manual loop instead of LINQ
    _enemyCache.Clear();
    for (int i = 0; i < allUnits.Count; i++)
    {
        if (allUnits[i].IsEnemy)
            _enemyCache.Add(allUnits[i]);
    }
    
    // NonAlloc version
    int hitCount = Physics.RaycastNonAlloc(ray, _hitBuffer);
}
```

#### 面试题：如何减少 GC 压力？

1. **对象池 (Object Pool)：** 复用对象而非频繁 new/destroy
2. **避免装箱：** 使用泛型集合代替非泛型集合
3. **缓存复用：** 预分配 buffer，避免每帧分配
4. **避免闭包捕获：** Lambda 捕获外部变量会生成堆对象
5. **struct 代替 class：** 小型数据用 struct 避免堆分配
6. **StringBuiler：** 替代字符串拼接
7. **NonAlloc API：** Unity 中使用 `RaycastNonAlloc` 等

### 2.3 Finalizer 与 IDisposable

```csharp
// Standard Dispose Pattern
class ResourceHolder : IDisposable
{
    private IntPtr _nativeHandle;
    private ManagedResource _managed;
    private bool _disposed = false;

    // Destructor / Finalizer: called by GC, non-deterministic
    ~ResourceHolder()
    {
        Dispose(false);
    }

    // IDisposable: called explicitly, deterministic
    public void Dispose()
    {
        Dispose(true);
        GC.SuppressFinalize(this);  // prevent finalizer from running
    }

    protected virtual void Dispose(bool disposing)
    {
        if (_disposed) return;
        
        if (disposing)
        {
            // Release managed resources
            _managed?.Dispose();
        }
        
        // Release unmanaged resources
        if (_nativeHandle != IntPtr.Zero)
        {
            NativeAPI.Release(_nativeHandle);
            _nativeHandle = IntPtr.Zero;
        }
        
        _disposed = true;
    }
}

// Usage with using statement
using (var holder = new ResourceHolder())
{
    // use holder...
}  // Dispose() called automatically here
```

#### 面试题：Finalizer 的问题？

- Finalizer 使对象至少多存活一代（进入 Finalization Queue → Freachable Queue）
- 执行时机不确定，由专门的 Finalizer 线程执行
- 执行顺序不确定
- 会显著增加 GC 开销
- **最佳实践：** 实现 `IDisposable`，用 `using` 语句确保及时释放

## 三、委托与事件

### 3.1 委托 (Delegate)

```csharp
// Delegate is a type-safe function pointer
delegate int MathOperation(int a, int b);

// Built-in delegate types
// Action: void return
Action<int, string> onLog;           // void Method(int, string)
// Func: with return value
Func<int, int, int> calculate;       // int Method(int, int)
// Predicate: returns bool
Predicate<int> isPositive;           // bool Method(int)

class DelegateExample
{
    // Multicast delegate
    public void Demo()
    {
        MathOperation op = Add;
        op += Multiply;  // multicast: both will be called
        
        int result = op(3, 4);  // result = 12 (last return value)
        
        op -= Add;  // remove Add from invocation list
    }
    
    static int Add(int a, int b) => a + b;
    static int Multiply(int a, int b) => a * b;
}
```

### 3.2 事件 (Event)

```csharp
// Event = restricted delegate (can only += / -= from outside)
class EventSystem
{
    // Event declaration
    public event Action<int> OnHealthChanged;
    public event Action OnDeath;
    
    private int _health = 100;
    
    public void TakeDamage(int damage)
    {
        _health -= damage;
        
        // Raise event (null check with ?. operator)
        OnHealthChanged?.Invoke(_health);
        
        if (_health <= 0)
        {
            OnDeath?.Invoke();
        }
    }
}

class UIManager
{
    private EventSystem _eventSystem;
    
    public void Subscribe()
    {
        _eventSystem.OnHealthChanged += UpdateHealthBar;
        _eventSystem.OnDeath += ShowGameOver;
    }
    
    public void Unsubscribe()
    {
        // IMPORTANT: always unsubscribe to prevent memory leaks
        _eventSystem.OnHealthChanged -= UpdateHealthBar;
        _eventSystem.OnDeath -= ShowGameOver;
    }
    
    void UpdateHealthBar(int hp) { /* ... */ }
    void ShowGameOver() { /* ... */ }
}
```

#### 面试题：delegate 和 event 的区别？

| 特性              | delegate           | event                  |
| ----------------- | ------------------ | ---------------------- |
| 外部赋值 `=`      | ✅ 可以             | ❌ 只能 `+=` / `-=`     |
| 外部调用 `Invoke` | ✅ 可以             | ❌ 只能在声明类内部调用 |
| 封装性            | 低（任何人可覆盖） | 高（发布-订阅模式）    |
| 用途              | 回调、策略模式     | 观察者模式、事件驱动   |

#### 面试题：事件不取消订阅会怎样？

**内存泄漏！** 事件持有订阅者的引用，导致订阅者无法被 GC 回收。在 Unity 中，`OnDestroy` 时必须取消订阅。

## 四、泛型 (Generics)

### 4.1 泛型基础

```csharp
// Generic class
class ObjectPool<T> where T : class, new()
{
    private Stack<T> _pool = new Stack<T>();
    
    public T Get()
    {
        return _pool.Count > 0 ? _pool.Pop() : new T();
    }
    
    public void Return(T obj)
    {
        _pool.Push(obj);
    }
}

// Generic method
T Max<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) > 0 ? a : b;
}

// Generic constraints
// where T : struct          // value type
// where T : class           // reference type
// where T : new()           // has parameterless constructor
// where T : BaseClass       // inherits from BaseClass
// where T : IInterface      // implements IInterface
// where T : unmanaged       // unmanaged type (no reference type fields)
```

#### 面试题：C# 泛型 vs Java 泛型？

| 特性           | C# 泛型                  | Java 泛型                    |
| -------------- | ------------------------ | ---------------------------- |
| 实现方式       | **具现化 (Reification)** | **类型擦除 (Type Erasure)**  |
| 运行时类型信息 | 保留                     | 擦除为 Object                |
| 值类型支持     | ✅ 无装箱                 | ❌ 必须装箱 (`List<Integer>`) |
| 性能           | 值类型零开销             | 值类型有装箱开销             |
| 反射           | 可获取泛型参数           | 运行时无法获取               |

### 4.3 协变与逆变

```csharp
// Covariance (out): can use derived type where base is expected
// IEnumerable<out T> is covariant
IEnumerable<string> strings = new List<string> { "a", "b" };
IEnumerable<object> objects = strings;  // OK: string -> object (covariant)

// Contravariance (in): can use base type where derived is expected
// Action<in T> is contravariant
Action<object> objectAction = obj => Console.WriteLine(obj);
Action<string> stringAction = objectAction;  // OK: object -> string (contravariant)

// Custom variance
interface IProducer<out T>  // covariant: T only in output positions
{
    T Produce();
    // void Consume(T item);  // ERROR: T cannot be in input position
}

interface IConsumer<in T>  // contravariant: T only in input positions
{
    void Consume(T item);
    // T Produce();  // ERROR: T cannot be in output position
}
```

### 5.2 Dictionary 内部原理

```
┌───────────────────────────────────────────┐
│           Dictionary<TKey, TValue>        │
│                                           │
│  buckets[] (int array)                    │
│  ┌───┬───┬───┬───┬───┬───┬───┐            │
│  │ 2 │-1 │ 0 │-1 │ 1 │-1 │ 3 │            │
│  └───┴───┴───┴───┴───┴───┴───┘            │
│    ↓           ↓       ↓       ↓          │
│  entries[] (Entry struct array)           │
│  ┌─────────────────────────────────┐      │  
│  │[0] hash=42 key="a" val=1 next=-1│      │
│  │[1] hash=17 key="b" val=2 next=-1│      │
│  │[2] hash=42 key="c" val=3 next=0 │←collision chain
│  │[3] hash=99 key="d" val=4 next=-1│      │
│  └─────────────────────────────────┘      │
│                                           │ 
│  Lookup: hash(key) % buckets.Length       │
│          → bucket index → entry chain     │
└───────────────────────────────────────────┘
```

#### 面试题：Dictionary 的 Key 需要满足什么条件？

1. **必须正确实现 `GetHashCode()`：** 相等的对象必须返回相同的哈希值
2. **必须正确实现 `Equals()`：** 用于解决哈希冲突时的精确比较
3. **Key 应该是不可变的：** 如果 Key 在放入 Dictionary 后被修改，哈希值改变，将无法找到

```csharp
class PlayerKey : IEquatable<PlayerKey>
{
    public readonly int Id;
    public readonly string Name;
    
    public override int GetHashCode()
    {
        // Combine hash codes
        return HashCode.Combine(Id, Name);
    }
    
    public override bool Equals(object obj) => Equals(obj as PlayerKey);
    
    public bool Equals(PlayerKey other)
    {
        if (other is null) return false;
        return Id == other.Id && Name == other.Name;
    }
}
```

```csharp
class ThreadSafeCounter
{
    private int _count = 0;
    private readonly object _lock = new object();
    
    // lock statement (Monitor.Enter / Monitor.Exit)
    public void Increment()
    {
        lock (_lock)
        {
            _count++;  // thread-safe
```

### 6.3 async/await

```csharp
// async/await is a state machine generated by compiler
async Task<string> DownloadDataAsync(string url)
{
    using var client = new HttpClient();
    
    // await: release current thread, resume when complete
    string data = await client.GetStringAsync(url);
    
    // This runs on the captured SynchronizationContext
    // (UI thread in WPF/WinForms, ThreadPool in ASP.NET Core)
    return data.ToUpper();
}

// ConfigureAwait(false): don't capture SynchronizationContext
async Task<string> LibraryMethodAsync(string url)
{
    using var client = new HttpClient();
    
    // Resume on any ThreadPool thread (better performance in libraries)
    string data = await client.GetStringAsync(url).ConfigureAwait(false);
    return data.ToUpper();
}
```

#### 面试题：async/await 的原理？

编译器将 async 方法转换为**状态机 (State Machine)**：

```
async Task Example()
{                           ┌───────────────────┐
    var a = await Step1();  │  State 0: Start   │
                            │  Call Step1()     │
                            │  If complete →    │
    var b = await Step2(a); │  State 1          │
                            │  Call Step2(a)    │
                            │  If complete →    │
    return b;               │  State 2: Done    │
}                           └───────────────────┘
```

- 每个 `await` 是一个状态切换点
- 如果 awaited Task 已完成，同步继续（无线程切换）
- 如果未完成，注册 continuation，释放当前线程
- Task 完成后，在捕获的 SynchronizationContext 上恢复执行

#### 面试题：async void 的问题？

```csharp
// BAD: async void
async void FireAndForget()
{
    await Task.Delay(1000);
    throw new Exception("Unobserved!");  // crashes the process!
}

// GOOD: async Task
async Task ProperAsync()
{
    await Task.Delay(1000);
    throw new Exception("Can be caught by caller");
}

// async void exceptions cannot be caught with try-catch
// They are posted to SynchronizationContext and crash the app
// Only acceptable for event handlers: async void Button_Click(...)
```

### 6.4 Task vs Thread

| 特性     | Thread               | Task                  |
| -------- | -------------------- | --------------------- |
| 抽象级别 | OS 线程              | 工作单元              |
| 创建开销 | 高（~1MB 栈）        | 低（ThreadPool 复用） |
| 返回值   | 无                   | `Task<T>` 支持        |
| 异常传播 | 线程内崩溃           | 可通过 await 传播     |
| 取消     | 手动 flag            | CancellationToken     |
| 组合     | 困难                 | `WhenAll`, `WhenAny`  |
| 适用场景 | 长时间运行的后台工作 | I/O 密集、短时计算    |

### 7.1 延迟执行 vs 立即执行


### 8.2 abstract vs interface

```csharp
// Abstract class: partial implementation, single inheritance
abstract class Weapon
{
    public string Name { get; protected set; }
    public int Damage { get; protected set; }
    
    // Can have implementation
    public virtual void Equip()
    {
        Console.WriteLine($"Equipped {Name}");
    }
    
    // Must be overridden
    public abstract void Attack();
}

// Interface: contract only (before C# 8), multiple inheritance
interface IDamageable
{
    int Health { get; }

}

interface IInteractable
{
    void Interact();
}

// Multiple interface implementation
class Enemy : Weapon, IDamageable, IInteractable  // ERROR: can't inherit 2 classes

{

    public void Interact() { /* ... */ }
}
```

| 特性       | abstract class      | interface         |
| ---------- | ------------------- | ----------------- |
| 多继承     | ❌ 单继承            | ✅ 多实现          |

### 8.3 游戏开发常用设计模式

```csharp
// 1. Singleton (use with caution in Unity)
class GameManager
{
    private static GameManager _instance;
    private static readonly object _lock = new object();
    
    public static GameManager Instance
    {
        get
        {
            if (_instance == null)
            {
                lock (_lock)
                {
                    if (_instance == null)
                        _instance = new GameManager();
                }
            }
            return _instance;
        }
    }
    
    private GameManager() { }
}

// 2. Observer Pattern (event-based)
class EventBus
{
    private static Dictionary<string, Action<object>> _events = new();
    
    public static void Subscribe(string eventName, Action<object> handler)
    {
        if (!_events.ContainsKey(eventName))
            _events[eventName] = null;
        _events[eventName] += handler;
    }
    
    public static void Publish(string eventName, object data = null)
    {
        if (_events.TryGetValue(eventName, out var handler))
            handler?.Invoke(data);
    }
    
    public static void Unsubscribe(string eventName, Action<object> handler)
    {
        if (_events.ContainsKey(eventName))
            _events[eventName] -= handler;
    }
}

// 3. Object Pool
class ObjectPool<T> where T : class, new()
{
    private readonly Stack<T> _pool;
    private readonly Action<T> _onGet;
    private readonly Action<T> _onReturn;
    
    public ObjectPool(int initialSize, Action<T> onGet = null, Action<T> onReturn = null)
    {
        _pool = new Stack<T>(initialSize);
        _onGet = onGet;
        _onReturn = onReturn;
        
        for (int i = 0; i < initialSize; i++)
            _pool.Push(new T());
    }
    
    public T Get()
    {
        T obj = _pool.Count > 0 ? _pool.Pop() : new T();
        _onGet?.Invoke(obj);
        return obj;
    }
    
    public void Return(T obj)
    {
        _onReturn?.Invoke(obj);
        _pool.Push(obj);
    }
}

// 4. State Pattern
abstract class CharacterState
{
    public abstract void Enter();
    public abstract void Update();
    public abstract void Exit();
}

class IdleState : CharacterState
{
    public override void Enter() { /* play idle animation */ }
    public override void Update() { /* check for input */ }
    public override void Exit() { /* cleanup */ }
}

class StateMachine
{
    private CharacterState _currentState;
    
    public void ChangeState(CharacterState newState)
    {
        _currentState?.Exit();
        _currentState = newState;
        _currentState.Enter();
    }
    
    public void Update() => _currentState?.Update();
}

// 5. Command Pattern (for undo/redo)
interface ICommand
{
    void Execute();
    void Undo();
}

class MoveCommand : ICommand
{
    private Vector3 _previousPosition;
    private Vector3 _newPosition;
    private Transform _target;
    
    public MoveCommand(Transform target, Vector3 newPos)
    {
        _target = target;
        _newPosition = newPos;
    }
    
    public void Execute()
    {
        _previousPosition = _target.Position;
        _target.Position = _newPosition;
    }
    
    public void Undo()
    {
        _target.Position = _previousPosition;
    }
}
```

## 九、关键字深度解析

### 9.1 ref / out / in

```csharp
// ref: pass by reference, must be initialized before call
void Swap(ref int a, ref int b)
{
    int temp = a;
    a = b;
    b = temp;
}

// out: pass by reference, must be assigned inside method
bool TryParse(string input, out int result)
{
    return int.TryParse(input, out result);
}

// in: pass by reference (readonly), prevents copying large structs
void Calculate(in Matrix4x4 matrix)  // no copy, cannot modify
{
    // matrix.M11 = 0;  // ERROR: cannot modify 'in' parameter
    float det = matrix.M11 * matrix.M22;
}

// ref return and ref local (C# 7+)
ref int FindMax(int[] array)
{
    int maxIndex = 0;
    for (int i = 1; i < array.Length; i++)
    {
        if (array[i] > array[maxIndex])
            maxIndex = i;
    }
    return ref array[maxIndex];  // return reference to array element
}
```

### 9.2 readonly vs const

```csharp
class Constants
{
    // const: compile-time constant, inlined at call site
    // Changing const requires recompiling ALL referencing assemblies
    public const float PI = 3.14159f;
    public const string Version = "1.0.0";
    
    // readonly: runtime constant, set in constructor or initializer
    // Changing readonly only requires recompiling this assembly
    public readonly DateTime StartTime;
    public static readonly int ProcessorCount = Environment.ProcessorCount;
    
    public Constants()
    {
        StartTime = DateTime.Now;  // OK: set in constructor
    }
}

// readonly struct: all fields are readonly, enables compiler optimizations
readonly struct Vector2
{
    public readonly float X;
    public readonly float Y;
    
    public Vector2(float x, float y) { X = x; Y = y; }
    
    // 'in' parameter with readonly struct: guaranteed no defensive copy
    public float Dot(in Vector2 other) => X * other.X + Y * other.Y;
}
```

### 9.3 sealed

```csharp
// sealed class: cannot be inherited
sealed class FinalClass { }
// class Derived : FinalClass { }  // ERROR

// sealed method: prevents further override
class Base
{
    public virtual void Method() { }
}

class Middle : Base
{
    public sealed override void Method() { }  // no further override allowed
}

class Derived : Middle
{
    // public override void Method() { }  // ERROR: Method is sealed
}

// Performance benefit: sealed enables devirtualization
// JIT can inline sealed virtual method calls
```

### 9.4 virtual / override / new

```csharp
class Animal
{
    public virtual void Speak() => Console.WriteLine("...");
    public void Walk() => Console.WriteLine("Animal walks");
}

class Dog : Animal
{
    public override void Speak() => Console.WriteLine("Woof!");  // polymorphic
    public new void Walk() => Console.WriteLine("Dog walks");     // hides base
}

Animal animal = new Dog();
animal.Speak();  // "Woof!"       - virtual dispatch (runtime type)
animal.Walk();   // "Animal walks" - static dispatch (compile-time type)

Dog dog = new Dog();
dog.Speak();     // "Woof!"
dog.Walk();      // "Dog walks"
```
