# 迭代器模式（Iterator Pattern）—— C# 实现

迭代器模式是一种行为型设计模式，它提供一种方法顺序访问一个聚合对象中的各个元素，而又不暴露该对象的内部表示。

## 结构

- **Iterator（迭代器接口）**：定义访问和遍历元素的方法。
- **ConcreteIterator（具体迭代器）**：实现迭代器接口，跟踪遍历位置。
- **Aggregate（聚合接口）**：定义创建迭代器对象的方法。
- **ConcreteAggregate（具体聚合）**：实现聚合接口，返回具体迭代器实例。

## UML 类图

```
[Client] ——> [Iterator]
          ↑          ↑
[ConcreteIterator] [Aggregate]
                           ↑
               [ConcreteAggregate]
```

## C# 示例代码

```csharp
using System;
using System.Collections.Generic;

// 迭代器接口
public interface IIterator<T>
{
    bool HasNext();
    T Next();
}

// 聚合接口
public interface IAggregate<T>
{
    IIterator<T> CreateIterator();
}

// 具体聚合
public class NameCollection : IAggregate<string>
{
    private List<string> _names = new List<string>();

    public void Add(string name) => _names.Add(name);

    public IIterator<string> CreateIterator() => new NameIterator(this);

    public int Count => _names.Count;
    public string this[int index] => _names[index];

    // 具体迭代器
    private class NameIterator : IIterator<string>
    {
        private NameCollection _collection;
        private int _index = 0;

        public NameIterator(NameCollection collection)
        {
            _collection = collection;
        }

        public bool HasNext() => _index < _collection.Count;
        public string Next() => _collection[_index++];
    }
}

// 使用示例
class Program
{
    static void Main()
    {
        NameCollection names = new NameCollection();
        names.Add("Alice");
        names.Add("Bob");
        names.Add("Charlie");

        IIterator<string> iterator = names.CreateIterator();
        while (iterator.HasNext())
        {
            Console.WriteLine(iterator.Next());
        }
    }
}
```

## 输出结果

```
Alice
Bob
Charlie
```

## 适用场景

- 需要访问一个集合对象而不暴露其内部结构时。
- 需要为集合提供多种遍历方式时。
- 需要为遍历过程提供统一接口时。

## 优缺点

**优点：**
- 支持不同方式遍历同一聚合结构。
- 封装了集合的遍历细节，客户端无需了解集合内部结构。

**缺点：**
- 增加了类的数量。
- 对于简单的集合，使用迭代器模式可能会略显繁琐。

## 总结

迭代器模式为遍历集合对象提供统一接口，封装了集合内部结构，是对集合对象遍历的优雅解决方案。
