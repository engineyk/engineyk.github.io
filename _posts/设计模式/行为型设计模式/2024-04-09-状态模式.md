# 状态模式（State Pattern）—— C# 实现

状态模式是一种行为型设计模式，允许对象在其内部状态发生变化时改变它的行为，使其看起来就像改变了对象所属的类一样。状态模式将与状态相关的行为抽取到独立的状态类中，使得状态切换更清晰、易于维护。

## 结构

- **State（状态接口）**：定义与状态相关的行为。
- **ConcreteState（具体状态类）**：实现状态接口，封装某一具体状态下的行为。
- **Context（上下文/环境类）**：持有当前状态对象，并将请求委托给当前状态对象处理。

## UML 类图

```
[Context] ——> [State]
                  ↑
        [ConcreteStateA]
        [ConcreteStateB]
```

## C# 示例代码

```csharp
using System;

// 状态接口
public interface IState
{
    void Handle(Context context);
}

// 具体状态A
public class StateA : IState
{
    public void Handle(Context context)
    {
        Console.WriteLine("当前状态：A，切换到状态B");
        context.SetState(new StateB());
    }
}

// 具体状态B
public class StateB : IState
{
    public void Handle(Context context)
    {
        Console.WriteLine("当前状态：B，切换到状态A");
        context.SetState(new StateA());
    }
}

// 上下文
public class Context
{
    private IState _state;

    public Context(IState state)
    {
        _state = state;
    }

    public void SetState(IState state)
    {
        _state = state;
    }

    public void Request()
    {
        _state.Handle(this);
    }
}

// 使用示例
class Program
{
    static void Main()
    {
        Context context = new Context(new StateA());
        context.Request(); // 当前状态：A，切换到状态B
        context.Request(); // 当前状态：B，切换到状态A
        context.Request(); // 当前状态：A，切换到状态B
    }
}
```

## 输出结果

```
当前状态：A，切换到状态B
当前状态：B，切换到状态A
当前状态：A，切换到状态B
```

## 适用场景

- 对象的行为依赖于它的状态，并且在运行时根据状态发生变化。
- 需要避免大量条件语句（如 if/else 或 switch）来管理状态切换。

## 优缺点

**优点：**
- 封装状态转换，避免条件判断。
- 状态切换更清晰，易于维护和扩展。

**缺点：**
- 增加了系统中类的数量。
- 状态过多时，类数量激增。

## 总结

状态模式将状态相关行为局部化到各个状态类中，使得状态切换逻辑更加清晰、可维护，适合处理状态多且行为复杂的对象。
