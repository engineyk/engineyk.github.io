# C# 实现模板方法模式（Template Method Pattern）

模板方法模式是一种常用的设计模式，用于定义一个算法的骨架，将具体的步骤延迟到子类实现。下面用 C# 给出一个简单示例。

```csharp
// 抽象类，定义算法骨架
public abstract class AbstractClass
{
    // 模板方法，定义算法结构
    public void TemplateMethod()
    {
        Step1();
        Step2();
        Step3();
    }

    protected abstract void Step1();
    protected abstract void Step2();

    // 钩子方法，可选择重写
    protected virtual void Step3()
    {
        Console.WriteLine("默认的 Step3 实现");
    }
}

// 具体子类A
public class ConcreteClassA : AbstractClass
{
    protected override void Step1()
    {
        Console.WriteLine("A 实现的 Step1");
    }

    protected override void Step2()
    {
        Console.WriteLine("A 实现的 Step2");
    }
}

// 具体子类B
public class ConcreteClassB : AbstractClass
{
    protected override void Step1()
    {
        Console.WriteLine("B 实现的 Step1");
    }

    protected override void Step2()
    {
        Console.WriteLine("B 实现的 Step2");
    }

    protected override void Step3()
    {
        Console.WriteLine("B 自定义的 Step3");
    }
}

// 测试代码
class Program
{
    static void Main(string[] args)
    {
        AbstractClass a = new ConcreteClassA();
        a.TemplateMethod();

        AbstractClass b = new ConcreteClassB();
        b.TemplateMethod();
    }
}
```

## 运行结果

```
A 实现的 Step1
A 实现的 Step2
默认的 Step3 实现
B 实现的 Step1
B 实现的 Step2
B 自定义的 Step3
```

## 总结

- **抽象类**定义了算法结构，并声明抽象方法。
- **具体子类**负责实现具体步骤。
- **模板方法**保证算法结构不变，子类可以扩展细节实现。
