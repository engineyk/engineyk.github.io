# 享元模式（Flyweight Pattern）—— C# 实现

享元模式是一种结构型设计模式，旨在通过共享大量细粒度对象，减少内存使用和提升性能。它适用于需要创建大量相似对象的场景。

## 结构

- **Flyweight（享元接口）**：定义享元对象的接口。
- **ConcreteFlyweight（具体享元）**：实现享元接口，封装可共享的状态。
- **FlyweightFactory（享元工厂）**：创建和管理享元对象，实现对象复用。
- **客户端（Client）**：通过享元工厂获取享元对象，可以增加外部状态。

## UML 类图

```
[Client] ——> [FlyweightFactory] ——> [Flyweight]
                                     └──> [ConcreteFlyweight]
```

## C# 示例代码

```csharp
using System;
using System.Collections.Generic;

// 享元接口
public interface IShape
{
    void Draw(string color); // color 是外部状态
}

// 具体享元
public class Circle : IShape
{
    private string _radius; // 内部状态

    public Circle(string radius)
    {
        _radius = radius;
    }

    public void Draw(string color)
    {
        Console.WriteLine($"Circle: radius = {_radius}, color = {color}");
    }
}

// 享元工厂
public class ShapeFactory
{
    private Dictionary<string, IShape> _circlePool = new Dictionary<string, IShape>();

    public IShape GetCircle(string radius)
    {
        if (!_circlePool.ContainsKey(radius))
        {
            _circlePool[radius] = new Circle(radius);
            Console.WriteLine($"Created new Circle with radius {radius}");
        }
        return _circlePool[radius];
    }
}

// 使用示例
class Program
{
    static void Main()
    {
        ShapeFactory factory = new ShapeFactory();

        IShape c1 = factory.GetCircle("10");
        c1.Draw("Red");

        IShape c2 = factory.GetCircle("10");
        c2.Draw("Blue");

        IShape c3 = factory.GetCircle("20");
        c3.Draw("Red");
    }
}
```

## 输出结果

```

Circle: radius = 10, color = Red
Circle: radius = 10, color = Blue
Created new Circle with radius 20
Circle: radius = 20, color = Red
```

## 适用场景

- 系统有大量对象，且这些对象的大部分状态可以共享。
- 需要节省内存和提升性能的场景（如文本编辑器中的字符对象、棋盘游戏中的棋子）。

## 优缺点

**优点：**
- 大幅减少对象数量，节约内存、提升性能。
- 享元对象可以被多处共享和复用。

**缺点：**
- 使系统结构更复杂，需要区分内部状态和外部状态。
- 共享对象需要考虑线程安全。

## 总结

享元模式通过共享可复用对象，极大减少了内存消耗。适合处理大量细粒度对象的场景，是性能优化的重要手段之一。
