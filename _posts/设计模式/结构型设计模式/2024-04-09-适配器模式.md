# 适配器模式（Adapter Pattern）

## 概念简介

适配器模式是一种结构型设计模式，它允许将一个类的接口转换成客户端期望的另一种接口。适配器模式使得原本由于接口不兼容而不能一起工作的类可以协同工作。该模式常用于系统需要使用一些现有的类，但这些类的接口又与系统的接口不兼容的场景。

---

## 结构图

```
┌─────────────┐      ┌─────────────┐
│   客户端    │◄────►│  目标接口   │
└─────────────┘      └─────▲───────┘
                           │
                    ┌──────┴───────┐
                    │   适配器     │
                    └─────▲───────┘
                          │
                    ┌─────┴───────┐
                    │被适配者类   │
                    └─────────────┘
```

---

## 代码示例（C#）

```csharp
// 目标接口
public interface ITarget
{
    void Request();
}

// 已有类（被适配者）
public class Adaptee
{
    public void SpecificRequest()
    {
        Console.WriteLine("被适配者方法调用");
    }
}

// 适配器
public class Adapter : ITarget
{
    private Adaptee adaptee;

    public Adapter(Adaptee adaptee)
    {
        this.adaptee = adaptee;
    }

    public void Request()
    {
        // 将目标接口的请求转换为被适配者的方法
        adaptee.SpecificRequest();
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        Adaptee adaptee = new Adaptee();
        ITarget target = new Adapter(adaptee);
        target.Request(); // 输出：被适配者方法调用
    }
}
```

---

## 优缺点

**优点：**
- 可以让原本接口不兼容的类一起工作。
- 增加了类的透明性和复用性。
- 符合开闭原则，对原有代码无侵入。

**缺点：**
- 过多使用适配器会使系统复杂，降低可读性。
- 适配器编写需要对原有接口和目标接口都比较了解。

---

## 应用场景

- 系统需要使用现有的类，且这些类的接口与系统要求不一致时。
- 希望复用一些现有的类，但接口又不符合要求时。
- 需要通过适配多个已有子系统的接口时。
