# 组合模式（Composite Pattern）—— C# 实现

组合模式用于将对象组合成树形结构，统一单个对象和组合对象的处理方式。

## 结构说明

- **Component（抽象组件）**：声明统一接口。
- **Leaf（叶子节点）**：实现 Component 接口，无子节点。
- **Composite（组合节点）**：实现 Component 接口，包含子节点。

## C# 示例代码

```csharp
using System;
using System.Collections.Generic;

// 抽象组件
abstract class FileComponent
{
    protected string name;
    public FileComponent(string name) { this.name = name; }
    public abstract void Show(int depth = 0);
}

// 叶子节点
class FileLeaf : FileComponent
{
    public FileLeaf(string name) : base(name) { }
    public override void Show(int depth = 0)
    {
        Console.WriteLine(new string('-', depth) + name);
    }
}

// 组合节点
class FolderComposite : FileComponent
{
    private List<FileComponent> children = new List<FileComponent>();
    public FolderComposite(string name) : base(name) { }

    public void Add(FileComponent component)
    {
        children.Add(component);
    }

    public void Remove(FileComponent component)
    {
        children.Remove(component);
    }

    public override void Show(int depth = 0)
    {
        Console.WriteLine(new string('-', depth) + "[Folder] " + name);
        foreach (var child in children)
        {
            child.Show(depth + 2);
        }
    }
}

// 使用示例
class Program
{
    static void Main()
    {
        FolderComposite root = new FolderComposite("Root");
        root.Add(new FileLeaf("File1.txt"));
        root.Add(new FileLeaf("File2.txt"));

        FolderComposite subFolder = new FolderComposite("SubFolder");
        subFolder.Add(new FileLeaf("SubFile1.txt"));
        subFolder.Add(new FileLeaf("SubFile2.txt"));

        root.Add(subFolder);

        root.Show();
    }
}
```

## 输出结果

```
[Folder] Root
--File1.txt
--File2.txt
--[Folder] SubFolder
----SubFile1.txt
----SubFile2.txt
```

## 适用场景

- 树形结构数据表示（如文件系统、公司组织架构、UI组件树等）。
- 需要统一处理单个对象和组合对象时。

## 优缺点

**优点：**
- 客户端统一处理叶子和组合对象，简化客户端代码。
- 易于扩展新类型的节点。

**缺点：**
- 过度抽象可能导致设计复杂。
- 类型约束不够细致。

## 总结

组合模式是处理层次结构对象的优雅方案，适合表示“部分-整体”关系的数据结构。
