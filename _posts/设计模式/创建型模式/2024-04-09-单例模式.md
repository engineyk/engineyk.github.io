# 单例模式（Singleton Pattern）

## 概念简介

单例模式是一种创建型设计模式，确保一个类在系统中只有一个实例，并提供一个全局访问点。常用于需要全局唯一对象的场景，例如系统配置、日志对象、线程池等。

---

## 结构图

```
┌──────────────┐
│   单例类     │
└─────┬────────┘
      │
      ▼

```

---

## 代码示例（C#）

```csharp
public class Singleton
{
    // 私有静态变量保存唯一实例
    private static Singleton instance;

    // 锁对象，确保线程安全
    private static readonly object locker = new object();

    // 私有构造函数，防止外部new
    private Singleton() { }

    // 公共静态方法，获取唯一实例
    public static Singleton Instance
    {
        get
        {
            if (instance == null)
            {
                lock (locker)
                {
                    if (instance == null)
                    {
                        instance = new Singleton();
                    }
                }
            }
            return instance;
        }
    }

    // 示例方法
    public void ShowMessage()
    {
        Console.WriteLine("Hello, Singleton!");
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        Singleton singleton1 = Singleton.Instance;
        Singleton singleton2 = Singleton.Instance;
        singleton1.ShowMessage();

        // 判断是否为同一个实例
        Console.WriteLine(object.ReferenceEquals(singleton1, singleton2)); // 输出True
    }
}
```

---

## 优缺点

**优点：**
- 全局唯一实例，节省资源。
- 只初始化一次，延迟加载。
- 提供全局访问点。

**缺点：**
- 不易扩展，违反单一职责原则。
- 多线程环境下实现不当会有线程安全问题。
- 可能隐藏依赖关系，影响测试。

---

## 应用场景

- 需要唯一对象的场景，如：配置对象、日志对象、数据库连接池、线程池等。
- 需要共享访问的资源管理。
