# 抗锯齿（Anti-Aliasing）技术总结

## 一、概述

抗锯齿（AA）是渲染管线中用于消除图像边缘锯齿的核心技术。
根据实现阶段和原理不同，主要分为以下几类：

| 技术 | 实现阶段   | 核心原理        | 硬件依赖     | 时间性信息 |
| ---- | ---------- | --------------- | ------------ | ---------- |
| MSAA | 光栅化阶段 | 多重采样合并    | 无           | ❌          |
| FXAA | 后处理阶段 | 边缘模糊        | 无           | ❌          |
| SMAA | 后处理阶段 | 形态学边缘检测  | 无           | ❌          |
| TAA  | 后处理阶段 | 时间性帧混合    | 无           | ✅          |
| DLSS | 后处理阶段 | 深度学习超分    | NVIDIA GPU   | ✅          |
| FSR  | 后处理阶段 | 空间/时间性超分 | 无（跨平台） | ✅（2.0+）  |

---

## 二、各技术详解

### 1. MSAA（Multi-Sample Anti-Aliasing）多重采样抗锯齿

**实现阶段**：光栅化阶段

**核心原理**：
- 对每个像素进行多次采样（2x / 4x / 8x）
- 将多个采样结果合并为最终像素颜色
- 在几何边缘处效果最佳

**缺点**：
- 不兼容延迟渲染(Deferred Rendering) 
  - 对延迟渲染（Deferred Rendering）不友好，G-Buffer 多重采样显存开销极大
  - 延迟渲染需要Geometry和Lighting两个pass，在光照阶段无法通过GBuffer获取片段的覆盖率信息
- 对透明物体效果差
- 随采样倍数增加，性能开销线性增长

---

### 2. FXAA（Fast Approximate Anti-Aliasing）快速近似抗锯齿

**实现阶段**：后处理阶段

**核心原理**：
- 检测图像中亮度变化明显的边缘像素
- 性能开销较小，适用于实时应用。
- 对检测到的边缘进行模糊处理
- 完全基于最终图像，不依赖几何信息

**优点**：性能开销极低，兼容所有渲染管线

**缺点**：
- 图像整体偏模糊
- 细节和文字容易丢失

---

### 3. SMAA（Enhanced Subpixel Morphological Anti-Aliasing）增强型子像素形态抗锯齿

**实现阶段**：后处理阶段

**核心原理**：
- 在 FXAA 基础上增加形态学（Morphological）分析
- 检测边缘的具体形状（L形、Z形等）
- 根据边缘形状进行更精确的混合处理

**优势**：比 FXAA 保留更多细节，模糊感更低，质量介于 FXAA 和 TAA 之间

---

### 4. TAA（Temporal Anti-Aliasing）时间性抗锯齿

**实现阶段**：后处理阶段

**核心思想**：利用历史帧信息与当前帧混合，将多帧采样结果积累，等效提升采样率

#### 4.1 Jitter（抖动采样）

- 每帧渲染时对相机投影矩阵施加微小偏移（Sub-pixel Jitter）
- 使每帧的采样位置略有不同，积累后覆盖更多子像素位置
- 使用 **Halton 序列**生成偏移量
  - Halton 序列是一种低差异序列（Low-Discrepancy Sequence）
  - 保证采样位置在子像素空间内均匀分布

#### 4.2 重投影（Reprojection）

- 将历史帧的像素重新投影到当前帧的屏幕空间
- 通过 **Motion Vector（运动向量）** 计算每个像素的运动偏移
- Motion Vector 记录了像素从上一帧到当前帧的屏幕空间位移

#### 4.3 混合（Blending）

$$Color_{output} = 0.1 \times Color_{current} + 0.9 \times Color_{history}$$

- 当前帧权重：**0.1**
- 历史帧权重：**0.9**
- 历史帧占比高，使多帧采样信息得以积累

#### 4.4 历史帧校正（Ghost 抑制）

由于历史帧可能包含过时信息，需要对历史帧颜色进行校正：

| 方法      | 原理                                                      | 特点                                 |
| --------- | --------------------------------------------------------- | ------------------------------------ |
| **Clamp** | 将历史帧颜色值限制（Clamp）在当前帧邻域颜色的 AABB 范围内 | 温和，减少鬼影，保留更多历史信息     |
| **Clip**  | 将历史帧颜色值裁剪（Clip）到当前帧邻域颜色的 AABB 范围内  | 激进，更强力减少鬼影，但可能引入闪烁 |

#### 4.5 常见问题及解决方案

| 问题                 | 原因                                         | 解决方案                                                                    |
| -------------------- | -------------------------------------------- | --------------------------------------------------------------------------- |
| **鬼影（Ghosting）** | 物体快速移动时，历史帧颜色信息与当前帧不匹配 | 使用 Clamp/Clip 限制历史帧颜色范围；根据 Motion Vector 大小动态调整混合权重 |
| **模糊（Blur）**     | 多帧混合导致图像整体模糊                     | 使用锐化滤波器（Sharpening Filter）在 TAA 后进行锐化处理                    |
| **半透明物体鬼影**   | 半透明物体无深度信息，无法正确重投影         | 对半透明物体单独处理，不参与 TAA 混合                                       |

### 5. DLSS（Deep Learning Super Sampling）深度学习超级采样

**实现阶段**：后处理阶段  
**硬件要求**：仅支持 NVIDIA GPU（依赖 Tensor Core 加速推理）

#### 5.1 基本原理

利用深度学习（CNN）将低分辨率图像超分辨率放大到目标分辨率，同时结合时间性信息提升质量。

**输入数据**：
- 低分辨率当前帧（含 Jitter）
- Motion Vector（运动向量）
- 深度缓冲区（Depth Buffer）
- 历史帧

**输出数据**：高分辨率当前帧

**网络结构**：卷积神经网络（CNN）

#### 5.2 版本演进

| 版本         | 核心特性                                                         | 质量               |
| ------------ | ---------------------------------------------------------------- | ------------------ |
| **DLSS 1.0** | 纯 CNN 空间放大，无时间性信息                                    | 较差，图像模糊     |
| **DLSS 2.0** | 引入时间性信息，结合历史帧混合，通用模型                         | 较好，图像清晰锐利 |
| **DLSS 3.0** | 新增**帧生成（Frame Generation）**，AI 生成中间帧提升帧率        | 帧率大幅提升       |
| **DLSS 3.5** | 新增**光线重建（Ray Reconstruction）**，深度学习重建光追降噪结果 | 光追质量显著提升   |

#### 5.3 渲染分辨率模式

| 模式                            | 渲染分辨率比例 | 输出分辨率 | 适用场景           |
| ------------------------------- | -------------- | ---------- | ------------------ |
| 质量模式（Quality）             | 67%            | 100%       | 追求画质           |
| 平衡模式（Balanced）            | 58%            | 100%       | 质量与性能平衡     |
| 性能模式（Performance）         | 50%            | 100%       | 追求帧率           |
| 超性能模式（Ultra Performance） | 33%            | 100%       | 极限性能，画质最差 |

**优点**：图像质量好，性能提升明显  
**缺点**：仅支持 NVIDIA GPU，需要游戏集成支持

---

### 6. FSR（FidelityFX Super Resolution）超级分辨率

**实现阶段**：后处理阶段  
**硬件要求**：无硬件限制，支持所有 GPU（AMD / NVIDIA / Intel）

#### 6.1 版本演进

| 版本        | 核心特性                                               | 类比           |
| ----------- | ------------------------------------------------------ | -------------- |
| **FSR 1.0** | 纯空间放大算法（EASU + RCAS），无时间性信息            | 高质量空间超分 |
| **FSR 2.0** | 引入时间性信息，结合历史帧混合，类似 TAA + 超分        | 对标 DLSS 2.0  |
| **FSR 3.0** | 新增**帧生成（Frame Generation）**，生成中间帧提升帧率 | 对标 DLSS 3.0  |

#### 6.2 FSR 1.0 核心算法

- **EASU**（Edge Adaptive Spatial Upsampling）：边缘自适应空间放大
  - 分析边缘方向，自适应地进行方向性放大
- **RCAS**（Robust Contrast Adaptive Sharpening）：鲁棒对比度自适应锐化
  - 在放大后对图像进行锐化，补偿放大带来的模糊

#### 6.3 FSR 2.0 输入数据

- 低分辨率当前帧
- Motion Vector（运动向量）
- 深度缓冲区（Depth Buffer）
- 历史帧

#### 6.4 渲染分辨率模式

| 模式                            | 渲染分辨率比例 | 输出分辨率 |
| ------------------------------- | -------------- | ---------- |
| 质量模式（Quality）             | 67%            | 100%       |
| 平衡模式（Balanced）            | 58%            | 100%       |
| 性能模式（Performance）         | 50%            | 100%       |
| 超性能模式（Ultra Performance） | 33%            | 100%       |

**优点**：跨平台，无硬件限制，性能提升明显  
**缺点**：整体效果不如 DLSS，需要游戏集成支持

## 三、不同渲染管线的应用

| -    | 适用管线   | 效果                                                       | 消耗           | 其它描述                         |
| ---- | ---------- | ---------------------------------------------------------- | -------------- | -------------------------------- |
| MSAA | 前向       | 清晰度高，抗锯齿好                                         | 带宽中，显存中 | 需要额外记录采样覆盖数据         |
| FXAA | 前向，延迟 | 清晰度较高，抗锯齿较好                                     | 带宽低，显存低 | 不需要额外记录数据，计算量较大   |
| TAA  | 延迟       | 清晰度较低，存在延时、闪烁、鬼影等，但静态画面抗锯齿非常好 | 带宽高，显存高 | 需要速度缓冲和额外记录历史帧数据 |

### 3.1 前向渲染（Forward Rendering）

| 技术     | 适用性 | 说明                                                       |
| -------- | ------ | ---------------------------------------------------------- |
| MSAA     | ✅ 最佳 | 前向渲染原生支持，性能开销可接受，是前向渲染的首选 AA 方案 |
| FXAA     | ✅      | 后处理，完全兼容，性能极低                                 |
| SMAA     | ✅      | 后处理，完全兼容，质量优于 FXAA                            |
| TAA      | ✅      | 需要额外的 Motion Vector Pass                              |
| DLSS     | ✅      | 需要 Motion Vector + 深度缓冲                              |
| FSR 2.0+ | ✅      | 需要 Motion Vector + 深度缓冲                              |

### 3.2 延迟渲染（Deferred Rendering）

| 技术     | 适用性         | 说明                                                     |
| -------- | -------------- | -------------------------------------------------------- |
| MSAA     | ⚠️ 不推荐       | G-Buffer 多重采样显存开销极大，实现复杂                  |
| FXAA     | ✅              | 后处理，完全兼容                                         |
| SMAA     | ✅              | 后处理，完全兼容                                         |
| TAA      | ✅ **主流方案** | G-Buffer 天然提供 Motion Vector 和深度信息，延迟渲染首选 |
| DLSS     | ✅              | G-Buffer 天然提供所有所需输入                            |
| FSR 2.0+ | ✅              | G-Buffer 天然提供所有所需输入                            |

> 💡 **延迟渲染中 TAA 是事实标准**：G-Buffer 在渲染过程中已经生成了 Motion Vector 和深度缓冲，TAA 可以直接利用这些数据，几乎没有额外的数据采集开销。

### 3.3 光线追踪管线（Ray Tracing）

| 技术     | 适用性     | 说明                                                                            |
| -------- | ---------- | ------------------------------------------------------------------------------- |
| TAA      | ✅          | 可用于降噪 + 抗锯齿，但对光追噪点处理能力有限                                   |
| DLSS 3.5 | ✅ **最佳** | Ray Reconstruction 专为光追设计，用深度学习重建光追降噪结果，效果远超传统降噪器 |
| FSR 2.0+ | ✅          | 兼容光追管线，但降噪效果不如 DLSS 3.5                                           |

## 四、DLSS vs FSR 对比

| 对比项   | DLSS               | FSR                  |
| -------- | ------------------ | -------------------- |
| 开发商   | NVIDIA             | AMD                  |
| 硬件要求 | 仅 NVIDIA GPU      | 所有 GPU             |
| 1.0 原理 | CNN 超分           | EASU + RCAS 空间放大 |
| 2.0 原理 | CNN + 时间性信息   | 时间性信息（类 TAA） |
| 3.0 特性 | 帧生成             | 帧生成               |
| 3.5 特性 | Ray Reconstruction | 无对应版本           |
| 整体质量 | 更好（AI 推理）    | 略逊，但跨平台       |
| 开源     | ❌ 闭源             | ✅ 开源               |

## 五、技术选型建议

```
移动端 / 低端硬件
    └── FXAA（性能开销最低）

PC 前向渲染
    └── MSAA（几何边缘质量最好）
        └── 性能不足时降级为 FXAA

PC 延迟渲染（主流）
    └── TAA（标配基础方案）
        ├── NVIDIA 用户 → DLSS 2.0+（质量最佳）
        └── 跨平台 / AMD 用户 → FSR 2.0+（性价比高）

光线追踪
    └── DLSS 3.5 Ray Reconstruction（最佳，仅 NVIDIA）
        └── FSR 2.0+（跨平台备选）

帧率提升需求
    ├── DLSS 3.0 Frame Generation（NVIDIA RTX 40系）
    └── FSR 3.0 Frame Generation（跨平台）
```

## 六、核心概念速查

| 概念                      | 说明                                                                         |
| ------------------------- | ---------------------------------------------------------------------------- |
| **Motion Vector**         | 记录每个像素从上一帧到当前帧的屏幕空间运动偏移，是 TAA/DLSS/FSR 重投影的基础 |
| **Halton 序列**           | 低差异序列，用于生成均匀分布的 Jitter 偏移量，保证子像素采样覆盖均匀         |
| **Ghosting（鬼影）**      | 历史帧信息不准确导致的残影，TAA 的主要问题之一                               |
| **Color AABB Clamp/Clip** | 通过限制历史帧颜色在当前帧邻域范围内来抑制鬼影的方法                         |
| **Frame Generation**      | 通过 AI 在两帧之间生成中间帧，提升显示帧率（DLSS 3.0 / FSR 3.0）             |
| **Ray Reconstruction**    | 用深度学习替代传统光追降噪器，重建光追结果（DLSS 3.5）                       |
| **EASU**                  | FSR 1.0 的边缘自适应空间放大算法                                             |
| **RCAS**                  | FSR 1.0 的鲁棒对比度自适应锐化算法                                           |
| **低差异序列**            | 一种采样序列，相比随机采样能更均匀地覆盖采样空间，减少聚集现象               |
