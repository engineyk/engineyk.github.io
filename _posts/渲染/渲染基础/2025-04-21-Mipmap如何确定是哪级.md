# Q1: Mipmap如何确定是哪级

## Mipmap 基本概念

Mipmap（Mipmapping）是计算机图形学中一种纹理映射技术，通过预先生成一系列不同分辨率的纹理层级（LOD），以解决远距离物体渲染时的纹理走样（Aliasing）问题。每一级 Mipmap 的分辨率是上一级的一半（长宽各减半）。

```
原始纹理：2048×2048
Mip 0: 2048×2048
Mip 1: 1024×1024  （1/4 像素）
Mip 2: 512×512    （1/16 像素）
Mip 3: 256×256    （1/64 像素）
...
Mip 11: 1×1       （最小层级）
```

## 层级确定的核心原理

Mipmap 层级的选择基于一个核心原理：**屏幕像素覆盖的纹理像素数量（texels per pixel）**。当这个比值远大于1时（一个屏幕像素覆盖多个纹理像素），就会产生走样；当这个比值远小于1时（多个屏幕像素覆盖一个纹理像素），就会产生模糊。理想情况是 texels/pixel ≈ 1。

## 数学公式推导

## 1. 基础公式：导数法（最准确）

在渲染管线中，GPU 通过计算纹理坐标 `(u,v)` 对屏幕坐标 `(x,y)` 的偏导数来确定 Mipmap 层级：

```c
// 计算纹理坐标在屏幕空间的梯度
float dudx = dFdx(u);  // u 在 x 方向的变化率
float dudy = dFdy(u);  // u 在 y 方向的变化率
float dvdx = dFdx(v);  // v 在 x 方向的变化率
float dvdy = dFdy(v);  // v 在 y 方向的变化率

// 计算屏幕像素在纹理空间覆盖的椭圆面积
float A = dudx * dvdy - dudy * dvdx;
float B = dudx * dudx + dudy * dudy;
float C = dvdx * dvdx + dvdy * dvdy;

// 最大轴长度（保守估计）
float lambda = 0.5 * log2(max(B, C));

// Mipmap 层级（浮点数）
float mipLevel = max(0.0, lambda);
```

## 2. 简化公式：距离法（快速估算）

对于透视投影下的平面纹理，可以通过屏幕空间距离近似：

```
// 假设纹理尺寸为 textureSize
// pixelCoverage = 纹理像素数 / 屏幕像素数
float distanceToCamera = length(cameraPos - worldPos);
float pixelCoverage = (textureSize * objectSize) / (distanceToCamera * screenResolution);

// Mipmap 层级公式
float mipLevel = 0.5 * log2(pixelCoverage);
```

## 3. 实用公式：屏幕空间覆盖率

更实用的计算方式：

```
// 计算纹理坐标在屏幕空间的差异（相邻像素间）
float2 duv_dx = ddx(texcoord);
float2 duv_dy = ddy(texcoord);

// 计算纹理空间的最大边长
float maxLength = max(length(duv_dx * textureSize), length(duv_dy * textureSize));

// Mipmap 层级
float mipLevel = log2(maxLength);
```

## GPU 硬件实现

现代 GPU（如 NVIDIA/AMD）在硬件层面实现了 Mipmap 层级计算：

1. **梯度计算单元**：每个像素着色器都有专门的硬件计算 `dFdx/dFdy`
2. **层级插值**：支持 trilinear filtering（三线性过滤），在相邻 Mip 层级间插值
3. **各向异性过滤**：当纹理投影严重扭曲时（如地面纹理），使用各向异性过滤采样多个方向

## 实践中的关键问题

## 1. 如何避免层级跳跃（Popping）

```c
// 使用 trilinear filtering（自动层级间插值）
sampler2D tex : register(s0);
float4 color = tex2D(tex, uv);  // 硬件自动三线性插值

// 或手动控制渐变
float mipLevel = calculateMipLevel();
float mipLevelFrac = frac(mipLevel);
float mipLevelInt = floor(mipLevel);
float4 colorLow = tex2Dlod(tex, float4(uv, 0, mipLevelInt));
float4 colorHigh = tex2Dlod(tex, float4(uv, 0, mipLevelInt + 1));
float4 color = lerp(colorLow, colorHigh, mipLevelFrac);
```

## 2. 如何确定最大最小层级

```c
// OpenGL / DirectX 设置
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAX_LOD, maxLod);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_LOD, minLod);

// Unity 设置
texture.maxMipMapLevel = maxLod;
texture.mipMapBias = bias;  // 全局偏移
```

## 3. 负层级（放大）的处理

当 `mipLevel < 0` 时，表示需要放大纹理（屏幕像素小于纹理像素）。硬件会自动使用 Mip 0 并插值，但会产生模糊。解决方案：
- 使用更高分辨率纹理
- 应用锐化后处理
- 使用细节贴图（Detail Map）

## 各向异性过滤与 Mipmap

各向异性过滤（Anisotropic Filtering）是对 Mipmap 的扩展：

```
常规 Mipmap：假设屏幕像素在纹理空间是圆形覆盖
各向异性：识别屏幕像素在纹理空间是椭圆形覆盖

// 各向异性程度计算
float aniso = max(length(duv_dx), length(duv_dy)) / min(length(duv_dx), length(duv_dy));

// 16x 各向异性 = 沿椭圆长轴采样 16 个点，取平均值
```

## 性能优化技巧

1. **Mipmap 生成时机**
   - 离线生成：美术导出时预生成（推荐）
   - 运行时生成：`glGenerateMipmap()`，有性能开销

2. **内存优化**
   ```
   // Mipmap 总内存 ≈ 原始纹理 × 4/3
   // 例如：2048×2048 RGBA8 = 16MB
   // 全部 Mipmap = 16MB × 4/3 ≈ 21.3MB
   ```

3. **流式加载优化**
   - 只加载当前需要的 Mip 层级
   - 远景物体使用低 Mip 层级
   - 近景物体使用高 Mip 层级

## 常见面试问题扩展

## Q1.1：为什么需要 Mipmap？
- 解决远处纹理的摩尔纹（Moiré Pattern）
- 减少缓存抖动（相邻像素访问同一 Mip 层级）
- 提高渲染质量

## Q1.2：Mipmap 与纹理过滤的关系？
- **最近邻（Nearest）**：无 Mipmap，直接采样
- **双线性（Bilinear）**：在单个 Mip 层级内插值
- **三线性（Trilinear）**：在两个相邻 Mip 层级间插值
- **各向异性（Anisotropic）**：沿椭圆长轴多采样

## Q1.3：如何手动计算 Mipmap 层级？
```cpp
// C++ 手动计算示例
float calculateMipLevel(int textureWidth, int textureHeight, 
                       float du_dx, float du_dy,
                       float dv_dx, float dv_dy) {
    // 计算纹理空间覆盖面积
    float pixelWidth = max(
        sqrt(du_dx * du_dx + dv_dx * dv_dx) * textureWidth,
        sqrt(du_dy * du_dy + dv_dy * dv_dy) * textureWidth
    );
    
    float pixelHeight = max(
        sqrt(du_dx * du_dx + dv_dx * dv_dx) * textureHeight,
        sqrt(du_dy * du_dy + dv_dy * dv_dy) * textureHeight
    );
    
    float maxDimension = max(pixelWidth, pixelHeight);
    
    // log2 计算层级
    if (maxDimension <= 1.0f) {
        return 0.0f;  // 不需要降采样
    }
    
    return log2(maxDimension);
}
```

## 总结

Mipmap 层级确定的核心是计算 **屏幕像素在纹理空间的覆盖面积**。现代 GPU 硬件自动通过纹理坐标的屏幕空间导数（`dFdx/dFdy`）来计算最合适的层级。正确使用 Mipmap 可以：

1. **消除远处纹理走样**
2. **提高缓存命中率**
3. **平衡质量与性能**
4. **支持各向异性过滤**

关键公式：`mipLevel = 0.5 * log2(texels_per_pixel)`
