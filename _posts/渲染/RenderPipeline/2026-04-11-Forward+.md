## 1. 技术概述

### 1.1 什么是 Forward+

Forward+（也称 Tiled Forward Shading）是一种混合渲染技术，结合了 Forward Rendering 的灵活性和 Deferred Rendering 的多光源高效处理能力。其核心思想是：

> **将屏幕划分为 Tile（瓦片），通过 Light Culling 预先计算每个 Tile 受影响的光源列表，然后在 Forward Pass 中只计算相关光源。**

### 1.2 发展历史

| 年份 | 里程碑 |
|------|--------|
| 2012 | AMD 在 GDC 发表 "Forward+: Bringing Deferred Lighting to the Next Level" |
| 2013 | Takahiro Harada 发表完整论文 |
| 2015 | Unreal Engine 4.14 引入 Forward Shading 路径 |
| 2017 | Unity 引入 URP (原 LWRP) 支持 Forward+ |
| 2022 | Unity URP 14 正式支持 Forward+ 渲染路径 |
| 2023 | 各主流引擎全面支持，成为移动端和 VR 的首选方案 |

### 1.3 核心优势一览

```
┌─────────────────────────────────────────────────────┐
│                  Forward+ 核心优势                    │
├─────────────────┬───────────────────────────────────┤
│  多光源支持      │ 支持数百甚至上千个动态光源           │
│  MSAA 兼容      │ 原生支持硬件 MSAA，无需额外处理       │
│  材质多样性      │ 不受 G-Buffer 格式限制               │
│  透明物体        │ 天然支持透明物体光照                  │
│  带宽友好        │ 无需写入/读取 G-Buffer，带宽开销低     │
│  内存友好        │ 无需多张全屏 G-Buffer RT              │
└─────────────────┴───────────────────────────────────┘
```

## 2. 核心原理

### 2.1 整体架构

Forward+ 渲染分为三个核心阶段：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Depth Pass  │────▶│ Light Culling│────▶│ Shading Pass │
│  (Pre-Z)     │     │  (Compute)   │     │  (Forward)   │
└──────────────┘     └──────────────┘     └──────────────┘
     Stage 1              Stage 2              Stage 3
  生成深度缓冲        Tile 光源剔除        前向着色计算
```

### 2.2 Stage 1: Depth Pre-Pass

**目的**：生成场景深度缓冲，为 Light Culling 提供每个 Tile 的深度范围。

```hlsl
// Depth Pre-Pass Vertex Shader
float4 DepthOnlyVert(float3 posOS : POSITION) : SV_POSITION
{
    return TransformObjectToHClip(posOS);
}

// Depth Pre-Pass Fragment Shader (optional, for alpha test)
void DepthOnlyFrag(float4 posCS : SV_POSITION)
{
    // For opaque objects: empty (depth-only write)
    // For alpha-tested objects: discard based on alpha
}
```

**关键点**：
- 只写入深度，不写入颜色
- Alpha Test 物体需要采样纹理判断是否 discard
- 为后续 Light Culling 提供 Tile 的 Zmin/Zmax

### 2.3 Stage 2: Light Culling (核心)

这是 Forward+ 的核心创新。使用 Compute Shader 将屏幕划分为 Tile，计算每个 Tile 的光源列表。

#### 2.3.1 Tile 划分

```
屏幕空间 (1920 x 1080)
┌────┬────┬────┬────┬────┬────┬─ ─ ─ ─┐
│ T0 │ T1 │ T2 │ T3 │ T4 │ T5 │  ...  │
├────┼────┼────┼────┼────┼────┼─ ─ ─ ─┤
│ T120│T121│T122│T123│T124│T125│  ...  │
├────┼────┼────┼────┼────┼────┼─ ─ ─ ─┤
│    │    │    │    │    │    │  ...  │
└────┴────┴────┴────┴────┴────┴─ ─ ─ ─┘

Tile Size: 通常 16x16 像素
Tile Count: ceil(1920/16) x ceil(1080/16) = 120 x 68 = 8160 tiles
```

#### 2.3.2 Tile 深度范围计算

```hlsl
// Compute Shader: Calculate per-tile depth bounds
groupshared uint tileMinDepthInt;
groupshared uint tileMaxDepthInt;

[numthreads(TILE_SIZE, TILE_SIZE, 1)]
void LightCullingCS(
    uint3 groupId    : SV_GroupID,
    uint3 groupTid   : SV_GroupThreadID,
    uint  groupIndex : SV_GroupIndex)
{
    // Step 1: Initialize shared memory
    if (groupIndex == 0)
    {
        tileMinDepthInt = 0xFFFFFFFF;
        tileMaxDepthInt = 0;
    }
    GroupMemoryBarrierWithGroupSync();

    // Step 2: Load depth and compute tile min/max
    uint2 pixelCoord = groupId.xy * TILE_SIZE + groupTid.xy;
    float depth = DepthTexture.Load(int3(pixelCoord, 0)).r;
    uint depthInt = asuint(depth); // Reversed-Z: larger float = closer

    // Atomic min/max on shared memory
    InterlockedMin(tileMinDepthInt, depthInt); // Farthest depth
    InterlockedMax(tileMaxDepthInt, depthInt); // Nearest depth
    GroupMemoryBarrierWithGroupSync();

    // Convert back to float
    float tileMinDepth = asfloat(tileMinDepthInt);
    float tileMaxDepth = asfloat(tileMaxDepthInt);
    // ...
}
```

#### 2.3.3 视锥体构建与光源相交测试

```hlsl
// Step 3: Construct tile frustum planes
// Each tile forms a frustum from camera through the tile's screen-space bounds
float4 frustumPlanes[6];
ConstructTileFrustum(groupId.xy, tileMinDepth, tileMaxDepth, frustumPlanes);

// Step 4: Cull lights against tile frustum
groupshared uint tileLightCount;
groupshared uint tileLightIndices[MAX_LIGHTS_PER_TILE];

if (groupIndex == 0)
    tileLightCount = 0;
GroupMemoryBarrierWithGroupSync();

// Each thread tests a subset of lights
for (uint i = groupIndex; i < totalLightCount; i += TILE_SIZE * TILE_SIZE)
{
    Light light = LightBuffer[i];

    bool intersects = false;

    if (light.type == LIGHT_TYPE_POINT)
    {
        // Sphere-Frustum intersection test
        intersects = SphereFrustumIntersect(
            light.positionVS, light.range, frustumPlanes);
    }
    else if (light.type == LIGHT_TYPE_SPOT)
    {
        // Cone-Frustum intersection test
        intersects = ConeFrustumIntersect(
            light.positionVS, light.directionVS,
            light.range, light.spotAngle, frustumPlanes);
    }

    if (intersects)
    {
        uint index;
        InterlockedAdd(tileLightCount, 1, index);
        if (index < MAX_LIGHTS_PER_TILE)
            tileLightIndices[index] = i;
    }
}
GroupMemoryBarrierWithGroupSync();

// Step 5: Write light list to global buffer
if (groupIndex == 0)
{
    uint tileIndex = groupId.y * numTilesX + groupId.x;
    uint offset = tileIndex * MAX_LIGHTS_PER_TILE;
    LightCountBuffer[tileIndex] = min(tileLightCount, MAX_LIGHTS_PER_TILE);

    for (uint j = 0; j < min(tileLightCount, MAX_LIGHTS_PER_TILE); j++)
        LightIndexBuffer[offset + j] = tileLightIndices[j];
}
```

### 2.4 Stage 3: Forward Shading Pass

```hlsl
// Forward+ Fragment Shader
float4 ForwardPlusLitFrag(Varyings input) : SV_Target
{
    // Step 1: Determine which tile this pixel belongs to
    uint2 tileCoord = uint2(input.positionCS.xy) / TILE_SIZE;
    uint tileIndex = tileCoord.y * numTilesX + tileCoord.x;

    // Step 2: Get light list for this tile
    uint lightCount = LightCountBuffer[tileIndex];
    uint lightOffset = tileIndex * MAX_LIGHTS_PER_TILE;

    // Step 3: Surface data
    SurfaceData surface = GetSurfaceData(input);
    BRDFData brdf = GetBRDFData(surface);

    // Step 4: Accumulate lighting from tile's light list
    float3 color = float3(0, 0, 0);

    // Directional lights (global, not per-tile)
    color += CalculateDirectionalLight(brdf, mainLight);

    // Per-tile additional lights
    for (uint i = 0; i < lightCount; i++)
    {
        uint lightIndex = LightIndexBuffer[lightOffset + i];
        Light light = AdditionalLightBuffer[lightIndex];
        color += CalculateAdditionalLight(brdf, light, input.positionWS);
    }

    // Step 5: Add ambient, emission, etc.
    color += CalculateGI(brdf, input);
    color += surface.emission;

    return float4(color, surface.alpha);
}
```

### 2.5 数学原理：Tile Frustum 构建

```
                    Near Plane
                   ┌─────────┐
                  ╱│         │╲
                ╱  │  Tile   │  ╲
              ╱    │ Frustum │    ╲
            ╱      │         │      ╲
  Camera  ◉────────┤ zMin    │────────  Far
            ╲      │ zMax    │      ╱
              ╲    │         │    ╱
                ╲  │         │  ╱
                  ╲│         │╱
                   └─────────┘

  Tile Frustum = 4 side planes + near plane (zMax) + far plane (zMin)
```

```hlsl
// Construct frustum for a tile
void ConstructTileFrustum(
    uint2 tileId,
    float minDepth, float maxDepth,
    out float4 planes[6])
{
    // Tile corners in NDC
    float2 tileMin = float2(tileId) * TILE_SIZE / screenSize * 2.0 - 1.0;
    float2 tileMax = float2(tileId + 1) * TILE_SIZE / screenSize * 2.0 - 1.0;
    tileMin.y = -tileMin.y; // Flip Y for NDC
    tileMax.y = -tileMax.y;

    // Convert to view-space rays
    float3 corners[4];
    corners[0] = ScreenToView(float2(tileMin.x, tileMin.y)); // Top-Left
    corners[1] = ScreenToView(float2(tileMax.x, tileMin.y)); // Top-Right
    corners[2] = ScreenToView(float2(tileMax.x, tileMax.y)); // Bottom-Right
    corners[3] = ScreenToView(float2(tileMin.x, tileMax.y)); // Bottom-Left

    // 4 side planes (normals point inward)
    planes[0] = CreatePlane(float3(0,0,0), corners[0], corners[1]); // Top
    planes[1] = CreatePlane(float3(0,0,0), corners[1], corners[2]); // Right
    planes[2] = CreatePlane(float3(0,0,0), corners[2], corners[3]); // Bottom
    planes[3] = CreatePlane(float3(0,0,0), corners[3], corners[0]); // Left

    // Near/Far planes from depth bounds
    float nearZ = LinearDepth(maxDepth); // Reversed-Z: max depth = near
    float farZ  = LinearDepth(minDepth); // Reversed-Z: min depth = far
    planes[4] = float4(0, 0, -1,  nearZ); // Near plane
    planes[5] = float4(0, 0,  1, -farZ);  // Far plane
}
```

## 3. 与其他渲染路径对比

### 3.1 Forward vs Deferred vs Forward+

| 特性 | Forward | Deferred | Forward+ |
|------|---------|----------|----------|
| **光源复杂度** | O(objects × lights) | O(pixels × lights) | O(pixels × lights_per_tile) |
| **多光源性能** | ❌ 差 (>8 光源急剧下降) | ✅ 好 | ✅ 好 |
| **MSAA 支持** | ✅ 原生支持 | ❌ 需要特殊处理 | ✅ 原生支持 |
| **透明物体** | ✅ 天然支持 | ❌ 需要额外 Forward Pass | ✅ 天然支持 |
| **G-Buffer 带宽** | ✅ 无 | ❌ 高 (80-160 bytes/pixel) | ✅ 无 |
| **材质多样性** | ✅ 无限制 | ❌ 受 G-Buffer 格式限制 | ✅ 无限制 |
| **Overdraw 开销** | ❌ 高 | ✅ 低 (deferred lighting) | ⚠️ 中 (Pre-Z 缓解) |
| **GPU 内存** | ✅ 低 | ❌ 高 (多张 RT) | ✅ 低 |
| **Compute Shader** | 不需要 | 不需要 | ✅ 需要 |
| **最低硬件要求** | DX9 / ES2.0 | DX10 / ES3.0 | DX11 / ES3.1 |

### 3.2 性能特征曲线

```
渲染耗时
  ▲
  │
  │     Forward
  │    ╱
  │   ╱
  │  ╱         Deferred
  │ ╱         ╱
  │╱     ────╱──────────────────
  │    ╱   ╱
  │   ╱  ╱    Forward+
  │──╱─╱─────────────────────────
  │ ╱╱
  │╱╱
  ┼──────────────────────────────▶ 光源数量
  0    8    32   128   512  1024

  Forward:  线性增长，>8 光源后急剧恶化
  Deferred: 基础开销高，但光源增长平缓
  Forward+: 基础开销中等，光源增长最平缓
```

### 3.3 带宽对比 (1080p)

| 渲染路径 | G-Buffer 写入 | G-Buffer 读取 | 总带宽开销 |
|----------|--------------|--------------|------------|
| Forward | 0 | 0 | ~基准 |
| Deferred | ~50 MB/frame | ~50 MB/frame | ~100 MB/frame 额外 |
| Forward+ | 0 | 0 | ~Light List 缓冲 (~1-2 MB) |

### 4.3 CPU 端准备工作

```csharp
// CPU-side preparation (pseudo-code)
public void PrepareForwardPlus(Camera camera, List<Light> lights)
{
    // 1. Upload light data to GPU buffer
    NativeArray<LightData> lightDataArray = new NativeArray<LightData>(lights.Count);
    for (int i = 0; i < lights.Count; i++)
    {
        lightDataArray[i] = new LightData
        {
            positionVS = camera.worldToCameraMatrix.MultiplyPoint(lights[i].position),
            color      = lights[i].color * lights[i].intensity,
            range      = lights[i].range,
            type       = lights[i].type,
            spotAngle  = lights[i].spotAngle,
            directionVS = camera.worldToCameraMatrix.MultiplyVector(lights[i].forward)
        };
    }
    lightBuffer.SetData(lightDataArray);

    // 2. Calculate tile dimensions
    int numTilesX = Mathf.CeilToInt(camera.pixelWidth  / (float)TILE_SIZE);
    int numTilesY = Mathf.CeilToInt(camera.pixelHeight / (float)TILE_SIZE);
    int totalTiles = numTilesX * numTilesY;

    // 3. Allocate/resize light index buffer if needed
    int bufferSize = totalTiles * MAX_LIGHTS_PER_TILE;
    if (lightIndexBuffer == null || lightIndexBuffer.count < bufferSize)
    {
        lightIndexBuffer?.Release();
        lightIndexBuffer = new ComputeBuffer(bufferSize, sizeof(uint));
    }

    // 4. Set compute shader parameters
    lightCullingCS.SetBuffer(0, "_LightBuffer", lightBuffer);
    lightCullingCS.SetBuffer(0, "_LightIndexBuffer", lightIndexBuffer);
    lightCullingCS.SetBuffer(0, "_LightCountBuffer", lightCountBuffer);
    lightCullingCS.SetTexture(0, "_DepthTexture", depthRT);
    lightCullingCS.SetInts("_NumTiles", numTilesX, numTilesY);
    lightCullingCS.SetInt("_TotalLightCount", lights.Count);
    lightCullingCS.SetMatrix("_InvProjection", camera.projectionMatrix.inverse);

    // 5. Dispatch compute shader
    lightCullingCS.Dispatch(0, numTilesX, numTilesY, 1);
}
```

## 5. 关键实现细节

### 5.1 Tile Size 选择

| Tile Size | 优点 | 缺点 | 适用场景 |
|-----------|------|------|----------|
| **8×8** | 更精细的剔除，光源利用率高 | Compute 开销大，线程组小 | 光源密集场景 |
| **16×16** | 平衡的剔除精度和开销 | 通用最佳选择 | **推荐默认值** |
| **32×32** | Compute 开销最小 | 剔除精度低，每 Tile 光源多 | 光源稀疏场景 |
| **64×64** | 极低 Compute 开销 | 剔除几乎无效 | 不推荐 |

**推荐**：16×16 是业界标准，256 个线程正好匹配一个 Wavefront/Warp 的整数倍。

### 5.2 Depth Range 优化：2.5D Culling

标准 Forward+ 每个 Tile 只有一个 [Zmin, Zmax] 范围，当场景深度不连续时（如近处有物体，远处有天空），会导致大量误判。

```
标准 Tile Depth Range 问题：

  Side View:
  ─────────────────────────────────────────
  Camera ◉ ──── [Near Object] ──── gap ──── [Far Object]
                 ▲                           ▲
                Zmax                        Zmin
                 │◄──── Tile Depth Range ───▶│
                 │     (包含大量空白区域)       │
                 │     光源在 gap 中也被算入    │
```

**2.5D Culling 解决方案**：将深度范围细分为多个 Cell。

```hlsl
// 2.5D Culling: Subdivide depth range into cells
#define DEPTH_CELLS 32

groupshared uint depthMask; // 32-bit mask, each bit = one depth cell

// In light culling:
// 1. Each pixel sets its corresponding depth cell bit
uint cellIndex = uint((linearDepth - tileMinDepth) / (tileMaxDepth - tileMinDepth) * DEPTH_CELLS);
InterlockedOr(depthMask, 1u << cellIndex);

// 2. For each light, compute which cells it overlaps
uint lightMinCell = ...;
uint lightMaxCell = ...;
uint lightMask = ((1u << (lightMaxCell - lightMinCell + 1)) - 1) << lightMinCell;

// 3. Only accept light if its cells overlap with actual geometry
if ((lightMask & depthMask) != 0)
{
    // Light affects this tile
}
```

### 5.3 Light Index Buffer 布局

```
方案 A: 固定大小数组 (简单但浪费内存)
┌──────────────────────────────────────────────────┐
│ Tile 0: [L3, L7, L12, -, -, -, -, -, ...]       │ MAX_LIGHTS_PER_TILE slots
│ Tile 1: [L3, L7, L8, L12, L15, -, -, -, ...]    │
│ Tile 2: [L8, -, -, -, -, -, -, -, ...]           │
│ ...                                              │
└──────────────────────────────────────────────────┘
Memory: numTiles × MAX_LIGHTS_PER_TILE × 4 bytes

方案 B: 紧凑列表 + 偏移表 (节省内存)
┌─────────────────────────────────────────┐
│ Offset Table:                           │
│ Tile 0: offset=0,  count=3              │
│ Tile 1: offset=3,  count=5              │
│ Tile 2: offset=8,  count=1              │
├─────────────────────────────────────────┤
│ Light Index List:                       │
│ [L3, L7, L12, L3, L7, L8, L12, L15, L8]│
└─────────────────────────────────────────┘
Memory: numTiles × 8 bytes + totalAssignments × 4 bytes
```

### 5.4 Sphere-Frustum 相交测试

```hlsl
// Point light: Sphere vs Frustum test
bool SphereFrustumIntersect(float3 center, float radius, float4 planes[6])
{
    for (int i = 0; i < 6; i++)
    {
        float dist = dot(planes[i].xyz, center) + planes[i].w;
        if (dist < -radius)
            return false; // Sphere is completely outside this plane
    }
    return true;
}

// Spot light: Conservative Cone vs Frustum test
bool ConeFrustumIntersect(
    float3 apex, float3 dir, float range, float angle, float4 planes[6])
{
    // Use bounding sphere of cone as conservative test
    float cosAngle = cos(angle * 0.5);
    float sinAngle = sin(angle * 0.5);

    // Bounding sphere center and radius
    float3 sphereCenter;
    float sphereRadius;

    if (cosAngle > 0.707) // angle < 90 degrees
    {
        sphereCenter = apex + dir * (range * cosAngle);
        sphereRadius = range * sinAngle;
    }
    else
    {
        sphereCenter = apex + dir * (range * 0.5);
        sphereRadius = range * 0.5;
    }

    return SphereFrustumIntersect(sphereCenter, sphereRadius, planes);
}
```

## 7. 注意事项与常见陷阱

### 7.1 ⚠️ Depth Pre-Pass 的必要性

**问题**：如果跳过 Depth Pre-Pass，Light Culling 没有深度信息，只能使用相机的 near/far 作为 Tile 深度范围，导致几乎所有光源都被分配到所有 Tile。

**解决**：Depth Pre-Pass 是 Forward+ 的**必要前提**，不可省略。

```
❌ 无 Depth Pre-Pass:
  Tile depth range = [camera.near, camera.far]
  → 几乎所有光源都通过测试
  → Forward+ 退化为普通 Forward

✅ 有 Depth Pre-Pass:
  Tile depth range = [actual Zmin, actual Zmax]
  → 精确剔除不相关光源
  → Forward+ 发挥正常性能
```

### 7.2 ⚠️ Alpha Test 物体与 Depth Pre-Pass

**问题**：Alpha Test 物体在 Depth Pre-Pass 中需要采样纹理，破坏 Early-Z 优化，增加 Pre-Pass 开销。

**解决方案**：

```
方案 1: 分离渲染
  Pass 1a: Opaque Depth Pre-Pass (fast, no texture sampling)
  Pass 1b: Alpha-Test Depth Pre-Pass (slower, with texture sampling)
  Pass 2:  Light Culling
  Pass 3:  Forward Shading (depth test = Equal for opaque)

方案 2: 忽略 Alpha-Test 物体的深度
  - Alpha-Test 物体不参与 Depth Pre-Pass
  - Light Culling 的深度范围可能不够精确
  - 但避免了 Pre-Pass 中的纹理采样开销
  - 适用于 Alpha-Test 物体较少的场景

方案 3: Alpha to Coverage (推荐)
  - 使用 MSAA + Alpha to Coverage 替代 Alpha Test
  - 不需要 discard/clip，不破坏 Early-Z
  - 视觉质量更好（抗锯齿）
```

### 7.3 ⚠️ Reversed-Z 深度处理

**问题**：现代引擎普遍使用 Reversed-Z（近处=1，远处=0），在计算 Tile 深度范围时容易搞反。

```hlsl
// ❌ 错误：假设标准深度 (near=0, far=1)
InterlockedMin(tileMinDepthInt, depthInt); // 这是最远的
InterlockedMax(tileMaxDepthInt, depthInt); // 这是最近的

// ✅ 正确：Reversed-Z 下
// depthInt 越大 = 深度值越大 = 越近 (Reversed-Z)
// tileMinDepthInt = 最小深度值 = 最远
// tileMaxDepthInt = 最大深度值 = 最近
float tileFarDepth  = asfloat(tileMinDepthInt); // 最远
float tileNearDepth = asfloat(tileMaxDepthInt); // 最近
```

### 7.4 ⚠️ Compute Shader 同步

**问题**：Light Culling Compute Shader 中大量使用 GroupShared Memory 和原子操作，同步不当会导致竞态条件。

```hlsl
// ❌ 错误：缺少 barrier
tileMinDepthInt = 0xFFFFFFFF;
// 其他线程可能在初始化完成前就开始写入
InterlockedMin(tileMinDepthInt, myDepth);

// ✅ 正确：确保初始化完成后再操作
if (groupIndex == 0)
{
    tileMinDepthInt = 0xFFFFFFFF;
    tileMaxDepthInt = 0;
    tileLightCount = 0;
}
GroupMemoryBarrierWithGroupSync(); // 等待初始化完成

InterlockedMin(tileMinDepthInt, myDepth);
GroupMemoryBarrierWithGroupSync(); // 等待所有线程完成深度写入

// Now safe to use tileMinDepthInt / tileMaxDepthInt
```

### 7.5 ⚠️ MAX_LIGHTS_PER_TILE 溢出

**问题**：当某个 Tile 的光源数超过 `MAX_LIGHTS_PER_TILE` 时，多余的光源被丢弃，导致光照缺失。

```hlsl
// ❌ 错误：不检查上限
InterlockedAdd(tileLightCount, 1, index);
tileLightIndices[index] = lightIdx; // 可能越界！

// ✅ 正确：检查上限
InterlockedAdd(tileLightCount, 1, index);
if (index < MAX_LIGHTS_PER_TILE)
    tileLightIndices[index] = lightIdx;
// else: light is dropped for this tile
```

**建议值**：
- 一般场景：`MAX_LIGHTS_PER_TILE = 256`
- 光源密集场景：`MAX_LIGHTS_PER_TILE = 512`
- 移动端：`MAX_LIGHTS_PER_TILE = 64-128`

### 7.6 ⚠️ 移动端 TBR 架构注意事项

**问题**：移动端 GPU（Mali, Adreno, PowerVR）使用 Tile-Based Rendering (TBR)，Depth Pre-Pass 会导致额外的 Tile 刷新。

```
移动端 TBR 下的 Forward+ 问题：

  Pass 1 (Depth Pre-Pass):
    GPU 内部 Tile → 写入 Depth → Flush to memory
                                    ↓
  Pass 2 (Light Culling):           ↓
    Read Depth from memory ←────────┘
    Compute light lists → Write to memory
                                    ↓
  Pass 3 (Forward Shading):         ↓
    GPU 内部 Tile → Read Depth (again!) + Read Light Lists
    → Full shading → Flush Color to memory

  问题：Depth Buffer 被写入内存又读回，破坏了 TBR 的 on-chip 优势
```

**移动端优化方案**：

```
方案 1: Cluster Forward (推荐移动端)
  - 不依赖 Depth Pre-Pass
  - 在 3D 空间中划分 Cluster 而非 2D Tile
  - 避免额外的 Depth Pass

方案 2: 利用 Subpass / Framebuffer Fetch
  - Vulkan Subpass 可以在同一 Render Pass 内传递深度
  - 避免 Depth Buffer 的 memory round-trip
  - 但 Light Culling 的 Compute Shader 仍需要单独 Pass

方案 3: 接受 Depth Pre-Pass 开销
  - 如果光源数量足够多 (>32)，Forward+ 的收益仍然大于开销
  - 确保 Depth Pre-Pass 使用最简 shader
```

### 7.7 ⚠️ 动态分辨率与 Tile 对齐

**问题**：动态分辨率变化时，Tile 数量和大小需要重新计算。

```csharp
// 每帧检查分辨率变化
int currentWidth  = (int)(camera.pixelWidth  * dynamicResScale);
int currentHeight = (int)(camera.pixelHeight * dynamicResScale);

int numTilesX = Mathf.CeilToInt(currentWidth  / (float)TILE_SIZE);
int numTilesY = Mathf.CeilToInt(currentHeight / (float)TILE_SIZE);

// 边缘 Tile 可能不满 TILE_SIZE，需要在 Compute Shader 中处理
// 确保边缘线程不越界读取深度
if (pixelCoord.x >= currentWidth || pixelCoord.y >= currentHeight)
    return; // Out of bounds
```

### 7.8 ⚠️ 光源闪烁问题

**问题**：当光源恰好在 Tile 边界时，相邻帧可能因为浮点精度导致光源在 Tile 间跳跃，产生闪烁。

**解决**：在相交测试中添加小的 epsilon 扩展。

```hlsl
// Add small padding to light radius for intersection test
float testRadius = light.range + LIGHT_CULLING_EPSILON; // e.g., 0.1
bool intersects = SphereFrustumIntersect(light.positionVS, testRadius, frustumPlanes);
```

### 9.2 Unity URP Forward+ Shader 关键代码

```hlsl
// URP Forward+ 片段着色器中的光照循环
half4 UniversalFragmentForwardPlus(InputData inputData, SurfaceData surfaceData)
{
    // Main directional light
    Light mainLight = GetMainLight(inputData.shadowCoord);
    half3 color = LightingPhysicallyBased(brdfData, mainLight, inputData);

    // Forward+ additional lights
    // URP uses _AdditionalLightsIndices and _AdditionalLightsPosition buffers
    uint pixelLightCount = GetAdditionalLightsCount();

    // Forward+ path: iterate per-tile light list
    LIGHT_LOOP_BEGIN(pixelLightCount)
        Light light = GetAdditionalLight(lightIndex, inputData.positionWS);
        color += LightingPhysicallyBased(brdfData, light, inputData);
    LIGHT_LOOP_END

    // GI
    color += GlobalIllumination(brdfData, inputData);

    return half4(color, surfaceData.alpha);
}
```

## 10. 最佳实践检查清单

### 10.1 实现检查清单

```
□ Depth Pre-Pass
  □ 不透明物体使用最简 depth-only shader
  □ Alpha-test 物体单独处理或使用 Alpha to Coverage
  □ 确认 Reversed-Z 深度方向正确

□ Light Culling
  □ Tile Size 设置为 16×16
  □ GroupShared Memory 正确初始化和同步
  □ 所有 GroupMemoryBarrierWithGroupSync() 位置正确
  □ MAX_LIGHTS_PER_TILE 设置合理 (256 for PC, 64-128 for mobile)
  □ 光源溢出有保护 (index < MAX_LIGHTS_PER_TILE)
  □ Reversed-Z 深度比较方向正确
  □ 边缘 Tile 越界检查
  □ 光源相交测试添加 epsilon 防闪烁

□ Forward Shading
  □ Depth Test 设置为 Equal (利用 Pre-Z 避免 overdraw)
  □ 正确读取 Tile Light List
  □ Tile 坐标计算正确 (pixelCoord / TILE_SIZE)
  □ 方向光单独处理 (不参与 Tile Culling)

□ 透明物体
  □ 使用相同的 Tile Light List
  □ Back-to-front 排序
  □ Depth Test 设置为 LessEqual, Depth Write Off
```

// Debug visualization: Light count per tile (heat map)
float4 DebugLightCountFrag(Varyings input) : SV_Target
{
    uint2 tileCoord = uint2(input.positionCS.xy) / TILE_SIZE;
    uint tileIndex = tileCoord.y * numTilesX + tileCoord.x;
    uint lightCount = LightCountBuffer[tileIndex];

    // Heat map: blue(0) → green(8) → yellow(16) → red(32+)
    float t = saturate(lightCount / 32.0);
    float3 color;
    if (t < 0.25)
        color = lerp(float3(0,0,1), float3(0,1,1), t * 4.0);
    else if (t < 0.5)
        color = lerp(float3(0,1,1), float3(0,1,0), (t - 0.25) * 4.0);
    else if (t < 0.75)
        color = lerp(float3(0,1,0), float3(1,1,0), (t - 0.5) * 4.0);
    else
        color = lerp(float3(1,1,0), float3(1,0,0), (t - 0.75) * 4.0);

    return float4(color, 1.0);
}

// Debug: Tile grid overlay
float4 DebugTileGridFrag(Varyings input) : SV_Target
{
    uint2 pixelCoord = uint2(input.positionCS.xy);
    bool isBorder = (pixelCoord.x % TILE_SIZE == 0) || (pixelCoord.y % TILE_SIZE == 0);
    return isBorder ? float4(1,1,0,0.5) : float4(0,0,0,0);
}
