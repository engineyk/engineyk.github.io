# 投影系统与贴花渲染

> 本文系统梳理 **阴影投影（Shadow Projection）** 和 **贴花（Decal）** 两大渲染技术，重点解决"场景特别多特效时如何高效贴花"的实战问题。基于 UE5 源码级分析。

---

## 一、投影（Projection）技术体系

### 1.1 投影的本质

```
投影 = 将一个空间的信息映射到另一个空间的表面上

常见投影类型：
├─ 阴影投影（Shadow Projection）
│   └─ 将 Shadow Map 深度信息投影到场景表面，判断遮挡关系
├─ 纹理投影（Texture Projection / Projector）
│   └─ 将一张纹理从某个视角投射到场景表面（如弹孔、血迹）
├─ 贴花投影（Decal Projection）
│   └─ 将贴花材质投影到场景几何体上（本质是纹理投影的延伸）
└─ 平面反射投影（Planar Reflection）
    └─ 将场景从反射平面的镜像视角渲染
```

### 1.2 投影的数学基础

```
投影矩阵的核心：将世界空间坐标变换到投影空间 UV 坐标

对于阴影投影：
  WorldPos → LightViewMatrix → LightProjectionMatrix → ShadowMapUV
  
  ShadowMapUV.xy = 用于采样 Shadow Map
  ShadowMapUV.z  = 当前像素在光源空间的深度
  ShadowMap.r    = Shadow Map 中存储的最近深度
  
  if (ShadowMapUV.z > ShadowMap.r + Bias)
      pixel is in shadow;

对于纹理投影：
  WorldPos → ProjectorViewMatrix → ProjectorProjMatrix → ProjectorUV
  
  ProjectorUV.xy = 用于采样投影纹理
  ProjectorUV.z  = 用于深度衰减（可选）
```

## 二、阴影投影（Shadow Projection）

### 2.1 UE5 阴影投影管线

```
UE5 Shadow Projection Pipeline:

1. Shadow Map Generation (Shadow Pass)
   ├─ FProjectedShadowInfo::RenderDepth()
   │   └─ FShadowDepthPassMeshProcessor → 渲染 Shadow Depth Map
   ├─ CSM (Cascaded Shadow Maps)        → 方向光
   ├─ Cube Shadow Map                   → 点光源（6面）
   └─ Spot Shadow Map                   → 聚光灯

2. Shadow Projection (Lighting Pass)
   ├─ FSceneRenderer::RenderShadowProjections()
   │   └─ 遍历所有 FProjectedShadowInfo
   │       ├─ RenderProjection()                  → 方向光/聚光灯
   │       └─ RenderOnePassPointLightProjection() → 点光源
   │
   ├─ TShadowProjectionPS<Quality, bUseFadePlane, bModulated, bTransmission, SubPixel>
   │   └─ 模板化像素着色器，控制阴影质量和特性
   │
   └─ 输出到 ScreenShadowMaskTexture（阴影遮罩纹理）

3. Shadow Application (Deferred Lighting)
   └─ 在光照计算中采样 ScreenShadowMaskTexture
       └─ FinalColor *= ShadowMask
```

### 2.2 TShadowProjectionPS 源码解析

```cpp
// UE5 Source: ShadowRendering.h
// 模板参数控制阴影投影的各种特性

template<uint32 Quality,           // 阴影质量等级 (1-5)
         bool bUseFadePlane,       // 是否使用淡出平面（CSM 级联过渡）
         bool bModulatedShadows,   // 调制阴影（移动端）
         bool bUseTransmission,    // 透射阴影（SSS 材质）
         bool SubPixelShadow>      // 子像素阴影（毛发）
class TShadowProjectionPS : public FShadowProjectionPixelShaderInterface
{
    // 核心参数
    FShadowProjectionShaderParameters ProjectionParameters;     // 投影矩阵等
    FShaderParameter ShadowFadeFraction;                        // 阴影淡出系数
    FShaderParameter ShadowSharpen;                             // 阴影锐化
    FShaderParameter LightPosition;                             // 光源位置 + 1/Radius

    void SetParameters(...)
    {
        // 设置投影参数（Shadow Map 纹理、投影矩阵、Bias 等）
        ProjectionParameters.Set(BatchedParameters, View, ShadowInfo, ...);
        
        // 阴影淡出（远处阴影渐隐）
        SetShaderValue(BatchedParameters, ShadowFadeFraction, 
                       ShadowInfo->FadeAlphas[ViewIndex]);
        
        // 阴影锐化（ShadowSharpen * 7 + 1）
        SetShaderValue(BatchedParameters, ShadowSharpen, 
                       LightProxy.GetShadowSharpen() * 7.0f + 1.0f);
    }
};
```

### 2.3 阴影投影的 Shader 核心逻辑

```hlsl
// ShadowProjectionPixelShader.usf (simplified)

float4 MainPS(float4 SvPosition : SV_POSITION) : SV_Target0
{
    // 1. Reconstruct world position from depth
    float SceneDepth = CalcSceneDepth(SvPosition.xy);
    float3 WorldPosition = ReconstructWorldPosition(SvPosition.xy, SceneDepth);
    
    // 2. Transform to shadow map space
    float4 ShadowPosition = mul(float4(WorldPosition, 1), ScreenToShadowMatrix);
    float2 ShadowUV = ShadowPosition.xy / ShadowPosition.w;
    float PixelDepth = ShadowPosition.z;
    
    // 3. Sample shadow map with PCF filtering
    float Shadow = 0;
    
    #if SHADOW_QUALITY == 1  // Low: 1 sample
        Shadow = ShadowDepthTexture.SampleCmpLevelZero(
            ShadowDepthSampler, ShadowUV, PixelDepth - DepthBias);
    
    #elif SHADOW_QUALITY == 5  // Ultra: 16+ samples PCF
        for (int i = 0; i < NumSamples; i++)
        {
            float2 Offset = PoissonDisk[i] * FilterRadius;
            Shadow += ShadowDepthTexture.SampleCmpLevelZero(
                ShadowDepthSampler, ShadowUV + Offset, PixelDepth - DepthBias);
        }
        Shadow /= NumSamples;
    #endif
    
    // 4. Apply fade and sharpen
    Shadow = saturate((Shadow - 0.5) * ShadowSharpen + 0.5);
    Shadow = lerp(1.0, Shadow, ShadowFadeFraction);
    
    // 5. PCSS (Percentage Closer Soft Shadows) - optional
    #if USE_PCSS
        float BlockerDepth = FindBlockerDepth(ShadowUV, PixelDepth);
        float PenumbraSize = (PixelDepth - BlockerDepth) / BlockerDepth * LightSize;
        // Use PenumbraSize to scale PCF filter radius
    #endif
    
    return Shadow;
}
```

### 2.4 各类阴影投影方案对比

| 方案             | 光源类型      | Shader 类                                        | 特点                   |
| ---------------- | ------------- | ------------------------------------------------ | ---------------------- |
| **Standard PCF** | 方向光/聚光灯 | `TShadowProjectionPS<Q>`                         | 基础 PCF 滤波          |
| **PCSS**         | 方向光        | `TDirectionalPercentageCloserShadowProjectionPS` | 软阴影，半影随距离变化 |
| **Spot PCSS**    | 聚光灯        | `TSpotPercentageCloserShadowProjectionPS`        | 聚光灯软阴影           |
| **Point Light**  | 点光源        | `TOnePassPointShadowProjectionPS`                | 6面 Cube Map 单 Pass   |
| **Modulated**    | 移动端        | `TModulatedShadowProjection`                     | 调制阴影（颜色可调）   |
| **Translucency** | 半透明        | `TShadowProjectionFromTranslucencyPS`            | 半透明自阴影           |
| **VSM**          | 方向光        | `RenderVirtualShadowMapProjection`               | 虚拟阴影贴图（UE5）    |

### 2.5 RenderShadowProjections 执行流程

```
FSceneRenderer::RenderShadowProjections()
│
├─ CheckShadowDepthRenderCompleted()  // 确保 Shadow Map 已渲染完成
│
├─ for each View:
│   ├─ View.BeginRenderView()
│   │
│   ├─ Setup CommonPassParameters:
│   │   ├─ SceneTextures (Depth, GBuffer)
│   │   ├─ HairStrands (if SubPixelShadow)
│   │   ├─ Substrate (if enabled)
│   │   └─ RenderTargets[0] = OutputTexture (ShadowMask)
│   │       DepthStencil = SceneDepth (Read) + Stencil (Write)
│   │
│   └─ for each FProjectedShadowInfo:
│       ├─ Skip if FadeAlpha < 1/256 (too faded)
│       │
│       ├─ if bOnePassPointLightShadow:
│       │   └─ RenderOnePassPointLightProjection()
│       │       └─ TOnePassPointShadowProjectionPS
│       │
│       └─ else:
│           └─ RenderProjection()
│               ├─ Setup Stencil (light volume masking)
│               ├─ Select shader based on Quality + Features
│               │   ├─ PCSS? → TDirectionalPercentageCloserShadowProjectionPS
│               │   ├─ Modulated? → TModulatedShadowProjection
│               │   ├─ Transmission? → TShadowProjectionPS<Q,F,false,true>
│               │   └─ Standard → TShadowProjectionPS<Q,F>
│               └─ Draw fullscreen quad / light volume
```

## 三、贴花（Decal）系统

### 3.1 贴花的本质

```
贴花 = 将一个材质投影到场景几何体表面

渲染方式：
  1. 渲染一个 Box/Sphere 体积
  2. 对体积内的每个像素，从深度缓冲重建世界坐标
  3. 将世界坐标变换到贴花的局部空间
  4. 用局部空间 UV 采样贴花材质
  5. 将结果混合到 GBuffer / SceneColor

核心优势：
  ✅ 不需要修改底层几何体
  ✅ 可以叠加在任意表面上
  ✅ 支持动态创建和销毁
  ✅ 支持修改 GBuffer 属性（颜色、法线、粗糙度等）
```

### 3.2 UE5 贴花渲染阶段（EDecalRenderStage）

```
UE5 Decal Render Stages:

EDecalRenderStage::BeforeBasePass (DBuffer Decals)
│   ├─ 在 Base Pass 之前渲染
│   ├─ 写入 DBuffer (DBufferA/B/C + Mask)
│   │   ├─ DBufferA: BaseColor + Opacity
│   │   ├─ DBufferB: Normal + Opacity  
│   │   ├─ DBufferC: Metallic/Specular/Roughness + Opacity
│   │   └─ DBuffer Mask: 标记哪些像素被贴花覆盖
│   ├─ Base Pass 读取 DBuffer 并混合
│   └─ ★ 最高质量，参与完整光照计算
│
EDecalRenderStage::BeforeLighting (Deferred Decals)
│   ├─ 在 Base Pass 之后、Lighting 之前
│   ├─ 直接修改 GBuffer (SceneColor + GBufferA/B/C)
│   ├─ 两种 RenderTarget 模式：
│   │   ├─ SceneColorAndGBuffer (写入法线)
│   │   └─ SceneColorAndGBufferNoNormal (不写法线)
│   └─ ★ 性能好，但法线修改不参与某些光照特性
│
EDecalRenderStage::Emissive
│   ├─ 只写入 SceneColor（自发光）
│   └─ 适合发光效果
│
EDecalRenderStage::AmbientOcclusion
│   ├─ 只写入 AO 通道
│   └─ 适合局部 AO 修改
│
EDecalRenderStage::Mobile / MobileBeforeLighting
│   ├─ 移动端专用阶段
│   └─ 简化的渲染路径
```

### 3.3 贴花渲染管线源码流程

```
PC Deferred Pipeline:

FDeferredShadingSceneRenderer::Render()
│
├─ RenderDBuffer()                          // DBuffer Decals (BeforeBasePass)
│   ├─ DecalVisibilityTaskData->FinishRelevantDecals(BeforeBasePass)
│   ├─ GetDeferredDecalPassTextures()       → 准备纹理资源
│   │   ├─ SceneTextures (Depth, GBuffer)
│   │   ├─ DBufferTextures (A/B/C/Mask)
│   │   └─ EyeAdaptationBuffer
│   └─ AddDeferredDecalPass()               → 提交 RDG Pass
│       ├─ Sort decals by SortOrder
│       ├─ for each visible decal:
│       │   ├─ Setup blend state (GetDecalBlendState_DBuffer)
│       │   ├─ Bind decal material
│       │   └─ Draw decal box volume
│       └─ RenderMeshDecals()               → Mesh Decals
│
├─ RenderBasePass()  // Base Pass reads DBuffer
│   └─ Shader reads DBufferA/B/C and blends with material
│
├─ AddDeferredDecalPass(BeforeLighting)     // GBuffer Decals
│   ├─ GetDeferredDecalPassTextures()
│   └─ Render decals directly to GBuffer
│
├─ Lighting Pass                            // Uses modified GBuffer
│
├─ AddDeferredDecalPass(Emissive)           // Emissive Decals
│
└─ AddDeferredDecalPass(AmbientOcclusion)   // AO Decals


Mobile Pipeline:

FMobileSceneRenderer::Render()
│
├─ RenderDBuffer()  // Mobile DBuffer (if enabled)
│   └─ Similar to PC but simplified
│
├─ RenderDeferredMultiPassDecals()  // Mobile Deferred Decals
│   ├─ Two render target modes:
│   │   ├─ SceneColorAndGBuffer
│   │   └─ SceneColorAndGBufferNoNormal
│   ├─ BuildMeshRenderingCommands() (if GPU Scene enabled)
│   └─ GraphBuilder.AddPass("Decals_XXX")
│       └─ RenderDecals()
│
└─ Post-processing
```

### 3.4 贴花混合模式详解

```
UE5 Decal Blend Modes:

1. Translucent (BLEND_Translucent)
   ├─ 最常用的模式
   ├─ Alpha Blend: SrcAlpha * Src + (1-SrcAlpha) * Dst
   ├─ 可写入：Emissive, Normal, Roughness/Metallic/Specular, BaseColor
   └─ 通过 RenderTargetWriteMask 控制写入哪些通道

2. Alpha Composite (BLEND_AlphaComposite)
   ├─ 预乘 Alpha: Src + (1-SrcAlpha) * Dst
   ├─ 不写入 Normal
   └─ 适合粒子特效贴花

3. Modulate (BLEND_Modulate)
   ├─ 调制混合: Dst * Src
   ├─ BaseColor 通道使用 DestColor * InvSrcAlpha
   └─ 适合污渍、脏迹效果

4. DBuffer Decals (BeforeBasePass)
   ├─ 写入独立的 DBuffer 纹理
   ├─ Base Pass 中读取并混合
   ├─ 支持完整的 PBR 属性修改
   └─ 性能开销较大（额外的 MRT）

混合状态由 GetDecalBlendState_DBuffer / GetDecalBlendState_SceneColorAndGBuffer 
根据 RenderTargetWriteMask 选择对应的 TStaticBlendState
```

## 四、场景特别多特效时如何高效贴花

> 这是实际项目中最常遇到的问题：MOBA/FPS 游戏中大量技能特效需要在地面留下痕迹，如何保证性能？

### 4.1 性能瓶颈分析

```
大量贴花的性能瓶颈：

1. Overdraw（最大瓶颈）
   ├─ 每个贴花渲染一个 Box Volume
   ├─ Box 内所有像素都要执行 Pixel Shader
   ├─ 重叠贴花导致同一像素被多次着色
   └─ 100 个重叠贴花 = 100x Overdraw

2. Draw Call
   ├─ 每个贴花 = 1 个 Draw Call
   ├─ 不同材质的贴花无法合批
   └─ 状态切换（Blend State、Material）开销

3. 带宽
   ├─ DBuffer Decals 需要额外的 MRT 读写
   ├─ GBuffer Decals 需要读写多个 GBuffer
   └─ 移动端带宽尤其敏感

4. Shader 复杂度
   ├─ 每个贴花需要深度重建 + 空间变换
   ├─ 复杂材质（法线贴图、多层混合）增加 ALU
   └─ PCF/PCSS 阴影采样次数多
```

### 4.2 优化方案总览

| 方案                        | 适用场景            | 性能收益 | 实现复杂度 |
| --------------------------- | ------------------- | -------- | ---------- |
| **贴花合批（Atlas）**       | 相同材质的大量贴花  | ★★★★★    | 中         |
| **贴花池（Pool）**          | 动态创建/销毁的贴花 | ★★★★     | 低         |
| **距离剔除**                | 远处贴花            | ★★★★     | 低         |
| **LOD 降级**                | 远处/低端设备       | ★★★      | 中         |
| **Screen Space Decal 优化** | 所有贴花            | ★★★★     | 高         |
| **Mesh Decal 替代**         | 静态贴花            | ★★★★★    | 中         |
| **Virtual Texture Decal**   | 大世界静态贴花      | ★★★★★    | 高         |
| **Render Target 烘焙**      | 地面痕迹            | ★★★★★    | 中         |

### 4.3 方案一：贴花纹理图集（Decal Atlas）

```
核心思路：将多个贴花纹理合并到一张 Atlas 中，减少材质切换和 Draw Call

实现方式：
  1. 离线打包：将所有贴花纹理打包到一张大纹理（2048x2048）
  2. 运行时通过 UV 偏移选择子纹理
  3. 所有使用同一 Atlas 的贴花共享同一材质实例
  4. 通过 Material Parameter Collection 传递 UV 参数

优化效果：
  ├─ 100 个贴花 → 1 个材质 → 可合批
  ├─ 减少 PSO 切换
  └─ 减少纹理绑定切换
```

```hlsl
// Decal Atlas Material (simplified)
float4 DecalAtlasPS(float2 DecalUV)
{
    // Atlas parameters: (OffsetU, OffsetV, ScaleU, ScaleV)
    float4 AtlasParams = DecalAtlasParameters;
    
    // Remap UV to atlas sub-region
    float2 AtlasUV = DecalUV * AtlasParams.zw + AtlasParams.xy;
    
    // Clamp to sub-region to prevent bleeding
    float2 MinUV = AtlasParams.xy;
    float2 MaxUV = AtlasParams.xy + AtlasParams.zw;
    AtlasUV = clamp(AtlasUV, MinUV + TexelSize, MaxUV - TexelSize);
    
    return AtlasTexture.Sample(LinearSampler, AtlasUV);
}
```

### 4.4 方案二：贴花对象池（Decal Pool）

```
核心思路：预分配固定数量的贴花组件，循环复用，避免频繁创建/销毁

实现方式（伪代码）：

class FDecalPool
{
    TArray<UDecalComponent*> Pool;  // 预分配的贴花组件
    int32 MaxDecals = 64;          // 最大贴花数量
    int32 NextIndex = 0;           // 环形缓冲索引
    
    void SpawnDecal(FVector Location, FRotator Rotation, 
                    UMaterialInterface* Material, float Size, float Lifetime)
    {
        UDecalComponent* Decal = Pool[NextIndex % MaxDecals];
        
        // Reuse existing component
        Decal->SetWorldLocationAndRotation(Location, Rotation);
        Decal->SetDecalMaterial(Material);
        Decal->DecalSize = FVector(Size);
        Decal->SetVisibility(true);
        Decal->FadeScreenSize = 0.01f;  // Distance fade
        
        // Setup auto-fade
        Decal->SetFadeOut(Lifetime, 1.0f);
        
        NextIndex++;
    }
};

优化效果：
  ├─ 零 GC 压力（无创建/销毁）
  ├─ 固定内存占用
  ├─ 自动淘汰最旧的贴花
  └─ 配合 Atlas 可进一步减少 Draw Call
```

### 4.5 方案三：距离剔除与 LOD

```
距离剔除策略：

1. FadeScreenSize（UE5 内置）
   Decal->FadeScreenSize = 0.01f;  // 屏幕占比 < 1% 时隐藏
   
2. 自定义距离分级：
   Distance < 10m  → 全质量（法线 + 颜色 + 粗糙度）
   Distance 10-30m → 中质量（仅颜色）
   Distance 30-50m → 低质量（简化材质）
   Distance > 50m  → 不渲染

3. 重要性排序：
   ├─ 玩家脚下的贴花 → 高优先级
   ├─ 技能命中点贴花 → 中优先级
   └─ 环境装饰贴花 → 低优先级
   
   当贴花数量超过预算时，优先剔除低优先级贴花

4. 视锥剔除（引擎自动）：
   UE5 自动对 Decal Volume 做视锥剔除
   但重叠检测需要自己实现
```

### 4.6 方案四：Mesh Decal 替代 Volume Decal

```
核心思路：对于静态贴花，用 Mesh Decal 替代 Volume Decal

Volume Decal（标准贴花）：
  ├─ 渲染一个 Box Volume
  ├─ 每个像素都要做深度重建 + 空间变换
  ├─ Overdraw 严重
  └─ 适合动态贴花

Mesh Decal（网格贴花）：
  ├─ 直接在目标表面上放置一个贴合的网格
  ├─ 不需要深度重建
  ├─ 没有 Volume 的 Overdraw 问题
  ├─ 可以参与 Mesh Draw Command 缓存
  └─ 适合静态/半静态贴花

UE5 中 Mesh Decal 的渲染路径：
  RenderMeshDecals()
  └─ DrawDecalMeshCommands()
      └─ 根据 DecalRenderStage 选择 RenderTargetMode
          ├─ BeforeBasePass → DBuffer
          ├─ BeforeLighting → SceneColorAndGBuffer
          ├─ Emissive → SceneColor
          └─ AmbientOcclusion → AO

性能对比（100 个贴花）：
  Volume Decal: ~100 Draw Calls, 高 Overdraw
  Mesh Decal:   可合批到 ~10 Draw Calls, 低 Overdraw
```

### 4.7 方案五：Render Target 烘焙（★ 推荐用于地面痕迹）

```
核心思路：将贴花渲染到一张 RT 上，作为地面材质的输入

适用场景：
  ├─ 技能在地面留下的痕迹（烧焦、冰冻、毒液）
  ├─ 脚印、车辙
  ├─ 弹坑、爆炸痕迹
  └─ 任何"地面累积效果"

实现流程：
  1. 创建一张 World-Space RT（如 4096x4096 覆盖整个战场）
  2. 特效触发时，将贴花绘制到 RT 上（2D 绘制，极快）
  3. 地面材质采样这张 RT 作为叠加层
  4. RT 可以有多个通道：颜色、法线偏移、粗糙度偏移

优势：
  ✅ 无论多少"贴花"，地面只有 1 个 Draw Call
  ✅ 零 Overdraw（贴花已经烘焙到 RT 中）
  ✅ 支持无限累积（不受贴花数量限制）
  ✅ 可以做渐变消散（RT 逐帧衰减）
  ✅ 移动端友好

劣势：
  ❌ 只适用于平面/近似平面的表面
  ❌ RT 分辨率限制了精度
  ❌ 不支持法线贴图的完整 PBR 效果
  ❌ 需要自定义地面材质
```

// Ground Material with RT Decal Layer
float4 GroundPS(float3 WorldPos, float3 Normal, float2 UV)
{
    // Base ground material
    float4 BaseColor = GroundTexture.Sample(Sampler, UV);
    float Roughness = GroundRoughness.Sample(Sampler, UV).r;
    
    // Sample decal RT (world-space UV)
    float2 DecalUV = (WorldPos.xy - WorldOrigin.xy) / WorldSize.xy;
    float4 DecalColor = DecalRT.Sample(LinearSampler, DecalUV);
    float DecalRoughness = DecalRoughnessRT.Sample(LinearSampler, DecalUV).r;
    float DecalMask = DecalColor.a;
    
    // Blend
    float4 FinalColor = lerp(BaseColor, DecalColor, DecalMask);
    float FinalRoughness = lerp(Roughness, DecalRoughness, DecalMask);
    
    // Fade over time (optional: RT alpha decays each frame)
    // DecalRT.a *= 0.995 per frame → gradual fade
    
    return FinalColor;
}

### 4.8 方案六：Virtual Texture Decal（大世界方案）

```
核心思路：利用 Runtime Virtual Texture (RVT) 存储贴花信息

适用场景：
  ├─ 大世界地面贴花（数千个）
  ├─ 地形上的道路、河流标记
  └─ 需要高精度且数量巨大的静态贴花

实现原理：
  1. 创建 RVT（Runtime Virtual Texture）
  2. 贴花渲染到 RVT 的 Physical Pages 中
  3. 地面材质通过 VT 采样获取贴花信息
  4. 只有可见的 Pages 才会被渲染（按需加载）

优势：
  ✅ 理论上支持无限数量的贴花
  ✅ 只渲染可见区域（LOD + 按需加载）
  ✅ 与地形系统完美集成
  ✅ 支持完整 PBR 属性

劣势：
  ❌ 实现复杂度高
  ❌ 需要 VT 硬件支持
  ❌ 动态贴花需要 Invalidate Pages（有延迟）
  ❌ 内存占用较大
```

### 4.9 方案七：Screen Space Decal 优化技巧

```
针对标准 Volume Decal 的优化技巧：

1. 减少 Volume 大小
   ├─ 贴花 Box 越小，覆盖的像素越少
   ├─ 使用 Tight Bounding Box 而非默认的 Cube
   └─ 对于地面贴花，Z 轴可以很薄

2. 深度测试优化
   ├─ 开启 Depth Test 剔除被遮挡的贴花像素
   ├─ 使用 Stencil 标记已渲染区域
   └─ 避免渲染完全被遮挡的贴花

3. 材质简化
   ├─ 远距离使用简化材质（无法线贴图）
   ├─ 只写入需要的通道（如只写 BaseColor）
   ├─ 避免复杂的材质节点
   └─ 使用 Unlit 模式的贴花（只写 Emissive）

4. 排序优化
   ├─ 按材质排序减少状态切换
   ├─ 按 SortOrder 控制渲染顺序
   └─ 相同材质的贴花尽量连续渲染

5. 选择正确的 RenderStage
   ├─ DBuffer (BeforeBasePass): 最高质量但最贵
   ├─ BeforeLighting: 性价比最高
   ├─ Emissive: 最便宜（只写 SceneColor）
   └─ 特效贴花推荐用 Emissive 或 BeforeLighting
```

## 五、特效贴花的最佳实践（MOBA/FPS 场景）

### 5.1 分层策略

```
特效贴花分层管理：

Layer 1: 静态环境贴花（地面裂缝、污渍等）
  ├─ 方案：Mesh Decal 或 Virtual Texture
  ├─ 数量：不限
  ├─ 更新频率：从不
  └─ 性能开销：极低

Layer 2: 半静态痕迹（技能痕迹、弹坑等）
  ├─ 方案：RT 烘焙（推荐）或 Decal Pool
  ├─ 数量：不限（RT 方案）/ 64-128（Pool 方案）
  ├─ 更新频率：触发时
  ├─ 生命周期：10-30 秒渐隐
  └─ 性能开销：低

Layer 3: 动态特效贴花（技能范围指示器、持续效果等）
  ├─ 方案：Decal Pool + Atlas
  ├─ 数量：16-32
  ├─ 更新频率：每帧
  ├─ 生命周期：跟随技能
  └─ 性能开销：中

Layer 4: 瞬时特效（命中闪光、爆炸等）
  ├─ 方案：粒子系统（Niagara）替代贴花
  ├─ 数量：按需
  ├─ 更新频率：每帧
  ├─ 生命周期：< 1 秒
  └─ 性能开销：低（粒子比贴花更适合瞬时效果）
```

### 5.2 移动端特殊优化

```
移动端贴花优化要点：

1. 带宽优先
   ├─ 避免 DBuffer Decals（额外 MRT 开销巨大）
   ├─ 优先使用 Emissive 阶段（只写 SceneColor）
   ├─ 或使用 MobileBeforeLighting 阶段
   └─ 减少 GBuffer 读写次数

2. Shader 简化
   ├─ 移动端使用 TModulatedShadowProjection（调制阴影）
   ├─ 贴花材质避免复杂节点
   ├─ 使用 Half Precision（FP16）
   └─ 减少纹理采样次数

3. 数量控制
   ├─ 移动端贴花预算：< 20 个同屏
   ├─ 使用更激进的距离剔除
   ├─ 低端设备直接关闭贴花
   └─ 用粒子替代部分贴花效果

4. RT 烘焙方案在移动端的优势
   ├─ 一次性绘制到 RT，后续零开销
   ├─ RT 分辨率可以根据设备档位调整
   ├─ 不需要额外的 MRT
   └─ 完美适配 Tile-Based GPU 架构
```

## 六、面试高频问题

### Q1: 阴影投影的原理是什么？Shadow Map 如何工作？

```
答：
阴影投影分两个阶段：

1. Shadow Map 生成（Shadow Pass）
   - 从光源视角渲染场景深度到 Shadow Map
   - 方向光用 CSM（级联阴影贴图），多个级联覆盖不同距离
   - 点光源用 Cube Shadow Map（6个面）
   - 聚光灯用单张 Shadow Map

2. Shadow Projection（投影阶段）
   - 对每个屏幕像素，从深度缓冲重建世界坐标
   - 将世界坐标变换到光源空间（乘以 ScreenToShadowMatrix）
   - 用变换后的 XY 采样 Shadow Map，获取最近深度
   - 比较当前像素深度和 Shadow Map 深度
   - 如果当前深度 > Shadow Map 深度 + Bias → 在阴影中
   - 通过 PCF 滤波实现软阴影边缘

UE5 中通过 TShadowProjectionPS 模板类实现，
模板参数控制质量等级、淡出平面、调制阴影、透射、子像素等特性。
输出到 ScreenShadowMaskTexture，在光照计算中采样使用。
```

### Q2: DBuffer Decal 和 GBuffer Decal 的区别？

```
答：
DBuffer Decal (BeforeBasePass):
  - 在 Base Pass 之前渲染到独立的 DBuffer 纹理
  - DBufferA/B/C 分别存储 BaseColor/Normal/Roughness + Opacity
  - Base Pass 中读取 DBuffer 并与材质混合
  - 优点：参与完整光照计算，效果最好
  - 缺点：额外的 MRT 开销，需要 DBuffer 内存

GBuffer Decal (BeforeLighting):
  - 在 Base Pass 之后、Lighting 之前直接修改 GBuffer
  - 通过 Alpha Blend 混合到 SceneColor + GBufferA/B/C
  - 优点：不需要额外的 DBuffer，性能更好
  - 缺点：法线修改不参与某些光照特性（如 SSS）

选择建议：
  - 需要高质量法线效果 → DBuffer
  - 性能优先 → GBuffer Decal
  - 只需要颜色/发光 → Emissive Decal（最便宜）
  - 移动端 → 避免 DBuffer，用 GBuffer 或 Emissive
```

### Q3: 场景中有大量特效贴花（如 MOBA 技能痕迹），如何优化？

```
答：核心策略是"分层 + 烘焙 + 池化"

1. RT 烘焙（最推荐）
   - 将地面痕迹渲染到一张 World-Space RT
   - 地面材质采样 RT 作为叠加层
   - 无论多少痕迹，地面只有 1 个 Draw Call
   - 支持渐变消散（RT alpha 逐帧衰减）

2. 贴花对象池
   - 预分配固定数量的 DecalComponent（如 64 个）
   - 环形缓冲复用，避免 GC
   - 配合 Atlas 纹理减少材质切换

3. 距离剔除 + LOD
   - FadeScreenSize 控制远距离隐藏
   - 远处贴花降级为简化材质
   - 超出预算时按优先级剔除

4. 选择正确的渲染阶段
   - 特效贴花优先用 Emissive（最便宜）
   - 需要影响光照的用 BeforeLighting
   - 避免在移动端使用 DBuffer

5. 用粒子替代瞬时效果
   - 命中闪光、爆炸等 < 1 秒的效果用 Niagara
   - 粒子比贴花更适合瞬时效果
```

### Q4: 纹理投影（Projector）的实现原理？

```
答：
纹理投影的核心是将一个纹理从某个视角"投射"到场景表面：

数学原理：
  1. 定义投影器的位置、朝向、FOV（类似一个虚拟摄像机）
  2. 构建 ProjectorViewMatrix 和 ProjectorProjectionMatrix
  3. 对场景中的每个像素：
     - 从深度缓冲重建世界坐标
     - WorldPos → ProjectorViewMatrix → ProjectorProjMatrix → ProjectorUV
     - 用 ProjectorUV.xy 采样投影纹理
     - 检查 ProjectorUV 是否在 [0,1] 范围内（投影范围裁剪）
     - 可选：用 ProjectorUV.z 做深度衰减

实现方式：
  方式 A：使用 UE5 的 Decal 系统（推荐）
    - DecalComponent 本质就是一个纹理投影器
    - 引擎自动处理投影矩阵和深度测试
    
  方式 B：自定义 SceneViewExtension
    - 在后处理中实现自定义投影效果
    - 更灵活但需要自己管理投影矩阵
    
  方式 C：材质中实现
    - 在材质 Shader 中直接做世界坐标到投影 UV 的变换
    - 适合简单的平面投影（如 Planar Projection）
```

### Q5: PCSS（Percentage Closer Soft Shadows）的原理？

```
答：
PCSS 实现物理正确的软阴影，半影大小随遮挡距离变化：

三个步骤：
1. Blocker Search（遮挡搜索）
   - 在 Shadow Map 中搜索当前像素附近的遮挡体
   - 计算平均遮挡深度 dBlocker
   - 搜索范围 = LightSize * (dReceiver - NearPlane) / dReceiver

2. Penumbra Estimation（半影估算）
   - PenumbraSize = LightSize * (dReceiver - dBlocker) / dBlocker
   - 遮挡体越远，半影越大（物理正确）
   - 遮挡体越近，阴影越锐利

3. PCF Filtering（百分比接近滤波）
   - 用 PenumbraSize 作为 PCF 滤波半径
   - 在该半径内做多次深度比较并平均
   - 采样数通常 16-64 次

UE5 中的实现：
  TDirectionalPercentageCloserShadowProjectionPS（方向光）
  TSpotPercentageCloserShadowProjectionPS（聚光灯）
  
  关键参数：
  - TanLightSourceAngle: 光源角度的正切值
  - MaxKernelSize: 最大软阴影内核大小（r.Shadow.MaxSoftKernelSize）
  - PCSSParameters = (TanLightSourceAngle * SZ/SW, MaxKernelSize/Resolution, 0, 0)
```

## 八、Unity Forward 渲染路径下的贴花实现

> Unity 的贴花系统因渲染管线不同而差异巨大。本章重点讲解 **Forward 渲染路径** 下各管线的贴花实现方案，以及没有 GBuffer 时如何高效实现贴花效果。

### 8.1 Forward 渲染的核心挑战

```
Forward 渲染 vs Deferred 渲染 对贴花的影响：

Deferred 渲染：
  ├─ 有完整的 GBuffer（BaseColor, Normal, Roughness...）
  ├─ 贴花可以直接读写 GBuffer → Screen Space Decal 天然支持
  ├─ 贴花在 Lighting 之前修改 GBuffer → 参与完整光照
  └─ 实现简单，效果好

Forward 渲染：
  ├─ ❌ 没有 GBuffer（或只有有限的 GBuffer）
  ├─ ❌ 无法直接修改表面属性（Normal, Roughness）
  ├─ ❌ 光照在 Base Pass 中一次性计算完成
  ├─ 贴花必须在光照之前注入，或者用其他方式实现
  └─ 需要特殊的技术方案

Forward 下贴花的可选方案：
  1. DBuffer Decal（Forward + DBuffer）→ URP/HDRP 推荐
  2. Projector 组件（内置管线）         → 额外 Pass 叠加
  3. Forward Decal Pass（额外渲染通道） → 自定义实现
  4. Mesh Decal（网格贴花）             → 所有管线通用
  5. RT 烘焙（材质混合）                → 所有管线通用
```

### 8.2 Unity 各管线贴花方案对比

| 管线           | Forward 贴花方案                           | 技术实现                | 是否支持法线 | 性能       |
| -------------- | ------------------------------------------ | ----------------------- | ------------ | ---------- |
| **内置管线**   | Projector 组件                             | 额外 Pass 叠加          | ❌            | 中         |
| **URP**        | Decal Renderer Feature (Forward + DBuffer) | DBuffer 或 Screen Space | ✅ (DBuffer)  | 高         |
| **URP**        | Decal Renderer Feature (Forward)           | 额外 Forward Pass       | ❌ 仅颜色     | 中         |
| **HDRP**       | Decal Projector (Forward + DBuffer)        | DBuffer                 | ✅            | 高         |
| **HDRP**       | Cluster Decal                              | 聚类贴花                | ✅            | 高         |
| **自定义 SRP** | 自定义实现                                 | 灵活                    | 取决于实现   | 取决于实现 |

### 8.3 内置管线：Projector 组件

```
Unity 内置管线中没有原生 Decal 系统，使用 Projector 组件实现贴花效果。

Projector 工作原理：
  1. Projector 定义一个视锥体（类似摄像机）
  2. 视锥体内的所有对象会被额外渲染一次
  3. 使用 Projector/Light 或 Projector/Multiply Shader
  4. 通过 Alpha Blend 叠加到已渲染的表面上

Projector Inspector 属性：
  ├─ Near Clip Plane    : 近裁剪面
  ├─ Far Clip Plane     : 远裁剪面
  ├─ Field Of View      : 视野角度（透视模式）
  ├─ Aspect Ratio       : 宽高比
  ├─ Orthographic       : 正交/透视模式
  ├─ Orthographic Size  : 正交投影大小
  ├─ Material           : 投影材质（必须用 Projector Shader）
  └─ Ignore Layers      : 忽略的层

Forward 渲染中的执行流程：
  Base Pass → 正常渲染对象（含光照）
  Projector Pass → 对视锥体内的对象额外渲染一次
    └─ 使用 Projector 材质叠加（Multiply/Additive）
```

```csharp
// Projector Shader (Projector/Multiply) - simplified
Shader "Projector/Multiply" {
    Properties {
        _ShadowTex ("Cookie", 2D) = "gray" {}
        _FalloffTex ("FallOff", 2D) = "white" {}
    }
    Subshader {
        Tags {"Queue"="Transparent"}
        Pass {
            ZWrite Off
            ColorMask RGB
            // Multiply blend: Dst * Src
            Blend DstColor Zero
            Offset -1, -1

            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #include "UnityCG.cginc"
            
            // Projector matrices (Unity auto-sets these)
            float4x4 unity_Projector;      // World → Projector space
            float4x4 unity_ProjectorClip;  // Projector clip matrix
            
            sampler2D _ShadowTex;
            sampler2D _FalloffTex;
            
            struct v2f {
                float4 pos : SV_POSITION;
                float4 uvShadow : TEXCOORD0;  // Projector UV
                float4 uvFalloff : TEXCOORD1; // Falloff UV
            };
            
            v2f vert (float4 vertex : POSITION) {
                v2f o;
                o.pos = UnityObjectToClipPos(vertex);
                // Transform vertex to projector space
                o.uvShadow = mul(unity_Projector, vertex);
                o.uvFalloff = mul(unity_ProjectorClip, vertex);
                return o;
            }
            
            fixed4 frag (v2f i) : SV_Target {
                // Sample cookie texture using projective UV
                fixed4 texS = tex2Dproj(_ShadowTex, 
                    UNITY_PROJ_COORD(i.uvShadow));
                texS.a = 1.0 - texS.a;
                
                // Sample falloff texture for distance attenuation
                fixed4 texF = tex2Dproj(_FalloffTex, 
                    UNITY_PROJ_COORD(i.uvFalloff));
                
                // Multiply blend: white = no effect, dark = shadow
                fixed4 res = lerp(fixed4(1,1,1,1), texS, texF.a);
                return res;
            }
            ENDCG
        }
    }
}
```

```
Projector 的局限性：
  ❌ 每个 Projector 对视锥体内的每个对象增加一个额外 Draw Call
  ❌ 不支持修改法线/粗糙度（只能叠加颜色）
  ❌ 不参与光照计算（在光照之后叠加）
  ❌ 性能随受影响对象数量线性增长
  ❌ 仅支持内置管线（URP/HDRP 不支持）
  
  ✅ 实现简单
  ✅ 支持动态投影（跟随光源/角色移动）
  ✅ 适合 Blob Shadow（简单圆形阴影）
```

### 8.4 URP：Decal Renderer Feature（Forward 模式）

```
URP 提供了 Decal Renderer Feature，支持两种 Forward 贴花技术：

技术 A: Screen Space Decal（仅 Deferred 可用）
  └─ Forward 模式下不可用

技术 B: DBuffer Decal（Forward + DBuffer）★ 推荐
  ├─ 在 Base Pass 之前渲染贴花到 DBuffer
  ├─ Base Pass 读取 DBuffer 并混合
  ├─ 支持修改 BaseColor、Normal、MAOS（Metallic/AO/Smoothness）
  ├─ 参与完整光照计算
  └─ 需要额外的 DBuffer MRT 开销

技术 C: Forward Decal（仅 Forward 可用）
  ├─ 在不透明物体渲染之后、透明物体之前
  ├─ 作为额外的 Forward Pass 渲染
  ├─ 只能修改 BaseColor（不支持法线/粗糙度）
  ├─ 不参与光照计算（在光照之后叠加）
  └─ 性能最好，但效果有限
```

```
URP Decal Renderer Feature 配置：

1. 在 URP Renderer Data 中添加 Decal Renderer Feature
2. 选择 Technique:
   ├─ Automatic     : 根据平台自动选择
   ├─ DBuffer       : 强制使用 DBuffer（需要 MRT 支持）
   ├─ Screen Space  : 强制使用 Screen Space（仅 Deferred）
   └─ 注意          ：Forward 模式下 Screen Space 不可用

3. DBuffer 设置:
   ├─ Surface Data              : 选择写入哪些通道
   │   ├─ Albedo                : 只写颜色
   │   ├─ Albedo Normal         : 写颜色 + 法线
   │   ├─ Albedo Normal MAOS    : 写颜色 + 法线 + PBR 属性（最完整）
   │   └─ 通道越多，MRT 开销越大
   ├─ Max Draw Distance: 最大绘制距离
   └─ Use GBuffer: 是否在 Deferred 模式下使用 GBuffer

4. 场景中添加 Decal Projector 组件:
   ├─ Material                  : 使用 Shader Graphs/Decal 着色器的材质
   ├─ Width/Height/Depth        : 贴花投影体积大小
   ├─ Pivot                     : 轴心偏移
   ├─ Opacity                   : 整体不透明度
   ├─ Draw Distance             : 绘制距离
   ├─ Start/End Fade Percentage : 淡入淡出
   └─ Angle Fade                : 角度淡出（避免侧面拉伸）
```

```
URP Forward + DBuffer 渲染流程：

UniversalRenderer.Render() [Forward Path]
│
├─ DepthPrepass (if needed)
│   └─ Render scene depth
│
├─ DBufferRenderPass (★ DBuffer Decals)
│   ├─ Create DBuffer textures:
│   │   ├─ DBuffer0 (RGBA): Albedo.rgb + Albedo.a (opacity)
│   │   ├─ DBuffer1 (RGBA): Normal.rgb + Normal.a (opacity)
│   │   └─ DBuffer2 (RGBA): MAOS.rgb + Smoothness.a (opacity)
│   │
│   ├─ for each visible DecalProjector:
│   │   ├─ Frustum cull against camera
│   │   ├─ Distance cull (DrawDistance)
│   │   ├─ Render decal box volume
│   │   │   ├─ VS: Transform box vertices
│   │   │   └─ PS: 
│   │   │       ├─ Reconstruct world pos from depth
│   │   │       ├─ Transform to decal local space
│   │   │       ├─ Check bounds [0,1]
│   │   │       ├─ Sample decal material (albedo, normal, MAOS)
│   │   │       ├─ Apply angle fade
│   │   │       └─ Write to DBuffer MRT
│   │   └─ Next decal
│   │
│   └─ DBuffer textures ready for Base Pass
│
├─ ForwardPass (Base Pass)
│   ├─ For each object:
│   │   ├─ Sample material properties
│   │   ├─ ★ Read DBuffer and blend:
│   │   │   ├─ albedo = lerp(materialAlbedo, DBuffer0.rgb, DBuffer0.a)
│   │   │   ├─ normal = lerp(materialNormal, DBuffer1.rgb, DBuffer1.a)
│   │   │   └─ smoothness = lerp(materialSmoothness, DBuffer2.a, DBuffer2.a_opacity)
│   │   ├─ Calculate lighting with blended properties
│   │   └─ Output final color
│   └─ All opaque objects rendered with decal influence
│
├─ DecalForwardEmissivePass (optional)
│   └─ Render emissive decals on top
│
└─ TransparentPass
    └─ Render transparent objects
```

```hlsl
// URP DBuffer Decal Shader (simplified core logic)
// Based on Packages/com.unity.render-pipelines.universal/Shaders/Decal.shader

// Vertex Shader: Transform decal box
Varyings DecalVert(Attributes input)
{
    Varyings output;
    output.positionCS = TransformObjectToHClip(input.positionOS.xyz);
    output.positionWS = TransformObjectToWorld(input.positionOS.xyz);
    
    // Pass screen UV for depth sampling
    output.screenUV = ComputeScreenPos(output.positionCS);
    return output;
}

// Fragment Shader: Project decal onto surface
void DecalFrag(Varyings input, 
    out half4 outDBuffer0 : SV_Target0,  // Albedo
    out half4 outDBuffer1 : SV_Target1,  // Normal
    out half4 outDBuffer2 : SV_Target2)  // MAOS
{
    // 1. Reconstruct world position from depth buffer
    float2 screenUV = input.screenUV.xy / input.screenUV.w;
    float depth = SampleSceneDepth(screenUV);
    float3 worldPos = ComputeWorldSpacePosition(screenUV, depth, 
                                                 UNITY_MATRIX_I_VP);
    
    // 2. Transform world pos to decal local space
    float3 localPos = TransformWorldToObject(worldPos);
    
    // 3. Clip: discard pixels outside decal volume [-0.5, 0.5]
    clip(0.5 - abs(localPos.xyz));
    
    // 4. Calculate decal UV from local XZ
    float2 decalUV = localPos.xz + 0.5;
    
    // 5. Angle fade: reduce opacity on steep surfaces
    float3 worldNormal = SampleSceneNormals(screenUV);
    float3 decalForward = TransformObjectToWorldDir(float3(0, 1, 0));
    float angleFade = saturate(dot(worldNormal, decalForward));
    angleFade = smoothstep(_AngleFadeStart, _AngleFadeEnd, angleFade);
    
    // 6. Sample decal material
    half4 albedo = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, decalUV);
    albedo *= _BaseColor;
    
    half3 normalTS = UnpackNormal(
        SAMPLE_TEXTURE2D(_NormalMap, sampler_NormalMap, decalUV));
    
    half metallic = SAMPLE_TEXTURE2D(_MetallicMap, sampler_MetallicMap, 
                                      decalUV).r * _Metallic;
    half smoothness = _Smoothness;
    
    // 7. Apply opacity and angle fade
    half opacity = albedo.a * _DecalOpacity * angleFade;
    
    // 8. Write to DBuffer
    outDBuffer0 = half4(albedo.rgb, opacity);           // Albedo + opacity
    outDBuffer1 = half4(normalTS * 0.5 + 0.5, opacity); // Normal + opacity
    outDBuffer2 = half4(metallic, 0, 0, smoothness * opacity); // MAOS
}
```

// URP Forward Lit Shader - DBuffer reading (simplified)
// How Base Pass reads and applies DBuffer decals

half4 ForwardLitFrag(Varyings input) : SV_Target
{
    // Normal material sampling
    SurfaceData surfaceData = InitializeSurfaceData(input);
    
    // ★ Apply DBuffer decals
    #if defined(_DBUFFER)
    {
        // Sample DBuffer textures at screen UV
        float2 screenUV = input.positionCS.xy * _ScreenSize.zw;
        
        half4 DBuffer0 = SAMPLE_TEXTURE2D(_DBuffer0, sampler_DBuffer0, screenUV);
        half4 DBuffer1 = SAMPLE_TEXTURE2D(_DBuffer1, sampler_DBuffer1, screenUV);
        half4 DBuffer2 = SAMPLE_TEXTURE2D(_DBuffer2, sampler_DBuffer2, screenUV);
        
        // Blend decal properties with material properties
        surfaceData.albedo = lerp(surfaceData.albedo, DBuffer0.rgb, DBuffer0.a);
        
        half3 decalNormal = DBuffer1.rgb * 2.0 - 1.0;
        surfaceData.normalTS = lerp(surfaceData.normalTS, decalNormal, DBuffer1.a);
        
        surfaceData.metallic = lerp(surfaceData.metallic, DBuffer2.r, DBuffer2.a);
        surfaceData.smoothness = lerp(surfaceData.smoothness, DBuffer2.a, DBuffer2.a);
    }
    #endif
    
    // Continue with normal lighting calculation
    // Lighting now uses decal-modified surface properties
    half4 color = UniversalFragmentPBR(inputData, surfaceData);
    
    return color;
}

### 8.5 URP Forward Decal（无 DBuffer 的轻量方案）

```
当设备不支持 MRT（如部分低端移动设备），URP 退回到 Forward Decal 模式：

Forward Decal 工作原理：
  1. 正常渲染所有不透明物体（含光照）
  2. 在不透明物体之后、透明物体之前
  3. 渲染贴花 Box Volume
  4. 从深度缓冲重建世界坐标
  5. 变换到贴花局部空间，采样贴花纹理
  6. Alpha Blend 叠加到 SceneColor 上

局限性：
  ❌ 不修改法线 → 贴花不影响光照方向
  ❌ 不修改粗糙度/金属度 → 贴花不影响高光
  ❌ 在光照之后叠加 → 贴花区域的光照是底层材质的光照
  ✅ 不需要 MRT → 兼容所有设备
  ✅ 性能开销小 → 只需要一个额外 Pass
  ✅ 适合简单的颜色贴花（弹孔、血迹、标记）

渲染流程：
  DepthPrepass → ForwardPass(Opaque) → ★ DecalForwardPass → TransparentPass
                                        │
                                        ├─ Render decal volumes
                                        ├─ Read depth → reconstruct world pos
                                        ├─ Sample decal texture
                                        └─ Alpha blend to SceneColor
```

```hlsl
// URP Forward Decal (no DBuffer) - simplified
half4 ForwardDecalFrag(Varyings input) : SV_Target
{
    // Reconstruct world position from depth
    float2 screenUV = input.positionCS.xy * _ScreenSize.zw;
    float depth = SampleSceneDepth(screenUV);
    float3 worldPos = ComputeWorldSpacePosition(screenUV, depth, UNITY_MATRIX_I_VP);
    
    // Transform to decal local space
    float3 localPos = TransformWorldToObject(worldPos);
    clip(0.5 - abs(localPos.xyz));
    
    float2 decalUV = localPos.xz + 0.5;
    
    // Sample decal color only (no normal/roughness modification)
    half4 decalColor = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, decalUV);
    decalColor *= _BaseColor;
    
    // Angle fade
    float3 worldNormal = SampleSceneNormals(screenUV);
    float3 decalUp = TransformObjectToWorldDir(float3(0, 1, 0));
    float angleFade = saturate(dot(worldNormal, decalUp));
    
    decalColor.a *= _DecalOpacity * angleFade;
    
    // Output: will be alpha blended onto SceneColor
    return decalColor;
}
```

### 8.6 HDRP：Forward + DBuffer Decal

```
HDRP 的 Forward 模式同样支持 DBuffer Decal，且功能更强大：

HDRP Decal 特性：
  ├─ Decal Projector    : 投影贴花（类似 URP）
  ├─ Mesh Decal         : 网格贴花
  ├─ Decal Layers       : 贴花层（控制哪些对象接收贴花）
  ├─ Cluster Decal      : 聚类贴花（Forward 专用优化）
  ├─ Surface Gradient   : 表面梯度法线混合（更精确）
  ├─ Emissive Decal     : 自发光贴花
  └─ Transparent Decal  : 半透明物体上的贴花

HDRP Forward 贴花渲染流程：
  DepthPrepass
  → DBuffer Pass (DBuffer Decals)
  → Forward Opaque Pass (reads DBuffer)
  → Cluster Decal Application (★ Forward 专用)
  → Transparent Pass

Cluster Decal（聚类贴花）：
  ├─ 类似 Clustered Forward Lighting 的思路
  ├─ 将视锥体划分为 3D 聚类（Cluster）
  ├─ 每个 Cluster 记录影响它的贴花列表
  ├─ Forward Pass 中每个像素查询所属 Cluster
  ├─ 只处理该 Cluster 中的贴花
  └─ 避免全屏遍历所有贴花 → 性能优化

  优势：
  ✅ 支持在半透明物体上渲染贴花
  ✅ 贴花数量多时性能更好（O(贴花/Cluster) vs O(总贴花数)）
  ✅ 与 Clustered Forward Lighting 共享基础设施
```

### 8.7 自定义 Forward 贴花实现（通用方案）

```
如果使用自定义 SRP 或需要在内置管线中实现更好的贴花效果，
可以自己实现 Forward 贴花系统。

方案 A: 额外 Forward Pass（最简单）
  ├─ 在 Opaque Pass 之后添加一个 Decal Pass
  ├─ 渲染贴花 Box Volume
  ├─ 从深度缓冲重建世界坐标
  ├─ Alpha Blend 到 SceneColor
  └─ 局限：不影响光照

方案 B: DBuffer 自实现（最完整）
  ├─ 在 Base Pass 之前创建 DBuffer MRT
  ├─ 渲染贴花到 DBuffer
  ├─ 修改 Base Pass Shader 读取 DBuffer
  ├─ 贴花参与完整光照
  └─ 需要修改所有材质 Shader

方案 C: 材质混合（最灵活）
  ├─ 在材质 Shader 中直接实现贴花混合
  ├─ 通过 Texture Array / Atlas 传入贴花数据
  ├─ 每个像素查询是否被贴花覆盖
  ├─ 在光照计算之前混合贴花属性
  └─ 适合特定场景（如地面材质）
```

```csharp
// Unity Custom SRP: Forward Decal Pass implementation
using UnityEngine;
using UnityEngine.Rendering;

public class ForwardDecalPass : ScriptableRenderPass
{
    private Material decalMaterial;
    private Mesh decalBoxMesh;
    private List<DecalData> visibleDecals;
    
    public override void Execute(
        ScriptableRenderContext context, 
        ref RenderingData renderingData)
    {
        CommandBuffer cmd = CommandBufferPool.Get("Forward Decals");
        
        // Get scene depth texture
        cmd.SetGlobalTexture("_CameraDepthTexture", depthTextureId);
        
        // Setup blend state: SrcAlpha OneMinusSrcAlpha
        // This blends decal color onto existing scene color
        
        foreach (var decal in visibleDecals)
        {
            // Set per-decal properties
            MaterialPropertyBlock props = new MaterialPropertyBlock();
            props.SetMatrix("_DecalWorldToLocal", 
                decal.transform.worldToLocalMatrix);
            props.SetTexture("_DecalTexture", decal.texture);
            props.SetFloat("_DecalOpacity", decal.opacity);
            props.SetFloat("_AngleFadeStart", decal.angleFadeStart);
            
            // Draw decal box volume
            cmd.DrawMesh(
                decalBoxMesh,           // Unit cube mesh
                decal.transform.localToWorldMatrix,
                decalMaterial,
                0,                      // Submesh index
                0,                      // Shader pass
                props
            );
        }
        
        context.ExecuteCommandBuffer(cmd);
        CommandBufferPool.Release(cmd);
    }
}
```

// Unity Built-in Pipeline: Using CommandBuffer for Forward Decals

public class ForwardDecalRenderer : MonoBehaviour
{
    public Material decalMaterial;
    public Mesh cubeMesh;
    
    private CommandBuffer cmdBuffer;
    
    void OnEnable()
    {
        cmdBuffer = new CommandBuffer { name = "Forward Decals" };
        
        // Add command buffer AFTER opaque rendering
        Camera.main.AddCommandBuffer(
            CameraEvent.AfterForwardOpaque, 
            cmdBuffer);
    }
    
    void OnDisable()
    {
        Camera.main.RemoveCommandBuffer(
            CameraEvent.AfterForwardOpaque, 
            cmdBuffer);
        cmdBuffer.Release();
    }
    
    void LateUpdate()
    {
        cmdBuffer.Clear();
        
        // Find all decal projectors in scene
        var decals = FindObjectsOfType<DecalProjectorComponent>();
        
        foreach (var decal in decals)
        {
            // Frustum culling
            if (!GeometryUtility.TestPlanesAABB(
                GeometryUtility.CalculateFrustumPlanes(Camera.main),
                decal.GetComponent<Renderer>().bounds))
                continue;
            
            // Set decal properties
            MaterialPropertyBlock props = new MaterialPropertyBlock();
            props.SetMatrix("_WorldToDecal", 
                decal.transform.worldToLocalMatrix);
            props.SetTexture("_DecalTex", decal.decalTexture);
            props.SetFloat("_Opacity", decal.opacity);
            
            // Draw decal volume
            cmdBuffer.DrawMesh(
                cubeMesh,
                decal.transform.localToWorldMatrix,
                decalMaterial,
                0, 0, props);
        }
    }
}

### 8.8 Forward 贴花的 Shader 核心技巧

```
Forward 贴花 Shader 的关键技术点：

1. 深度重建世界坐标
   ├─ 从 _CameraDepthTexture 采样深度
   ├─ 使用 VP 逆矩阵重建世界坐标
   └─ 注意：不同平台深度范围不同（0~1 或 -1~1）

2. 投影 UV 计算
   ├─ 世界坐标 → 贴花局部空间
   ├─ 局部空间 XZ → 贴花 UV
   └─ 局部空间 Y → 深度（用于裁剪）

3. 边界裁剪
   ├─ clip(0.5 - abs(localPos)) → 裁剪 Box 外的像素
   └─ 或使用 SDF 实现软边缘

4. 角度淡出（Angle Fade）
   ├─ 采样场景法线（需要 DepthNormals Prepass）
   ├─ dot(surfaceNormal, decalForward) → 角度因子
   ├─ 陡峭表面上贴花淡出，避免拉伸
   └─ smoothstep 实现平滑过渡

5. 深度偏移（Depth Bias）
   ├─ 避免 Z-Fighting
   ├─ Offset -1, -1 在 ShaderLab 中设置
   └─ 或在 VS 中手动偏移 clip space Z

6. 法线混合（仅 DBuffer 方案）
   ├─ Reoriented Normal Mapping (RNM)
   ├─ 或简单 lerp 混合
   └─ HDRP 使用 Surface Gradient 方法（最精确）
```

```hlsl
// Forward Decal: Complete vertex/fragment shader
Shader "Custom/ForwardDecal"
{
    Properties
    {
        _DecalTex ("Decal Texture", 2D) = "white" {}
        _Color ("Color", Color) = (1,1,1,1)
        _Opacity ("Opacity", Range(0,1)) = 1.0
        _AngleFadeStart ("Angle Fade Start", Range(0,1)) = 0.3
        _AngleFadeEnd ("Angle Fade End", Range(0,1)) = 0.7
    }
    
    SubShader
    {
        Tags { "Queue"="Transparent-1" "RenderType"="Transparent" }
        
        Pass
        {
            Name "ForwardDecal"
            
            ZWrite Off
            ZTest Greater  // Only render behind the decal box front face
            Cull Front      // Render back faces of the box
            Blend SrcAlpha OneMinusSrcAlpha
            
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            #include "UnityCG.cginc"
            
            sampler2D _DecalTex;
            sampler2D _CameraDepthTexture;
            sampler2D _CameraDepthNormalsTexture;
            
            float4 _Color;
            float _Opacity;
            float _AngleFadeStart;
            float _AngleFadeEnd;
            float4x4 _WorldToDecal; // Set via MaterialPropertyBlock
            
            struct appdata
            {
                float4 vertex : POSITION;
            };
            
            struct v2f
            {
                float4 pos : SV_POSITION;
                float4 screenPos : TEXCOORD0;
                float3 ray : TEXCOORD1; // Camera to vertex ray
            };
            
            v2f vert(appdata v)
            {
                v2f o;
                o.pos = UnityObjectToClipPos(v.vertex);
                o.screenPos = ComputeScreenPos(o.pos);
                
                // Ray from camera to vertex (for depth reconstruction)
                float3 worldPos = mul(unity_ObjectToWorld, v.vertex).xyz;
                o.ray = worldPos - _WorldSpaceCameraPos;
                
                return o;
            }
            
            half4 frag(v2f i) : SV_Target
            {
                // 1. Sample depth and reconstruct world position
                float2 screenUV = i.screenPos.xy / i.screenPos.w;
                float depth = SAMPLE_DEPTH_TEXTURE(_CameraDepthTexture, screenUV);
                float linearDepth = LinearEyeDepth(depth);
                
                // Reconstruct world position using ray
                float3 ray = i.ray * (linearDepth / i.screenPos.w);
                float3 worldPos = _WorldSpaceCameraPos + ray;
                
                // 2. Transform to decal local space
                float3 localPos = mul(_WorldToDecal, float4(worldPos, 1)).xyz;
                
                // 3. Clip outside decal volume
                clip(0.5 - abs(localPos.x));
                clip(0.5 - abs(localPos.y));
                clip(0.5 - abs(localPos.z));
                
                // 4. Calculate decal UV
                float2 decalUV = localPos.xz + 0.5;
                
                // 5. Sample decal texture
                half4 decalColor = tex2D(_DecalTex, decalUV) * _Color;
                
                // 6. Angle fade (optional, needs DepthNormals texture)
                #if defined(_ANGLE_FADE)
                float3 normal;
                float depthDummy;
                DecodeDepthNormal(
                    tex2D(_CameraDepthNormalsTexture, screenUV),
                    depthDummy, normal);
                normal = mul((float3x3)unity_CameraToWorld, normal);
                
                float3 decalForward = normalize(
                    mul((float3x3)unity_ObjectToWorld, float3(0,1,0)));
                float angleFactor = saturate(dot(normal, decalForward));
                angleFactor = smoothstep(
                    _AngleFadeStart, _AngleFadeEnd, angleFactor);
                decalColor.a *= angleFactor;
                #endif
                
                // 7. Apply opacity
                decalColor.a *= _Opacity;
                
                return decalColor;
            }
            ENDHLSL
        }
    }
}
```

### 8.9 Forward 贴花性能优化

```
Forward 贴花的性能优化策略：

1. 减少 Overdraw
   ├─ ZTest Greater + Cull Front 技巧：
   │   渲染 Box 的背面，只处理 Box 内部的像素
   │   避免 Box 前面被遮挡时仍然执行 PS
   ├─ Tight Bounding Box：贴花体积尽量贴合
   └─ 地面贴花 Y 轴尽量薄

2. 减少 Draw Call
   ├─ GPU Instancing：相同材质的贴花合批
   │   cmd.DrawMeshInstanced(cubeMesh, 0, material, 0, matrices)
   ├─ Atlas 纹理：多个贴花共享一张纹理
   └─ SRP Batcher：确保贴花 Shader 兼容

3. 剔除优化
   ├─ 视锥剔除（Frustum Culling）
   ├─ 距离剔除（Distance Culling）
   ├─ 遮挡剔除（Occlusion Culling）
   └─ 屏幕占比剔除（Screen Size Culling）

4. Shader 优化
   ├─ 移动端使用 half precision
   ├─ 减少纹理采样（不需要法线就不采样法线）
   ├─ 简化角度淡出计算
   └─ 使用 Texture Array 替代多次 SetTexture

5. DBuffer vs Forward Decal 选择
   ├─ 需要法线/PBR → DBuffer（开销大但效果好）
   ├─ 只需颜色 → Forward Decal（开销小）
   ├─ 移动端低端 → Forward Decal 或 RT 烘焙
   └─ 大量贴花 → RT 烘焙（零运行时开销）
```

```
Forward 贴花性能预算（Unity URP）：

PC (Forward + DBuffer):
  ├─ 同屏贴花：< 100
  ├─ DBuffer 额外开销：~0.3ms（MRT 创建 + 读写）
  ├─ 每个贴花 GPU 开销：~0.01ms
  └─ 总预算：< 1.5ms

Mobile (Forward + DBuffer):
  ├─ 同屏贴花：< 15
  ├─ DBuffer 额外开销：~0.5ms（MRT 在 TBDR 上较贵）
  ├─ 每个贴花 GPU 开销：~0.03ms
  └─ 总预算：< 1.0ms

Mobile (Forward Decal, no DBuffer):
  ├─ 同屏贴花：< 20
  ├─ 无 MRT 开销
  ├─ 每个贴花 GPU 开销：~0.02ms
  └─ 总预算：< 0.5ms

建议：
  ├─ 高端设备 → Forward + DBuffer（完整 PBR 贴花）
  ├─ 中端设备 → Forward Decal（仅颜色）
  ├─ 低端设备 → RT 烘焙 或 Mesh Decal
  └─ 所有设备 → 配合对象池 + 距离剔除
```

### 8.10 面试高频问题（Unity Forward 贴花）

### Q6: Unity Forward 渲染路径下如何实现贴花？和 Deferred 有什么区别？

```
答：
Forward 渲染没有完整的 GBuffer，贴花实现有三种方案：

1. DBuffer Decal（Forward + DBuffer）
   - 在 Base Pass 之前创建额外的 DBuffer MRT
   - 贴花写入 DBuffer（Albedo/Normal/MAOS）
   - Base Pass 读取 DBuffer 并混合到材质属性中
   - 贴花参与完整光照计算
   - 效果最好，但需要 MRT 支持

2. Forward Decal（额外 Pass）
   - 在 Opaque Pass 之后渲染贴花 Box Volume
   - 从深度缓冲重建世界坐标，采样贴花纹理
   - Alpha Blend 叠加到 SceneColor
   - 不影响法线/粗糙度，不参与光照
   - 性能好，兼容性强

3. Mesh Decal
   - 直接在表面放置贴合的网格
   - 不需要深度重建
   - 可以参与正常的 Forward 光照
   - 适合静态贴花

与 Deferred 的区别：
  Deferred: 贴花直接修改 GBuffer → 天然支持所有 PBR 属性
  Forward:  没有 GBuffer → 需要 DBuffer 额外 MRT 或放弃 PBR 修改
  
  Deferred Screen Space Decal 更简单高效，
  Forward DBuffer Decal 需要额外的 MRT 开销但效果等价。
```

### Q7: URP 中 DBuffer Decal 和 Forward Decal 如何选择？

```
答：
选择依据：

DBuffer Decal:
  ✅ 支持修改法线、粗糙度、金属度
  ✅ 贴花参与完整 PBR 光照
  ✅ 效果与 Deferred Decal 等价
  ❌ 需要 MRT 支持（部分低端设备不支持）
  ❌ 额外的 DBuffer 内存和带宽开销
  ❌ 移动端 TBDR 架构上 MRT 较贵
  → 适合：PC、高端移动设备、需要高质量贴花

Forward Decal:
  ✅ 不需要 MRT → 兼容所有设备
  ✅ 性能开销小
  ✅ 实现简单
  ❌ 只能修改颜色（不影响法线/粗糙度）
  ❌ 不参与光照计算
  → 适合：低端设备、只需颜色叠加的贴花

实际项目建议：
  - URP Decal Renderer Feature 设置 Technique = Automatic
  - 引擎会根据设备能力自动选择 DBuffer 或 Forward
  - 高端设备自动用 DBuffer，低端退回 Forward
  - 配合 Decal Layers 控制哪些对象接收贴花
```

### Q8: 内置管线的 Projector 和 URP 的 Decal 有什么本质区别？

```
答：
本质区别在于渲染时机和方式：

Projector（内置管线）：
  - 工作方式：对视锥体内的每个对象额外渲染一次
  - 每个受影响的对象 = 1 个额外 Draw Call
  - 使用 Projector/Multiply 或 Projector/Light Shader
  - 在对象的 Forward Pass 中作为额外 Pass 执行
  - 不修改 GBuffer/DBuffer → 只能叠加颜色
  - 性能：O(受影响对象数) 个额外 Draw Call

URP Decal（Decal Renderer Feature）：
  - 工作方式：渲染贴花 Box Volume（Screen Space 方式）
  - 每个贴花 = 1 个 Draw Call（与受影响对象数无关）
  - 使用 Decal Shader（支持 Shader Graph）
  - DBuffer 模式：在 Base Pass 之前写入 DBuffer
  - Forward 模式：在 Opaque 之后叠加
  - 性能：O(贴花数) 个 Draw Call

关键差异：
  Projector: 按对象渲染 → 10 个对象 × 1 个 Projector = 10 Draw Calls
  URP Decal: 按贴花渲染 → 1 个 Decal 覆盖 10 个对象 = 1 Draw Call
  
  URP Decal 在大场景中性能远优于 Projector。
```
