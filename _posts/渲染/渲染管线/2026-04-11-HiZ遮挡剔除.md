## 1. Hi-Z 原理

### 1.1 什么是 Hi-Z

Hi-Z (Hierarchical-Z) 遮挡剔除是一种 **GPU 驱动的遮挡剔除技术**，通过构建深度缓冲区的 Mipmap 金字塔（Depth Pyramid），在 Compute Shader 中快速判断物体是否被其他物体完全遮挡，从而跳过不可见物体的渲染。

```
核心思想:
  "如果一个物体的包围盒在屏幕上投影的最近深度，
   仍然比 Depth Buffer 中对应区域的最远深度还要远，
   那么这个物体一定被完全遮挡，可以安全跳过。"
```

### 1.2 为什么需要 Hi-Z

```
传统遮挡剔除方案的问题:

1. Unity Umbra (CPU Occlusion Culling)
   ❌ 需要预烘焙
   ❌ 不支持动态物体作为遮挡体
   ❌ 精度有限 (Cell-based)
   ❌ CPU 开销随场景复杂度增长

2. Software Rasterization (CPU)
   ❌ CPU 光栅化遮挡体，性能有限
   ❌ 精度取决于 CPU 光栅化分辨率
   ✅ 不需要预烘焙

3. GPU Query (硬件遮挡查询)
   ❌ 1-2 帧延迟 (GPU → CPU 回读)
   ❌ 每个物体一个 Query，Draw Call 爆炸
   ❌ Pipeline Stall 风险
   ✅ 精确

4. Hi-Z (GPU Driven) ✅
   ✅ 完全 GPU 端执行，无 CPU 回读
   ✅ 支持动态遮挡体
   ✅ 不需要预烘焙
   ✅ 可处理百万级物体
   ✅ 与 Indirect Rendering 完美配合
   ⚠️ 有 1 帧延迟 (使用上一帧深度)
   ⚠️ 保守剔除 (可能漏剔，但不会错剔)
```

### 1.4 数学原理

```
深度比较的数学基础:

给定一个物体的 AABB (Axis-Aligned Bounding Box):
  - 8 个顶点投影到 NDC 空间
  - 计算屏幕空间包围矩形: (minX, minY, maxX, maxY)
  - 计算物体最近深度: minZ = min(所有顶点的 NDC.z)

选择 Mip Level:
  - 包围矩形的像素大小: width = (maxX - minX) * screenWidth
  - 包围矩形的像素大小: height = (maxY - minY) * screenHeight
  - 需要的 Mip Level: mip = ceil(log2(max(width, height)))
  
  为什么选这个 Mip?
  → 在这个 Mip Level 下，包围矩形大约覆盖 1-4 个像素
  → 只需要 1-4 次纹理采样就能覆盖整个包围矩形
  → 每个像素存储的是对应区域的 MAX 深度

遮挡判断:
  sampledDepth = texture(depthPyramid, uv, mipLevel)
  
  Reversed-Z (Unity/UE 默认):
    if (objectMinZ > sampledDepth)  → 被遮挡 (近处深度值更大)
    
  Standard Z:
    if (objectMinZ < sampledDepth)  → 被遮挡 (近处深度值更小)

  注意: Depth Pyramid 存储的是 MAX 深度 (最远的深度)
  在 Reversed-Z 中, MAX 深度 = MIN 值 (最小的 float 值)
  所以 Depth Pyramid 的 downsample 操作是取 MIN!
```

## 2. Depth Pyramid 构建

### 2.1 构建算法

```hlsl
// === Depth Pyramid Downsample Compute Shader ===

// 输入: 上一级 Mip 的深度纹理
// 输出: 当前级 Mip 的深度纹理
// 操作: 取 2×2 区域的 "最远深度"

Texture2D<float> _InputDepth;     // 上一级 Mip
RWTexture2D<float> _OutputDepth;  // 当前级 Mip

int2 _InputSize;  // 上一级分辨率

[numthreads(8, 8, 1)]
void CSDownsample(uint3 id : SV_DispatchThreadID)
{
    // 当前像素对应上一级的 2×2 区域
    int2 baseCoord = id.xy * 2;
    
    // 采样 2×2 区域
    float d0 = _InputDepth[baseCoord + int2(0, 0)];
    float d1 = _InputDepth[baseCoord + int2(1, 0)];
    float d2 = _InputDepth[baseCoord + int2(0, 1)];
    float d3 = _InputDepth[baseCoord + int2(1, 1)];
    
    // 处理边界情况 (非 2 的幂分辨率)
    // 如果上一级宽度是奇数，需要额外采样第 3 列
    if (_InputSize.x & 1)
    {
        float d4 = _InputDepth[min(baseCoord + int2(2, 0), _InputSize - 1)];
        float d5 = _InputDepth[min(baseCoord + int2(2, 1), _InputSize - 1)];
        d1 = min(d1, d4); // Reversed-Z: min = 最远
        d3 = min(d3, d5);
    }
    
    // 如果上一级高度是奇数，需要额外采样第 3 行
    if (_InputSize.y & 1)
    {
        float d6 = _InputDepth[min(baseCoord + int2(0, 2), _InputSize - 1)];
        float d7 = _InputDepth[min(baseCoord + int2(1, 2), _InputSize - 1)];
        d2 = min(d2, d6);
        d3 = min(d3, d7);
    }
    
    // Reversed-Z: 取 MIN (最远深度的值最小)
    // Standard-Z: 取 MAX (最远深度的值最大)
    float maxDepth = min(min(d0, d1), min(d2, d3));
    
    _OutputDepth[id.xy] = maxDepth;
}
```

### 2.2 单 Pass vs 多 Pass 构建

```
方案 1: 多 Pass 构建 (简单但慢)
─────────────────────────────────
Mip 0 (原始) → Dispatch → Mip 1
Mip 1        → Dispatch → Mip 2
Mip 2        → Dispatch → Mip 3
...

每级一次 Dispatch，需要 log2(maxDim) 次 Dispatch
1920×1080 → 约 11 次 Dispatch

问题: 每次 Dispatch 之间有 barrier，GPU 利用率低


方案 2: SPD (Single Pass Downsample) ✅ 推荐
─────────────────────────────────────────────
使用 AMD FidelityFX SPD 算法
一次 Dispatch 生成所有 Mip Level
利用 LDS (Local Data Share / Shared Memory) 在 Workgroup 内传递数据

优势:
  - 1 次 Dispatch 代替 11 次
  - 减少 11 次 barrier 到 0 次
  - GPU 利用率大幅提升
  - 性能提升 3-5x

限制:
  - 需要 Shader Model 6.0+ (Wave Intrinsics)
  - 或使用 GroupMemoryBarrier + LDS 的兼容实现
```

### 2.3 SPD 核心思路

```hlsl
// === Single Pass Downsample 核心思路 ===

// 每个 Workgroup 处理 64×64 的区域
// 在 Workgroup 内部通过 LDS 逐级降采样

groupshared float sharedDepth[64][64]; // LDS

[numthreads(256, 1, 1)]
void CSSPD(uint3 groupId : SV_GroupID, uint localIdx : SV_GroupIndex)
{
    // Phase 1: 从全局内存加载 64×64 区域到 LDS
    // 每个线程加载 16 个像素 (256 threads × 16 = 4096 = 64×64)
    for (int i = 0; i < 16; i++)
    {
        int2 coord = ComputeCoord(groupId.xy, localIdx, i);
        sharedDepth[coord.y][coord.x] = _InputDepth[coord];
    }
    GroupMemoryBarrierWithGroupSync();
    
    // Phase 2: 在 LDS 中逐级降采样
    // Mip 1: 64×64 → 32×32 (256 threads, 每线程处理 4 个)
    if (localIdx < 256)
    {
        int2 coord = ...; // 32×32 中的坐标
        float d = min(min(sharedDepth[coord.y*2][coord.x*2],
                          sharedDepth[coord.y*2][coord.x*2+1]),
                      min(sharedDepth[coord.y*2+1][coord.x*2],
                          sharedDepth[coord.y*2+1][coord.x*2+1]));
        sharedDepth[coord.y][coord.x] = d;
        _MipOutput[1][globalCoord] = d; // 写入 Mip 1
    }
    GroupMemoryBarrierWithGroupSync();
    
    // Mip 2: 32×32 → 16×16 (64 threads)
    // Mip 3: 16×16 → 8×8 (16 threads)
    // Mip 4: 8×8 → 4×4 (4 threads)
    // Mip 5: 4×4 → 2×2 (1 thread)
    // Mip 6: 2×2 → 1×1 (1 thread)
    // ...
    
    // Phase 3: 跨 Workgroup 的更高 Mip
    // 使用 Global Atomic Counter 同步
    // 最后一个完成的 Workgroup 负责生成剩余 Mip
}
```

## 3. 遮挡测试算法

### 3.1 基本遮挡测试

```hlsl
// === Hi-Z Occlusion Test Compute Shader ===

struct InstanceData
{
    float3 boundsCenter;   // AABB 中心 (世界空间)
    float3 boundsExtents;  // AABB 半尺寸 (世界空间)
};

StructuredBuffer<InstanceData> _AllInstances;
RWStructuredBuffer<uint> _VisibleInstances;  // 可见实例索引
RWByteAddressBuffer _VisibleCount;           // 可见实例数量

Texture2D<float> _HiZTexture;  // Depth Pyramid
SamplerState sampler_point_clamp;

float4x4 _ViewProjectionMatrix;
float4 _ScreenParams;  // (width, height, 1/width, 1/height)
int _HiZMipCount;      // Mip 总数

[numthreads(64, 1, 1)]
void CSOcclusionTest(uint3 id : SV_DispatchThreadID)
{
    if (id.x >= _InstanceCount) return;
    
    InstanceData inst = _AllInstances[id.x];
    
    // === Step 1: 视锥剔除 (先做，快速排除大量物体) ===
    if (!FrustumTest(inst.boundsCenter, inst.boundsExtents))
        return;
    
    // === Step 2: 将 AABB 8 个顶点投影到 NDC ===
    float3 center = inst.boundsCenter;
    float3 extents = inst.boundsExtents;
    
    float3 corners[8];
    corners[0] = center + float3(-extents.x, -extents.y, -extents.z);
    corners[1] = center + float3( extents.x, -extents.y, -extents.z);
    corners[2] = center + float3(-extents.x,  extents.y, -extents.z);
    corners[3] = center + float3( extents.x,  extents.y, -extents.z);
    corners[4] = center + float3(-extents.x, -extents.y,  extents.z);
    corners[5] = center + float3( extents.x, -extents.y,  extents.z);
    corners[6] = center + float3(-extents.x,  extents.y,  extents.z);
    corners[7] = center + float3( extents.x,  extents.y,  extents.z);
    
    float minX = 1.0, minY = 1.0, minZ = 1.0;
    float maxX = 0.0, maxY = 0.0;
    
    bool behindCamera = false;
    
    for (int i = 0; i < 8; i++)
    {
        float4 clipPos = mul(_ViewProjectionMatrix, float4(corners[i], 1.0));
        
        // 检查是否在相机后面
        if (clipPos.w <= 0)
        {
            behindCamera = true;
            break;
        }
        
        // 透视除法 → NDC
        float3 ndc = clipPos.xyz / clipPos.w;
        
        // NDC → UV (0-1)
        float2 uv = ndc.xy * 0.5 + 0.5;
        #if UNITY_UV_STARTS_AT_TOP
            uv.y = 1.0 - uv.y;
        #endif
        
        minX = min(minX, uv.x);
        maxX = max(maxX, uv.x);
        minY = min(minY, uv.y);
        maxY = max(maxY, uv.y);
        
        // Reversed-Z: 近处 z=1, 远处 z=0
        minZ = min(minZ, ndc.z); // 最远点 (最小值)
    }
    
    // 如果有顶点在相机后面，保守地认为可见
    if (behindCamera)
    {
        AppendVisible(id.x);
        return;
    }
    
    // 如果完全在屏幕外，已被视锥剔除处理
    // 但这里做额外的屏幕边界检查
    if (maxX < 0 || minX > 1 || maxY < 0 || minY > 1)
        return;
    
    // Clamp to screen
    minX = saturate(minX);
    maxX = saturate(maxX);
    minY = saturate(minY);
    maxY = saturate(maxY);
    
    // === Step 3: 计算合适的 Mip Level ===
    float2 screenSize = float2(maxX - minX, maxY - minY) * _ScreenParams.xy;
    float mipLevel = ceil(log2(max(screenSize.x, screenSize.y)));
    mipLevel = clamp(mipLevel, 0, _HiZMipCount - 1);
    
    // === Step 4: 采样 Depth Pyramid ===
    // 在选定的 Mip Level 下，包围矩形大约覆盖 1-4 个像素
    // 采样 4 个角的深度，取最远的
    float2 uvCenter = float2((minX + maxX) * 0.5, (minY + maxY) * 0.5);
    
    // 方法 A: 采样 4 个角 (更精确)
    float d0 = _HiZTexture.SampleLevel(sampler_point_clamp, float2(minX, minY), mipLevel);
    float d1 = _HiZTexture.SampleLevel(sampler_point_clamp, float2(maxX, minY), mipLevel);
    float d2 = _HiZTexture.SampleLevel(sampler_point_clamp, float2(minX, maxY), mipLevel);
    float d3 = _HiZTexture.SampleLevel(sampler_point_clamp, float2(maxX, maxY), mipLevel);
    
    // Reversed-Z: 最远深度 = 最小值
    float furthestDepth = min(min(d0, d1), min(d2, d3));
    
    // === Step 5: 遮挡判断 ===
    // Reversed-Z: 物体最近深度 (最大 z 值) vs 场景最远深度 (最小 z 值)
    // 注意: 这里 minZ 是 AABB 投影后的最小 NDC.z，在 Reversed-Z 中是最远点
    // 我们需要的是 AABB 的最近点 = max(ndc.z)
    // 重新计算:
    float objectClosestZ = 0.0; // 需要在上面的循环中计算 maxZ
    
    // Reversed-Z: objectClosestZ > furthestDepth → 被遮挡
    // (物体最近点比场景最远点还远)
    if (objectClosestZ < furthestDepth) // Reversed-Z: 近处值大
    {
        // 被遮挡，不添加到可见列表
        return;
    }
    
    // 可见，添加到列表
    AppendVisible(id.x);
}

void AppendVisible(uint instanceIndex)
{
    uint idx;
    _VisibleCount.InterlockedAdd(0, 1, idx);
    _VisibleInstances[idx] = instanceIndex;
}
```

### 3.2 Reversed-Z 深度比较详解

```
=== Reversed-Z (Unity / Unreal 默认) ===

深度值范围: Near Plane = 1.0, Far Plane = 0.0

近处物体: z ≈ 1.0 (大)
远处物体: z ≈ 0.0 (小)

Depth Pyramid 存储: 每个像素区域的 "最远深度"
  最远深度 = 最小的 z 值
  所以 Downsample 操作 = MIN

遮挡判断:
  objectClosestZ = max(所有 AABB 顶点的 ndc.z)  // 最近点, 值最大
  sceneDepth = sample(HiZ, uv, mip)              // 区域最远深度, 值最小
  
  if (objectClosestZ < sceneDepth)
      → 物体最近点比场景最远点还远 → 被遮挡!
  
  等价于: 物体完全在深度缓冲的 "后面"


=== Standard-Z (OpenGL 默认) ===

深度值范围: Near Plane = 0.0, Far Plane = 1.0

近处物体: z ≈ 0.0 (小)
远处物体: z ≈ 1.0 (大)

Depth Pyramid 存储: 每个像素区域的 "最远深度"
  最远深度 = 最大的 z 值
  所以 Downsample 操作 = MAX

遮挡判断:
  objectClosestZ = min(所有 AABB 顶点的 ndc.z)  // 最近点, 值最小
  sceneDepth = sample(HiZ, uv, mip)              // 区域最远深度, 值最大
  
  if (objectClosestZ > sceneDepth)
      → 物体最近点比场景最远点还远 → 被遮挡!


⚠️ 关键: 搞混 Reversed-Z 和 Standard-Z 是 Hi-Z 最常见的 Bug!
```

### 3.3 球体包围盒优化

```hlsl
// 使用球体代替 AABB 可以减少计算量
// 8 个顶点投影 → 1 个中心点投影 + 半径计算

bool OcclusionTestSphere(float3 center, float radius)
{
    // 投影中心点
    float4 clipPos = mul(_ViewProjectionMatrix, float4(center, 1.0));
    if (clipPos.w <= 0) return false; // behind camera, visible
    
    float3 ndc = clipPos.xyz / clipPos.w;
    float2 uv = ndc.xy * 0.5 + 0.5;
    
    // 计算球体在屏幕上的像素半径
    // 近似: screenRadius = radius / distance * focalLength
    float dist = clipPos.w;
    float screenRadius = radius / dist * _ProjectionParams.x;
    float pixelRadius = screenRadius * _ScreenParams.y * 0.5;
    
    // Mip Level
    float mipLevel = ceil(log2(pixelRadius * 2.0));
    mipLevel = clamp(mipLevel, 0, _HiZMipCount - 1);
    
    // 采样
    float sceneDepth = _HiZTexture.SampleLevel(sampler_point_clamp, uv, mipLevel);
    
    // 物体最近深度 (中心 - 半径方向的最近点)
    float objectClosestZ = ndc.z; // 简化: 使用中心深度
    // 更精确: 考虑半径在深度方向的偏移
    // objectClosestZ = (clipPos.z + radius * _ProjectionParams.z) / clipPos.w;
    
    // Reversed-Z
    return objectClosestZ < sceneDepth; // true = occluded
}
```

## 4. 完整调用流程

### 4.1 单帧渲染流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Frame N 渲染流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────┐                   │
│  │ 1. 获取深度源                            │                   │
│  │    方案 A: 使用 Frame N-1 的深度 (1帧延迟)│                   │
│  │    方案 B: 先渲染大型遮挡体的 Pre-Z       │                   │
│  └─────────────────┬───────────────────────┘                   │
│                    │                                             │
│                    ▼                                             │
│  ┌─────────────────────────────────────────┐                   │
│  │ 2. 构建 Depth Pyramid                    │                   │
│  │    CS: Downsample Depth → Mip Chain      │                   │
│  │    输入: Depth Buffer                     │                   │
│  │    输出: Hi-Z Texture (all mips)          │                   │
│  │    耗时: ~0.1-0.3ms                       │                   │
│  └─────────────────┬───────────────────────┘                   │
│                    │                                             │
│                    ▼                                             │
│  ┌─────────────────────────────────────────┐                   │
│  │ 3. 视锥剔除 (Frustum Culling)            │                   │
│  │    CS: 测试所有实例的 AABB vs 6 个平面    │                   │
│  │    输入: 所有实例 AABB + VP 矩阵          │                   │
│  │    输出: 通过视锥测试的实例列表            │                   │
│  │    耗时: ~0.05-0.2ms                      │                   │
│  └─────────────────┬───────────────────────┘                   │
│                    │                                             │
│                    ▼                                             │
│  ┌─────────────────────────────────────────┐                   │
│  │ 4. Hi-Z 遮挡剔除                         │                   │
│  │    CS: 对通过视锥测试的实例做遮挡测试     │                   │
│  │    输入: 视锥可见实例 + Hi-Z Texture      │                   │
│  │    输出: 最终可见实例列表 + IndirectArgs   │                   │
│  │    耗时: ~0.05-0.2ms                      │                   │
│  └─────────────────┬───────────────────────┘                   │
│                    │                                             │
│                    ▼                                             │
│  ┌─────────────────────────────────────────┐                   │
│  │ 5. 渲染可见实例                           │                   │
│  │    DrawMeshInstancedIndirect              │                   │
│  │    或 BRG BatchDrawCommand                │                   │
│  │    只渲染通过所有剔除的实例                │                   │
│  └─────────────────────────────────────────┘                   │
│                                                                 │
│  ┌─────────────────────────────────────────┐                   │
│  │ 6. 保存当前帧深度 (供 Frame N+1 使用)     │                   │
│  └─────────────────────────────────────────┘                   │

└─────────────────────────────────────────────────────────────────┘
```

### 4.2 Unity URP 中的调用时序

```csharp
// === Unity URP 集成示例 ===

public class HiZOcclusionCullingPass : ScriptableRenderPass
{
    ComputeShader downsampleCS;
    ComputeShader cullingCS;
    
    RTHandle hiZTexture;       // Depth Pyramid
    RTHandle prevFrameDepth;   // 上一帧深度
    
    public override void OnCameraSetup(CommandBuffer cmd, ref RenderingData renderingData)
    {
        // 分配 Hi-Z Texture (带 Mipmap)
        var desc = new RenderTextureDescriptor(
            Screen.width, Screen.height, RenderTextureFormat.RFloat, 0);
        desc.useMipMap = true;
        desc.autoGenerateMips = false;
        desc.enableRandomWrite = true;
        desc.mipCount = CalculateMipCount(Screen.width, Screen.height);
        
        RenderingUtils.ReAllocateIfNeeded(ref hiZTexture, desc, name: "_HiZTexture");
    }
    
    public override void Execute(ScriptableRenderContext context, ref RenderingData renderingData)
    {
        CommandBuffer cmd = CommandBufferPool.Get("Hi-Z Occlusion Culling");
        
        // Step 1: Copy previous frame depth (or current depth prepass)
        cmd.CopyTexture(prevFrameDepth, hiZTexture, 0, 0); // Mip 0
        
        // Step 2: Build Depth Pyramid
        int mipCount = hiZTexture.rt.mipmapCount;
        for (int mip = 1; mip < mipCount; mip++)
        {
            int mipWidth = Mathf.Max(1, Screen.width >> mip);
            int mipHeight = Mathf.Max(1, Screen.height >> mip);
            
            cmd.SetComputeTextureParam(downsampleCS, 0, "_InputDepth", hiZTexture, mip - 1);
            cmd.SetComputeTextureParam(downsampleCS, 0, "_OutputDepth", hiZTexture, mip);
            cmd.SetComputeIntParams(downsampleCS, "_InputSize",
                Mathf.Max(1, Screen.width >> (mip - 1)),
                Mathf.Max(1, Screen.height >> (mip - 1)));
            
            cmd.DispatchCompute(downsampleCS, 0,
                Mathf.CeilToInt(mipWidth / 8f),
                Mathf.CeilToInt(mipHeight / 8f), 1);
        }
        
        // Step 3: Frustum + Hi-Z Culling
        cmd.SetComputeMatrixParam(cullingCS, "_VPMatrix",
            renderingData.cameraData.GetProjectionMatrix() *
            renderingData.cameraData.GetViewMatrix());
        cmd.SetComputeTextureParam(cullingCS, 0, "_HiZTexture", hiZTexture);
        cmd.SetComputeBufferParam(cullingCS, 0, "_AllInstances", allInstancesBuffer);
        cmd.SetComputeBufferParam(cullingCS, 0, "_VisibleInstances", visibleBuffer);
        cmd.SetComputeIntParam(cullingCS, "_HiZMipCount", mipCount);
        cmd.SetComputeVectorParam(cullingCS, "_ScreenParams",
            new Vector4(Screen.width, Screen.height, 1f / Screen.width, 1f / Screen.height));
        
        // Reset visible count
        cmd.SetBufferCounterValue(visibleBuffer, 0);
        
        cmd.DispatchCompute(cullingCS, 0,
            Mathf.CeilToInt(instanceCount / 64f), 1, 1);
        
        // Copy count to indirect args
        cmd.CopyCounterValue(visibleBuffer, argsBuffer, sizeof(uint));
        
        // Step 4: Draw
        cmd.DrawMeshInstancedIndirect(mesh, 0, material, 0, argsBuffer);
        
        context.ExecuteCommandBuffer(cmd);
        CommandBufferPool.Release(cmd);
    }
}
```

## 5. Two-Phase Occlusion Culling

### 5.1 为什么需要两阶段

```
单阶段 Hi-Z 的问题:

使用上一帧深度 → 1 帧延迟 → 可能出现闪烁

场景:
  Frame N-1: 物体 A 在相机前方，被渲染
  Frame N:   相机快速转向，物体 A 不在视野中
             但上一帧深度中有物体 A 的深度
             新进入视野的物体 B 可能被错误地判断为 "被 A 遮挡"
             → 物体 B 闪烁消失一帧!

解决方案: Two-Phase Occlusion Culling (两阶段遮挡剔除)
```

### 5.2 两阶段流程

```
┌─────────────────────────────────────────────────────────────────┐
│              Two-Phase Occlusion Culling                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Phase 1: 使用上一帧深度做保守剔除                               │
│  ─────────────────────────────────────                          │
│  输入: 上一帧的 Depth Pyramid                                    │
│  处理: 对所有实例做 Hi-Z 测试                                    │
│  输出:                                                          │
│    - 确定可见列表 (上一帧也可见的物体)                            │
│    - 不确定列表 (上一帧不可见，需要重新测试)                      │
│                                                                 │
│  判断逻辑:                                                      │
│    if (上一帧可见 && 本帧 Hi-Z 测试通过)                         │
│        → 确定可见 ✅                                             │
│    if (上一帧可见 && 本帧 Hi-Z 测试不通过)                       │
│        → 不确定 (可能是相机移动导致) → 放入不确定列表             │
│    if (上一帧不可见)                                             │
│        → 不确定 → 放入不确定列表                                 │
│                                                                 │
│                    │                                             │
│                    ▼                                             │
│  渲染 Phase 1 确定可见的物体                                     │
│  (这些物体的深度会写入当前帧的 Depth Buffer)                     │
│                                                                 │
│                    │                                             │
│                    ▼                                             │
│  Phase 2: 使用当前帧深度重新测试不确定物体                       │
│  ─────────────────────────────────────                          │
│  输入: 当前帧的 Depth Buffer (Phase 1 渲染后)                    │
│  处理: 重建 Depth Pyramid，对不确定列表做 Hi-Z 测试              │
│  输出: 额外的可见物体列表                                        │
│                                                                 │
│                    │                                             │
│                    ▼                                             │
│  渲染 Phase 2 新发现的可见物体                                   │
│                                                                 │
│  结果: 无闪烁! 所有可见物体都被正确渲染                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 两阶段实现

```hlsl
// === Phase 1: Conservative Culling ===

[numthreads(64, 1, 1)]
void CSPhase1Cull(uint3 id : SV_DispatchThreadID)
{
    uint idx = id.x;
    if (idx >= _InstanceCount) return;
    
    InstanceData inst = _AllInstances[idx];
    
    // Frustum test (current frame)
    if (!FrustumTest(inst.boundsCenter, inst.boundsExtents))
        return;
    
    // Hi-Z test using PREVIOUS frame depth
    bool occluded = HiZTest(inst.boundsCenter, inst.boundsExtents, _PrevHiZTexture);
    
    bool wasVisibleLastFrame = _VisibilityFlags[idx]; // 上一帧是否可见
    
    if (!occluded && wasVisibleLastFrame)
    {
        // Definitely visible: passed Hi-Z and was visible last frame
        AppendToPhase1Visible(idx);
        _VisibilityFlags[idx] = 1;
    }
    else
    {
        // Uncertain: need Phase 2 re-test
        AppendToPhase2Test(idx);
    }
}

// === Phase 2: Re-test with current frame depth ===

[numthreads(64, 1, 1)]
void CSPhase2Cull(uint3 id : SV_DispatchThreadID)
{
    uint idx = id.x;
    if (idx >= _Phase2Count) return;
    
    uint instanceIdx = _Phase2TestList[idx];
    InstanceData inst = _AllInstances[instanceIdx];
    
    // Hi-Z test using CURRENT frame depth (after Phase 1 rendering)
    bool occluded = HiZTest(inst.boundsCenter, inst.boundsExtents, _CurrentHiZTexture);
    
    if (!occluded)
    {
        AppendToPhase2Visible(instanceIdx);
        _VisibilityFlags[instanceIdx] = 1;
    }
    else
    {
        _VisibilityFlags[instanceIdx] = 0;
    }
}
```

## 6. 注意事项与陷阱

### 6.1 关键注意事项

| #   | 注意事项            | 严重程度 | 说明                                                      |
| --- | ------------------- | -------- | --------------------------------------------------------- |
| 1   | **Reversed-Z 方向** | 🔴 致命   | Unity/UE 使用 Reversed-Z，Downsample 取 MIN，比较方向相反 |
| 2   | **非 2 的幂分辨率** | 🔴 致命   | 1920×1080 不是 2 的幂，Downsample 时边界像素需要特殊处理  |
| 3   | **UV 坐标系**       | 🟡 严重   | Unity 的 UV 起点可能在顶部或底部，取决于平台              |
| 4   | **1 帧延迟**        | 🟡 严重   | 使用上一帧深度会导致快速移动时闪烁，需要 Two-Phase        |
| 5   | **保守性**          | 🟢 注意   | Hi-Z 是保守剔除，可能漏剔但不会错剔                       |
| 6   | **小物体**          | 🟡 严重   | 极小物体在高 Mip 下可能被错误遮挡                         |
| 7   | **透明物体**        | 🟡 严重   | 透明物体不写深度，不能作为遮挡体                          |
| 8   | **阴影 Pass**       | 🟢 注意   | Shadow Map 的视角不同，需要单独的 Hi-Z                    |

### 6.2 常见 Bug 与修复

```
Bug 1: 所有物体都被剔除
─────────────────────
原因: Reversed-Z 方向搞反
修复: 
  Downsample: 使用 MIN (不是 MAX)
  比较: objectClosestZ < sceneDepth (不是 >)


Bug 2: 物体在屏幕边缘闪烁
─────────────────────────
原因: AABB 投影到屏幕外时 UV clamp 不正确
修复:
  - 正确 clamp UV 到 [0, 1]
  - 如果 AABB 跨越屏幕边缘，使用更高的 Mip Level
  - 如果 AABB 部分在相机后面，标记为可见


Bug 3: 非 2 的幂分辨率下出现条纹
──────────────────────────────
原因: Downsample 时奇数行/列的像素被遗漏
修复:
  - 检测奇数维度，额外采样第 3 行/列
  - 或者将深度 Buffer 对齐到 2 的幂 (浪费内存)


Bug 4: 快速移动相机时物体闪烁
────────────────────────────
原因: 上一帧深度与当前帧视角差异大
修复:
  - 实现 Two-Phase Occlusion Culling
  - 或使用 Reprojection (将上一帧深度重投影到当前帧)


Bug 5: 小物体被错误剔除
─────────────────────
原因: Mip Level 选择过高，一个像素覆盖了太大的区域
修复:
  - 对小物体 (屏幕占比 < 2 像素) 跳过 Hi-Z 测试
  - 或使用更低的 Mip Level (mipLevel - 1)
  - 添加保守偏移: mipLevel = max(0, mipLevel - 1)


Bug 6: 阴影中物体消失
───────────────────
原因: Hi-Z 只针对主相机做了剔除，Shadow Map 的视角不同
修复:
  - Shadow Pass 不使用 Hi-Z 剔除
  - 或为每个 Shadow Cascade 单独构建 Hi-Z
  - 或使用保守策略: 如果物体在主相机视锥内，Shadow 也渲染
```

### 6.3 深度源选择

```
深度源选择对比:

方案 A: 上一帧完整深度
  ✅ 包含所有不透明物体的深度
  ✅ 不增加当前帧渲染负担
  ❌ 1 帧延迟，快速移动时闪烁
  ❌ 需要额外的深度 Copy
  适用: 大多数场景

方案 B: 当前帧 Depth Prepass
  ✅ 无延迟，当前帧精确深度
  ❌ 需要先渲染 Depth Prepass (额外开销)
  ❌ Depth Prepass 本身也需要剔除 (鸡生蛋问题)
  适用: 已经有 Depth Prepass 的管线 (HDRP)

方案 C: 大型遮挡体 Pre-Z
  ✅ 只渲染少量大型遮挡体 (地形、建筑)
  ✅ 开销小，深度质量好
  ❌ 需要标记哪些物体是 "大型遮挡体"
  ❌ 小物体的遮挡关系无法利用
  适用: 有明确大型遮挡体的场景

方案 D: Two-Phase (推荐)
  ✅ 结合上一帧深度和当前帧深度
  ✅ 无闪烁
  ❌ 实现复杂
  ❌ 两次 Hi-Z 构建 + 两次剔除
  适用: 高质量要求的项目
```

## 11. 高级优化技巧

### 11.1 空间分区加速

```
问题: 100K 实例逐个做 Hi-Z 测试太慢

解决: 层级剔除 (Hierarchical Culling)

┌─────────────────────────────────────────┐
│ Level 0: 场景 Quadtree/Octree 节点      │
│   → 先测试大节点的 AABB                 │
│   → 如果节点被遮挡，跳过所有子节点       │
│   → 如果节点可见，递归测试子节点          │
│                                         │
│ Level 1: Chunk (32×32 区域)             │
│   → 测试 Chunk 的 AABB                  │
│   → 被遮挡则跳过 Chunk 内所有实例        │
│                                         │
│ Level 2: 单个实例                        │
│   → 只对通过 Chunk 测试的实例做精确测试   │
└─────────────────────────────────────────┘

效果:
  100K 实例 → 1000 Chunk 测试 + ~5000 实例测试
  减少 95% 的 Hi-Z 测试次数!
```

### 11.2 Temporal Stability (时间稳定性)

```hlsl
// 使用上一帧的可见性结果作为 hint
// 上一帧可见的物体大概率本帧也可见

[numthreads(64, 1, 1)]
void CSCullWithTemporal(uint3 id : SV_DispatchThreadID)
{
    uint idx = id.x;
    bool wasVisible = _PrevVisibility[idx];
    
    // 上一帧可见的物体: 直接标记可见，跳过 Hi-Z 测试
    // (下一帧再验证)
    if (wasVisible)
    {
        AppendVisible(idx);
        return;
    }
    
    // 上一帧不可见的物体: 做完整的 Hi-Z 测试
    if (!HiZTest(...))
    {
        AppendVisible(idx);
        _PrevVisibility[idx] = 1;
    }
}

// 优势: 大幅减少每帧的 Hi-Z 测试次数
// 风险: 可能延迟 1 帧才剔除新被遮挡的物体
// 适用: 相机移动平滑的场景
```

### 11.3 Depth Reprojection

```hlsl
// 将上一帧的深度重投影到当前帧
// 减少 1 帧延迟的影响

[numthreads(8, 8, 1)]
void CSReprojectDepth(uint3 id : SV_DispatchThreadID)
{
    float2 uv = (id.xy + 0.5) * _InvScreenSize;
    float depth = _PrevDepth[id.xy];
    
    // 从上一帧的屏幕空间重建世界坐标
    float4 clipPos = float4(uv * 2.0 - 1.0, depth, 1.0);
    float4 worldPos = mul(_PrevInvViewProjection, clipPos);
    worldPos /= worldPos.w;
    
    // 投影到当前帧的屏幕空间
    float4 currClip = mul(_CurrViewProjection, worldPos);
    float2 currUV = (currClip.xy / currClip.w) * 0.5 + 0.5;
    float currDepth = currClip.z / currClip.w;
    
    // 写入重投影后的深度
    if (currUV.x >= 0 && currUV.x <= 1 && currUV.y >= 0 && currUV.y <= 1)
    {
        // Atomic MIN (Reversed-Z)
        uint depthUint = asuint(currDepth);
        InterlockedMin(_ReprojectedDepth[uint2(currUV * _ScreenSize)], depthUint);
    }
}
```
