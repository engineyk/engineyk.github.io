## 3. 间接渲染管线优化

### 3.1 位置贴图编码优化

#### 问题
从贴图读取草位置时，贴图格式和编码方式直接影响 Compute Shader 的采样效率。

#### 优化方案

```hlsl
// ❌ 低效：使用 RGBA32F 存储位置 (每像素 16 bytes)
float4 posData = positionTex.Load(int3(id.xy, 0)); // 16 bytes/pixel

// ✅ 优化方案 1：R16G16 存储 XZ 归一化坐标，Y 从高度图采样
// 每像素 4 bytes，节省 75% 带宽
float2 xzNorm = positionTex.Load(int3(id.xy, 0)).rg; // 4 bytes/pixel
float3 worldPos;
worldPos.xz = xzNorm * _TerrainSize.xz + _TerrainOffset.xz;
worldPos.y  = heightMap.SampleLevel(sampler_linear, xzNorm, 0).r * _TerrainSize.y;

// ✅ 优化方案 2：R32 打包 XZ (各 16bit) + 高度图
// 每像素 4 bytes，精度足够覆盖 256m x 256m 地形 (精度 ~0.004m)
uint packed = asuint(positionTex.Load(int3(id.xy, 0)).r);
float2 xzNorm = float2(
    (packed >> 16) / 65535.0,
    (packed & 0xFFFF) / 65535.0
);
```

#### 贴图格式对比

| 格式         | 每像素字节 | 精度    | 适用场景               |
| ------------ | ---------- | ------- | ---------------------- |
| RGBA32F      | 16         | 完美    | 调试用，生产环境不推荐 |
| RG16         | 4          | 0.0015% | ✅ 推荐，配合高度图     |
| R32 (packed) | 4          | 0.0015% | ✅ 推荐，单通道纹理     |
| RGBA8        | 4          | 0.39%   | 小地形可用 (< 50m)     |

### 3.2 Compute Shader 剔除优化

#### 3.2.1 多级剔除管线

```hlsl
// === Stage 1: Chunk 级粗剔除 (每 Chunk 64x64 草) ===
// 一个线程处理一个 Chunk，快速排除大量不可见区域
[numthreads(8, 8, 1)]
void CSChunkCull(uint3 id : SV_DispatchThreadID)
{
    // Chunk AABB vs 视锥体
    float3 chunkCenter = GetChunkCenter(id.xy);
    float3 chunkExtent = _ChunkSize * 0.5;
    
    if (!FrustumTestAABB(chunkCenter, chunkExtent))
        return;
    
    // 距离剔除 (Chunk 级)
    float dist = distance(chunkCenter, _CameraPos);
    if (dist > _MaxGrassDistance)
        return;
    
    // 通过粗剔除，加入 Stage 2 列表
    uint idx;
    InterlockedAdd(visibleChunkCount[0], 1, idx);
    visibleChunks[idx] = id.xy;
}

// === Stage 2: 实例级精细剔除 ===
[numthreads(64, 1, 1)]
void CSInstanceCull(uint3 groupId : SV_GroupID, uint3 localId : SV_GroupThreadID)
{
    uint2 chunkId = visibleChunks[groupId.x];
    uint localIdx = localId.x;
    
    // 从位置贴图采样
    uint2 texCoord = chunkId * 64 + uint2(localIdx % 8, localIdx / 8);
    float3 worldPos = DecodeGrassPosition(texCoord);
    
    // 1. 视锥剔除 (点 + 半径)
    if (!FrustumTestSphere(worldPos, _GrassHeight * 0.5))
        return;
    
    // 2. 距离剔除 + LOD 计算
    float dist = distance(worldPos, _CameraPos);
    if (dist > _MaxGrassDistance)
        return;
    
    uint lod = ComputeGrassLOD(dist);
    
    // 3. 密度衰减 (远处随机剔除)
    float density = saturate(1.0 - (dist - _DensityFadeStart) / (_DensityFadeEnd - _DensityFadeStart));
    uint hash = WangHash(texCoord.x + texCoord.y * 65536);
    float rand01 = hash / 4294967295.0;
    if (rand01 > density)
        return;
    
    // 4. 遮挡剔除 (Hi-Z)
    float4 clipPos = mul(_ViewProjMatrix, float4(worldPos, 1.0));
    float2 screenUV = clipPos.xy / clipPos.w * 0.5 + 0.5;
    float sceneDepth = HiZBuffer.SampleLevel(sampler_point, screenUV, _HiZMipLevel).r;
    
    #if UNITY_REVERSED_Z
        if (clipPos.z / clipPos.w < sceneDepth) // Reversed-Z: 草更远则被遮挡
            return;
    #else
        if (clipPos.z / clipPos.w > sceneDepth)
            return;
    #endif
    
    // 写入可见实例
    GrassInstance inst;
    inst.position = worldPos;
    inst.lod = lod;
    inst.rotation = DecodeGrassRotation(texCoord);
    
    uint appendIdx;
    InterlockedAdd(indirectArgs[lod * 5 + 1], 1, appendIdx); // instanceCount++
    visibleInstances[lod].Append(inst);
}
```

#### 3.2.2 Hi-Z 遮挡剔除细节

```hlsl
// Hi-Z Mip Level 选择：根据草在屏幕上的投影大小
float ComputeHiZMipLevel(float3 worldPos, float radius)
{
    float dist = distance(worldPos, _CameraPos);
    float screenRadius = radius / dist * _ScreenHeight * 0.5; // 屏幕像素半径
    float mipLevel = ceil(log2(max(screenRadius * 2.0, 1.0)));  // 覆盖的 mip
    return min(mipLevel, _HiZMaxMip);
}
```

### 3.3 Indirect Args 优化

```hlsl
// ❌ 每帧 CPU 回读 Buffer 来设置 Draw 参数
// GPU → CPU 回读会导致 pipeline stall

// ✅ 完全 GPU 驱动：CS 直接写入 IndirectArgs
RWByteAddressBuffer indirectArgsBuffer; // 5 uint per LOD

// 初始化 (每帧开始)
[numthreads(1, 1, 1)]
void CSResetArgs(uint3 id : SV_DispatchThreadID)
{
    // DrawIndexedInstancedIndirect 参数:
    // [0] indexCountPerInstance
    // [1] instanceCount (CS 剔除时累加)
    // [2] startIndexLocation  
    // [3] baseVertexLocation
    // [4] startInstanceLocation
    
    for (uint lod = 0; lod < LOD_COUNT; lod++)
    {
        uint offset = lod * 20; // 5 * 4 bytes
        indirectArgsBuffer.Store(offset + 0, _IndexCountPerLOD[lod]);
        indirectArgsBuffer.Store(offset + 4, 0); // instanceCount = 0
        indirectArgsBuffer.Store(offset + 8, _StartIndexPerLOD[lod]);
        indirectArgsBuffer.Store(offset + 12, 0);
        indirectArgsBuffer.Store(offset + 16, 0);
    }
}
```

## 4. 顶点阶段优化

### 4.1 草片 Mesh 精简

#### 4.1.1 顶点数对比

| 草片类型           | 顶点数 | 三角形数 | 适用 LOD   |
| ------------------ | ------ | -------- | ---------- |
| 十字交叉片 (Cross) | 8      | 4        | LOD0 近景  |
| 单 Quad            | 4      | 2        | LOD1 中景  |
| 单三角形           | 3      | 1        | LOD2 远景  |
| Billboard Quad     | 4      | 2        | LOD2+ 远景 |
| 点精灵 (Point)     | 1      | -        | LOD3 极远  |

#### 4.1.2 程序化生成草片顶点 (无需 Mesh)

```hlsl
// 完全在 VS 中程序化生成草片，省去 Vertex Buffer
// 使用 SV_VertexID 推算顶点位置

struct GrassVertex
{
    float3 posOS;
    float2 uv;
};

// 单 Quad 草片: 4 顶点, 6 索引 (2 三角形)
// 顶点布局:
//  2───3
//  │ \ │
//  0───1

GrassVertex GetGrassVertex(uint vertexID, float width, float height)
{
    GrassVertex v;
    
    // vertexID: 0=左下, 1=右下, 2=左上, 3=右上
    float x = (vertexID & 1) ? 0.5 : -0.5;      // 0,1 → left/right
    float y = (vertexID >> 1) ? 1.0 : 0.0;       // 0,1 → bottom/top
    
    v.posOS = float3(x * width, y * height, 0.0);
    v.uv = float2((vertexID & 1), (vertexID >> 1));
    
    return v;
}

// 十字交叉: 8 顶点, 12 索引 (4 三角形)
// 两个 Quad 旋转 90°交叉
GrassVertex GetCrossGrassVertex(uint vertexID, float width, float height)
{
    uint quadIdx = vertexID / 4;  // 0 or 1
    uint localVert = vertexID % 4;
    
    GrassVertex v = GetGrassVertex(localVert, width, height);
    
    // 第二个 Quad 绕 Y 轴旋转 90°
    if (quadIdx == 1)
    {
        float temp = v.posOS.x;
        v.posOS.x = v.posOS.z;
        v.posOS.z = temp;
    }
    
    return v;
}
```

### 4.2 VS 计算优化

#### 4.2.1 风力动画简化

```hlsl
// ❌ 昂贵：多层 sin/cos 叠加
float3 wind = float3(
    sin(_Time.y * 2.3 + worldPos.x * 0.5) * cos(_Time.y * 1.7 + worldPos.z * 0.3),
    0,
    cos(_Time.y * 1.9 + worldPos.z * 0.7) * sin(_Time.y * 2.1 + worldPos.x * 0.4)
) * _WindStrength;
// 代价: 4x sin + 2x cos = 6 个三角函数

// ✅ 优化：风力贴图 + 顶点高度权重
// 预烘焙滚动风力纹理 (R8G8, 64x64, 每帧滚动 UV)
float2 windUV = worldPos.xz * _WindTiling + _Time.y * _WindScrollSpeed;
float2 windDir = windTex.SampleLevel(sampler_linear_repeat, windUV, 0).rg * 2.0 - 1.0;

// 只有顶部顶点受风力影响 (uv.y 作为权重)
float windWeight = v.uv.y * v.uv.y; // 二次曲线，底部固定
float3 windOffset = float3(windDir.x, 0, windDir.y) * windWeight * _WindStrength;

// 保持草长度不变 (简化版)
worldPos.xz += windOffset.xz;
worldPos.y -= length(windOffset.xz) * 0.5 * windWeight; // 近似弯曲
// 代价: 1 次纹理采样 + 少量 ALU，远优于三角函数
```

## 5. Alpha Test 与 Early-Z 问题详解

### 5.1 问题根因深度分析

#### 5.1.1 Early-Z 工作原理

```
正常渲染管线 (Early-Z 生效):

  光栅化 → Early-Z Test → Fragment Shader → Late-Z Write
              ↓ 失败                ↓ 通过
           丢弃片元            写入颜色+深度
              
  优势: 被遮挡的片元在 FS 执行前就被丢弃，节省大量 PS 计算
```

```
Alpha Test 管线 (Early-Z 失效):

  光栅化 → [Early-Z 被禁用] → Fragment Shader → clip()/discard → Late-Z Test & Write
                                    ↓                              ↓
                              计算所有片元                    才知道是否可见
                              (包括被遮挡的)

  问题: 
  1. GPU 无法在 FS 前确定片元是否存活 (因为 alpha 值在 FS 中才知道)
  2. 所有片元都必须执行完整 FS
  3. 深度写入延迟到 Late-Z，后续 draw call 无法利用这些深度信息
```

#### 5.1.2 为什么 Alpha Test 破坏 Early-Z

```hlsl
// 当 shader 中包含 discard/clip 时:
half4 frag(v2f i) : SV_Target
{
    half4 col = tex2D(_MainTex, i.uv);
    clip(col.a - _Cutoff);  // ← 这一行导致 Early-Z 失效!
    return col;
}

// 原因:
// 1. discard 意味着某些片元会被动态丢弃
// 2. 如果 Early-Z 提前写入深度，被 discard 的片元的深度也被写入了
// 3. 这会导致后面的物体被错误遮挡 (深度错误)
// 4. 因此硬件必须禁用 Early-Z，等 FS 执行完才做深度测试
```

#### 5.1.3 Overdraw 灾难

```
场景: 100 片草重叠在同一像素

有 Early-Z 时:
  第 1 片: 执行 FS, 写入深度
  第 2-100 片: Early-Z 测试失败, 跳过 FS
  总 FS 执行: 1 次

无 Early-Z 时 (Alpha Test):
  第 1-100 片: 全部执行 FS, 然后 Late-Z 决定谁可见
  总 FS 执行: 100 次 ← Overdraw 100x!
```

### 5.2 解决方案矩阵

| 方案                     | 原理                | 效果  | 复杂度 | 推荐度       |
| ------------------------ | ------------------- | ----- | ------ | ------------ |
| Pre-Z Pass               | 先写深度再着色      | ⭐⭐⭐⭐⭐ | 中     | ✅ 首选       |
| Alpha to Coverage        | MSAA 硬件抖动       | ⭐⭐⭐⭐  | 低     | ✅ MSAA 场景  |
| 调整草片 UV 减少透明区域 | 减少无效 discard    | ⭐⭐⭐   | 低     | ✅ 美术配合   |
| 排序：从前到后渲染       | 最大化 Early-Z 命中 | ⭐⭐⭐   | 中     | ✅ 配合 Pre-Z |
| Dithered Opacity         | 抖动代替 Alpha Test | ⭐⭐⭐   | 低     | 远景 LOD     |
| 不用 Alpha Test          | 草片轮廓贴合 Mesh   | ⭐⭐⭐⭐⭐ | 高     | 顶点数允许时 |

### 5.3 方案详解

#### 5.3.1 Pre-Z Pass (深度预渲染)

**核心思想**：先用一个极简 Pass 只写深度（含 Alpha Test），然后正式渲染时利用已有深度做 Early-Z。

```hlsl
// === Pass 0: Depth Pre-Pass (只写深度) ===
Pass
{
    Name "GrassDepthPrepass"
    Tags { "LightMode" = "DepthOnly" }
    
    ColorMask 0          // 不写颜色
    ZWrite On
    ZTest LEqual
    Cull Off             // 草双面
    
    HLSLPROGRAM
    #pragma vertex vert_depth
    #pragma fragment frag_depth
    
    struct v2f_depth
    {
        float4 pos : SV_POSITION;
        float2 uv  : TEXCOORD0;
    };
    
    v2f_depth vert_depth(appdata v, uint instanceID : SV_InstanceID)
    {
        v2f_depth o;
        GrassInstance inst = instanceBuffer[instanceID];
        float3 worldPos = TransformGrassVertex(v, inst);
        o.pos = mul(UNITY_MATRIX_VP, float4(worldPos, 1.0));
        o.uv = v.uv;
        return o;
    }
    
    void frag_depth(v2f_depth i)
    {
        half alpha = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, i.uv).a;
        clip(alpha - _Cutoff); // Alpha Test 在这里执行
        // 不输出颜色，只写深度
    }
    ENDHLSL
}

// === Pass 1: Color Pass (利用 Pre-Z 深度) ===
Pass
{
    Name "GrassForward"
    Tags { "LightMode" = "UniversalForward" }
    
    ZWrite Off           // ← 关键: 不再写深度!
    ZTest Equal          // ← 关键: 只渲染深度完全匹配的片元!
    Cull Off
    
    HLSLPROGRAM
    #pragma vertex vert_color
    #pragma fragment frag_color
    
    half4 frag_color(v2f i) : SV_Target
    {
        // 不需要 clip()! ZTest Equal 已经保证只有可见片元到达这里
        half4 col = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, i.uv);
        
        // 完整光照计算
        half3 lighting = CalculateGrassLighting(i);
        col.rgb *= lighting;
        
        return col;
    }
    ENDHLSL
}
```

**Pre-Z 性能分析**：

```
代价:
  + 额外一个 Draw Call
  + Depth Pass 的 VS + 简单 FS 开销
  + 深度 Buffer 写入带宽

收益:
  - Color Pass 中 Early-Z 完美生效
  - 每个像素只执行一次完整 FS (Overdraw = 1.0x)
  - 对于草这种高 Overdraw 场景，收益远大于代价

实测数据 (1080p, 50000 草实例, 中端 GPU):
  无 Pre-Z:  GPU Time = 8.2ms (Overdraw ~6x)
  有 Pre-Z:  GPU Time = 3.1ms (Overdraw ~1.0x)
  Pre-Z 本身: GPU Time = 0.8ms
  净收益: 4.3ms (52% 提升)
```

#### 5.3.2 Alpha to Coverage (A2C)

**原理**：利用 MSAA 的多采样点，将 Alpha 值转换为覆盖掩码，无需 `discard`。

```hlsl
// 开启 Alpha to Coverage
Pass
{
    // 需要 MSAA 开启 (4x 或 8x)
    AlphaToMask On       // ← 关键
    ZWrite On            // 可以写深度!
    ZTest LEqual
    Cull Off
    
    HLSLPROGRAM
    half4 frag(v2f i) : SV_Target
    {
        half4 col = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, i.uv);
        
        // 不使用 clip()!
        // Alpha to Coverage 会根据 alpha 值决定哪些 MSAA 采样点被覆盖
        // alpha = 0.5 → 4x MSAA 中 2 个采样点被覆盖
        
        // 锐化 alpha 以减少半透明过渡带
        col.a = (col.a - _Cutoff) / max(fwidth(col.a), 0.0001) * 0.5 + 0.5;
        
        return col;
    }
    ENDHLSL
}
```

**A2C 优缺点**：

```
优点:
  ✅ 不使用 discard，Early-Z 正常工作
  ✅ 边缘比 Alpha Test 更平滑 (MSAA 抗锯齿)
  ✅ 实现简单

缺点:
  ❌ 必须开启 MSAA (额外内存和带宽)
  ❌ 4x MSAA 只有 4 级透明度 (0%, 25%, 50%, 75%, 100%)
  ❌ 移动端 MSAA 开销较大
  ❌ 与延迟渲染不兼容
```

#### 5.3.4 Dithered Opacity (抖动透明)

{
    half alpha = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, i.uv).a;
    
    // 4x4 Bayer 抖动矩阵
    static const float bayerMatrix[16] = {
         0.0/16.0,  8.0/16.0,  2.0/16.0, 10.0/16.0,
        12.0/16.0,  4.0/16.0, 14.0/16.0,  6.0/16.0,

        15.0/16.0,  7.0/16.0, 13.0/16.0,  5.0/16.0
    };
    
    uint2 pixelPos = uint2(i.pos.xy) % 4;
    float threshold = bayerMatrix[pixelPos.y * 4 + pixelPos.x];

    // 1. 远景草逐渐消失 (LOD 过渡)

    return half4(color, 1.0);

### 5.4 渲染顺序优化

```hlsl
// 草的渲染顺序对 Early-Z 效率至关重要

// ❌ 错误顺序: 从后往前 (Painter's Algorithm)
// 最远的草先渲染，写入深度
// 近处的草后渲染，覆盖远处深度
// → 所有片元都执行了 FS，Early-Z 无效

// ✅ 正确顺序: 从前往后 (Front-to-Back)
// 最近的草先渲染，写入深度
// 远处的草后渲染，被 Early-Z 剔除
// → 大量远处片元被跳过

// 在 Compute Shader 中按距离排序:
// 方案 1: 粗粒度排序 (按 Chunk 距离)
// 方案 2: Bitonic Sort on GPU (精确但昂贵)
// 方案 3: 分桶 (近/中/远 三个 Buffer，分别 Draw)

// 推荐方案 3: 分桶渲染
[numthreads(64, 1, 1)]
void CSBucketSort(uint3 id : SV_DispatchThreadID)
{
    GrassInstance inst = allInstances[id.x];
    float dist = distance(inst.position, _CameraPos);
    
    if (dist < _NearDist)
    {
        uint idx;
        InterlockedAdd(nearCount[0], 1, idx);
        nearBuffer[idx] = inst;
    }
    else if (dist < _MidDist)
    {
        uint idx;
        InterlockedAdd(midCount[0], 1, idx);
        midBuffer[idx] = inst;
    }
    else
    {
        uint idx;
        InterlockedAdd(farCount[0], 1, idx);
        farBuffer[idx] = inst;
    }
}

// 渲染顺序: nearBuffer → midBuffer → farBuffer
// 近处草先渲染，建立深度，远处草被 Early-Z 剔除
```

### 5.5 各 GPU 架构的 Early-Z 行为

| GPU 架构            | Early-Z 机制                 | Alpha Test 影响      | 特殊优化                        |
| ------------------- | ---------------------------- | -------------------- | ------------------------------- |
| NVIDIA (Desktop)    | Hi-Z + Early-Z               | discard 禁用 Early-Z | `conservative depth` 可部分恢复 |
| AMD (Desktop)       | Hi-Z + Early-Z               | 同上                 | Re-Z pass 自动优化              |
| Mali (Tile-based)   | Forward Pixel Kill (FPK)     | discard 禁用 FPK     | 使用 `gl_FragDepth` 需谨慎      |
| Adreno (Tile-based) | Low Resolution Z (LRZ)       | discard 禁用 LRZ     | LRZ 是 Adreno 最重要的优化      |
| Apple (Tile-based)  | Hidden Surface Removal (HSR) | discard 部分破坏 HSR | TBDR 架构天然减少 Overdraw      |

```hlsl
// NVIDIA/AMD: Conservative Depth Output
// 告诉 GPU: 我的 fragment 深度不会比光栅化深度更近
// 这样 Early-Z 可以安全地剔除更远的片元

// 需要 shader model 5.0+
// 声明输出深度不会小于光栅化深度
layout(depth_greater) out float gl_FragDepth; // GLSL
// 或 HLSL:
float frag(v2f i, out float depth : SV_DepthGreaterEqual) : SV_Target
{
    half4 col = tex2D(_MainTex, i.uv);
    clip(col.a - _Cutoff);
    depth = i.pos.z; // 输出原始深度
    return col;
}
// 效果: 即使有 discard，Early-Z 仍可剔除深度更大的片元
// 限制: 只能声明 depth >= rasterized depth
```

### 6.1 光照简化

```hlsl
// ❌ 完整 PBR 光照 (草不需要这么复杂)
half3 GrassLighting_Full(v2f i)
{
    half3 N = normalize(i.normalWS);
    half3 V = normalize(_WorldSpaceCameraPos - i.worldPos);
    half3 L = normalize(_MainLightDirection);
    half3 H = normalize(V + L);
    
    half NdotL = saturate(dot(N, L));
    half NdotH = saturate(dot(N, H));
    half NdotV = saturate(dot(N, V));
    
    // GGX specular (草不需要!)
    half D = D_GGX(NdotH, roughness);
    half G = G_SmithGGX(NdotL, NdotV, roughness);
    half3 F = F_Schlick(f0, VdotH);
    half3 spec = D * G * F / (4.0 * NdotL * NdotV + 0.001);
    
    return (diffuse + spec) * NdotL * _MainLightColor;
}

// ✅ 简化草光照
half3 GrassLighting_Optimized(v2f i)
{
    // 草使用简化法线: 半球法线或固定向上
    half3 N = half3(0, 1, 0); // 或从顶点颜色/实例数据获取
    half3 L = _MainLightDirection; // 已归一化，不需要 normalize
    
    // Wrapped Lambert (更柔和的明暗过渡)
    half NdotL = dot(N, L) * 0.5 + 0.5; // [0,1] 范围
    
    // 次表面散射近似 (草叶透光效果)
    half3 backLight = saturate(dot(-N, L)) * _SubsurfaceColor * i.uv.y;
    
    // 环境光遮蔽 (底部更暗)
    half ao = lerp(_AOBottom, 1.0, i.uv.y);
    
    // 最终颜色
    half3 color = _GrassColor * (NdotL * _MainLightColor + backLight) * ao;
    color += _AmbientColor * ao;
    
    return color;
}
```

## 9. 移动端专项优化

### 9.1 Tile-Based GPU 注意事项

```
Mali / Adreno / Apple GPU 都是 Tile-Based 架构:

1. Overdraw 代价更高:
   - 每个 Tile 内的片元都在 on-chip memory 中处理
   - Overdraw 直接消耗 Tile buffer 带宽
   - Alpha Test 破坏 Hidden Surface Removal

2. 优化策略:
   - Pre-Z Pass 在 TBDR 上收益更大
   - 减少 Tile 内的草片重叠
   - 使用 subpass (Vulkan) 减少 Tile store/load

3. LRZ (Adreno) / FPK (Mali) 保护:
   - 避免 discard 以保持 LRZ/FPK 有效
   - 使用 Alpha to Coverage 替代 Alpha Test
```

## 附录 B: 常见问题排查

| 现象           | 可能原因            | 排查方法                                    |
| -------------- | ------------------- | ------------------------------------------- |
| 草闪烁         | Z-Fighting          | 检查 Pre-Z 和 Color Pass 的顶点变换是否一致 |
| 草边缘锯齿     | Alpha Test 硬边     | 使用 A2C 或 fwidth 软化                     |
| 远处草突然消失 | LOD 跳变            | 添加 Dithered/Scale 过渡                    |
| 帧率不稳定     | GC 或 Buffer resize | 预分配 Buffer，避免每帧创建                 |
| 移动端发热     | Overdraw 过高       | 检查 Early-Z 是否生效，添加 Pre-Z           |
| 草穿过地形     | 高度采样精度不足    | 提高高度图精度或使用更高 mip                |
