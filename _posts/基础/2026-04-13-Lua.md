## 一、数据类型

### 1.1 八种基本类型

| 类型         | 说明                                        | 示例                        |
| ------------ | ------------------------------------------- | --------------------------- |
| **nil**      | 空值，表示无效值                            | `local a = nil`             |
| **boolean**  | 布尔值                                      | `true`, `false`             |
| **number**   | 双精度浮点数（Lua 5.3+ 区分 integer/float） | `42`, `3.14`                |
| **string**   | 不可变字节序列                              | `"hello"`, `'world'`        |
| **table**    | 唯一的数据结构（数组+哈希表）               | `{1, 2, 3}`, `{name="lua"}` |
| **function** | 一等公民，闭包                              | `function() end`            |
| **userdata** | C 数据在 Lua 中的表示                       | `io.open()` 返回值          |
| **thread**   | 协程                                        | `coroutine.create()`        |

```lua
-- type() returns type name as string
print(type(nil))           -- "nil"
print(type(42))            -- "number"
print(type("hello"))       -- "string"
print(type({}))            -- "table"
print(type(print))         -- "function"
print(type(true))          -- "boolean"

-- Lua 5.3+ integer vs float
print(type(42))            -- "number"
print(math.type(42))       -- "integer"
print(math.type(42.0))     -- "float"
```

### 1.2 真值与假值

```lua
-- ONLY nil and false are falsy
-- Everything else is truthy (including 0 and empty string!)

if 0 then print("0 is truthy!") end           -- prints!
if "" then print("empty string is truthy!") end -- prints!
if nil then print("nil") else print("nil is falsy") end  -- falsy
if false then print("false") else print("false is falsy") end -- falsy
```

> **面试陷阱：** 与 C/C++/Python 不同，Lua 中 `0` 和 `""` 都是真值！

### 1.3 string 内部机制

```lua
-- Strings are immutable and interned (short strings <= 40 bytes)
local a = "hello"
local b = "hello"
print(a == b)  -- true (value comparison, but also same object for short strings)

-- String interning: short strings are stored in global hash table
-- Benefit: O(1) equality comparison (pointer comparison)
-- Cost: creation requires hash table lookup

-- Long strings (> 40 bytes): NOT interned, compared byte by byte
local long1 = string.rep("a", 100)
local long2 = string.rep("a", 100)
print(long1 == long2)  -- true (but byte-by-byte comparison)

-- String concatenation creates new string every time
-- BAD: O(n^2) for n concatenations
local result = ""
for i = 1, 10000 do
    result = result .. tostring(i)  -- new string object each iteration
end

-- GOOD: use table.concat, O(n)
local parts = {}
for i = 1, 10000 do
    parts[#parts + 1] = tostring(i)
end
local result = table.concat(parts)
```

#### 面试题：Lua 中 number 的精度问题？

```lua
-- Lua 5.3 之前: 所有 number 都是 double (64-bit IEEE 754)
-- 精确表示整数范围: -2^53 ~ 2^53

-- Lua 5.3+: 区分 integer (64-bit signed) 和 float (double)
local i = 42       -- integer
local f = 42.0     -- float
print(i == f)      -- true (automatic conversion for comparison)
print(i .. "")     -- "42" (integer to string)
print(f .. "")     -- "42.0" (float to string)

-- Integer overflow wraps around (modular arithmetic)
local max = math.maxinteger  -- 2^63 - 1 = 9223372036854775807
print(max + 1)               -- -9223372036854775808 (wraps!)

-- Float precision issue
print(0.1 + 0.2 == 0.3)     -- false! (IEEE 754 floating point)
print(0.1 + 0.2)             -- 0.30000000000000004
```

## 二、Table 深度解析

### 2.1 Table 的双重结构

```
┌─────────────────────────────────────────────────────┐
│                    Table                             │
├─────────────────────┬───────────────────────────────┤
│   Array Part        │       Hash Part               │
│   (连续整数键)       │       (其他所有键)              │
│                     │                               │
│   [1] = "a"         │   ["name"] = "lua"            │
│   [2] = "b"         │   ["version"] = 5.4           │
│   [3] = "c"         │   [10000] = "sparse"          │
│                     │   [true] = "yes"              │
│   size: 3           │   size: next power of 2       │
│   O(1) access       │   O(1) average access         │
└─────────────────────┴───────────────────────────────┘
```

**Array Part 规则：**
- 存储从 1 开始的连续整数键
- 大小为 2 的幂次
- 超过一半的空间被使用时才分配

**Hash Part 规则：**
- 存储所有非连续整数键（包括字符串键、稀疏整数键）
- 开放寻址法（链地址法的变体，链表节点存在哈希表内部）
- 大小为 2 的幂次

```lua
-- Array part vs Hash part
local t = {}
t[1] = "a"    -- array part
t[2] = "b"    -- array part
t[3] = "c"    -- array part
t[100] = "d"  -- hash part (sparse, not contiguous)
t["x"] = 1   -- hash part (string key)

-- Table resize: when array or hash part is full
-- Resize is expensive: rehash all elements, O(n)
-- Pre-allocate to avoid resize:
local t = {}  -- empty, will resize multiple times
-- vs
local t = {nil, nil, nil, nil, nil, nil, nil, nil}  -- pre-allocate 8 array slots
-- or use table.create in LuaJIT/Roblox
```

### 2.2 Table 长度操作符 `#`

```lua
-- # operator returns a "border" of the table
-- A border is any index n where t[n] ~= nil and t[n+1] == nil

local t = {1, 2, nil, 4, 5}
print(#t)  -- UNDEFINED! Could be 2 or 5

-- Safe usage: only use # on sequences (no holes)
local seq = {1, 2, 3, 4, 5}
print(#seq)  -- 5 (guaranteed)

-- Common pitfall with table.remove
local t = {1, 2, 3, 4, 5}
t[3] = nil       -- creates a hole! # is now unreliable
table.remove(t, 3)  -- correct way: shifts elements down

-- For sparse tables, count manually
local function tableCount(t)
    local count = 0
    for _ in pairs(t) do
        count = count + 1
    end
    return count
end
```

#### 面试题：`#` 操作符的陷阱？

**答：** `#` 只对**序列（sequence）**有确定行为。序列是从 1 开始、无空洞的连续整数键集合。如果 table 中间有 `nil`（空洞），`#` 的返回值是**未定义的**——它可能返回任何一个 border。

### 2.3 Table 作为数组

```lua
-- Lua arrays are 1-indexed!
local arr = {"a", "b", "c", "d"}
print(arr[1])  -- "a" (first element)
print(arr[0])  -- nil (0 is NOT a valid array index)

-- Iteration methods
-- ipairs: iterate array part only, stops at first nil
for i, v in ipairs(arr) do
    print(i, v)  -- 1 "a", 2 "b", 3 "c", 4 "d"
end

-- pairs: iterate ALL key-value pairs (order not guaranteed for hash part)
local mixed = {1, 2, 3, name = "lua", version = 5}
for k, v in pairs(mixed) do
    print(k, v)  -- 1 1, 2 2, 3 3, "name" "lua", "version" 5 (hash part order varies)
end

-- Numeric for: fastest iteration
for i = 1, #arr do
    local v = arr[i]
end

-- Performance comparison (10000 elements):
-- numeric for  > ipairs > pairs
-- ~1x            ~1.2x    ~2x
```

### 2.4 Table 作为 OOP

```lua
-- Table + metatable = OOP

-- Class definition
local Animal = {}
Animal.__index = Animal

function Animal.new(name, sound)
    local self = setmetatable({}, Animal)
    self.name = name
    self.sound = sound
    return self
end

function Animal:speak()
    print(self.name .. " says " .. self.sound)
end

-- Inheritance
local Dog = setmetatable({}, {__index = Animal})
Dog.__index = Dog

function Dog.new(name)
    local self = Animal.new(name, "Woof")
    return setmetatable(self, Dog)
end

function Dog:fetch(item)
    print(self.name .. " fetches " .. item)
end

-- Usage
local dog = Dog.new("Rex")
dog:speak()         -- "Rex says Woof" (inherited from Animal)
dog:fetch("ball")   -- "Rex fetches ball"

-- Method lookup chain:
-- dog -> Dog -> Animal
-- dog.speak -> dog table (not found)
--          -> Dog.__index = Dog table (not found)
--          -> Dog's metatable.__index = Animal table (found!)
```

#### 面试题：`:` 和 `.` 调用的区别？

```lua
-- : is syntactic sugar that passes self as first argument
function Animal:speak()     -- equivalent to:
function Animal.speak(self) -- this

dog:speak()          -- equivalent to:
dog.speak(dog)       -- this

-- Common bug:
function Animal.speak()  -- forgot self!
    print(self.name)     -- ERROR: self is nil
end
```

## 三、元表与元方法 (Metatable & Metamethod)

### 3.1 核心概念

```lua
-- Every table can have a metatable that defines special behaviors
local t = {}
local mt = {}
setmetatable(t, mt)  -- set metatable
getmetatable(t)      -- get metatable (returns mt)
```

### 3.2 常用元方法

| 元方法       | 触发时机            | 示例                   |
| ------------ | ------------------- | ---------------------- |
| `__index`    | 访问不存在的键      | `t.missing_key`        |
| `__newindex` | 给不存在的键赋值    | `t.new_key = value`    |
| `__call`     | 把 table 当函数调用 | `t()`                  |
| `__tostring` | `tostring(t)`       | `print(t)`             |
| `__len`      | `#t`                | `#t`                   |
| `__add`      | `t + other`         | 加法运算               |
| `__sub`      | `t - other`         | 减法运算               |
| `__mul`      | `t * other`         | 乘法运算               |
| `__eq`       | `t == other`        | 相等比较               |
| `__lt`       | `t < other`         | 小于比较               |
| `__concat`   | `t .. other`        | 连接运算               |
| `__gc`       | GC 回收时           | 析构函数（Lua 5.2+）   |
| `__pairs`    | `pairs(t)`          | 自定义迭代（Lua 5.2+） |

### 3.3 `__index` 深度解析

```lua
-- __index can be a table or a function

-- As table: prototype chain (inheritance)
local defaults = {color = "red", size = 10}
local obj = setmetatable({}, {__index = defaults})
print(obj.color)  -- "red" (found in defaults)
print(obj.size)   -- 10
obj.color = "blue" -- sets in obj, does NOT modify defaults
print(obj.color)  -- "blue" (found in obj now)

-- As function: computed properties, lazy loading
local smart = setmetatable({}, {
    __index = function(self, key)
        if key == "timestamp" then
            return os.time()
        elseif key == "random" then
            return math.random()
        end
        return nil
    end
})
print(smart.timestamp)  -- current time (computed each access)
print(smart.random)     -- random number
```

### 3.4 `__newindex` 与只读表

```lua
-- __newindex: intercept assignment to new keys

-- Read-only table
local function readOnly(t)
    return setmetatable({}, {
        __index = t,
        __newindex = function(_, key, value)
            error("attempt to modify read-only table, key: " .. tostring(key))
        end,
        __len = function() return #t end
    })
end

local config = readOnly({maxHP = 100, maxMP = 50})
print(config.maxHP)     -- 100
config.maxHP = 200      -- ERROR: attempt to modify read-only table

-- Tracked table (log all changes)
local function tracked(t)
    local proxy = {}
    local mt = {
        __index = t,
        __newindex = function(_, key, value)
            print(string.format("[SET] %s = %s", tostring(key), tostring(value)))
            t[key] = value  -- write to original table, not proxy
        end
    }
    return setmetatable(proxy, mt)
end
```

### 3.5 运算符重载

```lua
-- Vector2 with operator overloading
local Vector2 = {}
Vector2.__index = Vector2

function Vector2.new(x, y)
    return setmetatable({x = x or 0, y = y or 0}, Vector2)
end

function Vector2.__add(a, b)
    return Vector2.new(a.x + b.x, a.y + b.y)
end

function Vector2.__sub(a, b)
    return Vector2.new(a.x - b.x, a.y - b.y)
end

function Vector2.__mul(a, b)
    if type(a) == "number" then
        return Vector2.new(a * b.x, a * b.y)
    elseif type(b) == "number" then
        return Vector2.new(a.x * b, a.y * b)
    end
    return a.x * b.x + a.y * b.y  -- dot product
end

function Vector2.__eq(a, b)
    return a.x == b.x and a.y == b.y
end

function Vector2.__tostring(v)
    return string.format("(%g, %g)", v.x, v.y)
end

function Vector2:length()
    return math.sqrt(self.x * self.x + self.y * self.y)
end

-- Usage
local a = Vector2.new(1, 2)
local b = Vector2.new(3, 4)
print(a + b)       -- (4, 6)
print(a * 2)       -- (2, 4)
print(a == Vector2.new(1, 2))  -- true
print(tostring(a)) -- (1, 2)
```

#### 面试题：`rawget` 和 `rawset` 的作用？

```lua
-- rawget/rawset bypass metamethods
local t = setmetatable({}, {
    __index = function() return "from metatable" end,
    __newindex = function() error("read only!") end
})

print(t.foo)          -- "from metatable" (triggers __index)
print(rawget(t, "foo"))  -- nil (bypasses __index)

-- t.bar = 1           -- ERROR (triggers __newindex)
rawset(t, "bar", 1)     -- OK (bypasses __newindex)
print(rawget(t, "bar")) -- 1

-- Use case: implement __newindex that actually stores values
local logged = setmetatable({}, {
    __newindex = function(self, key, value)
        print("Setting", key, "=", value)
        rawset(self, key, value)  -- must use rawset to avoid infinite recursion!
    end
})
```

## 四、闭包 (Closure)

### 4.1 闭包原理

```lua
-- A closure = function + upvalues (captured external local variables)

function makeCounter()
    local count = 0  -- upvalue: captured by the returned function
    return function()
        count = count + 1  -- accesses upvalue
        return count
    end
end

local counter1 = makeCounter()
local counter2 = makeCounter()
print(counter1())  -- 1
print(counter1())  -- 2
print(counter2())  -- 1 (independent upvalue)
```

### 4.2 Upvalue 机制

```
┌──────────────────────────────────────────┐
│              Lua VM                      │
│                                          │
│  makeCounter() call:                     │
│  ┌──────────────────────┐                │
│  │ Stack Frame          │                │
│  │ local count = 0  ────┼──┐             │
│  └──────────────────────┘  │             │
│                            │             │
│  When makeCounter returns: │             │
│  count moves to heap ──────┘             │
│                                          │
│  ┌──────────────┐   ┌──────────────┐     │
│  │  Closure 1   │   │  Closure 2   │     │
│  │  upvalue ────┼──→│ UpVal object │     │
│  └──────────────┘   │ value = 2    │     │
│                     └──────────────┘     │
│                                          │
│  Multiple closures from same call        │
│  SHARE the same upvalue!                 │
└──────────────────────────────────────────┘
```

```lua
-- Shared upvalue example
function makePair()
    local value = 0
    
    local function getter()
        return value
    end
    
    local function setter(v)
        value = v  -- same upvalue as getter
    end
    
    return getter, setter
end

local get, set = makePair()
set(42)
print(get())  -- 42 (shared upvalue)
```

### 4.3 闭包的常见陷阱

```lua
-- Classic loop closure bug
local functions = {}
for i = 1, 5 do
    functions[i] = function()
        return i  -- captures the SAME upvalue 'i'
    end
end
-- In Lua, loop variable is local to each iteration (unlike JavaScript!)
print(functions[1]())  -- 1 (correct in Lua!)
print(functions[5]())  -- 5 (correct in Lua!)
-- Lua's for loop creates a new local 'i' for each iteration
-- This is different from JavaScript's var!

-- But watch out with while loops:
local funcs = {}
local i = 1
while i <= 5 do
    funcs[i] = function() return i end  -- captures the SAME 'i'
    i = i + 1
end
print(funcs[1]())  -- 6! (all share same upvalue, i ended at 6)

-- Fix: create new scope
local funcs = {}
local i = 1
while i <= 5 do
    local captured = i  -- new local for each iteration
    funcs[i] = function() return captured end
    i = i + 1
end
print(funcs[1]())  -- 1 (correct)
```

#### 面试题：Lua 闭包和 C# 闭包的区别？

| 特性         | Lua 闭包           | C# 闭包                       |
| ------------ | ------------------ | ----------------------------- |
| 实现         | 函数 + UpVal 链表  | 编译器生成 DisplayClass       |
| for 循环变量 | 每次迭代新变量     | C# 5 前共享，C# 5+ 每次新变量 |
| GC 影响      | UpVal 对象在堆上   | DisplayClass 在堆上           |
| 性能         | 轻量（UpVal 很小） | 较重（类实例）                |

## 五、协程 (Coroutine)

### 5.1 协程基础

```lua
-- Coroutine: cooperative multitasking (not preemptive)
-- Only ONE coroutine runs at a time, explicitly yields control

local co = coroutine.create(function(a, b)
    print("start", a, b)       -- start 1 2
    local c = coroutine.yield(a + b)  -- yield 3, suspend here
    print("resume", c)         -- resume 10
    return "done"
end)

print(coroutine.status(co))           -- "suspended"
local ok, result = coroutine.resume(co, 1, 2)  -- start coroutine
print(ok, result)                     -- true 3
print(coroutine.status(co))           -- "suspended"
local ok, result = coroutine.resume(co, 10)    -- resume with value
print(ok, result)                     -- true "done"
print(coroutine.status(co))           -- "dead"
```

### 5.2 协程状态机

```
                  create()
                     │
                     ▼
              ┌──────────────┐
              │  suspended   │◄─────────────┐
              └──────┬───────┘              │
                     │ resume()             │ yield()
                     ▼                      │
              ┌──────────────┐              │
              │  running     │──────────────┘
              └──────┬───────┘
                     │ return / error
                     ▼
              ┌──────────────┐
              │  dead        │
              └──────────────┘

  + "normal": a coroutine that resumed another coroutine
```

### 5.3 协程实战：迭代器

```lua
-- Coroutine-based iterator (elegant but slower than closure iterator)
function range(start, stop, step)
    step = step or 1
    return coroutine.wrap(function()
        for i = start, stop, step do
            coroutine.yield(i)
        end
    end)
end

for v in range(1, 10, 2) do
    print(v)  -- 1, 3, 5, 7, 9
end

-- Producer-Consumer pattern
function producer()
    return coroutine.create(function()
        local data = {"apple", "banana", "cherry"}
        for _, item in ipairs(data) do
            print("Producing: " .. item)
            coroutine.yield(item)
        end
    end)
end

function consumer(prod)
    while coroutine.status(prod) ~= "dead" do
        local ok, item = coroutine.resume(prod)
        if ok and item then
            print("Consuming: " .. item)
        end
    end
end

consumer(producer())
```

### 5.4 协程 vs 线程

| 特性     | Lua 协程                 | OS 线程           |
| -------- | ------------------------ | ----------------- |
| 调度     | 协作式（手动 yield）     | 抢占式（OS 调度） |
| 并发     | 单线程内交替执行         | 真正并行          |
| 切换开销 | 极低（~100ns）           | 高（~1-10μs）     |
| 同步     | 不需要锁                 | 需要锁/原子操作   |
| 栈大小   | 可动态增长（~1KB 起）    | 固定（~1MB）      |
| 适用场景 | 状态机、迭代器、异步流程 | CPU 密集并行      |

#### 面试题：协程在游戏中的应用？

1. **AI 行为树 / 状态机：** 每帧 resume，yield 等待条件
2. **动画序列：** 按时间线 yield 执行动画步骤
3. **异步加载：** 分帧加载资源，每帧 yield 避免卡顿
4. **对话系统：** yield 等待玩家输入
5. **技能流程：** 前摇 → yield → 伤害判定 → yield → 后摇

```lua
-- Game AI example
function aiRoutine(entity)
    return coroutine.create(function()
        while entity.alive do
            -- Patrol
            for i = 1, 5 do
                entity:moveTo(patrolPoints[i])
                coroutine.yield()  -- wait until next frame
            end
            
            -- Check for enemies
            local target = entity:findNearestEnemy()
            if target then
                while target.alive and entity:distanceTo(target) < 10 do
                    entity:attack(target)
                    coroutine.yield()  -- wait for attack cooldown
                end
            end
        end
    end)
end

-- In game loop:
function update()
    for _, co in ipairs(aiCoroutines) do
        if coroutine.status(co) ~= "dead" then
            coroutine.resume(co)
        end
    end
end
```

## 六、垃圾回收 (GC)

### 6.1 GC 算法

```
Lua GC: Incremental Tri-color Mark and Sweep

┌─────────────────────────────────────────────────┐
│                                                 │
│  White (未标记)  ──Mark──▶  Gray (待扫描)        │
│       ▲                        │                │
│       │                   Scan children         │
│       │                        │                │
│       │                        ▼                │
│  Sweep (回收)  ◀──────  Black (已扫描)          │
│                                                 │
│  Incremental: GC work is spread across frames   │
│  Not stop-the-world (mostly)                    │
└─────────────────────────────────────────────────┘

Lua 5.4: Generational GC (optional)
- Young generation: frequently collected
- Old generation: rarely collected
- Similar concept to .NET generational GC
```

### 6.2 GC 控制

```lua
-- GC control functions
collectgarbage("count")    -- returns memory in KB
collectgarbage("collect")  -- full GC cycle
collectgarbage("stop")     -- stop automatic GC
collectgarbage("restart")  -- restart automatic GC
collectgarbage("step", n)  -- incremental step

-- GC tuning parameters
collectgarbage("setpause", 100)     -- pause between cycles (default 200)
collectgarbage("setstepmul", 200)   -- speed of GC relative to allocation

-- Lua 5.4 generational mode
collectgarbage("generational", minor_mul, major_mul)
collectgarbage("incremental", pause, stepmul, stepsize)

-- Game-friendly GC strategy:
-- Option 1: Manual stepping
```

### 6.3 弱引用表 (Weak Table)

```lua
-- Weak table: entries can be garbage collected
-- __mode = "k": weak keys
-- __mode = "v": weak values
-- __mode = "kv": both weak

-- Cache with weak values (auto-cleanup when no other references)
local cache = setmetatable({}, {__mode = "v"})

function getTexture(path)
    local tex = cache[path]
    if tex then
        return tex  -- cache hit
    end
    tex = loadTexture(path)  -- expensive operation
    cache[path] = tex        -- cache it (weak reference)
    return tex
end
-- When texture is no longer referenced elsewhere, GC can collect it
-- and the cache entry is automatically removed

-- Object tracking without preventing GC
local allObjects = setmetatable({}, {__mode = "k"})

function trackObject(obj)
    allObjects[obj] = true  -- weak key: won't prevent obj from being collected
end

function countLiveObjects()
    local count = 0
    for _ in pairs(allObjects) do
        count = count + 1
    end
    return count
end
```

## 八、热更新 (Hotfix)

### 8.1 模块热重载

```lua
-- package.loaded caches loaded modules
-- To hot-reload, clear the cache and re-require

function hotReload(moduleName)
    -- Clear cached module
    package.loaded[moduleName] = nil
    
    -- Clear related modules if needed
    for key in pairs(package.loaded) do
        if key:find(moduleName) then
            package.loaded[key] = nil
        end
    end
    
    -- Re-require
    local ok, module = pcall(require, moduleName)
    if ok then
        print("Hot reload success: " .. moduleName)
        return module
    else
        print("Hot reload failed: " .. module)
        return nil
    end
end

-- Usage
hotReload("game.combat")
```

### 8.3 热更新框架原理

```
┌────────────────────────────────────────────────┐
│              Hot Update Flow                   │
│                                                │
│  1. Download patch.lua from server             │
│  2. Load patch into sandbox environment        │
│  3. Replace target functions                   │
│  4. Preserve upvalues from old functions       │
│                                                │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │ Original │    │  Patch   │    │ Patched  │  │
│  │ Function │ →  │  Code    │ →  │ Function │  │
│  │          │    │          │    │          │  │
│  │ upval: a │    │ new body │    │ upval: a │  │
│  │ upval: b │    │          │    │ upval: b │  │
│  └──────────┘    └──────────┘    └──────────┘  │
│                                                │
│  Key: transfer upvalues to maintain state      │
└────────────────────────────────────────────────┘
```

```lua
-- Upvalue transfer for hot update
local function getUpvalues(func)
    local upvalues = {}
    local i = 1
    while true do
        local name, value = debug.getupvalue(func, i)
        if not name then break end
        upvalues[name] = {index = i, value = value}
        i = i + 1
    end
    return upvalues
end

local function transferUpvalues(oldFunc, newFunc)
    local oldUpvals = getUpvalues(oldFunc)
    local i = 1
    while true do
        local name = debug.getupvalue(newFunc, i)
        if not name then break end
        if oldUpvals[name] then
            debug.setupvalue(newFunc, i, oldUpvals[name].value)
        end
        i = i + 1
    end
end

-- Hot patch a function while preserving state
function hotPatch(obj, funcName, newFunc)
    local oldFunc = obj[funcName]
    if oldFunc then
        transferUpvalues(oldFunc, newFunc)
    end
    obj[funcName] = newFunc
end
```

#### 面试题：Lua 热更新的限制？

1. **不能修改 C 代码：** 只能替换 Lua 层函数
2. **Upvalue 类型变化：** 如果新函数的 upvalue 结构不同，转移可能失败
3. **协程中的函数：** 正在 yield 的协程中的函数无法安全替换
4. **元表缓存：** 某些元方法可能被 VM 缓存，替换后不立即生效
5. **IL2CPP 限制：** iOS 等平台禁止 JIT，需要用解释器模式

## 九、性能优化

### 9.1 局部变量 vs 全局变量

```lua
-- Local variables are stored in registers (fast)
-- Global variables require table lookup (slow)

-- BAD: global access every iteration
for i = 1, 1000000 do
    math.sin(i)  -- lookup _ENV["math"] then math["sin"] each time
end

-- GOOD: localize frequently used functions
local sin = math.sin
for i = 1, 1000000 do
    sin(i)  -- direct register access, ~30% faster
end

-- Performance comparison:
-- local variable access:  ~1 cycle (register)
-- upvalue access:         ~2-3 cycles (indirect)
-- global variable access: ~5-10 cycles (hash table lookup)
-- table field access:     ~5-10 cycles (hash table lookup)
```

### 9.2 Table 优化

```lua
-- 1. Pre-allocate table size
-- BAD: multiple resizes
local t = {}
for i = 1, 1000 do
    t[i] = i  -- triggers resize at 1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024
end

-- GOOD: pre-allocate (LuaJIT: table.new, standard Lua: fill with nils)
local t = {}
for i = 1, 1000 do t[i] = false end  -- pre-allocate array part
for i = 1, 1000 do t[i] = i end      -- no resize needed

-- 2. Avoid creating tables in hot loops
-- BAD:
for i = 1, 10000 do
    local pos = {x = 0, y = 0, z = 0}  -- 10000 table allocations
end

-- GOOD: reuse table
local pos = {x = 0, y = 0, z = 0}
for i = 1, 10000 do
    pos.x, pos.y, pos.z = 0, 0, 0  -- zero allocations
end

-- 3. Use array part (integer keys) instead of hash part (string keys)
-- Array access is faster than hash access
local arrayStyle = {10, 20, 30}       -- array part: fast
local hashStyle = {x=10, y=20, z=30}  -- hash part: slower
```

### 9.3 字符串优化

```lua
-- 1. Use table.concat instead of ..
local parts = {}
for i = 1, 1000 do
    parts[#parts + 1] = tostring(i)
end
local result = table.concat(parts, ",")

-- 2. Use string.format for complex formatting
local msg = string.format("Player %s has %d HP at (%.1f, %.1f)",
    name, hp, x, y)

-- 3. Avoid string keys when possible (use enum-like integers)
local STATE_IDLE = 1
local STATE_RUN  = 2
local STATE_JUMP = 3
-- Faster than: state = "idle" / "run" / "jump"
```

### 9.4 函数调用优化

```lua
-- 1. Avoid unnecessary function calls
-- BAD:
for i = 1, #t do
    table.insert(t2, t[i])  -- function call overhead
end

-- GOOD:
for i = 1, #t do
    t2[#t2 + 1] = t[i]  -- direct table operation
end

-- 2. Inline small functions in hot paths
-- BAD:
local function clamp(v, min, max)
    if v < min then return min end
    if v > max then return max end
    return v
end
for i = 1, 1000000 do
    result = clamp(values[i], 0, 1)  -- function call overhead
end

-- GOOD: inline in hot loop
for i = 1, 1000000 do
    local v = values[i]
    if v < 0 then v = 0 elseif v > 1 then v = 1 end
    result = v
end

-- 3. Tail call optimization
-- Lua supports proper tail calls (no stack growth)
function factorial(n, acc)
    acc = acc or 1
    if n <= 1 then return acc end
    return factorial(n - 1, n * acc)  -- tail call: no stack growth
end
print(factorial(1000000))  -- works! (no stack overflow)
```

### 9.5 LuaJIT 特有优化

```lua
-- LuaJIT: JIT compiled, much faster than standard Lua
-- But some patterns prevent JIT compilation ("NYI" = Not Yet Implemented)

-- NYI patterns (fall back to interpreter, 10-100x slower):
-- 1. pairs() iteration (use numeric for instead)
-- 2. unpack() with many values
-- 3. string.gsub with function replacement
-- 4. pcall/xpcall in hot loops
-- 5. Coroutine resume/yield in JIT-compiled code
-- 6. table.insert/table.remove (use t[#t+1] instead)
-- 7. next() function

-- FFI: Foreign Function Interface (fastest C interop)
local ffi = require("ffi")
ffi.cdef[[
    typedef struct { float x, y, z; } Vec3;
    double sin(double x);
]]

local v = ffi.new("Vec3", 1, 2, 3)  -- stack allocated, no GC
print(v.x, v.y, v.z)

-- FFI sin is as fast as C sin (no Lua->C bridge overhead)
local C = ffi.C
for i = 1, 1000000 do
    C.sin(i)  -- direct C call, no overhead
end
```

```lua
function deepCopy(orig, copies)
    copies = copies or {}
    if type(orig) ~= "table" then return orig end
    if copies[orig] then return copies[orig] end  -- handle circular reference
    
    local copy = {}
    copies[orig] = copy
    
    for k, v in pairs(orig) do
        copy[deepCopy(k, copies)] = deepCopy(v, copies)
    end
    
    setmetatable(copy, deepCopy(getmetatable(orig), copies))
    return copy
end

        if instance.init then
            instance:init(...)
        end

    end

end

        self._handlers[event] = {}

    self._handlers[event][#self._handlers[event] + 1] = handler
end

function EventEmitter:off(event, handler)

    for i = #handlers, 1, -1 do

            table.remove(handlers, i)
        end

function async(func)

        local function step(ok, ...)

            if coroutine.status(co) == "dead" then return ... end
            -- yielded value is a function that takes a callback
            local awaitable = ...
            awaitable(function(...)
                step(coroutine.resume(co, ...))
            end)
```

```lua
-- Lua 5.4 to-be-closed variable
local function readFile(path)
    local f <close> = io.open(path, "r")  -- auto-closed when out of scope
    if not f then return nil end
    return f:read("*a")
end  -- f:close() called automatically here (like C# using)

-- Lua 5.3+ bitwise operators
local flags = 0xFF
local mask = flags & 0x0F   -- AND: 0x0F
local shifted = flags >> 4  -- right shift: 0x0F
local combined = 0x0F | 0xF0  -- OR: 0xFF
local inverted = ~flags     -- NOT
```

## 十三、调试技巧

```lua
-- 1. debug.traceback: get call stack
function riskyFunction()
    error("something went wrong")
end

local ok, err = xpcall(riskyFunction, function(msg)
    return debug.traceback(msg, 2)
end)
print(err)  -- full stack trace

-- 2. debug.getinfo: inspect function info
local info = debug.getinfo(1, "Sln")
print(info.source)       -- file name
print(info.currentline)  -- current line
print(info.name)         -- function name

-- 3. debug.getlocal / debug.setlocal
function inspect()
    local x = 42
    local name, value = debug.getlocal(1, 1)  -- level 1, local #1
    print(name, value)  -- "x" 42
end

-- 4. Performance profiling
local function profile(func, ...)
    local start = os.clock()
    local results = {func(...)}
    local elapsed = os.clock() - start
    print(string.format("Elapsed: %.4f ms", elapsed * 1000))
    return table.unpack(results)
end

-- 5. Memory profiling
local before = collectgarbage("count")
-- ... do something ...
local after = collectgarbage("count")
print(string.format("Memory delta: %.2f KB", after - before))
```
