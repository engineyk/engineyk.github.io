## 一、复杂度分析

### 1.1 时间复杂度

```
常见时间复杂度（从快到慢）：

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

┌──────────────────────────────────────────────────┐
│  Time                                             │
│  ▲                                                │
│  │                                    O(2ⁿ)      │
│  │                                  ╱             │
│  │                          O(n²) ╱               │
│  │                        ╱     ╱                 │
│  │                O(n log n)  ╱                   │
│  │              ╱           ╱                     │
│  │          O(n)          ╱                       │
│  │        ╱             ╱                         │
│  │    O(log n)        ╱                           │
│  │  ╱               ╱                             │
│  │O(1)────────────────────────────────▶ n         │
└──────────────────────────────────────────────────┘
```

| 复杂度     | 名称     | n=10⁶ 时操作数 | 典型算法             |
| ---------- | -------- | -------------- | -------------------- |
| O(1)       | 常数     | 1              | 哈希表查找、数组索引 |
| O(log n)   | 对数     | 20             | 二分查找、平衡 BST   |
| O(n)       | 线性     | 10⁶            | 遍历、线性查找       |
| O(n log n) | 线性对数 | 2×10⁷          | 归并排序、快速排序   |
| O(n²)      | 平方     | 10¹²           | 冒泡排序、暴力匹配   |
| O(2ⁿ)      | 指数     | ∞              | 子集枚举、暴力递归   |

### 1.2 空间复杂度

```
O(1)     - in-place algorithm (e.g., bubble sort)
O(n)     - linear extra space (e.g., merge sort, hash table)
O(log n) - recursion stack depth (e.g., quick sort average)
O(n²)    - 2D matrix (e.g., adjacency matrix)
```

### 1.3 均摊分析 (Amortized Analysis)

```
Dynamic Array (std::vector / List<T>) push_back:

  Single push_back: O(1) amortized, O(n) worst case

  Capacity:  1 → 2 → 4 → 8 → 16 → 32 ...
  Copy cost: 0   1   2   4    8   16 ...

  Total cost for n insertions:
  n + (1 + 2 + 4 + ... + n/2) = n + n - 1 ≈ 2n
  Amortized per operation: 2n / n = O(1)

  Growth factor:
  - C++ std::vector: typically 2x
  - C# List<T>: 2x
  - Java ArrayList: 1.5x
  - Python list: ~1.125x
```

```
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  0  │  1  │  2  │  3  │  4  │  5  │  6  │  7  │
```

#### 面试题：数组 vs 链表，什么时候用哪个？

| 场景              | 选择        | 原因                     |
| ----------------- | ----------- | ------------------------ |
| 频繁随机访问      | 数组        | O(1) vs O(n)             |
| 频繁头部插入/删除 | 链表        | O(1) vs O(n)             |
| 缓存友好          | 数组        | 连续内存，CPU cache line |
| 大小未知且变化大  | 链表        | 无需扩容拷贝             |
| 游戏中的实体列表  | 数组        | 缓存友好，遍历快         |
| LRU 缓存          | 链表+哈希表 | O(1) 移动到头部          |

| 操作                 | 单链表     | 双链表 | 说明                 |
| -------------------- | ---------- | ------ | -------------------- |
| 头部插入             | O(1)       | O(1)   | 修改 head 指针       |

```cpp
// Classic interview: reverse a linked list
struct ListNode {
    int val;
    ListNode* next;
};

// Iterative
ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* curr = head;
    while (curr) {
        ListNode* next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}

// Recursive
ListNode* reverseListRecursive(ListNode* head) {
    if (!head || !head->next) return head;
    ListNode* newHead = reverseListRecursive(head->next);
    head->next->next = head;
    head->next = nullptr;
    return newHead;
}
```


### 2.3 栈 (Stack)

```
LIFO - Last In, First Out

        ┌─────┐

        ├─────┤
        │  4  │
        ├─────┤
        │  3  │

```

| 操作     | 时间复杂度 |
| -------- | ---------- |
| Push     | O(1)       |
| Pop      | O(1)       |
| Peek/Top | O(1)       |
| Search   | O(n)       |

**经典应用：**
- 函数调用栈
- 括号匹配
- 表达式求值（后缀表达式）
- 撤销/重做 (Undo/Redo)
- DFS 非递归实现
- 单调栈

```cpp
// Classic: valid parentheses
bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);
        } else {
            if (st.empty()) return false;
            char top = st.top();
            st.pop();
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{'))
                return false;
        }
    }
    return st.empty();
}

// Monotonic stack: next greater element
vector<int> nextGreaterElement(vector<int>& nums) {
    int n = nums.size();
    vector<int> result(n, -1);
    stack<int> st;  // stores indices
    for (int i = 0; i < n; i++) {
        while (!st.empty() && nums[i] > nums[st.top()]) {
            result[st.top()] = nums[i];
            st.pop();
        }
        st.push(i);
    }
    return result;
}
```

### 2.4 队列 (Queue)

```
FIFO - First In, First Out

  Enqueue →  ┌───┬───┬───┬───┬───┐  → Dequeue
             │ 5 │ 4 │ 3 │ 2 │ 1 │
             └───┴───┴───┴───┴───┘
            rear                front

Circular Queue (Ring Buffer):
         ┌───┐
       ╱ │ 3 │ ╲
     ╱   └───┘   ╲
   ┌───┐       ┌───┐
   │ 2 │       │ 4 │
   └───┘       └───┘
     ╲   ┌───┐   ╱
       ╲ │ 1 │ ╱
         └───┘
   front ↑     ↑ rear
```

| 操作       | 时间复杂度 |
| ---------- | ---------- |
| Enqueue    | O(1)       |
| Dequeue    | O(1)       |
| Peek/Front | O(1)       |
| Search     | O(n)       |

**队列变体：**

| 类型                      | 特点               | 应用                   |
| ------------------------- | ------------------ | ---------------------- |
| 普通队列                  | FIFO               | BFS、任务调度          |
| 双端队列 (Deque)          | 两端都可入出       | 滑动窗口最大值         |
| 优先队列 (Priority Queue) | 按优先级出队       | Dijkstra、A*、任务调度 |
| 环形队列 (Ring Buffer)    | 固定大小，循环使用 | 音频缓冲、网络包缓冲   |

```cpp
// Ring buffer implementation
template<typename T, size_t N>
class RingBuffer {
    T data[N];
    size_t head = 0, tail = 0, count = 0;
public:
    bool push(const T& val) {
        if (count == N) return false;  // full
        data[tail] = val;
        tail = (tail + 1) % N;
        count++;
        return true;
    }
    bool pop(T& val) {
        if (count == 0) return false;  // empty
        val = data[head];
        head = (head + 1) % N;
        count--;
        return true;
    }
    size_t size() const { return count; }
};

// BFS using queue
void bfs(vector<vector<int>>& graph, int start) {
    queue<int> q;
    vector<bool> visited(graph.size(), false);
    q.push(start);
    visited[start] = true;
    while (!q.empty()) {
        int node = q.front();
        q.pop();
        for (int neighbor : graph[node]) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                q.push(neighbor);
            }
        }
    }
}
```

## 三、哈希表 (Hash Table)

### 3.1 原理

```
Hash Function: key → hash value → index

  key "apple"  ──hash()──▶  0x7A3F  ──mod N──▶  index 5

┌─────┬─────────────────────────────────────┐
│  0  │ → null                              │
├─────┤                                     │
│  1  │ → ["banana", 2] → null              │
├─────┤                                     │
│  2  │ → null                              │
├─────┤                                     │
│  3  │ → ["cherry", 5] → ["date", 3] → null│  ← collision (chaining)
├─────┤                                     │
│  4  │ → null                              │
├─────┤                                     │
│  5  │ → ["apple", 1] → null               │
└─────┴─────────────────────────────────────┘
```

### 3.2 冲突解决

| 方法                             | 原理                  | 优点                 | 缺点                      |
| -------------------------------- | --------------------- | -------------------- | ------------------------- |
| **链地址法 (Chaining)**          | 每个桶存链表          | 简单，负载因子可 >1  | 指针开销，缓存不友好      |
| **开放寻址法 (Open Addressing)** | 冲突时探测下一个位置  | 缓存友好，无指针开销 | 负载因子必须 <1，删除复杂 |
| **线性探测**                     | hash+1, hash+2, ...   | 简单                 | 聚集问题                  |
| **二次探测**                     | hash+1², hash+2², ... | 减少聚集             | 二次聚集                  |
| **双重哈希**                     | hash1 + i×hash2       | 最少聚集             | 计算开销                  |
| **Robin Hood Hashing**           | 偷富济贫              | 方差小，查找快       | 插入稍慢                  |

```
Open Addressing - Linear Probing:

Insert "cherry" (hash = 3, but slot 3 is taken):

┌─────┬──────────┐
│  0  │  empty   │
│  1  │ "banana" │
│  2  │  empty   │
│  3  │ "date"   │ ← collision!
│  4  │ "cherry" │ ← probe to next empty slot
│  5  │ "apple"  │
└─────┴──────────┘
```

### 3.3 负载因子与扩容

```
Load Factor α = n / N  (elements / buckets)

Chaining:    α can be > 1, optimal around 0.75
Open Addr:   α must be < 1, optimal around 0.5-0.7

Rehash trigger:
  Java HashMap:     α > 0.75 → double capacity
  C# Dictionary:    α > 1.0  → next prime size
  C++ unordered_map: α > 1.0 → double capacity
  Python dict:      α > 2/3  → 4x or 2x capacity

Rehash cost: O(n) - must re-insert all elements
```

### 3.4 各语言实现对比

| 语言                | 类型   | 冲突解决        | 负载因子 | 有序性         |
| ------------------- | ------ | --------------- | -------- | -------------- |
| C++ `unordered_map` | 哈希表 | 链地址法        | 1.0      | 无序           |
| C++ `map`           | 红黑树 | N/A             | N/A      | 有序           |
| C# `Dictionary`     | 哈希表 | 开放寻址        | ~1.0     | 无序           |
| Java `HashMap`      | 哈希表 | 链地址法+红黑树 | 0.75     | 无序           |
| Python `dict`       | 哈希表 | 开放寻址        | 2/3      | 插入序（3.7+） |
| Lua `table`         | 哈希表 | 开放寻址        | ~1.0     | 无序           |

#### 面试题：如何设计一个好的哈希函数？

```
Good hash function properties:
1. Deterministic: same input → same output
2. Uniform distribution: minimize collisions
3. Avalanche effect: small input change → large output change
4. Fast to compute

Common hash functions:
- Division method:     h(k) = k mod m  (m should be prime)
- Multiplication:      h(k) = floor(m × (k×A mod 1))  (A ≈ 0.618)
- FNV-1a:             fast, good distribution (used in many languages)
- MurmurHash:         fast, excellent distribution (used in game engines)
- xxHash:             extremely fast, good for large data
- CityHash/FarmHash:  Google, optimized for short strings
```

## 四、树 (Tree)

### 4.1 二叉树 (Binary Tree)

```
          ┌───┐
          │ 1 │           Level 0 (root)
          └─┬─┘
        ╱       ╲
     ┌───┐     ┌───┐
     │ 2 │     │ 3 │      Level 1
     └─┬─┘     └─┬─┘
    ╱     ╲       ╲
 ┌───┐  ┌───┐   ┌───┐
 │ 4 │  │ 5 │   │ 6 │     Level 2
 └───┘  └───┘   └───┘

Properties:
- Height h: longest path from root to leaf
- Complete binary tree: all levels full except possibly last
- Full binary tree: every node has 0 or 2 children
- Perfect binary tree: all levels completely full
- Nodes at level i: up to 2^i
- Total nodes in perfect tree of height h: 2^(h+1) - 1
```

### 4.2 遍历方式

```
          1
        ╱   ╲
       2     3
      ╱ ╲     ╲
     4   5     6

Pre-order  (根左右): 1, 2, 4, 5, 3, 6
In-order   (左根右): 4, 2, 5, 1, 3, 6
Post-order (左右根): 4, 5, 2, 6, 3, 1
Level-order (BFS):   1, 2, 3, 4, 5, 6
```

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
};

// Recursive traversals
void preorder(TreeNode* root) {
    if (!root) return;
    visit(root);           // root
    preorder(root->left);  // left
    preorder(root->right); // right
}

void inorder(TreeNode* root) {
    if (!root) return;
    inorder(root->left);   // left
    visit(root);           // root
    inorder(root->right);  // right
}

void postorder(TreeNode* root) {
    if (!root) return;
    postorder(root->left);  // left
    postorder(root->right); // right
    visit(root);            // root
}

// Iterative in-order (using stack)
void inorderIterative(TreeNode* root) {
    stack<TreeNode*> st;
    TreeNode* curr = root;
    while (curr || !st.empty()) {
        while (curr) {
            st.push(curr);
            curr = curr->left;
        }
        curr = st.top();
        st.pop();
        visit(curr);
        curr = curr->right;
    }
}

// Level-order (BFS)
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> result;
    if (!root) return result;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int size = q.size();
        vector<int> level;
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front();
            q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        result.push_back(level);
    }
    return result;
}
```

### 4.3 二叉搜索树 (BST)

```
BST Property: left < root < right

          8
        ╱   ╲
       3     10
      ╱ ╲      ╲
     1   6     14
        ╱ ╲   ╱
       4   7 13

In-order traversal gives sorted sequence:
1, 3, 4, 6, 7, 8, 10, 13, 14
```

| 操作      | 平均     | 最坏（退化为链表） |
| --------- | -------- | ------------------ |
| 查找      | O(log n) | O(n)               |
| 插入      | O(log n) | O(n)               |
| 删除      | O(log n) | O(n)               |
| 最小/最大 | O(log n) | O(n)               |

```cpp
// BST search
TreeNode* search(TreeNode* root, int target) {
    if (!root || root->val == target) return root;
    if (target < root->val) return search(root->left, target);
    return search(root->right, target);
}

// BST insert
TreeNode* insert(TreeNode* root, int val) {
    if (!root) return new TreeNode(val);
    if (val < root->val) root->left = insert(root->left, val);
    else if (val > root->val) root->right = insert(root->right, val);
    return root;
}

// BST delete
TreeNode* deleteNode(TreeNode* root, int key) {
    if (!root) return nullptr;
    if (key < root->val) {
        root->left = deleteNode(root->left, key);
    } else if (key > root->val) {
        root->right = deleteNode(root->right, key);
    } else {
        // Node found
        if (!root->left) return root->right;   // case 1: no left child
        if (!root->right) return root->left;   // case 2: no right child
        // Case 3: two children - replace with in-order successor
        TreeNode* successor = root->right;
        while (successor->left) successor = successor->left;
        root->val = successor->val;
        root->right = deleteNode(root->right, successor->val);
    }
    return root;
}
```

### 4.4 平衡二叉树

#### AVL 树

```
Balance Factor = height(left) - height(right)
Must be in {-1, 0, 1} for every node

Rotations:

  Right Rotation (LL):        Left Rotation (RR):
       z                          z
      / \                        / \
     y   T4    →    y           T1   y       →      y
    / \            / \              / \            / \
   x   T3        x    z           T2   x         z    x
  / \           / \  / \              / \        / \  / \
 T1  T2       T1 T2 T3 T4           T3  T4    T1 T2 T3 T4

  Left-Right (LR):            Right-Left (RL):
  Left rotate y, then         Right rotate y, then
  right rotate z              left rotate z
```

#### 红黑树 (Red-Black Tree)

```
Properties:
1. Every node is red or black
2. Root is black
3. Every leaf (NIL) is black
4. Red node's children must be black (no two consecutive reds)
5. All paths from node to descendant leaves have same number of black nodes

→ Height ≤ 2 × log₂(n+1)
→ Less strictly balanced than AVL, but fewer rotations on insert/delete

Used in:
- C++ std::map, std::set
- Java TreeMap, TreeSet
- Linux kernel (CFS scheduler, memory management)
```

| 特性      | AVL                | 红黑树           |
| --------- | ------------------ | ---------------- |

### 4.5 B 树 / B+ 树

```
B-Tree (order m):
- Each node has at most m children
- Each non-root node has at least ⌈m/2⌉ children
- All leaves at same level
- Used in databases and file systems

B+ Tree:
- All data in leaf nodes
- Leaf nodes linked together (range queries)
- Internal nodes only store keys (more keys per node → shorter tree)

         ┌─────────────┐
         │   30 | 60   │         Internal nodes
         └──┬──┬──┬────┘
           ╱   │    ╲
    ┌──────┐ ┌──────┐ ┌──────┐

    └──┬───┘ └──┬───┘ └──┬───┘
       └────────┴────────┘        Leaf linked list

Why B+ Tree for databases:
- Disk I/O is expensive → minimize tree height
- Node size = disk page size (4KB/16KB)
- More keys per node → fewer disk reads
- Leaf linked list → efficient range scans
```

```cpp
class Trie {
    struct TrieNode {
        TrieNode* children[26] = {};
        bool isEnd = false;
    };
    TrieNode* root;
public:
    Trie() : root(new TrieNode()) {}
    
    void insert(const string& word) {
        TrieNode* node = root;
        for (char c : word) {
            int idx = c - 'a';
            if (!node->children[idx])
                node->children[idx] = new TrieNode();
            node = node->children[idx];
        }
        node->isEnd = true;
    }
    
    bool search(const string& word) {
        TrieNode* node = root;
        for (char c : word) {
            int idx = c - 'a';
            if (!node->children[idx]) return false;
            node = node->children[idx];
        }
        return node->isEnd;
    }
    
    bool startsWith(const string& prefix) {
        TrieNode* node = root;
        for (char c : prefix) {
            int idx = c - 'a';
            if (!node->children[idx]) return false;
            node = node->children[idx];
        }
        return true;
    }
};
```

### 5.1 二叉堆

```

              90
            ╱    ╲
          80      70
         ╱  ╲    ╱  ╲
       50   60  30   40
      ╱ ╲
    20  10
```

| 操作                  | 时间复杂度 | 说明                             |
| --------------------- | ---------- | -------------------------------- |
| Insert (push)         | O(log n)   | 加到末尾，上浮 (sift up)         |
| Extract max/min (pop) | O(log n)   | 取根，末尾放根，下沉 (sift down) |
| Peek                  | O(1)       | 返回根                           |
| Build heap            | O(n)       | 从最后一个非叶节点开始下沉       |
| Heapify (sift down)   | O(log n)   | 单次调整                         |

```cpp
// Min-heap implementation
class MinHeap {
    vector<int> data;
    
    void siftUp(int i) {
        while (i > 0) {
            int parent = (i - 1) / 2;
            if (data[i] < data[parent]) {
                swap(data[i], data[parent]);
                i = parent;
            } else break;
        }
    }
    
    void siftDown(int i) {
        int n = data.size();
        while (2 * i + 1 < n) {
            int child = 2 * i + 1;
            if (child + 1 < n && data[child + 1] < data[child])
                child++;
            if (data[i] > data[child]) {
                swap(data[i], data[child]);
                i = child;
            } else break;
        }
    }
    
public:
    void push(int val) {
        data.push_back(val);
        siftUp(data.size() - 1);
    }
    
    int pop() {
        int top = data[0];
        data[0] = data.back();
        data.pop_back();
        if (!data.empty()) siftDown(0);
        return top;
    }
    
    int peek() { return data[0]; }
    int size() { return data.size(); }
};
```

#### 面试题：Top K 问题

```cpp
// Find K largest elements from N elements
// Method 1: Sort - O(N log N)
// Method 2: Max-heap of all N, pop K times - O(N + K log N)
// Method 3: Min-heap of size K - O(N log K) ← optimal for large N
// Method 4: Quick Select - O(N) average

// Method 3: Min-heap of size K
vector<int> topK(vector<int>& nums, int k) {
    priority_queue<int, vector<int>, greater<int>> minHeap;
    for (int num : nums) {
        minHeap.push(num);
        if (minHeap.size() > k) {
            minHeap.pop();  // remove smallest
        }
    }
    vector<int> result;
    while (!minHeap.empty()) {
        result.push_back(minHeap.top());
        minHeap.pop();
    }
    return result;
}

// Method 4: Quick Select (Hoare's algorithm)
int quickSelect(vector<int>& nums, int left, int right, int k) {
    int pivot = nums[right];
    int i = left;
    for (int j = left; j < right; j++) {
        if (nums[j] <= pivot) {
            swap(nums[i], nums[j]);
            i++;
        }
    }
    swap(nums[i], nums[right]);
    
    if (i == k) return nums[i];
    if (i < k) return quickSelect(nums, i + 1, right, k);
    return quickSelect(nums, left, i - 1, k);
}
```

```cpp
// BFS - Breadth First Search
// Time: O(V + E), Space: O(V)
void bfs(vector<vector<int>>& adj, int start) {
    int n = adj.size();
    vector<bool> visited(n, false);
    queue<int> q;
    visited[start] = true;
    q.push(start);
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        // process u
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                q.push(v);
            }
        }
    }
}

// DFS - Depth First Search
// Time: O(V + E), Space: O(V)
void dfs(vector<vector<int>>& adj, int u, vector<bool>& visited) {
    visited[u] = true;
    // process u
    for (int v : adj[u]) {
        if (!visited[v]) {
            dfs(adj, v, visited);
        }
    }
}
```

```cpp
// Dijkstra with priority queue
vector<int> dijkstra(vector<vector<pair<int,int>>>& adj, int src) {
    int n = adj.size();
    vector<int> dist(n, INT_MAX);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    
    dist[src] = 0;
    pq.push({0, src});
    
    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();
        if (d > dist[u]) continue;  // skip outdated entries
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}

// A* algorithm (game pathfinding)
struct Node {
    int x, y;
    int g;  // cost from start
    int h;  // heuristic (estimated cost to goal)
    int f() const { return g + h; }  // total estimated cost
};

// Heuristic functions:
// Manhattan distance: |x1-x2| + |y1-y2|  (4-directional grid)
// Euclidean distance: sqrt((x1-x2)² + (y1-y2)²)
// Chebyshev distance: max(|x1-x2|, |y1-y2|)  (8-directional grid)

vector<pair<int,int>> aStar(Grid& grid, pair<int,int> start, pair<int,int> goal) {
    auto cmp = [](const Node& a, const Node& b) { return a.f() > b.f(); };
    priority_queue<Node, vector<Node>, decltype(cmp)> openSet(cmp);
    unordered_set<int> closedSet;
    unordered_map<int, pair<int,int>> cameFrom;
    
    openSet.push({start.first, start.second, 0, heuristic(start, goal)});
    
    while (!openSet.empty()) {
        Node current = openSet.top();
        openSet.pop();
        
        if (current.x == goal.first && current.y == goal.second) {
            return reconstructPath(cameFrom, goal);
        }
        
        int id = current.x * cols + current.y;
        if (closedSet.count(id)) continue;
        closedSet.insert(id);
        
        for (auto [nx, ny] : getNeighbors(current.x, current.y)) {
            int nid = nx * cols + ny;
            if (closedSet.count(nid) || !grid.walkable(nx, ny)) continue;
            int newG = current.g + moveCost(current, {nx, ny});
            openSet.push({nx, ny, newG, heuristic({nx, ny}, goal)});
            cameFrom[nid] = {current.x, current.y};
        }
    }
    return {};  // no path found
}
```

```cpp
// Topological Sort (Kahn's algorithm - BFS based)
// Used for: task scheduling, build systems, course prerequisites
vector<int> topologicalSort(int n, vector<vector<int>>& adj) {
    vector<int> inDegree(n, 0);
    for (int u = 0; u < n; u++)
        for (int v : adj[u])
            inDegree[v]++;
    
    queue<int> q;
    for (int i = 0; i < n; i++)
        if (inDegree[i] == 0)
            q.push(i);
    
    vector<int> order;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        order.push_back(u);
        for (int v : adj[u]) {
            if (--inDegree[v] == 0)
                q.push(v);
        }
    }
    
    if (order.size() != n) return {};  // cycle detected!
    return order;
}
```

```cpp
// Quick Sort (Lomuto partition)
void quickSort(vector<int>& arr, int lo, int hi) {
    if (lo >= hi) return;
    int pivot = arr[hi];
    int i = lo;
    for (int j = lo; j < hi; j++) {
        if (arr[j] < pivot) {
            swap(arr[i], arr[j]);
            i++;
        }
    }
    swap(arr[i], arr[hi]);
    quickSort(arr, lo, i - 1);
    quickSort(arr, i + 1, hi);
}

// Merge Sort
void mergeSort(vector<int>& arr, int lo, int hi) {
    if (lo >= hi) return;
    int mid = lo + (hi - lo) / 2;
    mergeSort(arr, lo, mid);
    mergeSort(arr, mid + 1, hi);
    
    vector<int> temp(hi - lo + 1);
    int i = lo, j = mid + 1, k = 0;
    while (i <= mid && j <= hi) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= hi) temp[k++] = arr[j++];
    for (int k = 0; k < temp.size(); k++)
        arr[lo + k] = temp[k];
}

// Heap Sort
void heapSort(vector<int>& arr) {
    int n = arr.size();
    // Build max-heap
    for (int i = n / 2 - 1; i >= 0; i--)
        siftDown(arr, i, n);
    // Extract elements
    for (int i = n - 1; i > 0; i--) {
        swap(arr[0], arr[i]);
        siftDown(arr, 0, i);
    }
}
```

#### 面试题：为什么快速排序比归并排序快？

1. **缓存友好：** 快排是原地排序，数据局部性好；归并需要额外数组
2. **常数因子小：** 快排的内循环只有比较和交换，归并需要拷贝
3. **分支预测：** 快排的分支模式更可预测
4. **空间：** 快排 O(log n) 栈空间 vs 归并 O(n) 额外空间
5. **但归并排序是稳定的，** 且最坏情况也是 O(n log n)

```cpp
struct AABB {
    float x, y, w, h;
    bool contains(float px, float py) const {
        return px >= x && px <= x + w && py >= y && py <= y + h;
    }
    bool intersects(const AABB& other) const {
        return !(x + w < other.x || other.x + other.w < x ||
                 y + h < other.y || other.y + other.h < y);
    }
};

class Quadtree {
    static const int MAX_OBJECTS = 8;
    static const int MAX_DEPTH = 6;
    
    int depth;
    AABB bounds;
    vector<Object*> objects;
    Quadtree* children[4] = {};
    
    void subdivide() {
        float hw = bounds.w / 2, hh = bounds.h / 2;
        float x = bounds.x, y = bounds.y;
        children[0] = new Quadtree(depth + 1, {x, y, hw, hh});         // NW
        children[1] = new Quadtree(depth + 1, {x + hw, y, hw, hh});    // NE
        children[2] = new Quadtree(depth + 1, {x, y + hh, hw, hh});    // SW
        children[3] = new Quadtree(depth + 1, {x + hw, y + hh, hw, hh}); // SE
    }
    
public:
    void insert(Object* obj) {
        if (!bounds.contains(obj->x, obj->y)) return;
        
        if (objects.size() < MAX_OBJECTS || depth >= MAX_DEPTH) {
            objects.push_back(obj);
            return;
        }
        
        if (!children[0]) subdivide();
        for (int i = 0; i < 4; i++)
            children[i]->insert(obj);
    }
    
    void query(const AABB& range, vector<Object*>& result) {
        if (!bounds.intersects(range)) return;
        
        for (auto* obj : objects) {
            if (range.contains(obj->x, obj->y))
                result.push_back(obj);
        }
        
        if (children[0]) {
            for (int i = 0; i < 4; i++)
                children[i]->query(range, result);
        }
    }
};
```

### 8.4 BVH (Bounding Volume Hierarchy)

```
Top-down construction:

              ┌─────────────────────┐
              │    Root AABB        │
              └──────────┬──────────┘
                    ╱          ╲
         ┌────────────┐  ┌────────────┐
         │  Left AABB │  │ Right AABB │
         └──────┬─────┘  └──────┬─────┘
            ╱      ╲          ╱      ╲
        ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
        │Obj A │ │Obj B │ │Obj C │ │Obj D │
        └──────┘ └──────┘ └──────┘ └──────┘

Split strategies:
- Midpoint: split at center of longest axis
- SAH (Surface Area Heuristic): minimize expected traversal cost
- Median: split at median object position

Ray-BVH intersection:
1. Test ray against root AABB
2. If hit, recursively test children
3. If leaf, test ray against actual geometry
4. Early termination when closer hit found

Used in:
- Ray tracing (primary acceleration structure)
- Physics engines (broad phase collision)
- GPU ray tracing (hardware BVH in RTX)
```

```cpp
struct BVHNode {

    BVHNode* right = nullptr;
    vector<int> objectIndices;  // leaf only

BVHNode* buildBVH(vector<Object>& objects, int start, int end, int depth) {
    BVHNode* node = new BVHNode();
    
    // Compute bounds
    node->bounds = computeBounds(objects, start, end);
    
    int count = end - start;
    if (count <= 4) {  // leaf
        for (int i = start; i < end; i++)
            node->objectIndices.push_back(i);
        return node;
    }
    
    // Find longest axis and split
    int axis = node->bounds.longestAxis();
    int mid = (start + end) / 2;
    nth_element(objects.begin() + start, objects.begin() + mid,
                objects.begin() + end,
                [axis](const Object& a, const Object& b) {
                    return a.center[axis] < b.center[axis];
                });
    
    node->left = buildBVH(objects, start, mid, depth + 1);
    node->right = buildBVH(objects, mid, end, depth + 1);
    return node;
}
```

### 8.7 空间哈希 (Spatial Hashing)

```
Divide space into uniform grid cells.
Hash cell coordinates to bucket index.

  Cell(x, y) → hash(x, y) → bucket index

  hash(x, y) = (x * 73856093 ^ y * 19349663) % TABLE_SIZE

┌─────┬─────┬─────┬─────┐
│(0,0)│(1,0)│(2,0)│(3,0)│
│  A  │     │  B  │     │
├─────┼─────┼─────┼─────┤
│(0,1)│(1,1)│(2,1)│(3,1)│
│     │ A,C │  C  │     │
├─────┼─────┼─────┼─────┤
│(0,2)│(1,2)│(2,2)│(3,2)│
│     │     │  D  │  D  │
└─────┴─────┴─────┴─────┘

Objects spanning multiple cells are inserted into all overlapping cells.

Advantages:
- O(1) insertion and query
- Simple implementation
- Good for uniform object distribution

Disadvantages:
- Cell size must match object size
- Poor for non-uniform distributions
- Memory waste for sparse worlds
```

```cpp
class SpatialHash {
    float cellSize;
    unordered_map<int64_t, vector<Object*>> buckets;
    
    int64_t hashKey(int cx, int cy) {
        return ((int64_t)cx * 73856093) ^ ((int64_t)cy * 19349663);
    }
    
    pair<int, int> toCell(float x, float y) {
        return {(int)floor(x / cellSize), (int)floor(y / cellSize)};
    }
    
public:
    SpatialHash(float cellSize) : cellSize(cellSize) {}
    
    void insert(Object* obj) {
        auto [cx, cy] = toCell(obj->x, obj->y);
        buckets[hashKey(cx, cy)].push_back(obj);
    }
    
    vector<Object*> query(float x, float y, float radius) {
        vector<Object*> result;
        int minCX = (int)floor((x - radius) / cellSize);
        int maxCX = (int)floor((x + radius) / cellSize);
        int minCY = (int)floor((y - radius) / cellSize);
        int maxCY = (int)floor((y + radius) / cellSize);
        
        for (int cx = minCX; cx <= maxCX; cx++) {
            for (int cy = minCY; cy <= maxCY; cy++) {
                auto it = buckets.find(hashKey(cx, cy));
                if (it != buckets.end()) {
                    for (auto* obj : it->second) {
                        float dx = obj->x - x, dy = obj->y - y;
                        if (dx * dx + dy * dy <= radius * radius)
                            result.push_back(obj);
                    }
                }
            }
        }
        return result;
    }
    
    void clear() { buckets.clear(); }
};
```

## 九、并查集 (Union-Find / Disjoint Set)

```
Operations:
- Find(x): find the root/representative of x's set
- Union(x, y): merge the sets containing x and y

Optimizations:
- Path compression: flatten tree during Find
- Union by rank: attach smaller tree under larger tree

With both optimizations: O(α(n)) per operation
α(n) = inverse Ackermann function ≈ O(1) for all practical n
```

```cpp
class UnionFind {
    vector<int> parent, rank;
public:
    UnionFind(int n) : parent(n), rank(n, 0) {
        iota(parent.begin(), parent.end(), 0);  // parent[i] = i
    }
    
    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);  // path compression
        return parent[x];
    }
    
    bool unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return false;  // already in same set
        // union by rank
        if (rank[rx] < rank[ry]) swap(rx, ry);
        parent[ry] = rx;
        if (rank[rx] == rank[ry]) rank[rx]++;
        return true;
    }
    
    bool connected(int x, int y) {
        return find(x) == find(y);
    }
};

// Use case: Kruskal's MST
int kruskal(int n, vector<tuple<int,int,int>>& edges) {
    sort(edges.begin(), edges.end());  // sort by weight
    UnionFind uf(n);
    int totalWeight = 0, edgeCount = 0;
    for (auto [w, u, v] : edges) {
        if (uf.unite(u, v)) {
            totalWeight += w;
            if (++edgeCount == n - 1) break;
        }
    }
    return totalWeight;
}
```

```cpp
// 1. LRU Cache
class LRUCache {
    int capacity;
    list<pair<int, int>> cache;  // doubly linked list: {key, value}
    unordered_map<int, list<pair<int, int>>::iterator> map;
    
public:
    LRUCache(int capacity) : capacity(capacity) {}
    
    int get(int key) {
        auto it = map.find(key);
        if (it == map.end()) return -1;
        cache.splice(cache.begin(), cache, it->second);  // move to front
        return it->second->second;
    }
    
    void put(int key, int value) {
        auto it = map.find(key);
        if (it != map.end()) {
            it->second->second = value;
            cache.splice(cache.begin(), cache, it->second);
            return;
        }
        if (cache.size() == capacity) {
            map.erase(cache.back().first);
            cache.pop_back();
        }
        cache.push_front({key, value});
        map[key] = cache.begin();
    }
};

class MinStack {

    stack<int> minStack;

    void push(int val) {
        data.push(val);
        if (minStack.empty() || val <= minStack.top())
            minStack.push(val);
    }

        if (data.top() == minStack.top())
            minStack.pop();
        data.pop();
    }

public:
    string serialize(TreeNode* root) {
        if (!root) return "#";
        return to_string(root->val) + "," +
               serialize(root->left) + "," +
               serialize(root->right);
    }
    
    TreeNode* deserialize(string data) {
        queue<string> tokens;
        stringstream ss(data);
        string token;
        while (getline(ss, token, ',')) tokens.push(token);
        return build(tokens);
    }
    
private:
    TreeNode* build(queue<string>& tokens) {
        string val = tokens.front();
        tokens.pop();
        if (val == "#") return nullptr;
        TreeNode* node = new TreeNode(stoi(val));
        node->left = build(tokens);
        node->right = build(tokens);
        return node;
    }
};

class MedianFinder {
    priority_queue<int> maxHeap;  // left half (max-heap)
    priority_queue<int, vector<int>, greater<int>> minHeap;  // right half (min-heap)
public:
    void addNum(int num) {
        maxHeap.push(num);
        minHeap.push(maxHeap.top());
        maxHeap.pop();
        if (minHeap.size() > maxHeap.size()) {
            maxHeap.push(minHeap.top());
            minHeap.pop();
        }
    }
    
    double findMedian() {
        if (maxHeap.size() > minHeap.size())
            return maxHeap.top();
        return (maxHeap.top() + minHeap.top()) / 2.0;
    }
};
```
