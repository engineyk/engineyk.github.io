#### 1.1 核心类关系

| 类名 | 模块 | 职责 |
| --- | --- | --- |
| `UMaterialInterface` | Engine | 材质抽象基类，提供统一接口 |
| `UMaterial` | Engine | 材质蓝图资产，包含节点图、编译逻辑 |
| `UMaterialInstance` | Engine | 材质实例基类，参数覆盖 |
| `UMaterialInstanceConstant` | Engine | 编辑器可编辑的材质实例 |
| `UMaterialInstanceDynamic` | Engine | 运行时可修改参数的材质实例 |
| `FMaterial` | Engine | 渲染线程材质数据，存储 ShaderMap |
| `FMaterialResource` | Engine | FMaterial 的唯一子类，实际实现 |
| `FMaterialRenderProxy` | Engine | 渲染代理，负责参数传递到 GPU |
| `FMaterialShaderMap` | Engine | 编译后的 Shader 缓存 |
| `FHLSLMaterialTranslator` | Engine | 材质蓝图 → HLSL 代码转译器 |

#### 1.2 Material vs MaterialInstance

```
UMaterial（母材质）：
  - 定义完整的材质逻辑（节点图）
  - 编译生成 Shader 代码
  - 修改后需要重新编译 Shader（耗时）
  - 一个 Material 可以很复杂，包含大量节点

UMaterialInstanceConstant（材质实例常量）：
  - 基于母材质，只覆盖暴露的参数（颜色、数值、纹理）
  - 共享母材质的 Shader 代码，不需要重新编译
  - 编辑器中创建和修改
  - 高效：多个实例共享同一 Shader，只有 CBuffer 参数不同

UMaterialInstanceDynamic（动态材质实例）：
  - 运行时通过代码创建和修改参数
  - UMaterialInstanceDynamic::Create(BaseMaterial, Outer)
  - SetScalarParameterValue / SetVectorParameterValue / SetTextureParameterValue
  - 注意：每个 Dynamic Instance 是独立对象，过多会影响合批
```

### 二、材质蓝图编译流程

```
材质编辑器（节点图）
    │
    ▼
UMaterial::CacheResourceShadersForRendering()
    │
    ▼
FMaterial::CacheShaders()
    │
    ▼
FMaterial::BeginCompileShaderMap()
    │
    ├─ 1. 创建 FHLSLMaterialTranslator
    │
    ├─ 2. MaterialTranslator.Translate()
    │     ├─ 遍历材质表达式节点
    │     ├─ 每个节点调用 Compile() → 生成 HLSL 代码片段
    │     └─ 填充到各属性槽（BaseColor, Normal, Roughness...）
    │
    ├─ 3. GetMaterialEnvironment()
    │     └─ 设置编译宏、Feature Level、Quality Level
    │
    ├─ 4. GetMaterialShaderCode()
    │     ├─ 将编译结果填充到 MaterialTemplate.ush 的 %s 占位符
    │     └─ 生成完整的材质 Shader 代码
    │
    └─ 5. FMaterialShaderMap::Compile()
          ├─ 遍历所有需要的 ShaderType × VertexFactoryType 组合
          ├─ 每个组合生成一个编译任务
          ├─ 异步提交到 Shader 编译线程池
          └─ 编译结果缓存到 FMaterialShaderMap
```

#### 2.1 Shader 代码组成

| Shader 类型 | 组成 |
| --- | --- |
| GlobalShader | `Shader_x.usf` |
| MaterialShader | `Shader_x.usf` + `MaterialTemplate_x.usf` |
| MeshMaterialShader | `Shader_x.usf` + `MaterialTemplate_x.usf` + `VertexFactory_x.usf` |

```
MaterialTemplate.ush 是材质 Shader 模板：
  - 包含大量 %s 占位符和待替换的宏
  - 由 FHLSLMaterialTranslator::GetMaterialShaderCode() 负责填充
  - 包含：基础 Shader 模块引用、宏定义、结构体定义、材质属性接口

关键文件：
  Engine/Shaders/Private/MaterialTemplate.ush    — 材质模板
  Engine/Shaders/Private/BasePassPixelShader.usf  — BasePass 像素着色器
  Engine/Shaders/Private/ShadingModels.ush        — Shading Model 实现
  Engine/Shaders/Private/DeferredShadingCommon.ush — 延迟着色公共代码
```

#### 2.2 材质表达式编译

```cpp
// 每个材质节点（UMaterialExpression）都有 Compile() 方法
// 例如 UMaterialExpressionAdd:
int32 UMaterialExpressionAdd::Compile(FMaterialCompiler* Compiler, int32 OutputIndex)
{
    int32 Arg1 = A.GetTracedInput().Expression ?
        A.Compile(Compiler) : Compiler->Constant(ConstA);
    int32 Arg2 = B.GetTracedInput().Expression ?
        B.Compile(Compiler) : Compiler->Constant(ConstB);
    return Compiler->Add(Arg1, Arg2);
    // 最终生成 HLSL: (A + B)
}

// FHLSLMaterialTranslator 将表达式序列化为 HLSL 代码片段
// 例如：
//   Compiler->Add(A, B)  → "(%s + %s)"
//   Compiler->Mul(A, B)  → "(%s * %s)"
//   Compiler->Dot(A, B)  → "dot(%s, %s)"
//   Compiler->TextureSample(Tex, UV) → "Texture2DSample(%s, %sSampler, %s)"
```

#### 2.3 Shader 跨平台编译 (HLSLCC)

```
UE 的 Shader 跨平台方案：
  1. 所有 Shader 用 HLSL (SM5.0) 编写
  2. HLSLCC (HLSL Cross Compiler) 将 HLSL 转换为目标平台语言

HLSLCC 流程：
  Preprocessing → Parsing (AST) → Compilation (Mesa IR)
  → Optimization → Uniform Packing → Final Optimization
  → Generate Target Code (GLSL / Metal / Vulkan SPIR-V)

目标平台映射：
  Windows  → HLSL (直接编译)
  macOS    → Metal Shading Language (via HLSLCC)
  Linux    → Vulkan SPIR-V (via DXC/HLSLCC)
  Android  → GLSL ES / Vulkan SPIR-V
  iOS      → Metal Shading Language
  Switch   → NVN / GLSL
```

| Subsurface Profile | 皮肤 SSS | 基于 Profile 的高质量 SSS |
| Preintegrated Skin | 预积分皮肤 | 移动端皮肤，LUT 查表 |

#### 3.2 自定义 Shading Model（添加新的光照模型）

```
添加自定义 Shading Model 的完整步骤：

Step 1: 定义枚举
  Engine/Source/Runtime/Engine/Classes/Engine/EngineTypes.h
  → EMaterialShadingModel 枚举中添加新值
  例如: MSM_ToonShading = 14

Step 2: 编辑器支持
  Engine/Source/Editor/UnrealEd/Private/MaterialEditor/
  → 在材质编辑器 UI 中添加新选项

Step 3: GBuffer 编码
  Engine/Shaders/Private/DeferredShadingCommon.ush
  → 在 GBuffer 中编码 Shading Model ID
  → SetGBufferForShadingModel() 函数

Step 4: 材质模板适配
  Engine/Shaders/Private/ShadingModelsMaterial.ush
  → 添加材质属性输出逻辑
  → 定义新 Shading Model 需要的额外数据（如 CustomData）

Step 5: 光照计算
  Engine/Shaders/Private/ShadingModels.ush
  → 实现新的 BRDF / 光照计算函数
  例如:
    FDirectLighting ToonBxDF(FGBufferData GBuffer, half3 N, half3 V,
                              half3 L, float Falloff, float NoL)
    {
        // 卡通阶梯化光照
        half ToonNoL = smoothstep(0.0, 0.01, NoL);
        half ToonStep = floor(ToonNoL * _Steps) / _Steps;
        ...
    }

Step 6: 延迟光照集成
  Engine/Shaders/Private/DeferredLightPixelShaders.usf
  → 在 GetDynamicLighting() 中添加 switch case
  → 调用新的 BxDF 函数

Step 7: C++ 端支持
  Engine/Source/Runtime/Renderer/Private/
  → 确保渲染器正确处理新 Shading Model 的 Pass 和状态
```

#### 3.3 GBuffer 布局（Deferred Rendering）

```
UE4 默认 GBuffer 布局：

GBufferA (RGBA8):
  R: WorldNormal.x (encoded)
  G: WorldNormal.y (encoded)
  B: WorldNormal.z (encoded)
  A: PerObjectGBufferData

GBufferB (RGBA8):
  R: Metallic
  G: Specular
  B: Roughness
  A: ShadingModelID (4bit) + SelectiveOutputMask (4bit)

GBufferC (RGBA8):
  R: BaseColor.r
  G: BaseColor.g
  B: BaseColor.b
  A: GBufferAO (Indirect AO)

GBufferD (RGBA8): — 可选，用于 CustomData
  R: CustomData0.x
  G: CustomData0.y
  B: CustomData1.x
  A: CustomData1.y

GBufferE: Precomputed Shadow Factors
GBufferF: Velocity (Motion Vector)

注意：
  - ShadingModelID 只有 4bit = 最多 16 种 Shading Model
  - CustomData 用于特殊 Shading Model 的额外数据
    例如 SSS: SubsurfaceColor 存在 CustomData
    例如 ClearCoat: ClearCoatRoughness 存在 CustomData
```

### 四、特殊材质效果实现

#### 4.1 皮肤 — Subsurface Scattering (SSS)

```
UE 提供两种 SSS 方案：

1. Subsurface Profile（高质量，PC/Console）
   - 基于 Burley 的 Normalized Diffusion Profile
   - 屏幕空间模糊：Separable SSS (Jorge Jimenez)
   - 每个 Profile 定义散射半径、颜色、衰减
   - 实现：
     a. BasePass 输出 Diffuse 到单独 RT
     b. 后处理阶段用 Profile Kernel 做屏幕空间模糊
     c. 合并回最终颜色

2. Preintegrated Skin（移动端）
   - 预计算 LUT: f(NdotL, Curvature) → 散射颜色
   - 曲率从法线变化率估算: curvature = length(fwidth(N))
   - 单次纹理查表，几乎无额外开销

HLSL 示例（预积分 SSS）:
  half curvature = length(fwidth(N)) / length(fwidth(worldPos));
  half NdotL = dot(N, L) * 0.5 + 0.5;
  half3 sss = tex2D(_SSSLut, float2(NdotL, curvature)).rgb;
  diffuse *= sss;
```

#### 4.2 头发 — Anisotropic Hair

```
UE 使用 Marschner Hair Model：
  - R lobe: 表面反射（Primary Specular）
  - TT lobe: 透射（Transmission）
  - TRT lobe: 内部反射后透射（Secondary Specular）

关键参数：
  - Tangent Direction: 沿发丝方向的切线
  - Shift: 高光偏移量（R 和 TRT 方向相反）
  - Roughness: 各向异性粗糙度

简化实现（Kajiya-Kay）:
  half StrandSpecular(half3 T, half3 H, half exponent) {
      half TdotH = dot(T, H);
      half sinTH = sqrt(1.0 - TdotH * TdotH);
      return pow(sinTH, exponent);
  }

  // 双高光
  half spec1 = StrandSpecular(T + N * shift1, H, exp1);
  half spec2 = StrandSpecular(T + N * shift2, H, exp2);
  half3 hairSpec = spec1 * specColor1 + spec2 * specColor2;
```

#### 4.3 布料 — Cloth Shading

```
UE 使用 Charlie Sheen 模型（Ashikhmin Cloth BRDF）：
  - 替代 GGX NDF，使用 Charlie Distribution
  - 更柔和的高光衰减，模拟织物微纤维散射
  - Fuzz Color: 绒毛颜色（掠射角可见）

Charlie NDF:
  float D_Charlie(float roughness, float NdotH) {
      float invR = 1.0 / roughness;
      float cos2h = NdotH * NdotH;
      float sin2h = 1.0 - cos2h;
      return (2.0 + invR) * pow(sin2h, invR * 0.5) / (2.0 * PI);
  }

UE Cloth 额外特性：
  - Cloth 不使用 Metallic（始终为 0）
  - 支持 Fuzz Color 参数（CustomData 传递）
  - 掠射角混合 Fuzz Color 和 Base Color
```

#### 4.4 眼睛 — Eye Shading

```
UE Eye Shading Model 特点：
  - 双层结构：角膜（Cornea）+ 虹膜（Iris）
  - 角膜：透明层，Specular 反射（Clear Coat 类似）
  - 虹膜：折射后的漫反射，带焦散效果
  - 瞳孔：根据光照强度动态缩放

关键技术：
  1. 视差偏移（Parallax）: 模拟虹膜深度
     float2 irisUV = uv + viewDir.xy * irisDepth / viewDir.z;
  2. 角膜折射: 折射光线采样虹膜纹理
  3. 环境反射: 角膜表面的环境映射
  4. 焦散: 光线聚焦在虹膜上的亮斑
```

#### 4.5 水面 — SingleLayerWater

```
UE5 SingleLayerWater Shading Model：
  - 单 Pass 水面渲染（替代传统多 Pass）
  - 支持折射、吸收、散射
  - 基于 Beer-Lambert 定律的水体吸收

关键参数：
  - Absorption: 水体吸收系数（RGB，控制水色）
  - Scattering: 散射系数
  - Phase Function: Henyey-Greenstein 相位函数

实现要点：
  1. 采样水下场景深度 → 计算水体厚度
  2. Beer-Lambert: transmittance = exp(-absorption * thickness)
  3. 散射: scattering = (1 - transmittance) * scatterColor
  4. 折射: 基于法线偏移 UV 采样水下颜色
  5. 反射: SSR + Reflection Probe fallback
  6. Fresnel: 控制反射/折射比例
```

#### 4.6 植被 — Two Sided Foliage

```
双面植被材质特点：
  - 正面：标准 PBR 光照
  - 背面：透光效果（Subsurface Transmission）
  - 背面光照 = 光源穿透叶片后的散射

实现：
  half backLighting = saturate(dot(-N, L));
  half3 transmission = backLighting * subsurfaceColor * thickness;
  finalColor = frontLighting + transmission;

UE 中的 Two Sided Foliage：
  - SubsurfaceColor 通过 CustomData 传递
  - 自动处理法线翻转
  - 支持风动画（World Position Offset）
```

### 五、材质系统设计要点（自研引擎参考）

#### 5.1 Uber Shader vs 模块化 Shader

```
Uber Shader（UE 方案）：
  - 一个巨大的 Shader 包含所有功能
  - 通过 #define / #if 控制功能开关
  - 优点：统一管理，编辑器友好
  - 缺点：变体爆炸（Permutation Explosion）

模块化 Shader（Unity URP/HDRP 方案）：
  - 功能拆分为独立模块（Lighting, Shadow, GI...）
  - 通过 #include 组合
  - 优点：灵活，变体可控
  - 缺点：组合复杂度高

自研引擎建议：
  - 核心 Pass（BasePass, Shadow, Depth）用 Uber Shader
  - 特殊效果（后处理、特效）用独立 Shader
  - 材质编辑器生成的代码填充到模板中（类似 UE 的 MaterialTemplate.ush）
```

#### 5.2 Shader 变体管理

```
变体爆炸问题：
  N 个 Feature × M 个 Quality × K 个 Platform = N×M×K 个变体
  例如：10 个 Feature 各 2 个选项 = 2^10 = 1024 个变体

UE 的解决方案：
  1. Static Switch: 编译期确定，生成不同变体
  2. Dynamic Branch: 运行时分支（GPU 友好时使用）
  3. Shader Permutation: 通过 FShaderPermutationParameters 管理
  4. Shader Stripping: Cook 时剔除未使用的变体

Unity 的解决方案：
  1. multi_compile: 生成所有组合（变体多）
  2. shader_feature: 只生成使用到的组合（需要收集）
  3. ShaderVariantCollection: 预收集需要的变体
  4. Shader Stripping: 构建时回调剔除

自研引擎建议：
  - 定义 Feature Flag 系统
  - 编辑器中收集实际使用的变体组合
  - 构建时只编译需要的变体
  - 运行时按需加载（Shader 缓存 + 异步编译）
```

#### 5.3 材质参数系统

```
CBuffer 布局设计：

// Per-Frame (全局，每帧更新一次)
cbuffer FrameConstants : register(b0) {
    float4x4 ViewProjection;
    float3   CameraPosition;
    float    Time;
    float4   LightDirection;
    float4   LightColor;
    ...
};

// Per-Material (每种材质一个，切换材质时更新)
cbuffer MaterialConstants : register(b1) {
    float4 BaseColor;
    float  Metallic;
    float  Roughness;
    float  EmissiveIntensity;
    ...
};

// Per-Object (每个物体一个，SRP Batcher 核心)
cbuffer ObjectConstants : register(b2) {
    float4x4 ObjectToWorld;
    float4x4 WorldToObject;
    ...
};

UE 的 SRP Batcher 等价物：
  - UE 通过 Uniform Buffer 分层管理
  - 相同 Shader 的不同材质实例只需更新 MaterialConstants
  - 减少 GPU 状态切换
```

#### 6.2 UE Shader 类型声明

```cpp
// Global Shader（全局着色器，不依赖材质）
class FMyGlobalShaderPS : public FGlobalShader
{
    DECLARE_GLOBAL_SHADER(FMyGlobalShaderPS)
    SHADER_USE_PARAMETER_STRUCT(FMyGlobalShaderPS, FGlobalShader)

    BEGIN_SHADER_PARAMETER_STRUCT(FParameters, )
        SHADER_PARAMETER(FVector4f, MyColor)
        SHADER_PARAMETER_TEXTURE(Texture2D, MyTexture)
        SHADER_PARAMETER_SAMPLER(SamplerState, MySampler)
        RENDER_TARGET_BINDING_SLOTS()
    END_SHADER_PARAMETER_STRUCT()
};
IMPLEMENT_GLOBAL_SHADER(FMyGlobalShaderPS,
    "/Engine/Private/MyShader.usf", "MainPS", SF_Pixel);

// Material Shader（材质着色器，依赖材质蓝图）
class FMyMaterialShaderPS : public FMaterialShader
{
    DECLARE_SHADER_TYPE(FMyMaterialShaderPS, Material)
```

### 七、材质性能优化

#### 7.1 Shader 复杂度控制

```
材质编辑器中查看 Shader 复杂度：
  View Mode → Shader Complexity
  绿色 → 低开销 | 红色 → 高开销 | 白色 → 极高开销

关键指标：
  - Instruction Count: 指令数（越少越好）
  - Texture Samples: 纹理采样次数（移动端 < 8）
  - Shader Model: 所需的最低 SM 版本

优化手段：
  1. 减少纹理采样：合并通道（RGBA 打包多个属性）
  2. 使用 half 精度：移动端 ALU 效率翻倍
  3. 避免动态分支：用 lerp 替代 if-else
  4. 预计算：将复杂计算移到顶点着色器
  5. Shader LOD：远距离使用简化材质
  6. 材质实例化：共享 Shader，减少编译和切换开销
```

#### 7.2 材质分级策略

```
Quality Level 分级（UE 内置支持）：

High（PC/Console）：
  - 完整 PBR + SSS + Clear Coat
  - 法线贴图 + 视差映射
  - 8+ 纹理采样
  - 全精度计算

Medium（中端移动）：
  - 简化 PBR（无 SSS）
  - 法线贴图
  - 4-6 纹理采样
  - half 精度

Low（低端移动）：
  - Lambert + Blinn-Phong
  - 无法线贴图
  - 2-3 纹理采样
  - 最小化指令数

实现方式：
  - UE: Quality Switch 节点 / Material Quality Level
  - 自研引擎: Shader LOD / Feature Flag 动态切换
```

#### 7.3 变体优化

```
变体数量 = Π(每个 Feature 的选项数)

优化策略：
  1. 合并相似 Feature：将低频 Feature 合并为一个
  2. 动态分支替代静态变体：GPU 分支预测好的情况下
  3. Shader Stripping：构建时剔除未使用变体
  4. 按需编译：运行时遇到新组合才编译（需要缓存）
  5. PSO 预热：启动时预编译常用变体组合

UE 变体管理：
  - FShaderPermutationDomain: 定义变体维度
  - ShouldCompilePermutation(): 过滤不需要的组合
  - ShaderPipelineCache: PSO 缓存和预热
```

### 八、面试高频问题 — Unreal 材质

```
Q: UMaterialInterface、FMaterialRenderProxy、FMaterial 的关联和区别？
A: UMaterialInterface 是游戏线程的材质抽象基类（UObject）；
   FMaterialRenderProxy 是渲染代理，负责将材质参数传递到渲染线程；
   FMaterial 是渲染线程的材质数据，存储编译后的 ShaderMap。
   三者分离是为了线程安全：游戏线程操作 UMaterial，
   渲染线程通过 FMaterialRenderProxy 获取 FMaterial 数据。

Q: 材质蓝图的编译流程？
A: UMaterial 修改 → CacheResourceShadersForRendering
   → FMaterial::CacheShaders → BeginCompileShaderMap
   → FHLSLMaterialTranslator::Translate()（遍历表达式节点生成 HLSL 片段）
   → GetMaterialShaderCode()（填充 MaterialTemplate.ush）
   → FMaterialShaderMap::Compile()（异步编译所有 ShaderType × VF 组合）

Q: 如何添加自定义 Shading Model？
A: 1) EMaterialShadingModel 添加枚举
   2) 编辑器 UI 添加选项
   3) GBuffer 编码（DeferredShadingCommon.ush）
   4) 材质属性输出（ShadingModelsMaterial.ush）
   5) 光照计算（ShadingModels.ush 实现新 BxDF）
   6) 延迟光照集成（DeferredLightPixelShaders.usf）

Q: Material 和 MaterialInstance 的区别？性能影响？
A: Material 定义完整逻辑，修改需重新编译 Shader；
   MaterialInstance 只覆盖参数，共享 Shader 代码，无需重编译。
   性能：多个 MaterialInstance 共享同一 Shader = 更好的合批。
   注意：过多 Dynamic MaterialInstance 会打断合批。

Q: 如何设计一个材质系统？（自研引擎）
A: 1) 材质模板：定义 Shader 模板（类似 MaterialTemplate.ush）
   2) 节点图编辑器：可视化编辑材质逻辑
   3) 表达式编译：节点 → HLSL 代码片段
   4) 变体管理：Feature Flag + 按需编译 + Stripping
   5) 参数系统：CBuffer 分层（Frame/Material/Object）
   6) 实例化：母材质 + 参数覆盖，共享 Shader
   7) 分级：Quality Level 动态切换
   8) 跨平台：HLSL → HLSLCC → 目标平台语言

Q: 皮肤 SSS 怎么实现？移动端怎么优化？
A: PC: Separable SSS（屏幕空间模糊，基于 Subsurface Profile）
   移动端: Pre-Integrated Skin（预积分 LUT，f(NdotL, Curvature)）
   LUT 查表只需一次纹理采样，几乎无额外开销。

Q: Shader 变体爆炸怎么解决？
A: 1) ShouldCompilePermutation() 过滤无效组合
   2) 动态分支替代低频静态变体
   3) 构建时 Shader Stripping 剔除未使用变体
   4) PSO Cache 预热避免运行时编译卡顿
   5) 合并相似 Feature 减少维度
```
