# Unreal Gameplay Framework 详解

> Unreal Engine 的 Gameplay 框架是一套完整的游戏逻辑架构体系，  
> 通过职责分离的设计，将游戏规则、玩家控制、角色表现、状态同步等模块解耦。

## 1. 核心概念

### Gameplay 框架三大支柱

| 支柱                 | 职责               | 典型应用                |
| -------------------- | ------------------ | ----------------------- |
| **GameMode**         | 游戏规则、流程控制 | 匹配开始/结束、得分规则 |
| **PlayerController** | 玩家输入与交互     | 鼠标点击、键盘操作      |
| **Pawn/Character**   | 游戏内实体         | 玩家角色、NPC           |

---

## 2. 基础类层次结构

```
UObject (最基础)
├── Actor (可放置在场景中的对象)
│   ├── Pawn (可被控制的实体)
│   │   └── Character (带移动组件的Pawn)
│   ├── Controller (控制Actor的逻辑)
│   │   ├── PlayerController (玩家控制)
│   │   └── AIController (AI控制)
│   └── Info (存放游戏信息)
│       ├── GameStateBase (游戏状态)
│       ├── PlayerStateBase (玩家状态)
│       └── GameModeBase (游戏模式)
├── Component (Actor的功能组件)
│   ├── SceneComponent (有位置变换的组件)
│   ├── ActorComponent (纯逻辑组件)
│   └── MovementComponent (移动组件)
└── GameInstance (全局单例，跨关卡)
```

## 3. GameMode 与 GameState

### GameModeBase

**职责**：定义游戏规则，**仅在服务器端存在**

```cpp
// MyGameMode.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/GameModeBase.h"
#include "MyGameMode.generated.h"

UCLASS()
class MYPROJECT_API AMyGameMode : public AGameModeBase
{
    GENERATED_BODY()

public:
    AMyGameMode();

    // Called when game starts
    virtual void BeginPlay() override;

    // Called when a player joins
    virtual void PostLogin(APlayerController* NewPlayer) override;

    // Called when a player leaves
    virtual void Logout(AController* Exiting) override;

    // End game logic
    UFUNCTION(BlueprintCallable)
    void EndGame(bool bIsPlayerWin);

    // Spawn player pawn
    virtual UClass* GetDefaultPawnClassForController_Implementation(
        AController* InController) override;

protected:
    UPROPERTY(EditDefaultsOnly, Category="Game Settings")
    int32 MaxPlayers = 4;

    UPROPERTY(EditDefaultsOnly, Category="Game Settings")
    float GameTimeLimit = 300.0f;

    UPROPERTY(EditDefaultsOnly, Category="Game Settings")
    TSubclassOf<APawn> DefaultPawnClass;

    UPROPERTY(EditDefaultsOnly, Category="Game Settings")
    TSubclassOf<APlayerController> DefaultPlayerControllerClass;
};
```

### GameStateBase

**职责**：同步游戏状态到所有客户端

```cpp
// MyGameState.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/GameStateBase.h"
#include "MyGameState.generated.h"

UCLASS()
class MYPROJECT_API AMyGameState : public AGameStateBase
{
    GENERATED_BODY()

public:
    // Is game currently active
    UPROPERTY(Replicated, BlueprintReadOnly, Category="Game State")
    bool bIsGameActive;

    // Remaining time (synced to all clients)
    UPROPERTY(Replicated, BlueprintReadOnly, Category="Game State")
    float RemainingTime;

    // Player score list
    UPROPERTY(Replicated, BlueprintReadOnly, Category="Game State")
    TArray<int32> PlayerScores;

    // Server update time
    UFUNCTION(Server, Reliable)
    void ServerUpdateRemainingTime(float NewTime);

    // Client receive time update
    UFUNCTION(Client, Reliable)
    void ClientUpdateRemainingTime(float NewTime);

    // Get player count
    UFUNCTION(BlueprintCallable, Category="Game State")
    int32 GetNumPlayers() const;

    // Get highest score
    UFUNCTION(BlueprintCallable, Category="Game State")
    int32 GetHighScore() const;

    virtual void GetLifetimeReplicatedProps(
        TArray<FLifetimeProperty>& OutLifetimeProps) const override;
};
```

### GameMode vs GameState 对比

| 维度           | GameMode           | GameState                |
| -------------- | ------------------ | ------------------------ |
| **存在位置**   | 仅服务器           | 服务器 + 所有客户端      |
| **主要职责**   | 规则定义、流程控制 | 状态存储、数据同步       |
| **典型数据**   | 规则参数、生成逻辑 | 比分、剩余时间、玩家列表 |
| **客户端访问** | ❌ 不可直接访问     | ✅ 可直接访问             |

## 4. PlayerController 与 PlayerState

### PlayerController

**职责**：处理玩家输入，协调客户端与服务器的通信

```cpp
// MyPlayerController.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/PlayerController.h"
#include "MyPlayerController.generated.h"

UCLASS()
class MYPROJECT_API AMyPlayerController : public APlayerController
{
    GENERATED_BODY()

public:
    AMyPlayerController();

    // Setup input bindings
    virtual void SetupInputComponent() override;

    // Local player initialization
    virtual void BeginPlay() override;

    // Server-side initialization
    virtual void OnPossess(APawn* InPawn) override;

    // Release control
    virtual void OnUnPossess() override;

    // Input bindings
    void MoveForward(float Value);
    void MoveRight(float Value);
    void LookUp(float Value);
    void Turn(float Value);
    void Jump();
    void StopJumping();
    void StartSprint();
    void StopSprint();
    void PrimaryAction();
    void SecondaryAction();

    // Server RPC: fire weapon
    UFUNCTION(Server, Reliable, WithValidation)
    void ServerFireWeapon(FVector Location, FVector Direction);

    // Client RPC: show hit effect
    UFUNCTION(Client, Reliable)
    void ClientShowHitEffect(FVector Location, FVector Normal);

    // UI interaction
    UFUNCTION(BlueprintCallable)
    void ShowMenu();

    UFUNCTION(BlueprintCallable)
    void HideMenu();

    // Camera settings
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Camera")
    float BaseTurnRate = 45.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Camera")
    float BaseLookUpRate = 45.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Camera")
    float MouseSensitivity = 1.0f;

protected:
    // Cached input component
    UPROPERTY()
    class UInputComponent* PlayerInputComponent;

    // Camera component
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category="Camera")
    class UCameraComponent* PlayerCamera;

    // Is sprinting
    bool bIsSprinting;
};
```

### PlayerController vs PlayerState 对比

| 维度             | PlayerController    | PlayerState        |
| ---------------- | ------------------- | ------------------ |
| **存在位置**     | 本地客户端 + 服务器 | 所有客户端         |
| **主要职责**     | 输入处理、UI控制    | 玩家数据存储       |

| **其他玩家可见** | ❌ 不可见            | ✅ 可见             |

### Pawn vs Character 对比

| 维度          | Pawn                     | Character                       |
| ------------- | ------------------------ | ------------------------------- |
| **移动组件**  | 无（需自定义）           | 自带 CharacterMovementComponent |
| **碰撞体**    | 无（需自定义）           | 自带 CapsuleComponent           |
| **骨骼网格**  | 无（需自定义）           | 自带 SkeletalMeshComponent      |
| **跳跃/蹲伏** | 需自实现                 | 内置支持                        |
| **网络移动**  | 需自实现                 | 内置预测与同步                  |
| **适用场景**  | 载具、飞行器、自定义实体 | 人形角色                        |

### ActorComponent vs SceneComponent

| 维度          | ActorComponent         | SceneComponent         |
| ------------- | ---------------------- | ---------------------- |
| **位置/旋转** | ❌ 无                   | ✅ 有 Transform         |
| **层级关系**  | ❌ 无父子关系           | ✅ 支持父子层级         |
| **典型用途**  | 逻辑组件（血量、技能） | 可视组件（网格、粒子） |
| **性能开销**  | 低                     | 略高（维护Transform）  |

## 8. 网络同步

### 复制属性类型

| 类型              | 描述        | 使用场景             |
| ----------------- | ----------- | -------------------- |
| `Replicated`      | 基础复制    | 简单的数值、布尔值   |
| `ReplicatedUsing` | 复制 + 回调 | 需要本地处理的属性   |
| `RepNotify`       | 通知复制    | 客户端需要响应的属性 |
| `OnRep` 函数      | 复制回调    | 属性变化时的逻辑     |

### RPC类型

| RPC类型        | 调用方向            | 可靠性              | 典型用途             |
| -------------- | ------------------- | ------------------- | -------------------- |
| `Server`       | 客户端 → 服务器     | Reliable/Unreliable | 玩家操作请求         |
| `Client`       | 服务器 → 特定客户端 | Reliable/Unreliable | 个人反馈（命中特效） |
| `NetMulticast` | 服务器 → 所有客户端 | Reliable/Unreliable | 全局事件（爆炸特效） |

### 网络同步示例

```cpp
// NetworkedActor.h
UCLASS()
class ANetworkedActor : public AActor
{
    GENERATED_BODY()

public:
    // Server authoritative position
    UPROPERTY(Replicated)
    FVector ServerPosition;

    // Client predicted position
    UPROPERTY()
    FVector ClientPredictedPosition;

    // Server RPC: request move
    UFUNCTION(Server, Unreliable)
    void ServerMove(FVector NewPosition);

    // Client RPC: sync position
    UFUNCTION(Client, Unreliable)
    void ClientUpdatePosition(FVector NewPosition);

    // Multicast RPC: play effect on all clients
    UFUNCTION(NetMulticast, Reliable)
    void MulticastPlayEffect(FVector Location);

    // Smooth interpolation
    void SmoothSync(float DeltaTime);

    // Replication setup
    virtual void GetLifetimeReplicatedProps(
        TArray<FLifetimeProperty>& OutLifetimeProps) const override;
};

// NetworkedActor.cpp
void ANetworkedActor::GetLifetimeReplicatedProps(
    TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);

    // Replicate to all clients
    DOREPLIFETIME(ANetworkedActor, ServerPosition);

    // Replicate only to owner
    DOREPLIFETIME_CONDITION(ANetworkedActor, ClientPredictedPosition,
        COND_OwnerOnly);
}
```

### 网络角色（Network Role）

| 角色                   | 描述           | 典型操作           |
| ---------------------- | -------------- | ------------------ |
| `ROLE_Authority`       | 服务器权威端   | 修改数据、执行逻辑 |
| `ROLE_AutonomousProxy` | 本地玩家客户端 | 输入预测、本地模拟 |
| `ROLE_SimulatedProxy`  | 其他客户端     | 插值同步、表现播放 |
| `ROLE_None`            | 无网络角色     | 纯本地对象         |

```cpp
void AMyCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    switch (GetLocalRole())
    {
        case ROLE_Authority:
            // Server: authoritative logic
            UpdateServerState(DeltaTime);
            break;

        case ROLE_AutonomousProxy:
            // Local player: input prediction
            ProcessLocalInput(DeltaTime);
            break;

        case ROLE_SimulatedProxy:
            // Other players: smooth interpolation
            SmoothRemoteMovement(DeltaTime);
            break;

        default:
            break;
    }
}
```

## 9. 生命周期管理

### Actor 生命周期流程

```
Spawn Actor
    ↓
Constructor()               // CDO创建，组件初始化
    ↓
PostInitializeComponents()  // 所有组件初始化完成
    ↓
BeginPlay()                 // 游戏开始，安全的初始化位置
    ↓
Tick(DeltaTime)             // 每帧更新（可禁用）
    ↓
EndPlay(Reason)             // 销毁前清理
    ↓
Destroy()                   // 销毁对象
```

### 重要事件顺序

```cpp
// 1. Constructor (CDO creation)
AMyActor::AMyActor()
{
    PrimaryActorTick.bCanEverTick = true;
    // Create components and set default values
    HealthComponent = CreateDefaultSubobject<UHealthComponent>(TEXT("HealthComponent"));
}

// 2. PostInitializeComponents()
void AMyActor::PostInitializeComponents()
{
    Super::PostInitializeComponents();
    // Called after all components are initialized
    // Safe to access component data here
}

// 3. BeginPlay()
void AMyActor::BeginPlay()
{
    Super::BeginPlay();
    // Game has started, safe initialization point
    // Bind delegates, start timers, etc.
    HealthComponent->OnDeath.AddDynamic(this, &AMyActor::OnDeath);
}

// 4. Tick() - every frame
void AMyActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
    // Per-frame update logic
}

// 5. EndPlay()
void AMyActor::EndPlay(const EEndPlayReason::Type EndPlayReason)
{
    Super::EndPlay(EndPlayReason);
    // Cleanup: clear timers, unbind delegates, release resources
    GetWorldTimerManager().ClearAllTimersForObject(this);
}
```

### EndPlay 触发原因

| 原因               | 描述             |
| ------------------ | ---------------- |
| `Destroyed`        | Actor 被显式销毁 |
| `LevelTransition`  | 关卡切换         |
| `EndPlayInEditor`  | 编辑器停止运行   |
| `RemovedFromWorld` | 从世界中移除     |
| `Quit`             | 游戏退出         |

#### 2. 组件缓存

```cpp
// Cache component references, avoid per-frame lookup
UPROPERTY()
UHealthComponent* CachedHealthComponent;

void AMyActor::BeginPlay()
{
    Super::BeginPlay();
    CachedHealthComponent = FindComponentByClass<UHealthComponent>();
    check(CachedHealthComponent); // Assert in debug
}
```

#### 3. 对象池

```cpp
// Reuse Actors with object pool
UCLASS()
class AProjectilePool : public AActor
{
    GENERATED_BODY()

public:
    AProjectile* AcquireProjectile()
    {
        if (!AvailableProjectiles.IsEmpty())
        {
            AProjectile* Projectile = AvailableProjectiles.Pop();
            Projectile->SetActorHiddenInGame(false);
            Projectile->SetActorEnableCollision(true);
            return Projectile;
        }
        // Pool exhausted: spawn new one
        return GetWorld()->SpawnActor<AProjectile>(ProjectileClass);
    }

    void ReturnProjectile(AProjectile* Projectile)
    {
        Projectile->SetActorHiddenInGame(true);
        Projectile->SetActorEnableCollision(false);
        Projectile->SetActorVelocity(FVector::ZeroVector);
        AvailableProjectiles.Push(Projectile);
    }

private:
    TArray<AProjectile*> AvailableProjectiles;

    UPROPERTY(EditDefaultsOnly)
    TSubclassOf<AProjectile> ProjectileClass;
};
```

### 架构设计原则

#### 1. 单一职责

```
GameMode        → 游戏规则（何时开始/结束，如何计分）
PlayerController → 输入处理（玩家如何操控）
Character       → 角色表现（如何移动、动画）
Component       → 特定功能（血量、武器、技能）
```
