# UE GAS (Gameplay Ability System) 实现原理

## 一、概述


Gameplay Ability System（GAS）是 Unreal Engine 提供的一套高度灵活、可扩展的框架，用于构建角色的**技能、属性、状态效果和交互逻辑**。它最初为大型 RPG/ARPG 项目设计，现已广泛应用于各类游戏。

---

## 二、核心组件架构

```
┌─────────────────────────────────────────────────┐
│               AbilitySystemComponent (ASC)       │
│  ┌───────────┐ ┌───────────┐ ┌───────────────┐  │
│  │ Gameplay   │ │ Gameplay  │ │ AttributeSet  │  │
│  │ Abilities  │ │ Effects   │ │ (Attributes)  │  │
│  └─────┬─────┘ └─────┬─────┘ └───────┬───────┘  │
│        │             │               │           │
│  ┌─────▼─────┐ ┌─────▼─────┐ ┌───────▼───────┐  │
│  │ Ability    │ │ Gameplay  │ │  Gameplay     │  │
│  │ Tasks      │ │ Cues      │ │  Tags         │  │
│  └───────────┘ └───────────┘ └───────────────┘  │
└─────────────────────────────────────────────────┘
```

| 组件                             | 职责                                   |
| -------------------------------- | -------------------------------------- |
| **AbilitySystemComponent (ASC)** | 中枢组件，管理所有能力、效果和属性     |
| **GameplayAbility (GA)**         | 技能/能力的逻辑载体                    |
| **GameplayEffect (GE)**          | 数据驱动的属性修改器                   |
| **AttributeSet (AS)**            | 属性数据容器（生命值、攻击力等）       |
| **GameplayTag**                  | 层级化标签系统，用于状态标记与条件判断 |
| **AbilityTask**                  | 异步操作执行单元（蒙太奇、等待输入等） |
| **GameplayCue**                  | 表现效果处理（特效、音效等）           |

## 三、各组件详解

### 3.1 AbilitySystemComponent (ASC)

ASC 是整个 GAS 的核心枢纽，负责：
- 管理和激活 GameplayAbility
- 应用和移除 GameplayEffect
- 持有和同步 AttributeSet
- 管理 GameplayTag 状态
- 处理网络复制与预测

#### 挂载位置选择

```cpp
// 方式1：挂载在 Character 上（简单，适合 AI / NPC）
AMyCharacter::AMyCharacter()
{
    AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("ASC"));
}

// 方式2：挂载在 PlayerState 上（推荐用于玩家角色，Pawn 切换时状态不丢失）
AMyPlayerState::AMyPlayerState()
{
    AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("ASC"));
}
```

#### 网络同步模式

| 模式      | 说明                                        | 适用场景         |
| --------- | ------------------------------------------- | ---------------- |
| `Full`    | 所有 GE 同步到所有客户端                    | AI / NPC         |
| `Mixed`   | GE 仅同步到拥有者客户端，Tag/Cue 同步到所有 | 玩家角色（推荐） |
| `Minimal` | 最少同步，仅 Tag/Cue 同步到所有             | AI（性能优先）   |

```cpp
AbilitySystemComponent->SetReplicationMode(EGameplayEffectReplicationMode::Mixed);
```

### 3.2 GameplayAbility (GA)

每个技能/能力对应一个 GA 类，定义了技能的**激活条件、执行逻辑、消耗和冷却**。

#### 生命周期

```
CanActivateAbility()
        │
        ▼
  ActivateAbility()
        │
        ▼
  CommitAbility()  ──→  Apply Cost & Cooldown
        │
        ▼
   执行技能逻辑 (AbilityTasks)
        │
        ▼
  EndAbility() / CancelAbility()
```

#### 代码示例

```cpp
UCLASS()
class UGA_FireBall : public UGameplayAbility
{
    GENERATED_BODY()

public:
    UGA_FireBall();

    // Check if ability can be activated
    virtual bool CanActivateAbility(const FGameplayAbilitySpecHandle Handle,
        const FGameplayAbilityActorInfo* ActorInfo,
        const FGameplayTagContainer* SourceTags,
        const FGameplayTagContainer* TargetTags,
        FGameplayTagContainer* OptionalRelevantTags) const override;

    // Main activation logic
    virtual void ActivateAbility(const FGameplayAbilitySpecHandle Handle,
        const FGameplayAbilityActorInfo* ActorInfo,
        const FGameplayAbilityActivationInfo ActivationInfo,
        const FGameplayEventData* TriggerEventData) override;
};

void UGA_FireBall::ActivateAbility(...)
{
    // 1. Commit cost and cooldown
    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, true);
        return;
    }

    // 2. Play montage using AbilityTask
    UAbilityTask_PlayMontageAndWait* MontageTask =
        UAbilityTask_PlayMontageAndWait::CreatePlayMontageAndWaitProxy(
            this, NAME_None, FireBallMontage, 1.0f);

    MontageTask->OnCompleted.AddDynamic(this, &UGA_FireBall::OnMontageCompleted);
    MontageTask->OnCancelled.AddDynamic(this, &UGA_FireBall::OnMontageCancelled);
    MontageTask->ReadyForActivation();
}
```

#### 激活策略 (Net Execution Policy)

| 策略              | 说明                               |
| ----------------- | ---------------------------------- |
| `LocalPredicted`  | 客户端预测执行，服务器验证（推荐） |
| `LocalOnly`       | 仅本地执行                         |
| `ServerInitiated` | 服务器发起，同步到客户端           |
| `ServerOnly`      | 仅服务器执行                       |

### 3.3 GameplayEffect (GE)

GE 是**纯数据驱动**的属性修改器，通常在蓝图中配置。

#### 持续类型

| 类型       | 说明                            | 典型用途           |
| ---------- | ------------------------------- | ------------------ |
| `Instant`  | 立即生效，一次性修改 BaseValue  | 伤害、治疗         |
| `Duration` | 持续一段时间，修改 CurrentValue | Buff/Debuff        |
| `Infinite` | 无限持续，需手动移除            | 被动技能、装备属性 |

#### 修改操作类型 (Modifier Op)

| 操作       | 公式                        |
| ---------- | --------------------------- |
| `Add`      | `CurrentValue += Magnitude` |
| `Multiply` | `CurrentValue *= Magnitude` |
| `Divide`   | `CurrentValue /= Magnitude` |
| `Override` | `CurrentValue = Magnitude`  |

#### Modifier 计算方式

```
Modifier Magnitude 计算方式：
├── ScalableFloat          → 固定值（可配曲线表）
├── AttributeBased         → 基于某个属性计算
├── CustomCalculationClass → 自定义 C++ 计算类
└── SetByCaller            → 运行时外部传入
```

#### SetByCaller 示例

```cpp
// Ability side: set damage value
FGameplayEffectSpecHandle SpecHandle = MakeOutgoingGameplayEffectSpec(DamageEffect, GetAbilityLevel());
SpecHandle.Data->SetSetByCallerMagnitude(FGameplayTag::RequestGameplayTag("Data.Damage"), 50.0f);

// Apply the effect
ApplyGameplayEffectSpecToTarget(Handle, ActorInfo, ActivationInfo, SpecHandle, {});
```

### 3.4 AttributeSet (AS)

属性集定义了角色的数值属性。

#### 核心概念：BaseValue vs CurrentValue

| 值             | 说明                                           |
| -------------- | ---------------------------------------------- |
| `BaseValue`    | 基础值，仅被 Instant GE 永久修改               |
| `CurrentValue` | 当前值 = BaseValue + Duration/Infinite GE 修改 |

#### 代码示例

```cpp
UCLASS()
class UMyAttributeSet : public UAttributeSet
{
    GENERATED_BODY()

public:
    // Health attribute
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_Health)
    FGameplayAttributeData Health;
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, Health)

    // MaxHealth attribute
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_MaxHealth)
    FGameplayAttributeData MaxHealth;
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, MaxHealth)

    // Attack Power
    UPROPERTY(BlueprintReadOnly, ReplicatedUsing = OnRep_AttackPower)
    FGameplayAttributeData AttackPower;
    ATTRIBUTE_ACCESSORS(UMyAttributeSet, AttackPower)

    // Called before attribute changes - for clamping
    virtual void PreAttributeChange(const FGameplayAttribute& Attribute, float& NewValue) override;

    // Called after GE execution - for post-processing logic
    virtual void PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data) override;

    UFUNCTION()
    void OnRep_Health(const FGameplayAttributeData& OldHealth);

    UFUNCTION()Z
    void OnRep_MaxHealth(const FGameplayAttributeData& OldMaxHealth);
};

void UMyAttributeSet::PreAttributeChange(const FGameplayAttribute& Attribute, float& NewValue)
{
    Super::PreAttributeChange(Attribute, NewValue);

    // Clamp health between 0 and MaxHealth
    if (Attribute == GetHealthAttribute())
    {
        NewValue = FMath::Clamp(NewValue, 0.0f, GetMaxHealth());
    }
}

void UMyAttributeSet::PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data)
{
    Super::PostGameplayEffectExecute(Data);

    if (Data.EvaluatedData.Attribute == GetHealthAttribute())
    {
        SetHealth(FMath::Clamp(GetHealth(), 0.0f, GetMaxHealth()));

        // Trigger death if health reaches 0
        if (GetHealth() <= 0.0f)
        {
            // Broadcast death event via GameplayEvent
            FGameplayEventData Payload;
            Payload.Instigator = Data.EffectSpec.GetEffectContext().GetInstigator();
            UAbilitySystemBlueprintLibrary::SendGameplayEventToActor(
                GetOwningActor(),
                FGameplayTag::RequestGameplayTag("Event.Death"),
                Payload);
        }
    }
}
```

### 3.5 GameplayTag

GAS 的**条件控制中枢**，使用层级化的标签进行状态管理。

#### 标签结构示例

```
Ability.Skill.FireBall
Ability.Skill.IceBlast
State.Dead
State.Stunned
State.Invincible
Damage.Physical
Damage.Magic.Fire
Cooldown.Skill.FireBall
```

#### 在 GA 中的标签配置

| 标签类型                 | 说明                           |
| ------------------------ | ------------------------------ |
| `AbilityTags`            | 标识该能力的标签               |
| `CancelAbilitiesWithTag` | 激活时取消拥有这些标签的能力   |
| `BlockAbilitiesWithTag`  | 激活期间阻止拥有这些标签的能力 |
| `ActivationOwnedTags`    | 激活期间给自身添加的标签       |
| `ActivationRequiredTags` | 激活所需的标签                 |
| `ActivationBlockedTags`  | 拥有这些标签时无法激活         |

### 3.6 AbilityTask

异步操作单元，在 GA 内部使用，用于处理耗时操作。

#### 常用内置 Task

| Task                           | 用途               |
| ------------------------------ | ------------------ |
| `PlayMontageAndWait`           | 播放动画蒙太奇     |
| `WaitGameplayEvent`            | 等待 GameplayEvent |
| `WaitTargetData`               | 等待目标选择数据   |
| `WaitInputPress`               | 等待输入按键       |
| `WaitDelay`                    | 等待延迟           |
| `WaitGameplayTagAdded/Removed` | 等待标签变化       |

#### 自定义 AbilityTask 示例

```cpp
UCLASS()
class UAbilityTask_WaitOverlap : public UAbilityTask
{
    GENERATED_BODY()

public:
    UPROPERTY(BlueprintAssignable)
    FWaitOverlapDelegate OnOverlap;

    UFUNCTION(BlueprintCallable, meta = (HidePin = "OwningAbility", DefaultToSelf = "OwningAbility"))
    static UAbilityTask_WaitOverlap* CreateWaitOverlap(UGameplayAbility* OwningAbility);

    virtual void Activate() override;
    virtual void OnDestroy(bool bInOwnerFinished) override;
};
```

### 3.7 GameplayCue

负责技能的**视觉/音频表现**，实现逻辑与表现分离。

#### 触发方式

```cpp
// Method 1: Via GameplayEffect (recommended)
// Set in GE's GameplayCue tag: GameplayCue.FireBall.Impact

// Method 2: Manual trigger in code
FGameplayCueParameters CueParams;
CueParams.Location = HitLocation;
AbilitySystemComponent->ExecuteGameplayCue(
    FGameplayTag::RequestGameplayTag("GameplayCue.FireBall.Impact"),
    CueParams);
```

#### Cue 类型

| 函数          | 触发时机                                      |
| ------------- | --------------------------------------------- |
| `OnExecute`   | 一次性触发（Instant GE）                      |
| `WhileActive` | 持续存在时触发（Duration/Infinite GE 添加时） |
| `OnRemove`    | GE 移除时触发                                 |

## 四、典型工作流

### 4.1 技能释放流程

```
Player Input
    │
    ▼
ASC->TryActivateAbility()
    │
    ▼
GA::CanActivateAbility()
  ├─ Check Tags (Required / Blocked)
  ├─ Check Cost (enough mana?)
  └─ Check Cooldown
    │
    ▼
GA::ActivateAbility()
    │
    ▼
GA::CommitAbility()
  ├─ Apply Cost GE (deduct mana)
  └─ Apply Cooldown GE
    │
    ▼
AbilityTask (PlayMontage, WaitEvent...)
    │
    ▼
Apply Damage GE to Target
    │
    ▼
Target ASC processes GE
  ├─ AttributeSet::PostGameplayEffectExecute()
  └─ Trigger GameplayCue
    │
    ▼
GA::EndAbility()
```

### 4.2 伤害计算流程

```
Source GA
    │
    ▼
Create GE Spec (Damage Effect)
  └─ Set Magnitude via SetByCaller / CustomCalc
    │
    ▼
Apply GE Spec to Target ASC
    │
    ▼
GameplayEffectExecutionCalculation (optional)
  ├─ Capture Source Attributes (AttackPower, CritRate...)
  ├─ Capture Target Attributes (Armor, Resistance...)
  └─ Calculate Final Damage
    │
    ▼
Target AttributeSet::PostGameplayEffectExecute()
  ├─ Clamp Health
  ├─ Check Death Condition
  └─ Trigger OnDamage Event
    │
    ▼
GameplayCue (Hit VFX, Damage Number...)
```

### 4.3 自定义 Execution Calculation

```cpp
UCLASS()
class UDamageExecutionCalc : public UGameplayEffectExecutionCalculation
{
    GENERATED_BODY()

public:
    UDamageExecutionCalc();

    virtual void Execute_Implementation(
        const FGameplayEffectCustomExecutionParameters& ExecutionParams,
        FGameplayEffectCustomExecutionOutput& OutExecutionOutput) const override;
};

UDamageExecutionCalc::UDamageExecutionCalc()
{
    // Declare captured attributes
    RelevantAttributesToCapture.Add(FGameplayEffectAttributeCaptureDefinition(
        UMyAttributeSet::GetAttackPowerAttribute(),
        EGameplayEffectAttributeCaptureSource::Source,
        true));

    RelevantAttributesToCapture.Add(FGameplayEffectAttributeCaptureDefinition(
        UMyAttributeSet::GetArmorAttribute(),
        EGameplayEffectAttributeCaptureSource::Target,
        true));
}

void UDamageExecutionCalc::Execute_Implementation(...) const
{
    float AttackPower = 0.f;
    ExecutionParams.AttemptCalculateCapturedAttributeMagnitude(
        ..., AttackPower);

    float Armor = 0.f;
    ExecutionParams.AttemptCalculateCapturedAttributeMagnitude(
        ..., Armor);

    // Damage formula
    float FinalDamage = FMath::Max(AttackPower - Armor * 0.5f, 0.0f);

    // Output the modifier
    OutExecutionOutput.AddOutputModifier(
        FGameplayModifierEvaluatedData(
            UMyAttributeSet::GetHealthAttribute(),
            EGameplayModOp::Additive,
            -FinalDamage));
}
```

## 五、网络同步与预测

### 客户端预测机制

```
Client                              Server
  │                                    │
  │── TryActivateAbility() ──────────▶│
  │   (Local Prediction)               │
  │   Create PredictionKey             │── Validate & Execute
  │   Execute Ability Locally          │   Apply GE (authoritative)
  │   Apply GE (predicted)            │
  │                                    │
  │◀─── Confirm / Reject ─────────────│
  │                                    │
  │ If Confirmed: Keep predicted state │
  │ If Rejected:  Rollback             │
```

关键点：
- **PredictionKey**：客户端生成的唯一标识，用于匹配预测和服务器确认
- **Scoped Prediction Window**：在此窗口内创建的效果都会被预测
- **Rollback**：服务器拒绝时，客户端自动回滚预测的状态

## 六、初始化流程

```cpp
// 1. In Character / PlayerState constructor
AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("ASC"));
AttributeSet = CreateDefaultSubobject<UMyAttributeSet>(TEXT("AttributeSet"));

// 2. Initialize ASC (typically in PossessedBy or BeginPlay)
void AMyCharacter::PossessedBy(AController* NewController)
{
    Super::PossessedBy(NewController);

    // Server-side: init ASC
    AbilitySystemComponent->InitAbilityActorInfo(this, this);

    // Grant default abilities
    for (TSubclassOf<UGameplayAbility>& Ability : DefaultAbilities)
    {
        AbilitySystemComponent->GiveAbility(
            FGameplayAbilitySpec(Ability, 1, INDEX_NONE, this));
    }

    // Apply default attributes via GE
    if (DefaultAttributeEffect)
    {
        FGameplayEffectContextHandle ContextHandle =
            AbilitySystemComponent->MakeEffectContext();
        ContextHandle.AddSourceObject(this);

        FGameplayEffectSpecHandle SpecHandle =
            AbilitySystemComponent->MakeOutgoingSpec(
                DefaultAttributeEffect, 1, ContextHandle);

        AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(*SpecHandle.Data.Get());
    }
}

// 3. Client-side: init in OnRep_PlayerState
void AMyCharacter::OnRep_PlayerState()
{
    Super::OnRep_PlayerState();
    AbilitySystemComponent->InitAbilityActorInfo(this, this);
}
```

## 七、设计哲学总结

| 原则               | 说明                                               |
| ------------------ | -------------------------------------------------- |
| **数据驱动**       | GE 和 AttributeSet 通过蓝图/数据表配置，无需硬编码 |
| **标签控制**       | GameplayTag 统一管理状态、条件、分类               |
| **逻辑与表现分离** | GameplayCue 独立处理视觉表现                       |
| **网络优先**       | 内置客户端预测和服务器权威机制                     |
| **可扩展性**       | 自定义 Task、Calculation、Cue 等扩展点丰富         |
