## 一、UE4 整体架构概览

### 1.1 引擎分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Game Project                             │
│                        游戏项目层                                │
│  (Your Game Logic, Blueprints, Content)                         │
├─────────────────────────────────────────────────────────────────┤
│                     Gameplay Framework                          │
│                     游戏性框架层                                 │
│  (GameMode, GameState, PlayerController, Pawn, HUD)             │
├─────────────────────────────────────────────────────────────────┤
│                      Engine Framework                           │
│                      引擎框架层                                  │
│  (Actor, Component, Level, World, Subsystem)                    │
├─────────────────────────────────────────────────────────────────┤
│                       Core Systems                              │
│                       核心系统层                                 │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐       │
│  │ Rendering│ Physics  │ Audio    │ Animation│ AI       │       │
│  │ 渲染     │ 物理     │ 音频      │ 动画     │ 人工智能  │       │
│  ├──────────┼──────────┼──────────┼──────────┼──────────┤       │
│  │ Network  │ Input    │ UI       │ Navigation│ Sequencer│      │
│  │ 网络     │ 输入      │ 界面     │ 导航      │ 过场     │       │
│  └──────────┴──────────┴──────────┴──────────┴──────────┘       │
├─────────────────────────────────────────────────────────────────┤
│                     UObject / Reflection                        │
│                     对象模型 / 反射系统                           │
│  (UObject, UClass, UPROPERTY, UFUNCTION, GC, Serialization)     │
├─────────────────────────────────────────────────────────────────┤
│                       Core / HAL                                │
│                       核心 / 硬件抽象层                          │
│  (Memory, Threading, Math, Containers, FileSystem, Platform)    │
```

### 1.2 引擎启动流程

```
程序入口 Program Entry
    │
    ▼
FEngineLoop::PreInit()
    │  ├─ 命令行解析 Command line parsing
    │  ├─ 模块加载 Module loading
    │  ├─ 配置文件读取 Config file reading
    │  └─ 平台初始化 Platform initialization
    ▼
FEngineLoop::Init()
    │  ├─ UEngine 创建 (GEngine)
    │  │   ├─ UGameEngine (独立运行 Standalone)
    │  │   └─ UEditorEngine (编辑器 Editor)
    │  ├─ 渲染设备初始化 RHI initialization
    │  ├─ 资产管理器初始化 Asset manager init
    │  └─ 世界创建 World creation
    ▼
FEngineLoop::Tick()  ← 主循环 Main Loop
    │
    ├─ GEngine->Tick()
    │   ├─ 世界 Tick World Tick
    │   │   ├─ Actor Tick (按 TickGroup 分组)
    │   │   ├─ Component Tick
    │   │   ├─ 物理模拟 Physics Simulation
    │   │   └─ Timer 回调 Timer Callbacks
    │   │
    │   ├─ 网络更新 Network Update
    │   │   ├─ 接收数据 Receive Data
    │   │   ├─ 属性复制 Property Replication
    │   │   └─ RPC 处理 RPC Processing
    │   │
    │   └─ 延迟操作 Deferred Actions
    │       ├─ 延迟销毁 Deferred Destroy
    │       └─ 关卡流送 Level Streaming
    │
    ├─ 渲染线程 Render Thread
    │   ├─ 场景遍历 Scene Traversal
    │   ├─ 可见性剔除 Visibility Culling
    │   ├─ 绘制调用 Draw Calls
    │   └─ 后处理 Post Processing
    │
    ├─ RHI 线程 RHI Thread
    │   └─ GPU 命令提交 GPU Command Submission
    │
    ├─ Slate 渲染 Slate Rendering (UI)
    │
    └─ GC 检查 Garbage Collection Check
        └─ 增量式 GC Incremental GC
```

### 1.3 帧循环详解 (Tick Order)

```
一帧的执行顺序 Single Frame Execution Order:

  ┌─────────────────────────────────────────────────────┐
  │ 1. 输入处理 Input Processing                         │
  │    PlayerInput → InputComponent → Bindings           │
  ├─────────────────────────────────────────────────────┤
  │ 2. TG_PrePhysics (物理前 Tick 组)                    │
  │    大多数 Actor/Component 在此 Tick                   │
  │    AI 决策、移动计算                                  │
  ├─────────────────────────────────────────────────────┤
  │ 3. TG_StartPhysics (物理开始)                        │
  │    物理模拟开始 Physics Simulation Start              │
  ├─────────────────────────────────────────────────────┤
  │ 4. TG_DuringPhysics (物理进行中)                     │
  │    与物理并行的逻辑                                   │
  ├─────────────────────────────────────────────────────┤
  │ 5. TG_EndPhysics (物理结束)                          │
  │    物理模拟完成 Physics Simulation Complete            │
  ├─────────────────────────────────────────────────────┤
  │ 6. TG_PostPhysics (物理后)                           │
  │    依赖物理结果的逻辑                                 │
  │    碰撞响应、物理动画                                 │
  ├─────────────────────────────────────────────────────┤
  │ 7. TG_PostUpdateWork (更新后)                        │
  │    相机更新、粒子系统、最终位置调整                    │
  ├─────────────────────────────────────────────────────┤
  │ 8. 渲染提交 Render Submission                        │
  │    场景代理更新 → 渲染命令入队                        │
  ├─────────────────────────────────────────────────────┤
  │ 9. Slate/UI 更新                                     │
  ├─────────────────────────────────────────────────────┤
  │ 10. 帧结束处理 Frame End                             │
  │     延迟销毁、GC 检查、帧同步                        │
  └─────────────────────────────────────────────────────┘
```

## 二、核心模块系统

### 2.1 模块架构

```
UE4 模块系统 Module System:

  模块 = 一个独立的编译单元 (.dll / .so)
  Module = An independent compilation unit

  ┌─────────────────────────────────────────────────────┐
  │                   Module Types                      │
  │                   模块类型                           │
  ├─────────────────────────────────────────────────────┤
  │                                                     │
  │  Runtime Module (运行时模块)                         │
  │    编辑器和游戏都加载                                 │
  │    例: Core, CoreUObject, Engine, InputCore         │
  │                                                     │
  │  Editor Module (编辑器模块)                          │
  │    仅编辑器加载，打包时不包含                         │
  │    例: UnrealEd, LevelEditor, PropertyEditor        │
  │                                                     │
  │  Developer Module (开发者模块)                       │
  │    开发阶段使用，发布时可选                           │
  │    例: ShaderCompileWorker, AssetTools              │
  │                                                     │
  │  ThirdParty Module (第三方模块)                      │
  │    外部库封装                                        │
  │    例: PhysX, Recast, FreeType                      │
  │                                                     │
  │  Plugin Module (插件模块)                            │
  │    可选功能，按需启用                                 │
  │    例: OnlineSubsystem, Niagara, Paper2D            │
  │                                                     │
  └─────────────────────────────────────────────────────┘
```

### 2.3 模块定义 (.Build.cs)

```csharp
// MyModule.Build.cs
// Module build configuration file

using UnrealBuildTool;

public class MyModule : ModuleRules
{
    public MyModule(ReadOnlyTargetRules Target) : base(Target)
    {
        // PCH usage mode
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        // Public dependencies (headers and linking exposed to dependent modules)
        PublicDependencyModuleNames.AddRange(new string[] {
            "Core",
            "CoreUObject",
            "Engine",
            "InputCore"
        });

        // Private dependencies (linking only, headers not exposed)
        PrivateDependencyModuleNames.AddRange(new string[] {
            "Slate",
            "SlateCore",
            "UMG"
        });

        // Public include paths
        PublicIncludePaths.Add("MyModule/Public");

        // Private include paths
        PrivateIncludePaths.Add("MyModule/Private");

        // Macro definitions
        PublicDefinitions.Add("MY_MODULE_API=1");

        // Conditional compilation
        if (Target.Platform == UnrealTargetPlatform.Win64)
        {
            PrivateDependencyModuleNames.Add("D3D11RHI");
        }
    }
}
```

## 三、UObject / UClass 反射系统

### 3.1 UObject 类层次

```
UObject 继承体系 UObject Hierarchy:

  UObjectBase
    │  最底层，存储 UClass* 指针和对象标志
    │  Stores UClass pointer and object flags
    │
    └─ UObjectBaseUtility
        │  提供辅助函数 (IsA, GetClass, GetName...)
        │
        └─ UObject
            │  完整的对象系统基类
            │  反射、序列化、GC、CDO
            │
            ├─ UField
            │   ├─ UStruct
            │   │   ├─ UClass        (类的元数据 class metadata)
            │   │   ├─ UScriptStruct (结构体元数据 USTRUCT)
            │   │   └─ UFunction     (函数元数据 UFUNCTION)
            │   ├─ UProperty (UE4) / FProperty (UE4.25+)
            │   │   ├─ UIntProperty
            │   │   ├─ UFloatProperty
            │   │   ├─ UObjectProperty
            │   │   └─ ... (one per type)
            │   └─ UEnum            (枚举元数据 UENUM)
            │
            ├─ AActor              (场景中的实体 entity in world)
            ├─ UActorComponent     (组件基类 component base)
            ├─ USubsystem          (子系统 subsystem)
            ├─ UGameInstance       (游戏实例 game instance)
            ├─ UWorld              (世界 world)
            ├─ ULevel              (关卡 level)
            ├─ UPackage            (资产包 asset package)
            ├─ UBlueprint          (蓝图资产 blueprint asset)
            ├─ UDataAsset          (数据资产 data asset)
            └─ UDeveloperSettings  (开发者设置 developer settings)
```

### 3.2 反射宏系统

```
UHT (Unreal Header Tool) 反射宏:

  ┌─────────────────────────────────────────────────────────┐
  │ UCLASS()         - 类声明 class declaration              │
  │ USTRUCT()        - 结构体声明 struct declaration         │
  │ UENUM()          - 枚举声明 enum declaration             │
  │ UPROPERTY()      - 属性声明 property declaration         │
  │ UFUNCTION()      - 函数声明 function declaration         │
  │ GENERATED_BODY() - 生成代码插入点 generated code slot    │
  └─────────────────────────────────────────────────────────┘

  编译流程 Compilation Flow:

  .h 文件 (带 UCLASS 等宏)
       │
       ▼
  UHT (Unreal Header Tool) 解析
       │  生成 .generated.h 和 .gen.cpp
       │  包含反射数据注册代码
       ▼
  C++ 编译器编译
       │
       ▼
  运行时: UClass 对象包含完整的反射信息
          属性偏移、函数指针、元数据
```

### 3.3 UPROPERTY 说明符

```cpp
// UPROPERTY common specifiers

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Combat")
float Health = 100.f;
// EditAnywhere:       editable anywhere in editor (CDO + instance)
// BlueprintReadWrite: readable and writable in Blueprint
// Category:           editor category grouping

UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
UStaticMeshComponent* MeshComp;
// VisibleAnywhere:  visible but not editable in editor
// BlueprintReadOnly: read-only in Blueprint

UPROPERTY(Replicated, ReplicatedUsing = OnRep_Health)
float Health;
// Replicated:      network replicated
// ReplicatedUsing: replication callback function

UPROPERTY(Transient)
float CachedValue;
// Transient: not serialized, not saved

UPROPERTY(SaveGame)
int32 PlayerScore;
// SaveGame: included in save game data

UPROPERTY(meta = (ClampMin = "0", ClampMax = "100"))
float Percentage;
// meta: metadata for editor constraints
```

### 3.4 UFUNCTION 说明符

```cpp
// UFUNCTION common specifiers

UFUNCTION(BlueprintCallable, Category = "Combat")
void TakeDamage(float Amount);
// BlueprintCallable: callable from Blueprint

UFUNCTION(BlueprintImplementableEvent)
void OnDeath();
// BlueprintImplementableEvent: declared in C++, implemented in Blueprint

UFUNCTION(BlueprintNativeEvent)
void OnHit();
// BlueprintNativeEvent: C++ provides default, Blueprint can override
// Requires OnHit_Implementation()

UFUNCTION(Server, Reliable, WithValidation)
void ServerFire();
// Server:         client calls, server executes (RPC)
// Reliable:       guaranteed delivery (TCP-like)
// WithValidation: requires ServerFire_Validate()

UFUNCTION(Client, Unreliable)
void ClientPlayHitEffect();
// Client:     server calls, owning client executes
// Unreliable: may be lost (UDP-like)

UFUNCTION(NetMulticast, Reliable)
void MulticastPlayExplosion();
// NetMulticast: server calls, all clients execute

UFUNCTION(Exec)
void DebugCommand();
// Exec: console command
```

### 3.5 CDO (Class Default Object) 机制

```
CDO 机制 Class Default Object:

  每个 UClass 都有一个 CDO 实例
  CDO 存储该类的默认属性值
  Every UClass has a CDO instance storing default property values

  UClass (AMyCharacter)
    │
    ├─ ClassDefaultObject ← CDO (default instance of AMyCharacter)
    │   │  Health = 100
    │   │  Speed = 600
    │   │  MeshComp = nullptr (components not created in CDO)
    │   │
    │   └─ Usage:
    │       ├─ Copy default values when creating new objects
    │       ├─ Reference for "Reset to Default" in editor
    │       ├─ Delta serialization (only save values different from CDO)
    │       └─ Source of parent class defaults during Blueprint compilation
    │
    ├─ Instance 1 (Health = 80, Speed = 600)  ← only serialize Health
    ├─ Instance 2 (Health = 100, Speed = 800) ← only serialize Speed
    └─ Instance 3 (Health = 100, Speed = 600) ← no serialization needed

  获取 CDO Getting CDO:
    AMyCharacter* CDO = GetDefault<AMyCharacter>();
    // Read-only access to defaults

    AMyCharacter* MutableCDO = GetMutableDefault<AMyCharacter>();
    // Writable access (use with caution!)
```

## 四、Actor / Component 模型

### 4.1 Actor 生命周期

```
Actor 生命周期 Actor Lifecycle:

  ┌─────────────────────────────────────────────────────┐
  │                   创建阶段 Creation                  │
  ├─────────────────────────────────────────────────────┤
  │                                                     │
  │  1. Constructor (构造函数)                           │
  │     │  ⚠️ Only called once for CDO creation          │
  │     │  Use for: setting defaults, creating components │
  │     │  Forbidden: accessing World, other Actors,      │
  │     │             game logic                          │
  │     │                                                 │
  │     │  CreateDefaultSubobject<T>() to create comps    │
  │     │  SetRootComponent() to set root                 │
  │     │  SetupAttachment() to set hierarchy             │
  │     ▼                                                 │
  │  2. PostInitializeComponents()                        │
  │     │  All components initialized                     │
  │     │  Safe to access components                      │
  │     ▼                                                 │
  │  3. BeginPlay()                                       │
  │     │  Actor officially enters the game world         │
  │     │  Safe to access World, other Actors             │
  │     │  Initialize game logic here                     │
  │                                                       │
  ├───────────────────────────────────────────────────────┤
  │                   运行阶段 Runtime                     │
  ├───────────────────────────────────────────────────────┤
  │                                                       │
  │  4. Tick(float DeltaTime)                             │
  │     │  Called every frame (if bCanEverTick = true)    │
  │     │  Main game logic update                         │ 
  │     │                                                 │
  │  5. Various Events                                    │
  │     │  NotifyHit() - collision                        │
  │     │  NotifyActorBeginOverlap() - overlap            │
  │     │  OnRep_XXX() - network replication callback     │
  │                                                       │
  ├───────────────────────────────────────────────────────┤
  │                   销毁阶段 Destruction                 │
  ├───────────────────────────────────────────────────────┤
  │                                                       │
  │  6. EndPlay(EEndPlayReason)                           │
  │     │  Actor about to leave the world                 │
  │     │  Reasons: Destroyed, LevelTransition,           │
  │     │           RemovedFromWorld, Quit                │
  │     ▼                                                 │
  │  7. Destroyed()                                       │
  │     │  Actor marked for pending destruction           │
  │     ▼                                                 │
  │  8. BeginDestroy()                                    │
  │     │  Begin async destruction                        │
  │     ▼                                                 │
  │  9. FinishDestroy()                                   │
  │        GC reclaims memory                             │
  │                                                       │
  └───────────────────────────────────────────────────────┘

  ⚠️ Key distinction:
  Destroy() → marks for destruction, deferred to frame end or GC
  Do NOT access the Actor immediately after Destroy()!
  Use IsValid(Actor) or IsValidLowLevel() to check
```

```
         │  DOREPLIFETIME_CONDITION(AMyActor, Ammo, COND_OwnerOnly)
         ▼
    Serialize changed properties
         │
         ▼
    Send via ActorChannel

  ─────── Network ───────
         │
  Client:
         ▼
    Deserialize properties
         │
         ▼

```

```
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  Server RPC (Client → Server)                               │
  │  ┌──────────┐         ┌──────────┐                          │
  │  │ Client   │ ──────→ │ Server   │                          │
  │  │ calls    │         │ executes │                          │

  │  ┌──────────┐         ┌──────────┐                          │
  │  │ Server   │ ──────→ │ Client   │                          │
  │  │ calls    │         │ (Owner)  │                          │
  │  │          │         │ executes │                          │
  │  └──────────┘         └──────────┘                          │
  │  UFUNCTION(Client, Reliable)                                │ 
  │  void ClientShowDamageNumber(float Damage);                 │
  │  Use: server notifies specific client (show damage, SFX)    │
  │                                                             │
  │  Multicast RPC (Server → All Clients)                       │
  │  ┌──────────┐    ┌──────────┐                               │
  │  │ Server   │ ──→│ Client 1 │                               │
  │  │ calls    │ ──→│ Client 2 │                               │
  │  │          │ ──→│ Client 3 │                               │
  │  └──────────┘    └──────────┘                               │
  │  UFUNCTION(NetMulticast, Unreliable)                        │
  │  void MulticastPlayExplosion();                             │
  │  Use: effects everyone needs to see (explosion, death)      │
  │                                                             │
  │  Reliable vs Unreliable:                                    │
  │  ├─ Reliable:   guaranteed delivery, ordered (TCP-like)     │
  │  │              Use for: critical game logic (fire, buy)    │
  │  │              ⚠️ Too many causes bandwidth issues         │
  │  │                                                          │
  │  └─ Unreliable: may be lost, unordered (UDP-like)           │
  │                 Use for: frequent updates (position, anim)  │
  │                 Loss is acceptable                          │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

```cpp
    bool bShouldStop = false;

public:
    void Start()
    {
        Thread = FRunnableThread::Create(this, TEXT("MyThread"));
    }
};
```

## 十四、内存管理与 GC

### 14.1 内存管理架构

```
UE4 Memory Management:

  ┌─────────────────────────────────────────────────────────────┐
  │                    UObject Memory Management                │
  │                                                             │
  │  UObject is managed by GC (Garbage Collection)              │
  │  Do NOT manually delete UObject!                            │
  │                                                             │
  │  Creating UObject:                                          │
  │    NewObject<T>()           - general creation              │
  │    CreateDefaultSubobject() - create component in ctor      │
  │    SpawnActor<T>()          - create Actor                  │
  │    DuplicateObject()        - duplicate object              │
  │                                                             │
  │  Preventing GC (keeping references):                        │
  │    UPROPERTY() marked member variables                      │
  │    TArray<UObject*> etc. containers (need UPROPERTY)        │
  │    AddToRoot() / RemoveFromRoot()                           │
  │    FGCObjectScopeGuard (temporary protection)               │
  │                                                             │
  │  Destruction:                                               │
  │    Actor: Destroy() → mark pending kill → GC reclaims       │
  │    Component: DestroyComponent()                            │
  │    Other UObject: remove all refs → GC auto-reclaims        │
  │    ConditionalBeginDestroy() → force begin destruction      │
  │                                                             │
  ├─────────────────────────────────────────────────────────────┤
  │                    Non-UObject Memory Management            │
  │                                                             │
  │  Regular C++ objects:                                       │
  │    TSharedPtr<T>  - shared pointer (reference counted)      │
  │    TWeakPtr<T>    - weak pointer (no ref count increment)   │
  │    TUniquePtr<T>  - unique pointer (exclusive ownership)    │
  │    ⚠️ Cannot be used for UObject!                          │
  │                                                             │
  │  TWeakObjectPtr<T> - UObject weak reference                 │
  │    Does not prevent GC collection                           │
  │    Must check IsValid() before access                       │
  │                                                             │
  │  FMalloc - low-level memory allocator                       │
  │    FMemory::Malloc() / Free()                               │
  │    Platform-specific impl (TBB, Binned, Binned2, Ansi)      │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

### 14.2 GC 机制

```
Garbage Collection Mechanism:

  UE4 uses Mark-Sweep GC

  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  Phase 1: Mark                                             │
  │    Traverse from GC root set                                │
  │    ├─ Global UObject list                                  │
  │    ├─ AddToRoot() objects                                  │
  │    ├─ UPROPERTY() reference chains                         │
  │    └─ Mark all reachable objects as "alive"                │
  │                                                             │
  │  Phase 2: Sweep                                            │
  │    Traverse all UObjects                                   │
  │    Unmarked objects → call BeginDestroy()                   │
  │    Wait for async destruction → FinishDestroy()             │
  │    Reclaim memory                                           │
  │                                                             │
  │  GC Trigger Timing:                                        │
  │    ├─ Periodic (default 60 seconds)                        │
  │    │   gc.TimeBetweenPurgingPendingKillObjects              │
  │    ├─ Under memory pressure                                │
  │    ├─ During level transitions                             │
  │    └─ Manual: GEngine->ForceGarbageCollection()            │
  │                                                             │
  │  Incremental GC:                                           │
  │    Does not complete all GC work in one frame              │
  │    Spread across multiple frames to reduce hitches         │
  │    gc.MaxObjectsNotConsideredByGC                           │
  │    gc.IncrementalBeginDestroyEnabled                        │
  │                                                             │
  │  Cluster GC:                                               │
  │    Group related objects into clusters                      │
  │    Objects in cluster marked/reclaimed together             │
  │    Reduces GC traversal overhead                           │
  │    Example: Actor + all Components = one cluster            │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  ⚠️ Common GC Pitfalls:
    1. Raw pointer to UObject without UPROPERTY
       → Object GC'd, dangling pointer!
    2. Lambda capturing UObject pointer
       → GC unaware Lambda holds reference
       → Use TWeakObjectPtr or ensure lifetime
    3. Circular references
       → UE4 GC handles cycles (mark-sweep)
       → But TSharedPtr cannot! (reference counting)
```

## 十六、面试高频题

### 16.1 概念理解题

```
Q1: UObject 和 AActor 的区别？
A:
  UObject:
    - UE4 对象系统的基类 (base class of object system)
    - 提供反射、序列化、GC (reflection, serialization, GC)
    - 不存在于世界中，没有变换 (no world presence, no transform)
    - 例: UDataAsset, UAnimInstance, USubsystem

  AActor:
    - 继承自 UObject (inherits from UObject)
    - 可以放置在世界 (UWorld) 中 (placeable in world)
    - 有生命周期 (BeginPlay, Tick, EndPlay)
    - 有组件系统 (Component system)
    - 支持网络复制 (network replication)
    - 有变换 (通过 RootComponent) (transform via RootComponent)

Q2: Actor 的 Tick 和 Component 的 Tick 谁先执行？
A:
  默认情况下 Actor Tick 先于其 Component Tick
  By default Actor Tick executes before its Component Tick
  可通过 bTickBeforeOwner 改变
  Can be changed via bTickBeforeOwner
  都受 TickGroup 影响
  Both affected by TickGroup
  同一 TickGroup 内: Actor → Component
  Within same TickGroup: Actor → Component

Q3: GameMode 和 GameState 的区别？
A:
  GameMode:
    - 仅存在于服务器 (server only)
    - 定义游戏规则 (defines game rules)
    - 处理玩家加入/离开 (handles player join/leave)
    - 客户端无法访问 (clients cannot access)

  GameState:
    - 存在于所有端 (all endpoints: server + all clients)
    - 存储需要所有人知道的游戏状态 (shared game state)
    - 比分、时间、玩家列表 (score, time, player list)
    - 通过网络复制同步 (synced via replication)

Q4: 什么是 CDO？有什么用？
A:
  CDO = Class Default Object
  每个 UClass 有一个默认实例
  Every UClass has one default instance
  用途 Usage:
    1. 新对象创建时复制默认值 (copy defaults for new objects)
    2. 序列化时差异比较 (delta serialization)
    3. 编辑器 "Reset to Default" 的参考 (editor reset reference)
    4. 蓝图编译时的父类默认值来源 (Blueprint compilation)
  获取: GetDefault<T>()

Q5: TSharedPtr 和 TWeakObjectPtr 的区别？
A:
  TSharedPtr<T>:
    - 用于非 UObject 的 C++ 对象 (for non-UObject C++ objects)
    - 引用计数 (reference counted)
    - 不能用于 UObject! (cannot use for UObject!)

  TWeakObjectPtr<T>:
    - 用于 UObject (for UObject)
    - 不阻止 GC 回收 (does not prevent GC)
    - 访问前需 IsValid() 检查 (check IsValid() before access)
    - 解决悬空指针问题 (solves dangling pointer issues)
```

```
  - Server authoritative (all modifications via Server RPC)
  - Property replication syncs to clients
  - Delegate-driven UI updates

Q7: 如何优化大世界的性能？
A:
  1. Level Streaming 关卡流送
     Load/unload sub-levels by region
     World Composition for auto management

  2. LOD System
     Lower detail models for distant objects
     HLOD merges distant objects

  3. Occlusion Culling 遮挡剔除
     Don't render invisible objects
     Software/Hardware Occlusion Query

  4. Instanced Rendering 实例化渲染
     Same mesh uses InstancedStaticMesh
     Reduces Draw Calls

  5. Virtual Texture 虚拟纹理
     Load texture tiles on demand
     Reduces VRAM usage

  6. Nanite (UE5) / Custom LOD
     Virtual geometry, automatic LOD

  7. Object Pooling 对象池
     Reuse frequently created/destroyed objects
     Reduces GC pressure

  8. Async Loading 异步加载
     Load assets asynchronously to avoid hitches
     Preload upcoming areas
```

### 16.3 调试与性能题

```
Q8: 如何排查 UE4 中的性能问题？
A:
  1. Built-in Tools:
     stat unit        - frame time breakdown (Game/Draw/GPU)
     stat fps         - frame rate
     stat scenerendering - rendering stats
     stat memory      - memory stats
     stat game        - game thread stats

  2. Unreal Insights
     Lightweight profiling tool
     Records CPU/GPU/Memory/Network

  3. GPU Profiler
     ProfileGPU command
     View per-pass render time

  4. Common Bottlenecks:
     CPU bottleneck:
       - Heavy Tick functions → reduce Tick frequency
       - Blueprint Tick → use Timer or event-driven
       - GC hitches → reduce UObject creation/destruction
       - Too many collision queries → optimize channels

     GPU bottleneck:
       - Too many Draw Calls → batching, instancing
       - Overdraw → reduce translucency
       - Shader complexity → simplify materials
       - High resolution → dynamic resolution

     Memory bottleneck:
       - Large textures → compression, Mipmap, streaming
       - Dense meshes → LOD
       - Assets not unloaded → check reference chains

Q9: 如何调试网络同步问题？
A:
  1. Tools:
     stat net          - network stats
     net.ListActors    - list replicated Actors
     p.NetShowCorrections 1 - show position corrections

  2. Simulate Network Conditions:
     Editor: Play → Advanced Settings
     → Set latency, packet loss, bandwidth limits

  3. Common Issues:
     - Property not replicated → check UPROPERTY(Replicated)
     - RPC not executing → check caller and executor sides
     - Position jitter → check movement component smoothing
     - High bandwidth → reduce replication frequency,
       use conditional replication
```
