# 抽象工厂模式（Abstract Factory Pattern）

## 概念简介

抽象工厂模式是一种创建型设计模式。它提供一个接口，用于创建一系列相关或相互依赖的对象，而无需指定它们具体的类。通常每个工厂对应不同的产品族，确保一族对象能一起工作。

```
      │
┌─────▼─────┐

└───────────┘
┌─────▼─────┐
│产品B2     │
└───────────┘
```

## 代码示例（C#）

```csharp
// 抽象产品A
public interface IProductA
{
    void Show();
}

// 抽象产品B
public interface IProductB
{
    void Display();
}

// 具体产品A1
public class ProductA1 : IProductA
{
    public void Show()
    {
        Console.WriteLine("产品A1");
    }
}

// 具体产品A2
public class ProductA2 : IProductA
{
    public void Show()
    {
        Console.WriteLine("产品A2");
    }
}

// 具体产品B1
public class ProductB1 : IProductB
{
    public void Display()
    {
        Console.WriteLine("产品B1");
    }
}

// 具体产品B2
public class ProductB2 : IProductB
{
    public void Display()
    {
        Console.WriteLine("产品B2");
    }
}

// 抽象工厂
public interface IFactory
{
    IProductA CreateProductA();
    IProductB CreateProductB();
}

// 具体工厂1
public class Factory1 : IFactory
{
    public IProductA CreateProductA()
    {
        return new ProductA1();
    }
    public IProductB CreateProductB()
    {
        return new ProductB1();
    }
}

// 具体工厂2
public class Factory2 : IFactory
{
    public IProductA CreateProductA()
    {
        return new ProductA2();
    }
    public IProductB CreateProductB()
    {
        return new ProductB2();
    }
}

// 客户端调用
class Program
{
    static void Main(string[] args)
    {
        IFactory factory = new Factory1();
        IProductA productA = factory.CreateProductA();
        IProductB productB = factory.CreateProductB();
        productA.Show();
        productB.Display();
    }
}
```

---

## 优缺点

**优点：**
- 易于交换产品系列，符合开闭原则。
- 保证产品族之间的一致性。

**缺点：**
- 新增产品等级结构时，需要修改所有工厂类，扩展不便。
- 结构复杂，类数量增多。

---

## 应用场景

- 系统需要保证产品族之间的相互兼容。
- 系统要求产品的创建独立于产品的实现。
